# LoRA 微调原理与调参实战 | 把大象装进消费级显卡

LoRA 是当前最主流的 PEFT 方法，让你用一张 4090 就能微调 7B 甚至 13B 模型。本文深入原理并给出调参指南。

## 一、LoRA 核心原理

### 直观理解
LoRA 的洞见：微调时权重的变化矩阵是**低秩**的。

```
完整微调：更新所有参数 W ← W + ΔW
LoRA：  冻结 W，只训练两个小矩阵 A 和 B，使 ΔW ≈ B × A

其中：
W ∈ R^(d × k)：原始权重矩阵（冻结，不训练）
B ∈ R^(d × r)：上投影矩阵（可训练）
A ∈ R^(r × k)：下投影矩阵（可训练）
r：秩（rank），远小于 d 和 k
```

### 参数量对比
```
7B 模型，r=8，在所有 Attention 层插入 LoRA：
完整微调参数量：7,000,000,000
LoRA 参数量：    约 33,000,000（0.5%）

比例：微调的参数量只有全量的 1/200
```

## 二、插入位置

### 标准配置（LLaMA 架构）
```python
TARGET_MODULES = [

    "gate_proj", # FFN 门控

```

| Target Modules | 可训练参数 | 效果 | 推荐 |
|---------------|-----------|------|------|

| Attention 全层 | ~22M | 92% | 常用配置 |

| 全模块（all） | ~33M | 100% | 默认推荐 |

## 三、Rank 选择

### Rank 的物理意义
```
r=4  → 低保真，适合简单任务（分类、风格调整）
r=8  → 最常用平衡点，适合指令微调
r=16 → 更高保真，适合领域知识注入
r=32 → 接近全量微调效果
r=64+→ 几乎等效全量（但参数仍少得多）
```

### 调参实验示例
```python
# 设计 LoRA 调参实验
experiments = [
    {"r": 4, "alpha": 8, "dropout": 0.05},
    {"r": 8, "alpha": 16, "dropout": 0.05},
    {"r": 16, "alpha": 32, "dropout": 0.05},
    {"r": 8, "alpha": 8, "dropout": 0.05},   # alpha=r
    {"r": 8, "alpha": 32, "dropout": 0.05},   # alpha=4r
    {"r": 8, "alpha": 16, "dropout": 0.1},
]

# 每个配置训练 1 epoch，对比验证集 loss
# 选最优配置跑完整训练
```

## 四、alpha 与 Dropout

### alpha 的选择
```python
# alpha 是 LoRA 缩放的幅度：ΔW = (alpha/r) × B × A
# 常用规则：alpha = 2 × r

# 如果 alpha = r：缩放因子 = 1，ΔW 的影响较弱
# 如果 alpha = 2r：缩放因子 = 2，ΔW 的影响适中
# 如果 alpha = 4r：缩放因子 = 4，ΔW 的影响较强

# 调整 alpha ≈ 调整学习率的感觉
# 但机制不同：alpha 控制 ΔW 的幅值上限
```

### Dropout 的作用
```python
# LoRA dropout 防止两个小矩阵过拟合
# 0.0：不开 dropout（数据量 > 1万 条时推荐）
# 0.05：默认值，适合大多数场景
# 0.1：数据量 < 500 条时推荐
```

## 五、完整训练代码

### PEFT 库实现
```python
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer
from peft import LoraConfig, get_peft_model, TaskType
import torch

model_name = "Qwen/Qwen2.5-7B-Instruct"

# 加载模型
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)
tokenizer = AutoTokenizer.from_pretrained(model_name)

# LoRA 配置
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,                     # 秩
    lora_alpha=16,           # 缩放
    lora_dropout=0.05,       # 丢弃率
    target_modules=[
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj"
    ],
    bias="none",
)

# 应用 LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 输出: trainable params: 33,554,432 || all params: 7,073,554,432 || trainable%: 0.4745
```

## 六、调参心法

### 常见问题诊断
```python
# 问题：训练 loss 降不下去
# 诊断步骤：
"""
1. 检查学习率是否合适（LoRA 推荐 1e-4 ~ 5e-4）
2. 增大 rank（8 → 16）
3. 增加 target_modules（加上 k_proj、gate_proj）
4. 增大 alpha（16 → 32）
"""

# 问题：loss 降很快但验证集效果差
"""
1. 减小 rank（16 → 8）
2. 增加 lora_dropout（0.05 → 0.1）
3. 减少 epoch
4. 增加数据量
"""

# 问题：效果和 baseline 差不多
"""
1. 数据量可能不够（< 200 条）
2. rank 太小（4 → 8）
3. 学习率太小
4. alpha 太小
"""
```

## 七、LoRA 变体

| 变体 | 改进点 | 使用场景 |
|------|--------|---------|
| LoRA+ | 对 A 和 B 使用不同学习率 | 追求更优收敛 |
| rsLoRA | rank 稳定缩放 | 大 rank 场景（r≥64） |
| DoRA | 分离方向和幅度 | 效果略优于 LoRA |
| VeRA | 共享随机矩阵 | 极致省参数 |

### DoRA 快速尝试
```python
from peft import LoraConfig

# DoRA 只需在 LoraConfig 中加一个参数
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,
    lora_alpha=16,
    use_dora=True,  # 开启 DoRA
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
)
```

## 八、LoRA 合并与卸载

```python
# 合并 LoRA 权重到基座模型（部署用）
model = model.merge_and_unload()

# 只卸载 LoRA 权重（切换 LoRA 用）

# 保存 LoRA 权重（只保存 adapter，文件很小 ~100MB）

# 加载 LoRA 权重
```

## 总结

> LoRA 的核心哲学：不要全量更新，只在低秩空间寻找最优扰动。

**记住这组数字：r=8, alpha=16, dropout=0.05, lr=2e-4 → 90% 的场景直接能用。**
