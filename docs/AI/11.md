# QLoRA | 消费级显卡微调大模型

QLoRA 是 LoRA + 4bit 量化的结合，让 RTX 4060 也能微调 7B 模型。本文教你榨干消费级显卡的最后 1GB 显存。

## 一、QLoRA 的核心创新

### 三项关键技术
```python
"""
QLoRA = 4bit NormalFloat（NF4）量化 
      + 双重量化（Double Quantization） 
      + 分页优化器（Paged Optimizer）
"""
```

| 技术 | 作用 | 显存节省 |
|------|------|---------|
| NF4 量化 | 将模型权重压缩到 4bit | ~75% |
| 双重量化 | 量化常数的二次压缩 | 额外 ~0.4GB |
| 分页优化器 | 梯度 checkpoint 溢出到 CPU | 防止 OOM |

## 二、显存需求对比

```
任务：微调 7B 模型，batch_size=4

全量微调（FP32）：  ~120GB ❌ 需要 A100
全量微调（BF16）：  ~60GB  ❌ 需要 A100
LoRA（BF16）：      ~24GB  ✅ RTX 4090
QLoRA（NF4）：      ~10GB  ✅ RTX 4060 / 4070
QLoRA（NF4）batch=2: ~6GB  ✅ RTX 3060 / 笔记本 GPU
```

## 三、QLoRA 配置详解

### BitsAndBytes 配置
```python
from transformers import BitsAndBytesConfig
import torch

# QLoRA 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,                # 启用 4bit 量化
    bnb_4bit_quant_type="nf4",       # 量化类型 nf4
    bnb_4bit_use_double_quant=True,  # 双重量化
    bnb_4bit_compute_dtype=torch.bfloat16,  # 计算时用 BF16
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B-Instruct",
    quantization_config=bnb_config,
    device_map="auto",
    trust_remote_code=True,
)
```

### 完整 QLoRA 训练代码
```python
from transformers import (
    AutoModelForCausalLM, AutoTokenizer,
    TrainingArguments, Trainer, DataCollatorForLanguageModeling,
    BitsAndBytesConfig
)
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from datasets import load_dataset

# 1. 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_use_double_quant=True,
    bnb_4bit_compute_dtype=torch.bfloat16,
)

# 2. 加载模型
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B-Instruct",
    quantization_config=bnb_config,
    device_map="auto",
)

# 3. 预处理（QLoRA 必须）
model = prepare_model_for_kbit_training(model)

# 4. LoRA 配置
lora_config = LoraConfig(
    r=8,
    lora_alpha=16,
    lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
    bias="none",
    task_type="CAUSAL_LM",
)

# 5. 应用 LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

# 6. 训练参数（针对小显存优化）
training_args = TrainingArguments(
    output_dir="./qlora-output",
    per_device_train_batch_size=2,       # 小 batch
    gradient_accumulation_steps=8,       # 目标 batch=16
    gradient_checkpointing=True,         # 节省显存
    optim="paged_adamw_8bit",            # 8bit 优化器
    learning_rate=2e-4,
    num_train_epochs=3,
    warmup_ratio=0.1,
    lr_scheduler_type="cosine",
    bf16=True,
    logging_steps=10,
    save_steps=500,
    save_total_limit=1,                  # 只保留 1 个 checkpoint
    ddp_find_unused_parameters=False,
    report_to="none",
)

trainer = Trainer(
    model=model,
    args=training_args,

    data_collator=data_collator,
)
trainer.train()
```

## 四、显存优化技巧

### 逐级省显存
```python
# 从高到低的显存优化梯度

# Level 1: 基础 QLoRA
batch_size = 2
gradient_accumulation_steps = 8

# Level 2: 开启梯度检查点
gradient_checkpointing = True

# Level 3: 使用 8bit 优化器
optim = "paged_adamw_8bit"

# Level 4: 减少 max_seq_length
max_seq_length = 1024  # 原本 2048

# Level 5: 减少 LoRA rank
lora_r = 4  # 原本 8

# Level 6: 减少 target modules
target_modules = ["q_proj", "v_proj"]  # 最精简

# Level 7: 使用 CPU offload
device_map = "auto"  # 会自动 offload 到 CPU
```

### 显存预算表
```
7B 模型显存预算（batch=2, max_len=1024）：
├── 模型权重（4bit）：    ~4GB
├── LoRA 参数：           ~0.1GB
├── 优化器状态（8bit）：  ~0.2GB
├── 激活值（grad checkpoint）： ~2GB
├── KV Cache：            ~0.5GB
├── 其他开销：            ~1GB
└── 总计：                ~8GB
```

## 五、常见踩坑指南

### 坑 1：忘了 prepare_model_for_kbit_training
```python
# ❌ 错误：直接应用 LoRA 到量化模型
model = get_peft_model(model, lora_config)

# ✅ 正确：先预处理
model = prepare_model_for_kbit_training(model)
model = get_peft_model(model, lora_config)
```

### 坑 2：混合精度不匹配
```python
# ✅ QLoRA 推荐配置
bnb_4bit_compute_dtype = torch.bfloat16  # 量化内部计算精度
training_args.bf16 = True                 # 训练使用 BF16

# 如果显卡不支持 BF16（如老款 1080Ti）
bnb_4bit_compute_dtype = torch.float16
training_args.fp16 = True
```

### 坑 3：Batch Size 过大导致 OOM
```python
# QLoRA 从小 batch 开始，逐步加大
suggested_batch = {
    "8GB VRAM": 1,
    "12GB VRAM": 2,
    "16GB VRAM": 4,
    "24GB VRAM": 8,
}

# 配合梯度累积达到目标 batch
gradient_accumulation_steps = target_batch // per_device_batch
```

## 六、与标准 LoRA 的对比

| 维度 | LoRA | QLoRA |
|------|------|-------|
| 模型精度 | BF16/FP16 | NF4 |
| 显存需求（7B）| ~24GB | ~8GB |
| 训练速度 | 1.0x | ~0.7x（量化有开销）|
| 效果 | baseline | 接近 LoRA（95-98%） |
| 适合显卡 | 4090/5090 | 4060/4070 |
| 加载速度 | 快 | 慢（需要量化模型）|

## 七、效果试验验证

QLoRA 效果是否真的接近全量微调？以下来自社区实践：
- Qwen2.5-7B 在中文指令跟随任务上，QLoRA r=8 相对 LoRA r=8 效果损失 < 3%
- 对于分类等简单任务，QLoRA 几乎无损失
- 对于复杂推理任务（数学、代码），建议 r 适当增大到 16

## 总结

> QLoRA = 降低的是硬件门槛，而不是效果下限。

RTX 4060（8GB）+ QLoRA + batch=1 + 梯度累积 = 7B 模型微调自由。
