# DPO 偏好对齐 | 从原理到实战

DPO（Direct Preference Optimization）是目前最主流的对齐方法，比 RLHF 更简单、更稳定。

## 一、为什么需要对齐

### 模型的能力 vs 行为
```
能力（预训练）：“模型知道什么”
行为（对齐后）：“模型愿意怎么回答”

问题：
❌ 知道却不肯说："我不能回答这个问题"（过度安全）
❌ 知道却说错了（幻觉）
❌ 知道但不说人话（太学术、难懂）
❌ 知道但说的方式不恰当（不礼貌、啰嗦）
```

### 对齐的目标
```
helpful（有用）+ harmless（无害）+ honest（诚实）= HHH 对齐
```

## 二、DPO 原理

### 与传统 RLHF 的对比
```
RLHF 三步走：
Step 1: 收集偏好数据（A > B > C）
Step 2: 训练奖励模型（Reward Model）
Step 3: PPO 强化学习优化策略

DPO 一步到位：
Step 1: 收集偏好数据（A > B）
Step 2: 直接优化策略（跳过奖励模型）
```

### DPO 的核心公式
```python
# DPO Loss 的直觉
# 最大化好回答相对于差回答的 log 概率比

def dpo_loss(model, ref_model, prompt, chosen, rejected, beta=0.1):
    """
    beta: 偏离参考模型的程度（β越小，越能偏离）
    """
    # 当前模型的对数概率
    chosen_logp = model.log_prob(prompt, chosen)
    rejected_logp = model.log_prob(prompt, rejected)
    
    # 参考模型的对数概率（通常是原始 SFT 模型）
    ref_chosen_logp = ref_model.log_prob(prompt, chosen)
    ref_rejected_logp = ref_model.log_prob(prompt, rejected)
    
    # 隐式的奖励差
    chosen_reward = beta * (chosen_logp - ref_chosen_logp)
    rejected_reward = beta * (rejected_logp - ref_rejected_logp)
    
    # DPO loss
    loss = -torch.log(torch.sigmoid(chosen_reward - rejected_reward)).mean()
    
    return loss
```

## 四、DPO 训练实战

### LLaMA-Factory 配置

## 五、DPO 关键参数调优

### Beta（β）的选择
```
β = 0.01  → 大幅偏离 SFT 模型，激进对齐（数据好时用）
β = 0.1   → 适中偏离，最常用 ← 推荐
β = 0.5   → 保守偏离，数据质量不确定时用
β = 1.0   → 非常保守，几乎不改变行为
```

### 学习率
```
DPO 学习率比 SFT 小得多：
SFT：1e-5 ~ 5e-5
DPO：1e-7 ~ 5e-6  ← 小 10-100 倍

原因：DPO 是在最优策略附近微调，步子太大会破坏已有能力
```

### 数据量
```
最少：50-100 个偏好对（验证效果）
推荐：500-2000 个偏好对（有可见改善）
充足：5000+ 个偏好对（显著对齐提升）
```

## 六、DPO vs RLHF vs SFT

| 维度 | SFT | RLHF | DPO |
|------|-----|------|-----|
| 数据需求 | 指令-回答对 | 偏好排序 + 奖励模型 | 偏好排序 |
| 训练复杂度 | 低 | 高（3步） | 中（2步） |
| 训练稳定性 | 高 | 低（PPO 敏感） | 高 |
| 显存需求 | 1× 模型 | 4× 模型 | 2× 模型 |
| 对齐效果 | 基础 | 最好（上限） | 很好（接近 RLHF） |
| 上手难度 | ⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ |

## 七、常见问题

### DPO 的陷阱
1. **偏好数据不够**：可能对齐了错误的内容
2. **chosen/rejected 差异太小**：模型学不到有效信号
3. **学习率太大**：出现灾难性遗忘
4. **数据 pipeline 脏**：chosen 里混入了低质量
5. **只用单维度偏好**：只有 format 没有 content

### 验证 DPO 效果
