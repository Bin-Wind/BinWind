# Prompt Engineering 提示词工程精要 | 从入门到精通

Prompt Engineering 是用好大模型的最基础技能。好的 prompt 能让效果翻倍，差的 prompt 让最强模型翻车。

## 一、Prompt 基础框架

### 黄金公式
```
好的 Prompt = 角色设定 + 明确任务 + 格式要求 + 示例引导 + 约束条件
```

### 基础模板
```markdown
## 角色
你是一个{专业角色}

## 任务
{明确的任务描述}

## 要求
- {具体约束1}
- {具体约束2}

## 输出格式
{期望的输出格式}

## 示例
输入：{示例输入}
输出：{示例输出}
---
现在开始：
输入：{实际输入}
```

## 四、角色设定

### 高质量角色 Prompt
```markdown
你是一个资深 Python 后端开发工程师，拥有 10 年经验。你的特点是：
1. 用通俗的语言解释技术概念，就像跟一个聪明的新人同事说话
2. 给出实用的代码示例，而不是理论介绍
3. 主动指出常见陷阱和最佳实践
4. 当有多种方案时，对比优劣并给出推荐

在回答时，先给出核心答案，再补充背景知识。
```

## 五、高级技巧

### 1. 分步引导
```markdown
我需要你帮我写一份项目计划。请分步进行：

第1步：先问我 3 个关键问题，澄清项目需求
第2步：根据我的回答，列出计划大纲
第3步：逐步展开每个部分的细节

现在开始，先问第1个问题。
```

### 2. 自我修正
```markdown
请回答以下问题，然后：
1. 检查你的答案是否有事实错误
2. 指出哪些部分可能不确定
3. 如果有不确定的地方，请标注清楚

问题：{question}
```

### 3. 多角度思考
```markdown
从以下三个角度分析这个决策：
1. 🟢 乐观角度：最好的结果是什么？
2. 🔴 悲观角度：最坏的风险是什么？
3. 🔵 中立角度：最可能发生的情况是什么？

决策：{decision}
```

## 七、提示词优化方法

### 迭代优化循环
```python
def optimize_prompt(base_prompt, test_cases, model, iterations=5):
    """通过测试反馈优化 prompt"""
    prompt = base_prompt
    
    for i in range(iterations):
        # 1. 批量测试
        results = []
        for case in test_cases:
            output = model.generate(prompt.format(input=case["input"]))
            results.append({"input": case["input"], "output": output, "expected": case["expected"]})
        
        # 2. 分析失败案例
        failures = [r for r in results if not is_correct(r)]
        print(f"Iteration {i}: {len(failures)}/{len(results)} failures")
        
        if not failures:
            break  # 完美！
        
        # 3. 自动优化
        prompt = auto_improve_prompt(prompt, failures, model)
    
    return prompt
```

## 总结

> Prompt Engineering 的本质是：用清晰的语言，消除模型理解的不确定性。

**记住：好的 Prompt 是迭代出来的，不是一次写成的。测试 → 分析失败 → 修改 → 再测试。**
