# 数据标注全流程指南 | 从手工到自动化

数据标注是监督学习的核心环节。标注质量直接决定模型效果，本文将系统讲解标注方案的全流程。

## 一、标注策略选择

| 策略 | 说明 | 成本 | 质量 |
|------|------|------|------|
| 人工标注 | 专业标注团队逐条标注 | 高 | 最高 |
| 众包标注 | 通过平台分发任务 | 中 | 中 |
| 模型预标注+人工复核 | 先用模型打标，人工修正 | 中低 | 高 |
| 自动标注 | 用大模型（如 GPT-4）生成标签 | 低 | 中高 |
| 弱监督 | 用规则/关键词自动生成标签 | 最低 | 低 |

## 二、标注规范制定

### 标注文档模板
```markdown
## 标注任务：情感分类

### 任务描述
判断给定文本的情感倾向。

### 标签定义
- 正面：表达积极、满意、喜悦等正向情绪
- 负面：表达消极、不满、愤怒等负向情绪
- 中性：客观陈述，无明显情感倾向

### 标注示例
| 文本 | 标签 | 说明 |
|------|------|------|
| 这家店的服务太好了！ | 正面 | 明确的正向情感 |
| 等了半小时还没上菜 | 负面 | 明显的负面体验 |
| 今天是周三 | 中性 | 客观事实陈述 |

### 边界情况处理
- 反讽语句："真是太棒了（表情🙄）" → 负面
- 混合情感："菜好吃但服务差" → 以主要情感为准，标注为负面
- 条件句："如果能改进就更好了" → 中性偏正面，建议标注为中性

### 标注质量要求
- 每条文本至少 2 人标注
- 一致性低于 90% 需第三人仲裁
- 每日标注量不超过 200 条（防止疲劳）
```

## 四、标注质量控制

### 1. 预标注 + 人工复核
```python
from openai import OpenAI

client = OpenAI()

def pre_label(texts: List[str], task: str) -> List[Dict]:
    """用大模型预标注，人工再复核"""
    results = []
    for text in texts:
        resp = client.chat.completions.create(
            model="gpt-4o-mini",  # 用便宜的模型预标注
            messages=[{
                "role": "system",
                "content": f"你是一个标注员。任务：{task}。请只返回标注结果，不要解释。"
            }, {
                "role": "user",
                "content": text
            }]
        )
        results.append({
            "text": text,
            "pre_label": resp.choices[0].message.content,
            "status": "pending_review"
        })
    return results
```

### 2. 一致性检查
```python
def calc_agreement(annotator_a: List[str], annotator_b: List[str]) -> float:
    """计算标注一致率"""
    if len(annotator_a) != len(annotator_b):
        raise ValueError("长度不一致")
    agree = sum(1 for a, b in zip(annotator_a, annotator_b) if a == b)
    return agree / len(annotator_a)

# 一致率低于阈值 → 需要重标或仲裁
```

### 3. 黄金标准测试
- 在标注数据中混入 5%-10% 已知答案的"黄金条目"
- 标注者对这些条目的准确率应 > 95%
- 低于阈值 → 需要重新培训

## 五、标注数据管理

```python
import json
from datetime import datetime

class AnnotationManager:
    """标注数据管理器"""
    
    def __init__(self, db_path: str):
        self.db_path = db_path
    
    def export_to_alpaca(self, annotations: List[Dict]) -> List[Dict]:
        """导出为 Alpaca 训练格式"""
        return [{
            "instruction": ann["instruction"],
            "input": ann.get("input", ""),
            "output": ann["output"]
        } for ann in annotations if ann.get("status") == "approved"]
    
    def export_to_sharegpt(self, annotations: List[Dict]) -> List[Dict]:
        """导出为 ShareGPT 对话格式"""
        return [{
            "conversations": ann["conversations"]
        } for ann in annotations if ann.get("status") == "approved"]
    
    def get_statistics(self, annotations: List[Dict]) -> Dict:
        """标注进度统计"""
        total = len(annotations)
        return {
            "总数": total,
            "已完成": sum(1 for a in annotations if a["status"] == "done"),
            "已审核": sum(1 for a in annotations if a["status"] == "approved"),
            "待修改": sum(1 for a in annotations if a["status"] == "revision"),
            "完成率": f"{sum(1 for a in annotations if a['status'] == 'approved') / total * 100:.1f}%"
        }

# 使用示例
manager = AnnotationManager("./annotations.db")
stats = manager.get_statistics(all_annotations)
print(json.dumps(stats, ensure_ascii=False, indent=2))
```

## 六、成本控制

| 环节 | 省钱方法 |
|------|---------|
| 数据选择 | 优先标注模型最不确定的样本（主动学习） |
| 预标注 | 用 GPT-4o-mini 或 Qwen 等低成本模型先打标 |
| 分批标注 | 先标 200 条训练，看效果再决定是否追加 |
| 复用数据 | 一个标注数据集适配多个相近任务 |

## 总结

> 好的数据标注 = 清晰规范 + 合适工具 + 严格质控 + 持续迭代

标注不是一次性工作，而是与模型训练交替进行的循环优化过程。
