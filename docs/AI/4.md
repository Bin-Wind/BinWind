## 三、按场景选型

### 场景 1：微调后部署到业务
```
推荐：Qwen2.5-7B / Llama-3.1-8B
理由：7B 级别在效果与成本间最佳平衡
      LoRA 微调只需 24GB 显存
      推理延迟低，适合在线服务
```

### 场景 2：中文知识库问答
```
推荐：Qwen2.5-14B / DeepSeek-V2-Lite
理由：中文理解能力强
      14B 级别在 RAG 场景性价比最优
```

### 场景 3：代码生成助手
```
推荐：DeepSeek-Coder-V2 / CodeQwen1.5
理由：专为代码优化
      FIM（Fill-in-the-Middle）能力强
```

### 场景 4：轻量边缘部署
```
推荐：Qwen2.5-1.5B / Gemma-2B
理由：可在手机、IoT 设备运行
      量化后仅需 1-2GB 内存
```

## 四、选型 Checklist

```markdown
## 功能性需求
- [ ] 任务类型：生成 / 分类 / 抽取 / 对话 / 代码？
- [ ] 输入输出长度要求？（长文本 → 选择支持大窗口的模型）
- [ ] 需要多语言支持吗？
- [ ] 需要多模态（图片、语音）吗？

## 非功能性需求
- [ ] 延迟要求？（实时场景 → 小模型 + vLLM 加速）
- [ ] 并发量？（高并发 → 小模型 + 多实例）
- [ ] 可用显存？（<24GB → QLoRA 微调 7B）
- [ ] 数据安全要求？（数据不出域 → 开源模型本地部署）
- [ ] 授权许可？（商用 → 检查开源协议）

## 成本预算
- [ ] API 调用量预估？（高调用量 → 自部署更经济）
- [ ] 训练预算？（全量微调 vs LoRA，成本差 10 倍）
- [ ] 运维成本？（自建 vs 云服务）
```

## 五、模型评测方法

### 快速 A/B 测试
```python
def quick_benchmark(models: List[str], test_cases: List[Dict]) -> Dict:
    """对多个模型进行快速对比评测"""
    results = {}
    for model_name in models:
        scores = []
        for case in test_cases:
            # 调用模型获取输出
            output = call_model(model_name, case["input"])
            # 评分（可用 GPT-4 做 judge）
            score = evaluate_output(output, case["expected"])
            scores.append(score)
        results[model_name] = {
            "平均分": sum(scores) / len(scores),
            "最高分": max(scores),
            "最低分": min(scores),
        }
    return results
```

## 六、决策树

```
开始
│
├─ 是否需要多模态？ → 是 → GPT-4o / Qwen-VL / InternVL
│
├─ 是否必须本地部署？ 
│  ├─ 是 → 显存 < 24GB？
│  │  ├─ 是 → Qwen2.5-7B / Llama-8B + QLoRA
│  │  └─ 否 → Qwen2.5-72B / Llama-70B + LoRA
│  └─ 否 → 调用 API
│     ├─ 中文为主 → Qwen-Max / DeepSeek API
│     └─ 英文为主 → GPT-4o / Claude API
│
└─ 是否需要代码能力？ → 是 → DeepSeek-Coder / CodeQwen
```

## 总结

> 选模型没有银弹，合适的才是最好的。

**黄金法则：先用 API 验证可行性，后用开源模型降低成本。不要一上来就训练，先看看 prompt 能不能解决。**
