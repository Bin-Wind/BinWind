# 模型训练关键超参数详解 | 调参的科学与艺术

超参数是模型训练的"方向盘"。理解每个参数的作用，才能高效地调出好模型。

## 一、学习率（Learning Rate）

学习率是最重要的超参数，直接决定训练能否收敛。

### 推荐范围
```
预训练（从头训练）：1e-4 ~ 5e-4
全量微调：           1e-5 ~ 5e-5
LoRA 微调：          1e-4 ~ 5e-4
QLoRA 微调：         2e-4 ~ 5e-4
```

### 学习率调度策略
```python
from transformers import get_cosine_schedule_with_warmup, get_linear_schedule_with_warmup

# Cosine 衰减（最常用）
scheduler = get_cosine_schedule_with_warmup(
    optimizer,
    num_warmup_steps=100,     # warmup 步数
    num_training_steps=1000    # 总训练步数
)

# 线性衰减
scheduler = get_linear_schedule_with_warmup(
    optimizer,
    num_warmup_steps=100,
    num_training_steps=1000
)

# Constant（恒定 lr）适合短期微调
```

### 学习率诊断
```python
# 学习率查找器（Learning Rate Finder）
def lr_finder(model, train_loader, start_lr=1e-8, end_lr=1e-1):
    """找到合适的学习率范围"""
    lrs = []
    losses = []
    lr = start_lr
    
    for batch in train_loader:
        # 指数增长 lr
        lr *= (end_lr / start_lr) ** (1 / len(train_loader))
        loss = train_one_batch(model, batch, lr)
        
        lrs.append(lr)
        losses.append(loss)
        
        # loss 开始剧烈上升时停止
        if loss > min(losses) * 4:
            break
    
    # 最佳 lr：loss 下降最快的点 ÷ 10
    return lrs, losses
```

## 二、批次大小（Batch Size）

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 全量微调 7B | 4-16 | 受显存限制 |
| LoRA 微调 7B | 8-32 | LoRA 参数少，可以大一些 |
| QLoRA 微调 7B | 16-64 | 量化后更省显存 |
| 预训练 | 越大越好 | 受硬件约束 |

### 梯度累积
小显存的救星：
```python
# 目标 batch_size = 64，但显存只够 batch_size = 4
per_device_train_batch_size = 4
gradient_accumulation_steps = 16   # 4 × 16 = 64

# 效果等同于 batch_size = 64，但更省显存
```

## 三、训练轮数（Epochs）

```
微调推荐：2-5 epochs
少量数据（<1000条）：3-5 epochs
大量数据（>10万条）：1-2 epochs
```

### 早停（Early Stopping）
```python
from transformers import EarlyStoppingCallback

# 验证 loss 连续 3 轮不降就停止
trainer = Trainer(
    model=model,
    args=training_args,
    callbacks=[EarlyStoppingCallback(early_stopping_patience=3)],
    # ...
)
```

## 四、LoRA 专有参数

```python
# LLaMA-Factory / PEFT 配置
lora_config = {
    "r": 8,              # rank：越大容量越大，8~64 常用
    "lora_alpha": 16,    # 缩放系数，通常 = r × 2
    "lora_dropout": 0.05,# 丢弃率，0.05~0.1
    "target_modules": [  # 插入 LoRA 的模块
        "q_proj", "k_proj", "v_proj", "o_proj",  # Attention
        "gate_proj", "up_proj", "down_proj"       # FFN
    ],
    "task_type": "CAUSAL_LM"
}
```

### Rank 选择指南
| Rank | 适用场景 |
|------|---------|
| r=4 | 简单分类、风格调整 |
| r=8 | 指令微调（最常用） |
| r=16 | 领域知识注入 |
| r=32-64 | 复杂任务、追求效果上限 |
| r=128 | 几乎等效全量微调（但参数更少） |

## 五、优化器

```python
# AdamW（最主流）
from transformers import AdamW
optimizer = AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)

# SGD with Momentum（某些场景更稳定）
optimizer = torch.optim.SGD(model.parameters(), lr=1e-3, momentum=0.9)

# AdaFactor（省显存，T5/LLaMA 预训练用）
from transformers import Adafactor
optimizer = Adafactor(model.parameters(), scale_parameter=True, relative_step=True)

# Lion（新秀，收敛更快）
from lion_pytorch import Lion
optimizer = Lion(model.parameters(), lr=1e-4, weight_decay=1e-2)
```

## 七、调参方法论

```python
# 调参优先级（从最重要到最不重要）
"""
1. 学习率          → 先找到合适的量级
2. 训练轮数        → 观察 loss 曲线，避免过拟合
3. Batch Size      → 在显存允许范围内尽量大
4. LoRA Rank       → 8 起步，有效果瓶颈再加大
5. Warmup          → 通常 5%-10% 总步数
6. Weight Decay    → 默认 0.01 通常够用
7. Dropout         → 有明显过拟合再加
"""
```

## 总结

> 调参不是玄学，是实验科学。每次只改一个变量，记录实验结果，逐步逼近最优。

**新手推荐：直接用 LLaMA-Factory 默认参数跑 baseline，效果不满意再微调。**
