# 模型评估指标与基准测试 | 如何科学评价模型好坏

模型训练完成了，但它到底好不好？本文系统讲解大模型的评估方法论。

## 一、评估维度全景

```
模型评估
├── 自动评估（客观指标）
│   ├── 生成质量：Perplexity、BLEU、ROUGE
│   ├── 分类准确率：Accuracy、F1、AUC
│   └── 推理能力：基准数据集得分
├── 人工评估（主观指标）
│   ├── 单模型评分：Likert 1-5 分
│   ├── A/B 对比：哪个回答更好？
│   └── 盲测：消除品牌偏见
└── 业务评估（最终标准）
    ├── 用户采纳率
    ├── 任务完成率
    └── 用户满意度（NPS）
```

## 二、核心评估指标

### 1. Perplexity（困惑度）
衡量模型对文本的"熟悉程度"，越低越好。

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

def calc_perplexity(model, tokenizer, text: str) -> float:
    """计算单条文本的困惑度"""
    inputs = tokenizer(text, return_tensors="pt")
    with torch.no_grad():
        outputs = model(**inputs, labels=inputs["input_ids"])
        loss = outputs.loss
        perplexity = torch.exp(loss)
    return perplexity.item()

# 典型值参考：
# GPT-2: ~30-40
# LLaMA-7B: ~8-12
# GPT-4: ~3-5（估计值）
```

### 3. LLM-as-Judge（用模型评估模型）
```python
from openai import OpenAI

client = OpenAI()

def gpt_judge(prompt: str, response: str, criteria: str = "准确性和完整性") -> dict:
    """用 GPT-4 打分"""
    system_prompt = f"""你是一个严格的评审专家。请按以下标准给回答评分：
标准：{criteria}
评分范围：1-5 分（1=极差，5=完美）
输出 JSON：{{"score": X, "reason": "评分理由"}}"""
    
    resp = client.chat.completions.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": f"问题：{prompt}\n回答：{response}"}
        ],
        response_format={"type": "json_object"}
    )
    return eval(resp.choices[0].message.content)
```

```python
def build_test_set(annotations: List[Dict], n_per_category=50) -> List[Dict]:
    """构建分类覆盖的测试集"""
    categories = {}
    for item in annotations:
        cat = item.get("category", "general")
        if cat not in categories:
            categories[cat] = []
        categories[cat].append(item)
    
    test_set = []
```

## 七、评估注意事项

1. **测试集绝对不能被训练**：数据泄露等于作弊
2. **不要只看一个指标**：综合 BLEU + 人工评估 + 业务指标
3. **基准测试只是参考**：通用基准高 ≠ 你的场景好
4. **持续评估**：每次迭代都要和上一版对比
5. **建立回归测试集**：确保新版本不会搞坏已有能力

## 总结
