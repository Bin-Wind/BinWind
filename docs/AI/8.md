# vLLM 高性能推理部署 | 从模型到 API 服务

vLLM 是目前最主流的大模型推理引擎，用 PagedAttention 技术实现高吞吐低延迟。

## 一、为什么选 vLLM

| 特性 | 说明 |
|------|------|
| PagedAttention | 类似操作系统虚拟内存，KV Cache 利用率接近 100% |
| 连续批处理 | 动态合并请求，吞吐量提升 10-20 倍 |
| 量化支持 | 原生 AWQ、GPTQ 量化模型 |
| 兼容性 | 兼容 Hugging Face 模型，零改动部署 |
| OpenAI 兼容 API | 可直接替换 OpenAI SDK 的后端 |

## 二、安装与环境

```bash
# 安装 vLLM
pip install vllm

# 验证安装
python -c "import vllm; print(vllm.__version__)"
```

## 三、快速上手

### 启动 API 服务
```bash
# 基础启动
vllm serve Qwen/Qwen2.5-7B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --max-model-len 4096

# 生产级配置
vllm serve Qwen/Qwen2.5-7B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.90 \
  --max-num-seqs 256 \
  --tensor-parallel-size 1 \
  --dtype auto \
  --enable-prefix-caching
```

### 调用 API
```python
from openai import OpenAI

# 使用 OpenAI SDK 调用 vLLM
client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="not-needed"  # vLLM 默认不需要 API key
)

response = client.chat.completions.create(
    model="Qwen/Qwen2.5-7B-Instruct",
    messages=[
        {"role": "user", "content": "用三句话介绍一下杭州"}
    ],
    temperature=0.7,
    max_tokens=500,
    stream=True
)

for chunk in response:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
```

## 四、核心参数详解

### 显存相关
```
--gpu-memory-utilization 0.90    # GPU 显存利用率，建议 0.85-0.92
--max-model-len 4096             # 最大序列长度（输入+输出），该值越大显存占用越高
--max-num-seqs 256               # 最大并发请求数
--max-num-batched-tokens 8192    # 单批最大 token 数
```

### 多卡并行
```
--tensor-parallel-size 2         # 张量并行（模型拆分到 2 张卡），适合 70B 模型
--pipeline-parallel-size 1       # 流水线并行（模型按层拆分），一般用默认 1
```

### 量化加载
```bash
# AWQ 量化模型
vllm serve Qwen/Qwen2.5-7B-Instruct-AWQ \
  --quantization awq

# GPTQ 量化模型
vllm serve TheBloke/Llama-2-7B-GPTQ \
  --quantization gptq
```

### 性能优化
```bash
# 开启 prefix caching（适合 RAG 场景，相同 system prompt 可复用 KV Cache）
--enable-prefix-caching

# Chunked Prefill（将长 prompt 分块处理，降低首 token 延迟）
--enable-chunked-prefill

# Speculative Decoding（投机解码，提升吞吐量）
--speculative-model [draft-model-path]
```

## 五、Python 脚本直接使用

```python
from vllm import LLM, SamplingParams

# 初始化模型
llm = LLM(
    model="Qwen/Qwen2.5-7B-Instruct",
    max_model_len=4096,
    gpu_memory_utilization=0.90,
    trust_remote_code=True,
)

# 生成参数
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.9,
    max_tokens=512,
    stop=["</s>", "<|im_end|>"],
)

# 批量推理
prompts = [
    "介绍一下深度学习",
    "Python 的 GIL 是什么？",
    "如何用 Docker 部署服务？",
]

outputs = llm.generate(prompts, sampling_params)

for output in outputs:
    print(f"Prompt: {output.prompt}")
    print(f"Response: {output.outputs[0].text}")
    print("-" * 50)
```

## 六、生产部署架构

```
                    ┌──────────────┐
                    │   Nginx      │ (反向代理 + 限流)
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        ┌─────▼─────┐ ┌───▼────┐ ┌────▼─────┐
        │ vLLM-8000 │ │vLLM-8001│ │vLLM-8002 │ (多实例)
        │  GPU 0    │ │ GPU 1  │ │ GPU 2    │
        └───────────┘ └────────┘ └──────────┘
```

### Nginx 负载均衡
```nginx
upstream vllm_backend {
    least_conn;
    server 127.0.0.1:8000 max_fails=3 fail_timeout=30s;
    server 127.0.0.1:8001 max_fails=3 fail_timeout=30s;
    server 127.0.0.1:8002 max_fails=3 fail_timeout=30s;
}

server {
    listen 80;
    
    location /v1/ {
        proxy_pass http://vllm_backend;
        proxy_set_header Host $host;
        proxy_read_timeout 300s;    # 长连接超时
        proxy_buffering off;         # 流式输出需要关闭缓冲
        
        # 限流
        limit_req zone=api_limit burst=50 nodelay;
    }
}

limit_req_zone $binary_remote_addr zone=api_limit:10m rate=100r/s;
```


## 八、常见问题

| 问题 | 解决方案 |
|------|---------|
| OOM（显存不足） | 降低 `gpu-memory-utilization` 或 `max-model-len` |
| 请求排队太久 | 增加 `max-num-seqs` 或部署多实例 |
| 首 token 延迟高 | 开启 chunked-prefill |
| 吞吐量不理想 | 增大 batch size、启用 prefix caching |
| AWQ 模型加载失败 | 确认 `pip install autoawq` 已安装 |

## 总结

> vLLM = 最高效的大模型推理引擎，没有之一。

**一句话部署：`vllm serve model-name --port 8000` 就有一个 OpenAI 兼容的 API 了。**
