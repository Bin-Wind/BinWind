# 模型监控与持续迭代方案 | 上线不是终点

模型上线只是起点。线上表现、数据回流、持续迭代，才是真正的持久战。

## 一、监控体系架构

```yaml
监控层:
  基础设施层:
    - GPU 利用率、显存、温度
    - 请求 QPS、延迟 P50/P95/P99
    - 错误率、超时率
  
  模型质量层:
    - 输出长度分布（是否有漂移）
    - 回复拒绝率（模型是不是"不想回答"）
    - Token 生成速度（tokens/s）
    - 输入/输出 Token 比例
  
  业务指标层:
    - 用户点赞/点踩率
    - 任务完成率
    - 对话轮次分布
    - 用户留存率
```

## 二、日志系统搭建

### 请求日志格式
```python
import json
import time
from typing import Optional

class InferenceLogger:
    """推理日志记录器"""
    
    @staticmethod
    def log_request(
        request_id: str,
        prompt: str,
        response: str,
        model_version: str,
        latency_ms: int,
        input_tokens: int,
        output_tokens: int,
        user_id: Optional[str] = None,
        feedback: Optional[str] = None
    ):
        log_entry = {
            "timestamp": int(time.time()),
            "request_id": request_id,
            "model_version": model_version,
            "user_id": user_id,
            "input_tokens": input_tokens,
            "output_tokens": output_tokens,
            "latency_ms": latency_ms,
            "prompt_snippet": prompt[:200],  # 只存摘要用于分析
            "response_snippet": response[:200],
            "feedback": feedback,
        }
        
        # 写入日志文件或数据库
        with open("inference.log", "a") as f:
            f.write(json.dumps(log_entry, ensure_ascii=False) + "\n")
```

```python
    active_requests.inc()
    try:
        result = model.generate(prompt)
        token_counter.labels(type="output").inc(count_tokens(result))
        return result
    finally:
        active_requests.dec()
```

### 输出分布漂移检测
```python
import numpy as np
from scipy import stats

def detect_distribution_shift(
    reference_avg_len: float,
    reference_std_len: float,
    current_lengths: List[int],
    threshold: float = 0.05
) -> bool:
    """
    检测输出长度分布是否发生显著漂移
    使用 t 检验判断
    """
    current_mean = np.mean(current_lengths)
    t_stat, p_value = stats.ttest_1samp(current_lengths, reference_avg_len)
    return p_value < threshold  # p < 0.05 表示有显著漂移
```

## 五、数据回流

### 高质量数据收集
```python
def collect_quality_data(requests: List[Dict]) -> List[Dict]:
    """从线上请求中筛选高质量训练数据"""
    quality_data = []
    
    for req in requests:
        score = 0
        
        # 用户明确点赞
        if req.get("feedback") == "thumbs_up":
            score += 3
        
        # 对话轮次多（说明用户持续使用）
        if req.get("turns", 1) >= 3:
            score += 1
        
        # 回复长度适中（不太短不冗长）
        resp_len = len(req.get("response", ""))
        if 100 < resp_len < 2000:
            score += 1
        
        # 响应正常（没有报错 / 拒绝）
        if not req.get("is_refusal") and not req.get("error"):
            score += 1
        
        if score >= 4:  # 高质量阈值
            quality_data.append({
                "instruction": req["prompt"],
                "output": req["response"]
            })
    
    return quality_data
```

## 六、迭代流程

### 持续微调流水线
```
线上观察（2周）
  │
  ├── 收集用户反馈（点赞/点踩/举报）
  ├── 分析 Bad Case（错误类型、分布）
  └── 筛选高质量对话数据
  │
  ▼
数据标注与清洗（1周）
  │
  ├── Bad Case 人工分析
  ├── 高质量数据标注复核
  └── 补充边缘场景数据
  │
  ▼
模型重新微调（3-5天）
  │
  ├── 合并新旧数据集
  ├── LoRA/全量微调
  └── 离线评估达标
  │
  ▼
灰度验证（1周）
  │
  ├── 5% 流量 → 监控各项指标
  ├── 20% 流量 → A/B 对比
  └── 50% 流量 → 确认稳定性
  │
  ▼
全量上线
```

### A/B 实验框架
```python
import hashlib

def ab_routing(user_id: str, experiment: str) -> str:
    """基于用户 ID 的 A/B 分流"""
    hash_val = int(hashlib.md5(f"{experiment}:{user_id}".encode()).hexdigest(), 16)
    bucket = hash_val % 100
    
    if bucket < 50:
        return "control"   # 对照组（旧模型）
    else:
        return "treatment" # 实验组（新模型）

# 使用
model_version = ab_routing(user_id="user_123", experiment="finetune_v2")
```

```python
                "deployed_at": "2026-02-20",
                "metrics": {"accuracy": 0.87, "latency_p99": 1350}
            },
            "v3.0-beta": {
                "path": "/models/qwen-7b-finetuned-v3",
                "status": "staging",
                "deployed_at": "2026-03-01",
                "metrics": {"accuracy": 0.89, "latency_p99": 1400}
            },
        }
    
    def rollback(self) -> str:
        """回滚到上一个生产版本"""
        return self.versions["v1.0"]["path"]
    
    def get_production_model(self) -> str:
        for version, info in self.versions.items():
            if info["status"] == "production":
                return info["path"]
```
