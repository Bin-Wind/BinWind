# 版本控制 GUI：Fork / Sourcetree / GitKraken 怎么选？

命令行 Git 是开发者的基本功，但有些操作在命令行里实在费眼——比如查看分支图谱、逐行暂存改动、可视化解决合并冲突，这些场景 GUI 工具的效率和体验远超命令行。

本文对比三款主流 Git GUI 客户端，帮你找到最适合自己的那一款。

## 为什么需要 Git GUI

坦白讲，日常的 `git commit`、`git push`、`git pull`、创建分支，命令行完全够用，甚至更快。但以下场景，GUI 是降维打击：

- **可视化分支历史**：看到一张彩色的分支图谱，比在终端里翻 `git log --graph` 直观一百倍。谁合并了谁、谁分叉了多久，一目了然。
- **逐行暂存（Stage Hunks）**：GUI 可以只暂存文件中的某几行改动，而不是整个文件。这意味着你可以把一个文件里的改动拆成多个语义清晰的 commit，命令行做这件事非常痛苦。
- **合并冲突可视化**：左右两栏对比、一键选择保留哪边、甚至直接在中间编辑最终结果——比在代码里手动找 `<<<<<<<` 标记优雅太多。
- **交互式 rebase**：拖拽排序 commit、合并、修改 commit message，GUI 上操作是真正的所见即所得。

## Fork

Fork 是我目前的主力 Git GUI。它是 macOS 和 Windows 原生应用，启动飞快，界面干净，交互流畅。

**核心功能：**

- **交互式 Rebase**：直接拖拽 commit 来重新排序，右键选择 squash、edit、reword，全程可视化。不再需要对着 `pick` `squash` `fixup` 列表脑补结果。
- **Blame 视图**：每行代码标注最后修改者和时间，带热度图颜色（越近的改动颜色越深），快速定位代码责任人。
- **Stash 浏览器**：可以像翻书一样浏览每个 stash 的内容，不用 `git stash show -p` 一个字节一个字节地看。
- **Submodule 支持**：子模块的状态、更新、切换一目了然，不需要频繁输入 `git submodule update --init --recursive`。
- **Dark Theme**：自带的暗色主题质量很高，夜间写代码不刺眼。

**价格**：\$59.99 一次性买断。免费评估版无时间限制，但会不定时弹窗提醒购买——相当于有干扰的免费版。

**适合谁**：想要快速、简洁、永久授权的个人开发者。尤其适合 macOS 和 Windows 用户。

**不足**：Linux 不支持；持续买断模式意味着大版本升级可能需要重新付费（历史上有过）；团队协作功能较弱。

## Sourcetree

Sourcetree 是 Atlassian 出品的免费 Git GUI，历史悠久，用户基数大。

**核心功能：**

- 完整的 Git 操作覆盖，分支管理、合并、stash、cherry-pick 等一应俱全
- 与 Bitbucket 深度集成，支持 Jira issue 关联
- 内置 Git Flow 和 Hg Flow 支持
- 交互式 rebase 也有可视化界面

**谁在用**：Atlassian 生态用户、预算为零的开发者。

**值得吐槽的地方**：Sourcetree 基于 Electron（其实是嵌入了一个浏览器运行时），启动慢、内存占用高。在大型仓库（几万次 commit 以上）拉取完整历史时经常卡顿。UI 风格停留在 2015 年，和现代 macOS 应用格格不入。另外需要注册 Atlassian 账号才能使用，不是完全离线。

**一句话总结**：功能全面、不要钱，但体验粗糙。如果用 Bitbucket 工作流，勉强算是加分项。否则 Fork 的免费评估版都更舒服。

## GitKraken

GitKraken 是三款中最「好看」的，UI 设计精美，交互生动，对新手友好。

**核心功能：**

- **精美的提交图谱**：彩色分支线、commit 节点可点击展开详情、拖拽操作，视觉呈现业界最佳。
- **内置合并冲突编辑器**：三栏视图（ours / output / theirs），点击即选，实时预览合并结果。
- **GitKraken Boards**：看板功能，可以关联 commit 到卡片，做简单的项目管理。
- **团队功能**：共享 Profile、团队看板、协作编辑 commit message 等。
- **跨平台**：Mac / Windows / Linux 全部支持，体验一致。

**价格**：订阅制，Pro 版 \$4.95/月。免费版仅支持公开仓库。

**不足**：同样是 Electron 应用，重量级选手——启动慢、吃内存。订阅制对于不常用的用户来说成本不低。而且所有操作需要账号登录，数据同步到 GitKraken 服务器（即便是本地仓库也会收集使用数据）。隐私敏感的用户要注意。

**适合谁**：视觉学习者、Git 新手、需要跨平台统一体验的团队。

## Git CLI + Git GUI：最佳实践

与其争论 GUI 还是 CLI，不如一起用。我推荐的分工是：CLI 负责日常操作（commit / push / pull / branch），GUI 负责复杂操作（交互式 rebase、冲突解决、历史浏览）。这样既保持了命令行的速度和可脚本化能力，又在真正需要可视化的时候不折磨自己。

日常我同时开着 VS Code 里的 Git Graph 扩展（轻量，在侧栏就能看分支图）和 Fork（需要交互式操作时切过去）。

## 轻量级替代品

如果你只是想快速看一眼分支图，不必打开一个完整的 GUI：

- **Git Graph（VS Code 扩展）**：在 VS Code 侧栏显示分支图谱，点击 commit 看详情，右键操作，非常轻量。免费且和编辑器零距离。
- **tig**：终端里的 Git 浏览器。`brew install tig` 之后在仓库里输入 `tig`，就能用 vim 式快捷键浏览 log、diff、blame。纯文本，但比原生命令好太多。

## 选购建议

| 场景 | 推荐 |
|---|---|
| Mac/Windows 个人开发，追求爽快 | **Fork**（一次性付费，永久使用） |
| 预算有限，零成本 | **Sourcetree**（免费，忍忍体验） |
| 团队协作、视觉需求 | **GitKraken**（订阅制，UI 最好看） |
| 任何场景的底线 | **先学会 Git CLI**，GUI 只是加速器 |

最后一句忠告：不要因为用了 GUI 就放弃学习 Git 原理。看懂 reflog、理解 detached HEAD、知道 merge 和 rebase 的区别——这些知识在 GUI 坏掉或不在身边的时候会救你一命。
