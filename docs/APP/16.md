# 跨平台文件传输方案：从 AirDrop 到 LocalSend，选对工具不再折腾

文件传输似乎是个已解决的问题，但问问自己：iPhone 拍了一段 4K 视频想传到 Windows 电脑，同事的 Android 手机想发个 APK 安装包给你的 Mac，或者你要把一个 30GB 的文件夹发给外地的客户——这些场景你真的能快速搞定吗？邮件附件有限制、微信会压缩、U 盘要翻箱倒柜、云盘还要上传下载两步走。

本文梳理主流的跨平台文件传输方案，按场景匹配最优解。

## 问题的本质

文件传输的瓶颈从来不是单个工具，而是四个维度的冲突：平台兼容性、文件大小限制、传输速度、操作步骤数。理想的方案应该是：不分平台、不限大小、本地网络速度、两步以内完成。目前没有哪个方案完美覆盖所有维度，所以需要按场景选择。

## AirDrop：苹果生态的黄金标准

AirDrop 是苹果设备之间的零配置传输方案。不需要设置、不需要密码、不需要 Wi-Fi 密码——只要两台设备靠得足够近，选中文件 → 点击分享 → 选接收者，几秒钟完成。底层使用 Wi-Fi Direct + 蓝牙发现，速度可达几百 Mbps。

**优点**：零配置、速度快、端到端加密、不限文件大小、支持所有文件类型。体验好到让 Android 和 Windows 用户羡慕。

**缺点**：只支持苹果设备。iPhone 传 Android 不行，iPhone 传 Windows 不行。这个「围墙花园」是故意的。

## LocalSend：开源版 AirDrop，真正的全平台

LocalSend 是目前最好的 AirDrop 替代方案。开源、免费、无广告、不上传任何数据到云端。

**原理**：和 AirDrop 一样，依赖本地局域网（Wi-Fi 或热点）在设备之间直传。不需要互联网连接，不需要注册账号。数据在局域网内传输，端到端加密。

**支持的平台**：iPhone、Android、Windows、macOS、Linux——全部都有客户端。这意味着你终于可以：
- iPhone ↔ Android 互传照片
- Mac ↔ Windows 传文件直接秒到
- Linux 机器也能加入传文件群聊

**体验**：打开 App，自动扫描局域网内其他设备，选中文件，点发送，接收端确认。简洁到可以教爸妈用。没有文件大小限制（取决于你硬盘空间和网络稳定性）。

**局限**：两台设备必须在同一局域网（连同一个 Wi-Fi 或其中一台开热点）；不在同一个网络就用不了——这是所有本地传输工具的共同局限。

**一句话评价**：如果你只装一个跨平台文件传输工具，装 LocalSend。

## Snapdrop：浏览器就能传，零安装

Snapdrop 更进一步——连 App 都不需要装。两台设备在同一个局域网时，分别用浏览器打开 snapdrop.net，就能看到对方，拖拽文件即可传输。页面极简，功能单一纯粹。

**优点**：零安装，手机、平板、电脑、甚至访客的陌生设备都能传。基于 WebRTC，文件不经过服务器，P2P 直传。

**缺点**：
- **需要互联网加载页面**：snapdrop.net 是远程托管的网页，首次打开必须有网络连接才能加载页面本身（加载完后传输走局域网）
- **不够稳定**：有时候设备互相发现不了，需要刷新页面几次
- **不能传文件夹**：只能传单个文件，文件夹需要先压缩

**改进版**：PairDrop（pairdrop.net）是 Snapdrop 的 fork，增加了设备配对、断点续传、临时公网传输等特性，稳定性也略好。如果要用浏览器传文件，优先选 PairDrop。

## NearDrop：让 Android 伪装成 AirDrop

NearDrop 是一个有趣的「单向桥接」方案：在 Android 手机上安装 NearDrop 后，你的 Mac 会在 AirDrop 发送列表中看到这台 Android 设备。选它、发送、Android 端接收。仅支持 Mac → Android 方向，不能反向。

**适用场景**：主力 Mac + 备用 Android 的双机用户。有一定实用价值，但不如直接装 LocalSend 省事。

## 云端传输方案：不需要在同一网络

本地传输很方便——但前提是两台设备在同一个地方。如果要把文件传给远程的对方，或者你和接收者不在同一个城市，就只能走云端。

**Wormhole（magic-wormhole）**

命令行工具，极客最爱。安装：`brew install magic-wormhole`（Mac）或 `pip install magic-wormhole`。发送文件时生成一个一次性短语（如 `7-guitarist-revenge`），接收者输入相同短语，文件通过中继服务器端到端加密传输。不需要公网 IP、不需要注册、不依赖云盘。适合开发者之间互传敏感文件。

**Swiss Transfer**

云端传输的宝藏工具。免费、无需注册、单次最多 50GB（没错，50 个 G）、文件保留 30 天。网页操作、自动生成分享链接。界面设计精美（瑞士人做的）。适合一次性发大文件给不熟悉技术的人。

**WeTransfer**

老牌免费传输工具，知名度最高。免费版 2GB 上限、7 天保留期、邮箱发送。付费版 WeTransfer Pro：200GB 上限、密码保护、自定义过期时间。适合偶尔发个 PDF 或设计稿给客户。

**Send Anywhere**

韩国公司出品，跨平台 App。6 位数字密钥分享（类似 wormhole 但带图形界面），文件保留 48 小时。免费版 10GB 上限。优点是对方不需要装 App，浏览器输入密钥即可下载。适合经常需要跨设备传文件的轻度用户。

## 场景速查表

| 场景 | 最佳方案 | 理由 |
|---|---|---|
| iPhone ↔ Mac | **AirDrop** | 体验完美，没理由不用 |
| iPhone ↔ Android | **LocalSend** | 全平台、免费、快 |
| Mac ↔ Windows 同一网络 | **LocalSend** | 不需要 U 盘 |
| 一次性大文件给远程客户 | **Swiss Transfer** | 50GB 免费、30 天 |
| 开发者之间传敏感代码 | **magic-wormhole** | 端到端加密、CLI |
| 对方不想装 App | **Snapdrop / PairDrop** | 浏览器即用 |
| 日常 2GB 以内传输 | **WeTransfer** | 品牌可靠、操作简单 |
| 同一网络、懒得选 | **LocalSend** | 统一方案 |

## 终极建议

如果你受够了「我的 iPhone 怎么把视频发到同事的 Android」或「Mac 和 Windows 传文件还要插 U 盘」这种日常摩擦，装机三件套：**AirDrop 管苹果生态内 + LocalSend 管跨平台 + Swiss Transfer 管远程大文件**。全部免费，从此告别文件传输焦虑。
