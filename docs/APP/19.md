# Markdown 编辑器全景评测

Markdown 已经是写作者和开发者的通用语言——纯文本、可版本控制、任意工具都能打开、一键导出为 HTML/PDF/Word。但选择哪个 Markdown 编辑器？这个问题引出的分歧比选择代码编辑器还要大。本文从写作体验、功能完整度、价格三个维度横评市面上最主流的 Markdown 编辑器。

## 为什么用 Markdown 而不是 Word

先回答一个根本问题：为什么不直接用 Word 或者 Pages？Word 是排版工具，Markdown 是写作工具。用 Markdown 写文章时，你的注意力在内容上——`# 标题` 就是标题，`**加粗**` 就是加粗，不需要在工具栏里找按钮、调整样式、处理格式错乱。写完内容后，一键导出为目标格式，排版由模板统一控制。更重要的是，Markdown 的文件是纯文本——你可以用 Git 追踪每一处修改，在任何平台、任何编辑器里打开，永远不会被某个软件的专有格式绑死。

## 编辑器分类

市面上的 Markdown 编辑器大致分三种流派：

- **WYSIWYG（所见即所得）**：打字的同时渲染样式，没有源码和预览的分离。代表：Typora、MarkText
- **源码模式**：只显示 Markdown 源码，没有任何即时渲染。代表：iA Writer
- **双栏 / 混合模式**：源码在左、预览在右，或同一窗口切换。代表：MacDown、VS Code、Obsidian（Live Preview）

选择哪个流派取决于你的写作偏好——喜欢看纯文本的选源码模式，习惯了富文本打字体验的选 WYSIWYG。

## Typora：WYSIWYG 品类的标杆

Typora 是「所见即所得」Markdown 编辑器中做得最好的。你在 Typora 里输入 `# 标题`，回车后它立刻变成大号粗体，光标所在行的 `#` 消失。输 `**文字**`，加粗即时生效。没有源代码栏和预览栏之间的切换——整个界面就是你的文章本身。

**特色功能**：

- **主题系统**：内置多款精美主题（Github、Newsprint、Pixyll 等），也可以下载社区主题。一键切换全局风格。
- **文件树侧栏**：可以当作文档项目管理器，管理一个文件夹下的所有 .md 文件。
- **专注模式**：只高亮当前编辑的段落，其余文字淡出，非常适合沉浸式长文写作。
- **多格式导出**：PDF（保留 CSS 样式）、HTML、Word、LaTeX、图片，导出质量好。
- **表格编辑器**：直接在表格里填内容、拖拽调整列宽，和 Excel 一样直观。
- **数学公式**：LaTeX 数学公式即时渲染，学术写作友好。

**价格**：正式版 \$14.99 一次性买断。Typora 在 Beta 阶段免费了多年，正式版收费后旧版依然可用（但不更新）。考虑到一次付费永久使用，价格合理。

**不足**：Typora 的底层是 Electron，启动速度不算快，大文档（万行级）可能会出现卡顿。插件生态弱（对比 Obsidian），不是一个笔记管理工具——它只是一个编辑器。

**最佳场景**：专注写作者——写博客文章、技术文档、读书笔记。想要看输出结果又不想切窗的人。

## MarkText：开源版 Typora

MarkText 和 Typora 的定位几乎一模一样：WYSIWYG、主题、文件树、多格式导出。区别在于 MarkText 完全开源且免费。

**对比 Typora**：

- 同样 WYSIWYG 体验，渲染品质接近但稍逊
- 同样支持多种主题和专注模式
- 同样有表格编辑器
- 开源意味着不担心突然收费或停止更新
- 活跃开发中，版本迭代较快

**哪里不如 Typora**：界面细节和字体渲染的精细度略低；导出的 PDF 排版不如 Typora 稳定；大型文档的性能表现稍差。

**最佳场景**：想要 WYSIWYG Markdown 编辑体验但不想花钱、或者偏好开源软件的用户。

## Obsidian：不只是编辑器，是知识管理系统

Obsidian（已在第 2 篇详细介绍）本质上是一个知识库管理工具，但它也是一个非常强大的 Markdown 编辑器。它的 Live Preview 模式接近于 WYSIWYG——输入 Markdown 语法即时渲染，但光标仍在「源码」位置。

**作为编辑器的优势**：插件生态极其丰富——Table of Contents、Admonition（提示框）、Kanban、Excalidraw、Dataview 等可以把 Markdown 变成各种形态的内容。双向链接让文章之间产生网络化关联。

**作为编辑器的劣势**：如果你只是要写一篇文章然后导出 PDF，Obsidian 太重了。启动速度、内存占用都高于 Typora 和 iA Writer。WYSIWYG 体验不如 Typora 流畅。

**最佳场景**：你同时需要笔记管理和 Markdown 编辑功能，两者不分开。

## iA Writer：极简主义的极致追求

iA Writer（已在第 10 篇详细介绍）是源码模式 Markdown 编辑器的代表。屏幕上只有纯文本，没有任何渲染——你看到的就是源码。它的设计哲学是：写作时，什么都不要看，只看文字本身。

**差异化特色**：「句法高亮」——自动把形容词、副词、动词、名词、连词用不同颜色标注，帮助你在写作层面审视自己的句式结构。这个功能在英文写作中是真正的生产力工具（中文支持一般）。

**最佳场景**：追求纯净写作体验的人、长篇小说或论文写作者。不适合需要表格编辑、图片拖拽等「富」功能的人。

## MacDown：经典双栏派

MacDown 是 macOS 上最经典的开源双栏 Markdown 编辑器：左边是源码，右边实时预览。界面朴素、启动飞快（原生 macOS 应用，不是 Electron）、支持自定义 CSS 渲染风格。

**优势**：极轻量，打开即写，不占内存。对于从 Mou（MacDown 的前辈，已停止更新）时代过来的老用户来说，这个双栏模式是最熟悉的 Markdown 写作方式。

**劣势**：功能相对基础，没有主题系统（除 CSS 自定义外）、没有文件管理、没有导出到 Word 等格式。

**最佳场景**：想要轻量原生 Mac 应用、习惯双栏预览模式的开发者。

## VS Code + Markdown 插件：开发者的终极方案

如果你日常已经大量时间在 VS Code 里，那么直接在 VS Code 里写 Markdown 是最省事的。

**推荐插件组合**：

- **Markdown All in One**：快捷键（加粗、列表、表格格式化）、自动目录、自动补全
- **markdownlint**：Markdown 语法规范检查，帮你养成良好的书写习惯
- **Paste Image**：截图后直接粘贴到 Markdown 文件，自动保存图片到相对路径
- **Markdown Preview Enhanced**：增强预览，支持流程图、甘特图、幻灯片导出等高级功能
- **Git Graph + GitLens**：写 Markdown 不需要，但管理文档 Git 仓库方便

**优势**：免费、无需额外安装、配合 Git 完美、所有 VS Code 快捷键和习惯都能复用。

**劣势**：预览在分栏而非 WYSIWYG；排版微调不敌 Typora；表格编辑不如专用工具方便。

## 选购建议

| 场景 | 推荐 | 价格 |
|---|---|---|
| WYSIWYG 首选 | Typora | \$14.99 一次买断 |
| WYSIWYG + 免费开源 | MarkText | 免费 |
| 知识管理 + 编辑一体 | Obsidian | 免费（同步收费） |
| 极简纯文本写作 | iA Writer | \$49.99 macOS |
| 轻量双栏原生 macOS | MacDown | 免费 |
| 开发者已在用 VS Code | VS Code + 插件 | 免费 |

总结：如果你要专注写文章且希望所见即所得，Typora 的付费是值得的投入。如果不想花钱，MarkText 能给你 90% 的体验。如果你已经在用 Obsidian 做知识管理，就在 Obsidian 里写，不需要切工具。如果日常就在 VS Code 里生活，装几个插件就够了——代码和文档在同一个环境里效率最高。
