# 团队协作工具：Linear / Notion / Lark

团队工具选型是个很奇怪的事——很多时候不是「选最好的工具」，而是「选大家都能接受的最大公约数」。但好的工具确实能减少沟通摩擦、让信息有序流动，而糟糕的工具会让团队在「找文件」「查进度」「翻聊天记录」中浪费每一天。

本文对比三款各有所长的团队协作工具：Linear（工程团队的轻量项目管理）、Notion（灵活的信息中枢）、Lark（飞书，中国市场的 All-in-One）。同时简要提及 Slack 和 Discord 作为聊天层的补充。

## Linear：工程团队的项目管理，由工程师为工程师设计

Linear 的 slogan 是「Linear is a better way to build software」，它的核心假设是：Jira 太臃肿了，工程团队只需要一个快、轻、纯粹的任务追踪工具。

**使用体验**：

- **键盘优先**：所有操作都可以通过快捷键完成——创建 Issue、切换视图、分配人、改状态、添加标签。不需要鼠标。打开 Linear 就像打开 Vim——学习的头一个小时有摩擦，之后手指不会离开键盘。
- **速度极快**：桌面端使用本地渲染，页面切换几乎零延迟。和 Jira 的「点一下翻个页等 3 秒」形成鲜明对比。
- **视图多样**：Board（看板）、List（列表）、Calendar（日历）、Timeline（甘特图式的时间线）——同一个项目可以按不同视图查看。
- **Cycles**：Linear 版本的 Sprint，但更灵活。Cycle 不是强制的时间盒，可以设置负载阈值（比如每个 Cycle 每人不超过 8 个 Issue），防止过度承诺。Cycle 结束后未完成的 Issue 自动滚入下一个 Cycle，不需要手动挪。
- **Roadmaps**：面向外部的路线图视图，可以和公司官网同步。让团队有一个「对外的计划」，和内部的 Cycles 视图分开。
- **Triage**：所有新 Issue 先进入 Triage 分类，不会被直接扔进 Backlog 污染视野。强制养成「先过滤再处理」的习惯。
- **集成深度**：GitHub / GitLab PR 自动关联 Issue 并更新状态；Slack 通知格式化得好看；Figma 文件可以嵌入 Issue；Sentry 错误可以自动创建 Issue。

**价格**：免费版对小型团队够用，付费版 \$8/用户/月。

**不足**：它不擅长非工程场景——没有文档功能、没有 OKR、没有审批流。Linear 作为任务管理工具是一流的，但它不是一个公司运营系统。

**最佳场景**：软件工程团队（尤其是中小型团队）的项目管理。如果你对 Jira 恨之入骨，Linear 会是你的救赎。

## Notion：可以搭建任何东西的乐高积木

Notion 是一个「ALL-in-One workspace」——文档、数据库、Wiki、项目管理、日历，都在一个平台里。它不像传统 SaaS 那样给你预设好的功能模块，而是给了一套积木让你自己搭。

**核心能力**：

- **数据库（Database）**：Notion 的核心。表格、看板、日历、画廊、列表——同一个数据库可以根据场景切换视图。一个「任务」数据库可以同时以看板展示 Sprint，以日历展示排期，以表格导出 Excel。
- **页面嵌套**：页面里可以放页面，无限嵌套。Wiki 结构自然生长，不需要提前设计信息架构。
- **模板系统**：Notion 的市场里有大量社区模板——产品需求文档、会议纪要、招聘管道、OKR 仪表盘、个人图书馆等。

**团队典型用法**：

- 会议纪要（Meeting Notes）：用数据库按日期和主题组织，关联到相关的项目页面
- 产品需求文档（PRD）：页面嵌入 Figma 设计稿 + 技术方案链接 + 评论讨论
- 招聘管道：候选人卡片按面试阶段在看板里拖动
- 公司 Wiki：新人手册、福利政策、团队架构、技术风格指南
- OKR：每个团队的目标和 KR 在一个数据库中，对全公司透明

**价格**：Plus \$10/用户/月。免费版限制文件大小和协作人数。

**最大的优点也是最大的缺点**：灵活意味着你可以搭建任何东西，也意味着你可以搭出一个一团糟的信息迷宫。Notion 不给结构——你需要自己设计和维护结构。没有专职的信息架构师，Notion 工作区会迅速退化成一堆找不到的文件堆。

**最佳场景**：创业公司（一工具搞定所有）、需要 Wiki / 文档 / 轻量项目管理的非工程团队。

## Lark（飞书）：中国市场的办公超级 App

飞书是字节跳动出品的办公协作平台，在中国市场和东南亚有极强的渗透率。和 Notion 的「给你积木自己搭」不同，飞书是「给你全套装修好的房间」——开箱即用，模块齐全。

**模块一览**：

- **即时通讯（IM）**：类似 Slack 的频道式消息，支持消息已读/未读、话题串（Thread）、跨时区消息降噪
- **飞书文档**：在线协作文档，支持插入表格、流程图、思维导图、投票。比 Google Docs 丰富，比 Notion 结构化
- **飞书日历**：和 IM 深度集成——在聊天里定了时间，自动在日历创建
- **视频会议（飞书妙记）**：视频会议自动录制、自动生成文字稿、自动翻译。会议结束后直接生成会议纪要和待办事项。这是飞书最被低估的功能——会开完了，笔记也写好了
- **多维表格（Bitable）**：飞书版的数据库，类似 Airtable + Notion Database。按需切换表格 / 看板 / 表单视图
- **OKR 和绩效**：飞书内置 OKR 对齐功能，可以直接关联到人力和审批流程
- **审批（OA）**：请假、报销、采购等企业审批流，不需要额外采购 OA 系统

**价格**：标准版免费（有限存储和功能），企业版 ¥25/用户/月。

**适用场景**：中国市场的公司（尤其是百人以上规模），需要 IM + 文档 + 审批 + 视频会议一体化的场景。

**不足**：中国市场外渗透率低（虽然国际版叫 Lark，但功能不如国内飞书完善）；信息量大容易淹没；对于只想用一个模块的小团队来说太重；数据存储在中国境内，有数据本地化要求的跨国公司需要评估合规性。

## 沟通层：Slack 和 Discord

不管用 Linear、Notion 还是飞书做项目和文档，你还需要一个实时沟通工具：

**Slack**：即时通讯的事实标准（在中国市场外）。Huddles 快速语音通话、Canvas 内嵌文档、Workflow Builder 自动化工作流。Slack 的核心优势是频道（Channel）组织模式——每个话题一个频道，不会像微信群一样几百条消息都糊在一个窗口里。Pro 版 \$7.25/用户/月。

**Discord**：本来是游戏社交平台，但语音频道的「持久化」设计（进语音频道就能说话，不需要打电话 / 接电话）让它在远程办公和虚拟协作中越来越受欢迎。很多开源社区和独立开发者团队用 Discord 替代 Slack，因为它是免费的。不适合需要强安全和合规的企业。

## 组合推荐

不同阶段和类型的团队，推荐的工具组合不同：

| 团队类型 | 推荐方案 |
|---|---|
| 小型工程团队（5-20 人） | Linear（项目管理）+ Notion（文档）+ Slack（沟通） |
| 全功能创业公司 | Notion（文档+项目管理）+ Slack |
| 中国 / 东南亚市场公司 | 飞书（全部）+ Linear（如果工程团队坚持） |
| 开源 / 社区团队 | Discord（主战场）+ GitHub Projects（任务） |
| 中型以上企业 | 按模块选最佳——项目管理用 Linear、Wiki 用 Notion、IM 用 Slack |

关键是不要让单一工具做所有事。Linear 管不好 Wiki，Notion 做不了聊天，飞书不能替代专业的项目管理。找到每个领域的最佳工具，用集成把它们连起来，比用一个「万能工具」把所有事都做 70 分要高效得多。
