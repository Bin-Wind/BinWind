# 系统监控工具：iStat Menus / Stats / htop

想知道你的 Mac 为什么突然风扇狂转、电池掉电飞快、内存压力飙红？系统监控工具就是你的仪表盘——让你看到 CPU、内存、磁盘、网络、电池、传感器的实时状态。有了这些工具，你不需要打开活动监视器就能一眼发现问题。

Mac 上三款主流系统监控工具：iStat Menus（全功能商业软件）、Stats（免费开源替代）、htop（终端经典）。本文逐一分析适用场景。

## iStat Menus：菜单栏里的指挥中心

iStat Menus 是 Bjango 出品的老牌 Mac 系统监控工具，把 CPU、GPU、内存、磁盘、网络、电池、传感器、时间等十几种监控模块塞进了菜单栏。安装后，菜单栏右侧多出一排微型图表，鼠标悬停弹出详情面板，点击进入完整仪表盘。

**核心模块**：

- **CPU & GPU**：每个核心的占用率、频率、温度。可以按核心显示（P 核 vs E 核分开展示），也可以看整体使用率。MacBook Pro 的 Apple Silicon 用户可以看到能效核心和性能核心分别在工作时的分布——这在活动监视器中是看不到的。
- **内存**：使用量、缓存、Swap 占用、内存压力等级（绿色/黄色/红色）。Swap 区一旦被敲到 2GB 以上，就该考虑关几个 Chrome 标签页了。
- **磁盘**：读写速度、IOPS、剩余容量、SMART 健康状态。外接硬盘的吞吐量实时显示，在做大文件传输时可以确认实际速度。
- **网络**：上下行速率、总流量、当前连接的 IP。想知道为什么网速慢，看哪个进程在占用带宽——iStat Menus 的网络模块能列出每个进程的网络活动。
- **传感器**：CPU 核心温度、风扇转速、SSD 温度、主板温度、电池温度。做视频导出或编译大项目时，风扇转速从静音的 1200 RPM 飙到 6000+ RPM，你就知道该给 Mac 垫个散热架了。
- **电池**：电量、健康度、循环次数、最大容量、放电功率。MacBook 用户可以精确追踪电池老化进度。
- **天气**：菜单栏显示当前气温和天气预报——不算监控功能但放在菜单栏里确实实用。

**自定义能力**：每个模块的显示方式都可以细调——数字、百分比、柱状图、折线图、弧线图。配色方案、字体大小、菜单位置全部可调。你可以把不关心的指标隐藏，只显示几个关键读数。

**通知与告警**：可以设置阈值告警——CPU 超过 90% 持续 1 分钟、电池健康度低于 80%、磁盘剩余空间低于 10GB，自动弹出通知。这些在跑长时间任务时非常有用：编译到一半不用频繁切回来看进度，温度异常它会告诉你。

**价格**：iStat Menus 售价 **$11.99 一次性买断**（大版本更新可能需要重新付费）。对于一个你可能每天都在菜单栏里看的工具，价格合理。免费试用 14 天。

**不足**：默认开启全部模块会让菜单栏拥挤不堪，需要花时间精简；传感器读取偶尔有 1-2 秒延迟；不支持远程监控。

**最适合**：Mac 用户，愿意为精致、全面、菜单栏集成的系统监控付费。

## Stats：免费开源，iStat Menus 的最佳平替

Stats 是一款完全免费开源的 Mac 菜单栏监控工具，GitHub 上超过 25k Star。它的功能定位和 iStat Menus 高度重合：CPU、GPU、内存、磁盘、网络、传感器、电池——全部模块都有。如果你不想花钱买 iStat Menus，Stats 就是答案。

**和 iStat Menus 的差异**：

- **UI**：Stats 的菜单栏图表更简单朴素，没有 iStat Menus 那么多视觉定制选项。对于「看数字就行」的用户这不是缺点，对追求菜单栏美感的用户则是差距。
- **传感器模块**：Stats 的传感器读取兼容性略低于 iStat Menus——少数 Mac 型号上某些传感器读数不可用（通常是新款 Mac 的驱动适配滞后导致）。大部分用户不受影响。
- **风扇控制**：Stats 内置风扇转速控制，可以手动设置固定转速。iStat Menus 没有这个功能，需要额外安装 Macs Fan Control。
- **价格**：完全免费。用 `brew install stats` 一行命令安装，零付费、零广告、零账号。

**稳定性**：开源社区维护活跃，macOS 大版本更新后通常几周内适配。偶尔出现读取数据卡顿，重启应用即可恢复。

**最适合**：不想付费的用户、喜欢开源软件、对 UI 精致度要求不高的实用主义者。两个字：够用。

## htop：终端里的系统监控

htop 是一个交互式进程查看器，在终端中运行，是 `top` 命令的现代替代品。跨平台（macOS / Linux / BSD），通过 `brew install htop` 安装。

**终端监控的优势**：

- **SSH 到远程服务器直接运行**——不需要 GUI 工具也能看系统状态
- **极度轻量**——几 MB 内存，启动秒开，不占用菜单栏空间
- **键盘驱动**——F-key 排序（按 CPU、内存、运行时间）、搜索进程、kill 进程、显示线程树，全键盘操作效率极高
- **进程树**——可以展开进程的父子关系，查哪个进程 fork 出了子进程集群
- **颜色可配**——CPU 条、内存条、Swap 的颜色在 htoprc 中自由定义

**典型用法**：

```bash
htop                       # 启动
# 然后:
# F6 → 选择排序方式（CPU%、MEM%、TIME+）
# F4 → 搜索进程（输入进程名过滤显示）
# F9 → 发送信号给选中进程（9=SIGKILL, 15=SIGTERM）
# F5 → 切换树形视图（看父子进程关系）
# t   → 切换进程树模式
```

**htop 的不足**：缺少菜单栏常驻显示（你要主动打开终端才能看）；没有历史记录和告警功能；没有磁盘/网络/电池监控（htop 只关注 CPU 和内存）。htop 不是 iStat Menus 的竞争对手——它们是互补工具。

**推荐组合**：日常使用 iStat Menus 或 Stats 监控 Mac 本地系统的全局状态。SSH 到服务器或需要定位异常进程时用 htop。两者同时使用不冲突。

## 其他值得知道的系统监控工具

- **活动监视器（Activity Monitor）**：macOS 自带。作为临时应急和强制退出应用的工具够用。缺少菜单栏常驻和告警，不适合持续监控。
- **Macs Fan Control**：专攻风扇控制。可以为每个风扇设置基于温度传感器的自动调速曲线。iStat Menus + Macs Fan Control 是 Mac 散热管理的最佳双人组。
- **bmon / iftop / nload**：终端里的网络流量监控工具。htop 不显示网络吞吐，这三个补上。`brew install bmon` 后 `bmon` 查看各网络接口的实时流量图形。
- **btop**：htop 的升级版。用 C++ 重写，GPU 加速渲染，界面更美观（有 CPU/内存/网络/磁盘的 TUI 图表）。如果你觉得 htop 的界面太复古，`brew install btop` 试试。支持鼠标点击，配色方案丰富，比 htop 信息密度更高。

## 怎么选

| 场景 | 推荐 | 价格 |
|---|---|---|
| Mac 本地全面监控 | **iStat Menus** | $11.99 买断 |
| 免费开源方案 | **Stats** | 免费 |
| 终端/远程服务器 | **htop** 或 **btop** | 免费 |
| 只关注风扇噪音 | **Macs Fan Control** | 免费版够用 |

我自己用 iStat Menus + htop。菜单栏常驻 CPU 占用率和温度——看一眼就知道系统有没有异常负载。SSH 到服务器时 `htop` 打开，定位哪个进程吃光了内存。Stats 和 btop 也在我的机器上，偶尔切着用——不同的工具在不同的场景闪亮。
