# VS Code 必备插件清单 (2025)

VS Code 的成功有一半归功于其插件生态。但插件装多了会影响启动速度和编辑器性能，如何选择真正值得安装的插件是一门学问。这份清单基于我日常前端/全栈开发的经验，按类别精选了 2025 年仍然活跃、质量可靠的插件。

## 主题与视觉

**Catppuccin** 是过去两年最受欢迎的配色主题。柔和的莫兰迪色系（Latte、Frappe、Macchiato、Mocha 四种风味可选），所有主流语言都有精心配色的语法高亮，长时间编码不易疲劳。**One Dark Pro** 是经典的 Material Design 风格深色主题，如果你习惯 Atom 编辑器的配色，它就是最直接的延续。

**Material Icon Theme** 为文件和文件夹添加识别度很高的图标。`.tsx` 文件有 React 图标，`package.json` 有 NPM 图标，文件夹也会根据名称显示不同图标（`components`、`utils`、`hooks` 各有专属图标）。文件树一下子变得可扫描了。

## 效率必备

**Prettier** 是代码格式化的事实标准。配合 `"editor.formatOnSave": true`，每次保存自动统一缩进、引号风格、分号规则。团队成员之间再也不用争论格式问题，`.prettierrc` 就是最终裁定。

**ESLint** 在格式化基础上提供代码质量检查：未使用的变量、潜在 bug 模式、最佳实践违反。Prettier 管格式，ESLint 管逻辑，两者配合使用。

**GitLens** 是目前最强的 Git 可视化插件。光标放在任意一行，自动显示该行的最后修改者、修改时间和 commit message（inline blame）。点击可查看完整 commit 详情和 diff。侧边栏提供 commit 时间线、文件历史、分支对比、代码搜索。对于多人协作项目，GitLens 是理解代码演变过程的必备工具。

**Todo Tree** 收集代码中所有的 `TODO`、`FIXME`、`HACK`、`BUG` 等标记，在侧边栏集中展示。你可以快速浏览哪些地方有未完成的工作，点击直接跳转到对应位置。对于维护大型项目或者代码 Review，这是极好的待办事项清单。

## AI 辅助

**GitHub Copilot** 是目前最强的 AI 代码助手。上下文理解准确，多语言支持完善，不仅补全代码，还能根据注释生成整个函数。免费套餐每月提供 2000 次补全，对学生和开源维护者完全免费。付费版 $10/月，对于专业开发者来说物超所值。

**Codeium**（现改名 Windsurf）是 Copilot 的免费替代品。代码补全质量接近 Copilot，聊天功能可以对整个项目提问。免费版没有次数限制，适合预算有限或想尝试 AI 编程的开发者。

**Continue** 是一款开源的 AI 编码助手，可以接入本地模型（Ollama）或各种 API（OpenAI、Anthropic、Azure 等），自由度极高。如果你关注隐私或想使用本地模型，Continue 是唯一的选择。

## 语言专项

- **Volar**（Vue）：Vue 3 官方推荐的语言支持插件，替代了旧的 Vetur。TypeScript 支持、模板语法高亮、自动补全。
- **Rust Analyzer**：Rust 官方推荐的 LSP 实现，自动补全、类型提示、行内错误、代码重构。比旧的 Rust 插件快得多。
- **Python / Pylance**：微软官方的 Python 语言支持，智能提示、类型检查、自动导入、Jupyter Notebook 集成。
- **Go**（Go Team at Google）：Google 官方 Go 插件，代码补全、调试、测试、代码导航。

## 实用工具

**Better Comments** 给注释加颜色分类：`!` 红色警告、`?` 蓝色疑问、`TODO` 橙色待办、`*` 绿色强调、`//` 灰色删除线。阅读代码时，不同类型的注释一眼可辨。

**Error Lens** 把错误和警告直接显示在代码行尾，而不是只在问题面板里。漏了分号？那一行末尾直接看到红色的 `; expected`。不需要悬停查看，错误信息零距离。

**Peacock** 给不同项目的 VS Code 窗口设置不同颜色标识。当你同时打开多个 VS Code 窗口（前端项目一个，后端项目一个，文档站一个），靠颜色区分比靠标题栏文字快得多。

**Project Manager** 管理多个项目的快捷入口。收藏常用项目，快捷键快速切换。配合 Peacock 使用效果更佳。

## 远程开发

**Remote - SSH** 让你在 VS Code 中直接编辑远程服务器上的文件。SSH 连接后，VS Code 的终端、插件、Git 操作全部在远程服务器上运行，体验几乎和本地开发相同。对于需要编译环境在远端、或者项目必须在特定服务器环境运行的场景，这是颠覆性的工作方式。

**Dev Containers** 把项目运行环境封装在 Docker 容器中。新成员 clone 项目后，VS Code 自动构建容器、安装依赖、配置环境。再也没有「我电脑上跑不起来」的问题。团队统一开发环境的最佳实践。

## Markdown

**Markdown All in One** 提供快捷键（快捷键生成目录、加粗、列表等）、自动补全、数学公式支持、导出 HTML 等全套 Markdown 编辑增强。

**markdownlint** 检查 Markdown 语法规范：标题层级、列表缩进、链接格式、空白行等。写技术文档时保持风格统一。

**Paste Image** 截图后直接在 Markdown 文件中粘贴，自动将图片保存到指定目录并插入正确的相对路径。写文档时插入截图最便捷的方案。

## 管理建议

不要一次安装所有插件。我的维护策略：
- 只在当前工作区需要的插件，设为工作区推荐扩展（`.vscode/extensions.json`），不在全局启用。
- 每半年审视一次已安装插件列表，卸载不再使用的。尤其注意是否有功能已被 VS Code 内置替代的插件。
- 插件加载时间过长会影响启动速度。在 VS Code 的 `Cmd+Shift+P → Developer: Show Running Extensions` 可以查看每个插件的激活时间和原因。

保持插件清单精简，让 VS Code 始终快速响应。工具是为你服务的，不要把时间花在调试工具上。
