# 日期时间处理

JDK 8 引入的 `java.time`（JSR-310）彻底解决了旧 `Date`/`Calendar` API 线程不安全、设计混乱、API 难用的问题。现代 Java 项目应全面使用 `java.time`。

## LocalDate / LocalTime / LocalDateTime

不带时区的日期时间表示，适用于绝大多数业务场景：

```java
import java.time.*;

// === LocalDate：日期 ===
LocalDate today = LocalDate.now();
System.out.println(today);                  // 2026-06-29

LocalDate newYear = LocalDate.of(2026, Month.JANUARY, 1);
LocalDate parsed = LocalDate.parse("2026-12-25");
System.out.println(parsed.getDayOfWeek());  // FRIDAY
System.out.println(parsed.getYear());       // 2026

// === LocalTime：时间 ===
LocalTime now = LocalTime.now();
LocalTime lunch = LocalTime.of(12, 0, 0);   // 12:00
System.out.println(lunch.plusMinutes(30));  // 12:30

// === LocalDateTime：日期 + 时间 ===
LocalDateTime dateTime = LocalDateTime.now();
LocalDateTime meeting = LocalDateTime.of(2026, 7, 1, 14, 30);
System.out.println(meeting);  // 2026-07-01T14:30
```

## 日期计算

```java
LocalDate today = LocalDate.now();

// 增减
LocalDate nextWeek = today.plusWeeks(1);
LocalDate lastMonth = today.minusMonths(1);
LocalDate tenDaysLater = today.plus(Period.ofDays(10));

// 比较
boolean before = today.isBefore(nextWeek);
boolean after = today.isAfter(lastMonth);

// 调整器（TemporalAdjuster）
LocalDate firstDayOfMonth = today.with(TemporalAdjusters.firstDayOfMonth());
LocalDate lastDayOfYear = today.with(TemporalAdjusters.lastDayOfYear());
LocalDate nextFriday = today.with(TemporalAdjusters.next(DayOfWeek.FRIDAY));

System.out.println(firstDayOfMonth);  // 2026-06-01
System.out.println(nextFriday);       // 下个周五
```

## 格式化与解析

```java
import java.time.format.DateTimeFormatter;

LocalDateTime now = LocalDateTime.now();

// 内置格式
System.out.println(DateTimeFormatter.ISO_LOCAL_DATE.format(now));  // 2026-06-29

// 自定义格式
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String formatted = now.format(fmt);
System.out.println(formatted);  // 2026-06-29 14:30:15

DateTimeFormatter cnFmt = DateTimeFormatter.ofPattern("yyyy年M月d日 EEEE");
System.out.println(now.format(cnFmt));  // 2026年6月29日 星期一

// 解析字符串
LocalDate parsedDate = LocalDate.parse("2026-07-01");
LocalDate customParsed = LocalDate.parse("2026/12/25",
    DateTimeFormatter.ofPattern("yyyy/MM/dd"));
```

## ZonedDateTime（带时区）

需要处理跨时区场景时使用：

```java
import java.time.ZonedDateTime;
import java.time.ZoneId;
import java.util.Set;

// 获取时区
ZoneId tokyoZone = ZoneId.of("Asia/Tokyo");
ZoneId nyZone = ZoneId.of("America/New_York");

// 使用时区
ZonedDateTime now = ZonedDateTime.now();             // 系统默认时区
ZonedDateTime tokyo = ZonedDateTime.now(tokyoZone);
ZonedDateTime ny = ZonedDateTime.now(nyZone);

System.out.println(now);    // 2026-06-29T10:00+08:00[Asia/Shanghai]
System.out.println(tokyo);  // 2026-06-29T11:00+09:00[Asia/Tokyo]
System.out.println(ny);     // 2026-06-28T22:00-04:00[America/New_York]

// 时区转换
ZonedDateTime shanghai = ZonedDateTime.now();
ZonedDateTime newYork = shanghai.withZoneSameInstant(nyZone);

// 所有可用时区
Set<String> zones = ZoneId.getAvailableZoneIds();
// zones.forEach(System.out::println);
```

## Period 与 Duration

- **Period** — 基于日期的量（年、月、日）
- **Duration** — 基于时间的量（时、分、秒、纳秒）

```java
// Period
LocalDate start = LocalDate.of(2023, 1, 1);
LocalDate end = LocalDate.of(2026, 6, 29);
Period period = Period.between(start, end);

System.out.println(period.getYears());   // 3
System.out.println(period.getMonths());  // 5
System.out.println(period.getDays());    // 28

// Duration
Instant t1 = Instant.now();
Instant t2 = t1.plusSeconds(7263);
Duration duration = Duration.between(t1, t2);

System.out.println(duration.toHours());         // 2
System.out.println(duration.toMinutes() % 60);  // 1
System.out.println(duration.getSeconds() % 60); // 3
// 即 2小时1分3秒
```

## Instant 与时间戳

`Instant` 表示时间线上的一个瞬间点（纳秒精度），适合存储时间戳：

```java
Instant now = Instant.now();
System.out.println(now);                             // 2026-06-29T02:00:00.123Z

// 时间戳（毫秒/秒）
long epochMilli = now.toEpochMilli();                // 毫秒时间戳
long epochSecond = now.getEpochSecond();              // 秒时间戳

// 从时间戳恢复
Instant fromEpoch = Instant.ofEpochMilli(epochMilli);

// Instant 与 ZonedDateTime 互转
ZonedDateTime zdt = now.atZone(ZoneId.of("Asia/Shanghai"));
Instant back = zdt.toInstant();
```

## 新旧 API 互转

```java
import java.time.*;

// Date -> Instant
java.util.Date oldDate = new java.util.Date();
Instant instant = oldDate.toInstant();

// Instant -> Date
java.util.Date newDate = java.util.Date.from(Instant.now());

// LocalDateTime -> Date
LocalDateTime ldt = LocalDateTime.now();
java.util.Date date = java.util.Date.from(
    ldt.atZone(ZoneId.systemDefault()).toInstant());
```

> `java.time` 类都是不可变的（immutable）且线程安全的，可以放心地在多线程环境下使用，不会再出现 `SimpleDateFormat` 的并发问题。
