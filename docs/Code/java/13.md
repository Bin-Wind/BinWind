# 注解与反射

注解（Annotation）是 Java 的元数据机制，反射（Reflection）则让程序在运行时检查和操作自身结构。两者结合是框架（Spring、JUnit、MyBatis）实现魔法的基础。

## 内置注解

```java
// @Override — 编译检查是否真正重写了父类方法
@Override
public String toString() {
    return "MyObject";
}

// @Deprecated — 标记方法已过时
@Deprecated
public void oldMethod() { }

// @SuppressWarnings — 抑制编译器警告
@SuppressWarnings("unchecked")
List<String> list = (List<String>) someUntypedList();

// @FunctionalInterface — 确保接口只有一个抽象方法
@FunctionalInterface
public interface Converter {
    String convert(int value);
}
```

## 元注解与自定义注解

元注解（meta-annotation）是注解的注解，用于定义注解的生命周期和行为：

```java
import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME)  // 保留到运行时（反射可读）
@Target(ElementType.FIELD)           // 仅用于字段
public @interface NotNull {
    String message() default "不能为 null";
}

@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.METHOD})  // 可用于类和方法
public @interface Loggable {
    String value() default "";       // 默认属性名 value
    Level level() default Level.INFO;

    enum Level { DEBUG, INFO, WARN, ERROR }
}

// 使用注解
@Loggable(value = "UserService", level = Loggable.Level.DEBUG)
public class UserService {

    @NotNull(message = "用户名不能为空")
    private String username;

    @Loggable("findById")
    public User findById(Long id) {
        return new User(id, username);
    }
}
```

`@Retention` 的三种策略：
- `SOURCE` — 仅源码，编译时丢弃（如 `@Override`）
- `CLASS` — 保留在字节码，运行时不可读（默认值）
- `RUNTIME` — 运行时可通过反射读取（框架常用）

## 反射基础 — Class 对象

每个类在 JVM 中对应一个 `Class` 对象，是反射的入口：

```java
// 获取 Class 对象的三种方式
Class<?> clazz1 = String.class;                       // 类字面量
Class<?> clazz2 = "hello".getClass();                 // 对象的 getClass()
Class<?> clazz3 = Class.forName("java.lang.String");  // 全限定类名

System.out.println(clazz1.getName());        // java.lang.String
System.out.println(clazz1.getSimpleName());  // String
System.out.println(clazz1.getPackageName()); // java.lang
System.out.println(clazz1.getSuperclass());  // class java.lang.Object

// 实现的接口
Class<?>[] interfaces = ArrayList.class.getInterfaces();
for (Class<?> iface : interfaces) {
    System.out.println(iface.getSimpleName());  // List, RandomAccess, Cloneable, ...
}
```

## 反射操作成员

```java
public class Person {
    private String name;
    private int age;

    public Person() { }
    public Person(String name, int age) { this.name = name; this.age = age; }

    public String getName() { return name; }
    private void secretMethod() { System.out.println("secret!"); }
}

// === 构造方法 ===
Constructor<Person> ctor = Person.class.getDeclaredConstructor(String.class, int.class);
Person p = ctor.newInstance("李四", 28);

Constructor<Person> noArgCtor = Person.class.getDeclaredConstructor();
Person p2 = noArgCtor.newInstance();

// === 字段 ===
Field nameField = Person.class.getDeclaredField("name");
nameField.setAccessible(true);                    // 突破 private 限制
System.out.println(nameField.get(p));             // 李四
nameField.set(p, "王五");

// === 方法 ===
Method getName = Person.class.getMethod("getName");    // public 方法（包括继承的）
System.out.println(getName.invoke(p));                  // 王五

Method secret = Person.class.getDeclaredMethod("secretMethod");  // 任意方法
secret.setAccessible(true);
secret.invoke(p);  // 输出: secret!
```

## 反射与注解结合 — 简单校验框架

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface NotBlank {
    String message() default "不能为空";
}

static class Validator {
    public static List<String> validate(Object obj) throws Exception {
        List<String> errors = new ArrayList<>();
        for (Field field : obj.getClass().getDeclaredFields()) {
            if (field.isAnnotationPresent(NotBlank.class)) {
                field.setAccessible(true);
                NotBlank ann = field.getAnnotation(NotBlank.class);
                Object value = field.get(obj);
                if (value == null || value.toString().isBlank()) {
                    errors.add(field.getName() + ": " + ann.message());
                }
            }
        }
        return errors;
    }
}

// 使用
class UserForm {
    @NotBlank(message = "用户名不能为空")
    private String username;

    @NotBlank
    private String password;

    public UserForm(String username, String password) {
        this.username = username;
        this.password = password;
    }
}

UserForm form = new UserForm("admin", "");
List<String> errors = Validator.validate(form);
System.out.println(errors);  // [password: 不能为空]
```

## 反射实战场景

1. **框架核心** — Spring 依赖注入、MyBatis ORM 映射、Hibernate entity 扫描
2. **序列化** — JSON 库（Gson、Jackson）通过反射读写对象字段
3. **动态代理** — `java.lang.reflect.Proxy` 创建运行时代理对象
4. **单元测试** — JUnit 通过反射调用 `@Test` 方法
5. **插件机制** — `Class.forName` 动态加载驱动类（如 JDBC）

## 反射的代价

```java
// 反射调用慢于直接调用
long start = System.nanoTime();
for (int i = 0; i < 1_000_000; i++) {
    p.getName();            // 直接调用
}
long direct = System.nanoTime() - start;

Method method = Person.class.getMethod("getName");
start = System.nanoTime();
for (int i = 0; i < 1_000_000; i++) {
    method.invoke(p);       // 反射调用
}
long reflected = System.nanoTime() - start;

System.out.printf("直接调用: %dms, 反射: %dms%n",
    TimeUnit.NANOSECONDS.toMillis(direct),
    TimeUnit.NANOSECONDS.toMillis(reflected));
```

代价：反射绕过了编译时类型检查，丢失了 IDE 重构支持，且性能开销较大。**能用直接调用的场景，不要用反射**。框架和库中有节制地使用，业务代码中尽量避免。

## `getDeclared*` vs `get*` 对比

| 方法                    | 范围                       |
|-------------------------|----------------------------|
| `getDeclaredMethods()`  | 本类声明的所有方法（含 private） |
| `getMethods()`          | public 方法（含继承的）    |
| `getDeclaredFields()`   | 本类声明的所有字段         |
| `getFields()`           | public 字段（含继承的）    |
| `getDeclaredConstructors()` | 本类所有构造方法        |
| `getConstructors()`     | public 构造方法            |

理解注解和反射，你就能看穿框架的魔法 —— 所有那些看似自动完成的功能，底层都是这两个机制在运作。
