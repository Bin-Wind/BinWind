# Gradle 构建工具入门

Gradle 是继 Maven 之后的新一代构建工具，使用 Groovy 或 Kotlin DSL 来定义构建逻辑。它采用**有向无环图（DAG）**模型组织 Task 的执行顺序，支持增量构建和构建缓存，性能远优于 Maven 的顺序生命周期模型。Android 官方从 Gradle 2.x 开始全面采用它作为默认构建系统。

## 安装与 Wrapper

通过 Gradle Wrapper（`gradlew`）可以确保团队所有成员和 CI 环境使用一致的 Gradle 版本，无需手动安装。

```bash
# 在项目中生成 Wrapper
gradle wrapper --gradle-version 8.5

# 常用命令
./gradlew build          # 编译 + 测试 + 打包
./gradlew test            # 仅运行测试
./gradlew bootRun         # 启动 Spring Boot 应用
./gradlew clean           # 清理构建产物
./gradlew dependencies    # 查看依赖树
```

Wrapper 核心文件：`gradle-wrapper.properties` 指定 Gradle 版本和下载地址；`gradlew` / `gradlew.bat` 是跨平台的执行脚本。

## build.gradle (Groovy DSL)

```groovy
plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

group = 'com.example'
version = '1.0.0'

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

repositories {
    // 优先使用阿里云镜像加速
    maven { url 'https://maven.aliyun.com/repository/public' }
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    runtimeOnly 'com.mysql:mysql-connector-j'
    compileOnly 'org.projectlombok:lombok'
    annotationProcessor 'org.projectlombok:lombok'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
    testRuntimeOnly 'org.junit.platform:junit-platform-launcher'
}

tasks.named('test') {
    useJUnitPlatform()
}
```

依赖配置说明：`implementation`（编译和运行时可用，但不暴露给消费者）、`compileOnly`（仅编译期）、`runtimeOnly`（仅运行时）、`testImplementation`（仅测试）、`annotationProcessor`（注解处理器）。

## build.gradle.kts (Kotlin DSL)

Kotlin DSL 是 Gradle 推荐的现代写法，提供类型安全和 IDE 自动补全。

```kotlin
plugins {
    java
    id("org.springframework.boot") version "3.2.0"
    id("io.spring.dependency-management") version "1.1.4"
}

group = "com.example"
version = "1.0.0"

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

repositories {
    maven { url = uri("https://maven.aliyun.com/repository/public") }
    mavenCentral()
}

dependencies {
    implementation("org.springframework.boot:spring-boot-starter-web")
    implementation("org.springframework.boot:spring-boot-starter-data-jpa")
    runtimeOnly("com.mysql:mysql-connector-j")
    compileOnly("org.projectlombok:lombok")
    annotationProcessor("org.projectlombok:lombok")
    testImplementation("org.springframework.boot:spring-boot-starter-test")
}

tasks.withType<Test> {
    useJUnitPlatform()
}
```

## 自定义 Task

Task 是 Gradle 的原子执行单元。每个 Task 定义输入和输出，Gradle 据此判断是否需要重新执行。

```groovy
// 自定义 Task：复制配置文件
tasks.register('copyConfig', Copy) {
    from 'src/main/resources'
    into "${buildDir}/config"
    include 'application-*.yml'
}

// 自定义 Task：执行 Shell 脚本
tasks.register('runSqlScript', Exec) {
    workingDir "${projectDir}"
    commandLine 'bash', 'scripts/init-db.sh'
}

// Task 依赖关系
tasks.named('build') {
    dependsOn copyConfig
}
```

## 多模块项目

在 `settings.gradle` 中声明所有子模块：

```groovy
// settings.gradle
rootProject.name = 'my-platform'
include 'common', 'service', 'web'
```

子模块 `build.gradle` 中可以引用兄弟模块：

```groovy
dependencies {
    implementation project(':common')
    implementation project(':service')
}
```

### 统一依赖版本（Version Catalog）

在 `gradle/libs.versions.toml` 中集中管理版本：

```toml
[versions]
spring-boot = "3.2.0"
mysql = "8.0.33"

[libraries]
spring-boot-starter-web = { group = "org.springframework.boot", name = "spring-boot-starter-web", version.ref = "spring-boot" }
mysql-connector = { group = "com.mysql", name = "mysql-connector-j", version.ref = "mysql" }
```

然后在 `build.gradle.kts` 中引用 `libs.spring.boot.starter.web`。

## Gradle vs Maven 对比

| 特性 | Maven | Gradle |
|------|-------|--------|
| 构建模型 | 固定生命周期 | DAG 任务模型 |
| 配置语言 | XML | Groovy / Kotlin DSL |
| 构建速度 | 较慢（无增量） | 快（增量 + 缓存） |
| 依赖管理 | 传递依赖，最近优先 | 传递依赖，可配置策略 |
| 灵活性 | 低（插件覆盖生命周期） | 高（自定义 Task 灵活组合） |
| 学习曲线 | 平缓 | 稍陡（DSL 概念较多） |

> 最佳实践：新项目推荐使用 Kotlin DSL；必须保留 `gradlew` 脚本版本控制；使用 `implementation` 而非 `compile`（已废弃）；多模块项目使用 Version Catalog 统一管理版本。
