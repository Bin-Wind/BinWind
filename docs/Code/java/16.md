# JUnit 5 单元测试

JUnit 5 是 Java 单元测试的事实标准，相比 JUnit 4 有了革命性变化：模块化架构（JUnit Platform + Jupiter + Vintage）、Lambda 风格断言、参数化测试和扩展模型。一个合格的单元测试应当**快速、独立、可重复、自验证、及时（F.I.R.S.T.）**。

## 快速开始

```java
// 依赖：junit-jupiter, mockito-junit-jupiter
import org.junit.jupiter.api.*;

class CalculatorTest {

    @Test
    @DisplayName("1 + 1 应该等于 2")  // 中文可读的测试名称
    void shouldAddTwoNumbers() {
        int result = 1 + 1;
        assertEquals(2, result, "1 + 1 must be 2");
    }

    @Test
    void shouldThrowWhenDividingByZero() {
        // 断言抛出异常
        assertThrows(ArithmeticException.class,
            () -> { int x = 10 / 0; });
    }
}
```

## 核心注解

| 注解 | 说明 |
|------|------|
| `@Test` | 标记测试方法 |
| `@DisplayName` | 自定义测试显示名称 |
| `@BeforeEach` | 每个测试方法前执行 |
| `@AfterEach` | 每个测试方法后执行 |
| `@BeforeAll` | 所有测试方法前执行一次（必须 static） |
| `@AfterAll` | 所有测试方法后执行一次（必须 static） |
| `@Disabled` | 禁用测试 |
| `@Timeout` | 超时自动失败 |

```java
class LifecycleDemoTest {

    private List<String> items;

    @BeforeAll
    static void initGlobal() {
        System.out.println("Global setup — 只执行一次");
    }

    @BeforeEach
    void setUp() {
        items = new ArrayList<>();
        items.add("A");
        items.add("B");
    }

    @AfterEach
    void tearDown() {
        items.clear();
    }

    @Test
    void shouldHaveTwoItems() {
        assertEquals(2, items.size());
    }
}
```

## 强大的断言体系

JUnit 5 提供了丰富的断言方法，支持分组断言和超时断言：

```java
import static org.junit.jupiter.api.Assertions.*;

class AssertionsDemo {

    @Test
    void standardAssertions() {
        assertEquals(4, 2 + 2);
        assertTrue("hello".startsWith("h"));
        assertFalse(5 > 10);
        assertNull(null);
        assertNotNull(new Object());

        // 数组断言
        assertArrayEquals(
            new int[]{1, 2, 3},
            new int[]{1, 2, 3}
        );
    }

    @Test
    void groupedAssertions() {
        // 分组断言：一次执行所有断言，不会因第一个失败而终止
        assertAll("user",
            () -> assertEquals("John", "John"),
            () -> assertTrue(30 > 18)
        );
    }

    @Test
    @Timeout(1)  // 1 秒超时
    void shouldCompleteWithinOneSecond() throws InterruptedException {
        Thread.sleep(500);
    }
}
```

## @ParameterizedTest 参数化测试

避免重复编写相似逻辑的测试数据，一行注解驱动多组输入：

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;

class ParameterizedTestDemo {

    // 单参数值源
    @ParameterizedTest
    @ValueSource(strings = {"hello", "world", "JUnit"})
    void shouldNotBeBlank(String input) {
        assertFalse(input.isBlank());
    }

    // 多参数 CSV 格式
    @ParameterizedTest
    @CsvSource({
        "1, 2, 3",
        "-5, 5, 0",
        "100, 200, 300"
    })
    void shouldSum(int a, int b, int expected) {
        assertEquals(expected, a + b);
    }

    // 方法源：从工厂方法获取测试参数
    @ParameterizedTest
    @MethodSource("provideStrings")
    void shouldHaveExpectedLength(String input, int expectedLength) {
        assertEquals(expectedLength, input.length());
    }

    static Stream<Arguments> provideStrings() {
        return Stream.of(
            Arguments.of("cat", 3),
            Arguments.of("elephant", 8)
        );
    }

    // CSV 文件源
    @ParameterizedTest
    @CsvFileSource(resources = "/test-data.csv", numLinesToSkip = 1)
    void shouldParseValidEmails(String email, boolean expected) {
        assertEquals(expected, email.contains("@"));
    }
}
```

## Mockito 入门：隔离依赖

Mockito 是 Java 最流行的 Mock 框架，用于创建模拟对象，隔离外部依赖（数据库、网络、文件系统）。

```java
import static org.mockito.Mockito.*;
import static org.mockito.BDDMockito.*;

@ExtendWith(MockitoExtension.class)  // 启用 Mockito 注解
class UserServiceTest {

    @Mock
    private UserRepository repository;  // 创建 mock 对象

    @InjectMocks
    private UserService service;  // 自动注入 mock

    @Test
    void shouldFindUserById() {
        User mockUser = new User(1L, "Alice");
        when(repository.findById(1L)).thenReturn(Optional.of(mockUser));

        User found = service.findById(1L);

        assertNotNull(found);
        assertEquals("Alice", found.getName());
        verify(repository, times(1)).findById(1L);
    }

    @Test
    void shouldHandleNotFound() {
        when(repository.findById(999L)).thenReturn(Optional.empty());

        assertThrows(UserNotFoundException.class,
            () -> service.findById(999L));
    }
}
```

> 核心原则：单元测试不依赖外部资源（数据库、网络、文件）；使用 Mock 隔离被测对象的外部依赖；每个测试方法只验证一个行为；测试覆盖率 80% 以上是务实目标。
