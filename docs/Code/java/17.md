# 多线程与并发基础

Java 从诞生之日起就内置了多线程支持（Thread 类、synchronized 关键字），JVM 将操作系统的原生线程映射为 Java Thread。但直接操作 Thread 容易出错，Java 5 引入 `java.util.concurrent` 包（JUC），提供了线程池、同步器、并发集合等高级抽象。

## Thread 与 Runnable

```java
// 方式 1：继承 Thread 类
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " is running");
    }
}

// 方式 2：实现 Runnable 接口（推荐，避免单继承限制）
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " is running");
    }
}

public class ThreadDemo {
    public static void main(String[] args) {
        // 启动线程必须调用 start()，而非 run()
        new MyThread().start();
        new Thread(new MyTask()).start();

        // Lambda 简化（Runnable 是函数式接口）
        new Thread(() -> System.out.println("Lambda thread")).start();
    }
}
```

## ExecutorService 线程池

直接创建线程开销大且难以管理。**线程池**复用已有线程，提供了任务提交流、运行、结束的完整生命周期。

```java
import java.util.concurrent.*;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // 固定大小线程池
        ExecutorService pool = Executors.newFixedThreadPool(4);

        // 提交 Runnable（无返回值）
        pool.execute(() -> {
            System.out.println("Task running in: " + Thread.currentThread().getName());
        });

        // 提交 Callable（有返回值）
        Future<String> future = pool.submit(() -> {
            Thread.sleep(1000);
            return "Done";
        });

        try {
            String result = future.get(2, TimeUnit.SECONDS);
            System.out.println("Result: " + result);
        } catch (TimeoutException e) {
            future.cancel(true);
        } catch (Exception e) {
            e.printStackTrace();
        }

        pool.shutdown();  // 优雅关闭：不再接受新任务，等待已有任务完成
    }
}
```

**ThreadPoolExecutor 核心参数**：`corePoolSize`（常驻线程数）、`maximumPoolSize`（最大线程数）、`keepAliveTime`（空闲线程存活时间）、`workQueue`（任务队列类型——ArrayBlockingQueue、LinkedBlockingQueue、SynchronousQueue）、`RejectedExecutionHandler`（拒绝策略）。

## synchronized 关键字

synchronized 是 Java 内置的**互斥锁**，保证同一时刻只有一个线程执行被保护的代码块。

```java
class Counter {
    private int count = 0;

    // 修饰方法：锁住当前实例（this）
    public synchronized void increment() {
        count++;
    }

    // 修饰静态方法：锁住类的 Class 对象
    public static synchronized void staticMethod() {
        // ...
    }

    // 修饰代码块：可指定任意锁对象，粒度更细
    public void safeBlock() {
        synchronized (this) {
            count++;
        }
    }

    public int getCount() {
        return count;
    }
}
```

synchronized 底层通过对象头中的 **Mark Word** 记录锁状态，经历偏向锁（Biased Locking）、轻量级锁（CAS）、重量级锁（OS Mutex）的逐级膨胀过程。JDK 15 后默认禁用偏向锁。

## volatile 关键字

volatile 保证变量的**可见性**和**有序性**（禁止指令重排），但不保证**原子性**。适合一写多读的标志位场景。

```java
class FlagExample {
    volatile boolean running = true;

    void stop() {
        running = false;  // 写入对所有线程立即可见
    }

    void doWork() {
        while (running) {
            // ...
        }
    }
}
```

## wait / notify / notifyAll

这三个方法是 Object 类的方法，必须在 synchronized 块中调用。wait 释放锁并等待通知，notify 随机唤醒一个等待线程。

```java
class ProducerConsumer {
    private final Queue<String> queue = new LinkedList<>();
    private final int MAX_SIZE = 5;

    public synchronized void produce(String item) throws InterruptedException {
        while (queue.size() == MAX_SIZE) {
            wait();  // 队列满，等待消费者消费
        }
        queue.add(item);
        notifyAll();  // 唤醒所有等待线程
    }

    public synchronized String consume() throws InterruptedException {
        while (queue.isEmpty()) {
            wait();  // 队列空，等待生产者生产
        }
        String item = queue.poll();
        notifyAll();
        return item;
    }
}
```

实际开发中应优先使用 `java.util.concurrent` 包中的高级工具：`CountDownLatch`（等待一组操作完成）、`CyclicBarrier`（多线程互相等待到齐）、`Semaphore`（限流控制）、`ConcurrentHashMap`（线程安全 Map）、`BlockingQueue`（阻塞队列）。

## 线程安全的实践准则

1. **优先使用不可变对象**（final、Immutable Collections）
2. **使用并发集合**替代 synchronized 包装（ConcurrentHashMap > Collections.synchronizedMap）
3. **锁粒度最小化**（synchronized 代码块 > synchronized 方法）
4. **避免在锁中执行 I/O 操作**
5. **使用 ThreadLocal** 隔离线程私有的可变状态

```java
// ThreadLocal：每个线程持有独立副本，天然线程安全
private static final ThreadLocal<SimpleDateFormat> DATE_FORMAT =
    ThreadLocal.withInitial(() -> new SimpleDateFormat("yyyy-MM-dd"));
```

> 并发编程的两个核心痛点：**竞态条件**（Race Condition）和**可见性问题**。synchronized 解决原子性 + 可见性，volatile 解决可见性，原子类（AtomicInteger 等）通过 CAS 无锁实现原子操作。
