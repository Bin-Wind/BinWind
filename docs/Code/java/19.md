# JDBC 数据库操作

JDBC（Java Database Connectivity）是 Java 访问关系型数据库的标准 API，定义在 `java.sql` 和 `javax.sql` 包中。它屏蔽了不同数据库的差异，通过 Driver 接口统一操作 MySQL、PostgreSQL、Oracle 等数据库。虽然现在更多项目使用 MyBatis 或 JPA 等 ORM 框架，但 JDBC 仍是它们的底层基础。

## 核心组件

JDBC 的核心接口包括：`DriverManager`（驱动管理器，管理数据库驱动）、`Connection`（数据库连接）、`Statement` / `PreparedStatement`（执行 SQL）、`ResultSet`（结果集）。

## 数据库连接

```java
import java.sql.*;

public class JdbcConnection {

    // JDBC URL 格式：jdbc:mysql://[host]:[port]/[database]?[params]
    private static final String URL =
        "jdbc:mysql://localhost:3306/test_db?useSSL=false&serverTimezone=Asia/Shanghai";
    private static final String USER = "root";
    private static final String PASSWORD = "123456";

    public static void main(String[] args) {
        // try-with-resources 自动关闭资源（JDK 7+）
        try (Connection conn = DriverManager.getConnection(URL, USER, PASSWORD)) {
            System.out.println("连接成功：" + conn.getCatalog());
        } catch (SQLException e) {
            System.err.println("连接失败：" + e.getMessage());
        }
    }
}
```

从 JDBC 4.0 起，驱动类通过 **SPI（ServiceLoader）** 自动注册，不再需要 `Class.forName("com.mysql.cj.jdbc.Driver")`。

## PreparedStatement 防 SQL 注入

**永远不要拼接 SQL 字符串**。使用 `PreparedStatement` 进行参数化查询，既可防止 SQL 注入，又支持预编译提升性能。

```java
// 错误示例——SQL 注入风险
// String sql = "SELECT * FROM users WHERE name = '" + userInput + "'";

// 正确示例——参数化查询
public class PreparedStatementDemo {

    // 查询操作
    public User findByName(String name) {
        String sql = "SELECT id, name, email, age FROM users WHERE name = ?";
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {

            ps.setString(1, name);  // 参数索引从 1 开始
            try (ResultSet rs = ps.executeQuery()) {
                if (rs.next()) {
                    return new User(
                        rs.getLong("id"),
                        rs.getString("name"),
                        rs.getString("email"),
                        rs.getInt("age")
                    );
                }
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return null;
    }

    // 插入操作，带自增主键返回
    public Long insertUser(User user) {
        String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)";
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql,
                 Statement.RETURN_GENERATED_KEYS)) {

            ps.setString(1, user.getName());
            ps.setString(2, user.getEmail());
            ps.setInt(3, user.getAge());
            int affected = ps.executeUpdate();

            if (affected > 0) {
                try (ResultSet keys = ps.getGeneratedKeys()) {
                    if (keys.next()) {
                        return keys.getLong(1);
                    }
                }
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return null;
    }

    // 批量操作
    public void batchInsert(List<User> users) {
        String sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)";
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {

            for (User user : users) {
                ps.setString(1, user.getName());
                ps.setString(2, user.getEmail());
                ps.setInt(3, user.getAge());
                ps.addBatch();
            }
            int[] results = ps.executeBatch();  // 批量执行
            System.out.println("批量插入 " + results.length + " 条记录");
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }

    // 更新 / 删除
    public int updateAge(long userId, int newAge) {
        String sql = "UPDATE users SET age = ? WHERE id = ?";
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setInt(1, newAge);
            ps.setLong(2, userId);
            return ps.executeUpdate();  // 返回受影响行数
        } catch (SQLException e) {
            e.printStackTrace();
            return 0;
        }
    }
}
```

## 事务管理

JDBC 默认**自动提交**（auto-commit），每条 SQL 独立成一个事务。业务中需要手动控制事务边界：

```java
public class TransactionDemo {

    public void transfer(Long fromId, Long toId, BigDecimal amount) {
        Connection conn = null;
        try {
            conn = getConnection();
            conn.setAutoCommit(false);  // 关闭自动提交

            // 扣款
            String deductSQL = "UPDATE accounts SET balance = balance - ? WHERE id = ?";
            try (PreparedStatement ps = conn.prepareStatement(deductSQL)) {
                ps.setBigDecimal(1, amount);
                ps.setLong(2, fromId);
                ps.executeUpdate();
            }

            // 加款
            String addSQL = "UPDATE accounts SET balance = balance + ? WHERE id = ?";
            try (PreparedStatement ps = conn.prepareStatement(addSQL)) {
                ps.setBigDecimal(1, amount);
                ps.setLong(2, toId);
                ps.executeUpdate();
            }

            conn.commit();  // 提交事务
        } catch (SQLException e) {
            if (conn != null) {
                try {
                    conn.rollback();  // 回滚
                    System.err.println("事务回滚");
                } catch (SQLException ex) {
                    ex.printStackTrace();
                }
            }
            e.printStackTrace();
        } finally {
            if (conn != null) {
                try {
                    conn.setAutoCommit(true);  // 恢复默认
                    conn.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}
```

## 连接池简介

每次创建物理连接开销极大。**数据库连接池**维护一组长连接供应用复用，避免频繁创建和销毁。常用连接池实现：

- **HikariCP**：Spring Boot 2.x 默认，性能最优，字节码级别优化
- **Druid**（Alibaba）：国人常用，内置监控和 SQL 防火墙
- **C3P0 / DBCP2**：老牌框架，使用渐少

```java
// HikariCP 连接池示例
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:mysql://localhost:3306/test_db");
config.setUsername("root");
config.setPassword("123456");
config.setMaximumPoolSize(20);
config.setMinimumIdle(5);
config.setConnectionTimeout(30000);
config.setIdleTimeout(600000);

HikariDataSource dataSource = new HikariDataSource(config);
Connection conn = dataSource.getConnection();  // 从池中获取连接
```

> 最佳实践：始终使用 PreparedStatement 防 SQL 注入；生产环境必须配置连接池；事务粒度尽可能小，避免长事务锁表；大数据量场景使用游标或分批查询，防止 ResultSet 撑爆内存。
