# MyBatis 持久层框架

MyBatis 是 Java 生态中最流行的半自动 ORM（Object Relational Mapping）框架。与 Hibernate 的全自动映射不同，MyBatis 允许开发者手动编写 SQL，同时提供灵活的结果映射，特别适合复杂查询和遗留数据库场景。Spring Boot 集成 MyBatis 后的依赖：`mybatis-spring-boot-starter`。

## 核心组件

- **SqlSessionFactory**：线程安全，全局唯一，负责创建 SqlSession
- **SqlSession**：非线程安全，一次数据库会话，使用后必须关闭
- **Mapper 接口**：DAO 层接口，方法与 SQL 映射绑定
- **Mapper XML**：存放 SQL 和结果映射配置

## 快速配置

```xml
<!-- application.yml -->
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/myapp
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver

mybatis:
  mapper-locations: classpath:mapper/*.xml
  type-aliases-package: com.example.entity  # 别名扫描
  configuration:
    map-underscore-to-camel-case: true       # 下划线转驼峰
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # SQL 日志
```

## Mapper XML 基本用法

```xml
<!-- resources/mapper/UserMapper.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.UserMapper">

    <!-- resultMap：自定义结果映射，解决列名与属性名不一致 -->
    <resultMap id="BaseResultMap" type="com.example.entity.User">
        <id column="id" property="id"/>
        <result column="user_name" property="userName"/>
        <result column="email" property="email"/>
        <result column="create_time" property="createTime"/>
    </resultMap>

    <!-- 查询单条 -->
    <select id="findById" parameterType="long" resultMap="BaseResultMap">
        SELECT id, user_name, email, create_time
        FROM users
        WHERE id = #{id}
    </select>

    <!-- 查询列表 -->
    <select id="findAll" resultMap="BaseResultMap">
        SELECT id, user_name, email, create_time FROM users
    </select>

    <!-- 插入 + 返回自增主键 -->
    <insert id="insert" parameterType="User" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO users (user_name, email, create_time)
        VALUES (#{userName}, #{email}, NOW())
    </insert>

    <!-- 更新 -->
    <update id="update" parameterType="User">
        UPDATE users
        SET user_name = #{userName}, email = #{email}
        WHERE id = #{id}
    </update>

    <!-- 删除 -->
    <delete id="deleteById" parameterType="long">
        DELETE FROM users WHERE id = #{id}
    </delete>
</mapper>
```

对应的 Mapper 接口：

```java
package com.example.mapper;

import com.example.entity.User;
import org.apache.ibatis.annotations.Mapper;

@Mapper
public interface UserMapper {
    User findById(Long id);
    List<User> findAll();
    int insert(User user);
    int update(User user);
    int deleteById(Long id);
}
```

## 注解方式（简单 SQL）

简单 SQL 可直接用注解写在 Mapper 接口上，免去 XML 文件：

```java
import org.apache.ibatis.annotations.*;

@Mapper
public interface UserAnnoMapper {

    @Select("SELECT * FROM users WHERE id = #{id}")
    @Results({
        @Result(column = "user_name", property = "userName"),
        @Result(column = "create_time", property = "createTime")
    })
    User findById(Long id);

    @Insert("INSERT INTO users (user_name, email) VALUES (#{userName}, #{email})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(User user);

    @Update("UPDATE users SET user_name = #{userName} WHERE id = #{id}")
    int update(User user);

    @Delete("DELETE FROM users WHERE id = #{id}")
    int deleteById(Long id);
}
```

## 动态 SQL

MyBatis 最强大的特性之一，通过标签实现条件拼接，避免在 Java 代码中组装 SQL 字符串：

```xml
<select id="searchUsers" resultMap="BaseResultMap">
    SELECT id, user_name, email, create_time FROM users
    <where>
        <if test="keyword != null and keyword != ''">
            AND (user_name LIKE CONCAT('%', #{keyword}, '%')
                 OR email LIKE CONCAT('%', #{keyword}, '%'))
        </if>
        <if test="minAge != null">
            AND age >= #{minAge}
        </if>
        <if test="createTime != null">
            AND create_time >= #{createTime}
        </if>
    </where>
    ORDER BY create_time DESC
</select>

<!-- 批量插入（动态 SQL foreach） -->
<insert id="batchInsert">
    INSERT INTO users (user_name, email) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.userName}, #{user.email})
    </foreach>
</insert>

<!-- 批量删除 -->
<delete id="deleteByIds">
    DELETE FROM users WHERE id IN
    <foreach collection="ids" item="id" open="(" close=")" separator=",">
        #{id}
    </foreach>

<!-- 选择更新 -->
<update id="updateSelective">
    UPDATE users
    <set>
        <if test="userName != null">user_name = #{userName},</if>
        <if test="email != null">email = #{email},</if>
        <if test="age != null">age = #{age},</if>
    </set>
    WHERE id = #{id}
</update>
```

## #{} vs ${} — 安全关键区别

| 符号 | 处理方式 | SQL 注入 | 使用场景 |
|------|---------|---------|---------|
| `#{}` | 预编译占位符（PreparedStatement） | 防注入 | 所有参数值：`WHERE id = #{id}` |
| `${}` | 直接拼接字符串 | 不安全 | 动态表名、列名、ORDER BY：`ORDER BY ${columnName}` |

```java
// 错误：${} 用于参数值，有 SQL 注入风险
@Select("SELECT * FROM users WHERE name = '${name}'")

// 正确：#{} 用于参数值
@Select("SELECT * FROM users WHERE name = #{name}")

// 例外：动态排序需要用 ${}，但必须先做白名单校验
// 在 Service 层校验 columnName 只能为 id/name/email 之一
@Select("SELECT * FROM users ORDER BY ${columnName} ${direction}")
```

## 分页查询（PageHelper）

```java
// 引入 pagehelper-spring-boot-starter 后：
@Service
public class UserService {
    @Autowired
    private UserMapper userMapper;

    public PageInfo<User> findByPage(int pageNum, int pageSize) {
        PageHelper.startPage(pageNum, pageSize);
        List<User> list = userMapper.findAll();
        return new PageInfo<>(list);
        // PageInfo 包含 total, pages, list 等分页信息
    }
}
```

> 最佳实践：复杂查询用 XML，简单 CRUD 用注解；使用 `mapUnderscoreToCamelCase` 开启自动驼峰转换；`${}` 仅用于动态表名 / 列名且必须白名单校验；生产环境关闭 `StdOutImpl` 日志，避免 SQL 日志拖垮性能。
