# Spring Boot 快速上手

Spring Boot 是 Spring 生态的快速启动框架，核心理念是**约定大于配置（Convention over Configuration）**和**开箱即用（Out of the Box）**。它通过自动配置（Auto-Configuration）和起步依赖（Starter）将传统的 XML 地狱压缩为一个注解和一份 YAML 文件。创建一个可运行的 Web 应用只需几行代码。

## 第一个 Spring Boot 应用

最小的 Spring Boot 入口类只需要一个 `main` 方法和一个注解：

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication  // 等价于 @Configuration + @EnableAutoConfiguration + @ComponentScan
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

## application.yml 配置

Spring Boot 支持 `.properties` 和 `.yml` 两种格式，推荐后者（层次清晰、支持多文档）：

```yaml
# application.yml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  application:
    name: my-app
  datasource:
    url: jdbc:mysql://localhost:3306/myapp
    username: root
    password: ${DB_PASSWORD}  # 支持环境变量
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: update        # none / update / create / create-drop
    show-sql: true

# 自定义配置
app:
  jwt:
    secret: my-secret-key
    expiration: 86400000     # 24小时
```

可通过 `@Value` 或 `@ConfigurationProperties` 读取配置：

```java
// 方式 1：@Value 注入单个属性
@Value("${app.jwt.expiration}")
private long jwtExpiration;

// 方式 2：@ConfigurationProperties 批量绑定（推荐）
@Component
@ConfigurationProperties(prefix = "app.jwt")
@Data
public class JwtProperties {
    private String secret;
    private long expiration;
}
```

## 分层架构与核心注解

Spring Boot 经典三层架构：

```
Controller → Service → Repository (DAO) → Database
   ↓            ↓            ↓
@RestController @Service  @Repository
```

```java
// === Controller 层：接收 HTTP 请求 ===
@RestController  // = @Controller + @ResponseBody
@RequestMapping("/users")
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }

    @PostMapping
    public User createUser(@RequestBody @Valid UserCreateRequest request) {
        return userService.create(request);
    }
}

// === Service 层：业务逻辑 ===
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public User findById(Long id) {
        return userRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("User not found"));
    }

    @Transactional  // 声明式事务
    public User create(UserCreateRequest request) {
        User user = new User();
        user.setName(request.getName());
        user.setEmail(request.getEmail());
        return userRepository.save(user);
    }
}

// === Repository 层：数据访问 ===
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);
    boolean existsByEmail(String email);
}
```

## Bean 生命周期与依赖注入

Spring 通过 **IoC 容器（Inversion of Control）** 管理 Bean 的创建、注入和销毁。常用注入方式：

```java
// 推荐：构造器注入（不可变性、方便测试）
@RestController
public class UserController {
    private final UserService userService;

    public UserController(UserService userService) {  // @Autowired 可省略
        this.userService = userService;
    }
}

// 字段注入（简洁但不利于单元测试）
@Autowired
private UserService userService;

// Setter 注入（可选依赖）
@Autowired
public void setUserService(UserService userService) {
    this.userService = userService;
}
```

Bean 的作用域（Scope）：

```java
@Component
@Scope("singleton")  // 默认：全局唯一实例
public class SingletonBean {}

@Component
@Scope("prototype")  // 每次注入创建新实例
public class PrototypeBean {}

@Component
@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)  // 每个 HTTP 请求一个实例（Web 环境）
public class RequestBean {}

@Component
@Scope(value = "session", proxyMode = ScopedProxyMode.TARGET_CLASS)  // 每个 Session 一个实例
public class SessionBean {}
```

## 自动配置原理（Auto-Configuration）

自动配置是 Spring Boot 的核心魔法。运作流程：

1. `@SpringBootApplication` 内部包含 `@EnableAutoConfiguration`
2. `@EnableAutoConfiguration` 通过 `@Import(AutoConfigurationImportSelector.class)` 加载配置类
3. `AutoConfigurationImportSelector` 读取 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 文件中声明的所有自动配置类
4. 每个配置类上都有 `@ConditionalOnClass`（类存在）、`@ConditionalOnMissingBean`（Bean 不存在）、`@ConditionalOnProperty`（配置项存在）等条件注解
5. 仅当所有条件满足时，配置类中的 `@Bean` 方法才会生效

```java
// 自动配置源码片段（概念示意）
@AutoConfiguration
@ConditionalOnClass({ DataSource.class, EmbeddedDatabaseType.class })
@ConditionalOnMissingBean(DataSource.class)
@EnableConfigurationProperties(DataSourceProperties.class)
public class DataSourceAutoConfiguration {

    @Bean
    @ConditionalOnProperty(name = "spring.datasource.url")
    public DataSource dataSource(DataSourceProperties properties) {
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl(properties.getUrl());
        ds.setUsername(properties.getUsername());
        ds.setPassword(properties.getPassword());
        return ds;
    }
}
```

因此只需引入 `mysql-connector-j` jar 包并配置 `spring.datasource.url`，Spring Boot 就会自动创建 `DataSource` Bean，无需手动定义。

> 如果你想知道"哪些自动配置生效了"，在 `application.yml` 中设置 `debug: true`，或启动时加 `--debug` 参数，控制台会打印 Auto-configuration Report。
