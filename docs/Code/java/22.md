# Spring Boot 数据访问层

Spring Boot 通过 **Spring Data** 系列项目统一了各类数据源的访问方式。其中 JPA（Java Persistence API）+ Hibernate 是 RDBMS 领域最主流的 ORM 方案。配合 Spring Data JPA 的 Repository 接口，CRUD 操作可以零 SQL 实现，复杂查询则通过 `@Query` 或方法命名约定自动生成。

## JPA + Hibernate 基础

Hibernate 是 JPA 规范的实现者，Spring Data JPA 在 Hibernate 之上又封装了 Repository 抽象层。依赖 `spring-boot-starter-data-jpa` 即可自动引入 Hibernate 和连接池。

### @Entity 实体定义

```java
import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "users")  // 指定表名，省略则默认类名小写
@Data  // Lombok
@NoArgsConstructor
@AllArgsConstructor
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // 自增主键
    private Long id;

    @Column(nullable = false, length = 50)
    private String name;

    @Column(unique = true, nullable = false)
    private String email;

    @Enumerated(EnumType.STRING)  // 枚举映射为字符串而非序数
    @Column(length = 20)
    private UserStatus status;

    private Integer age;

    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @Column(name = "updated_at")
    private LocalDateTime updatedAt;

    // 实体生命周期回调
    @PrePersist
    protected void onCreate() {
        this.createdAt = LocalDateTime.now();
        this.updatedAt = LocalDateTime.now();
    }

    @PreUpdate
    protected void onUpdate() {
        this.updatedAt = LocalDateTime.now();
    }
}
```

### 关联关系

```java
// 一对多（One-to-Many）
@Entity
public class Order {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String orderNo;

    @ManyToOne(fetch = FetchType.LAZY)  // LAZY：用到时才加载，EAGER：立即加载
    @JoinColumn(name = "user_id")
    private User user;

    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<OrderItem> items = new ArrayList<>();

    // 添加双向关联的辅助方法
    public void addItem(OrderItem item) {
        items.add(item);
        item.setOrder(this);
    }
}

// 多对一（Many-to-One）
@Entity
public class OrderItem {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String productName;
    private BigDecimal price;
    private Integer quantity;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "order_id")
    private Order order;
}
```

### 常用 JPA 注解速查

| 注解 | 说明 |
|------|------|
| `@Entity` | 标记 JPA 实体 |
| `@Table(name="...")` | 指定表名 |
| `@Id` | 主键 |
| `@GeneratedValue` | 主键生成策略（IDENTITY / SEQUENCE / UUID） |
| `@Column` | 字段映射（name / nullable / length / unique） |
| `@Enumerated(EnumType.STRING)` | 枚举映射 |
| `@Transient` | 标记非持久化字段 |
| `@ManyToOne` / `@OneToMany` | 关联关系 |
| `@JoinColumn` | 外键列 |
| `fetch = FetchType.LAZY` | 延迟加载（性能优化关键） |

## Repository 接口

Spring Data JPA 的 Repository 无须实现类即可提供 CRUD 方法：

```java
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // === 方法命名自动生成 SQL ===
    Optional<User> findByEmail(String email);
    List<User> findByNameLike(String namePattern);
    List<User> findByAgeGreaterThan(int age);
    long countByStatus(UserStatus status);
    boolean existsByEmail(String email);
    void deleteByEmail(String email);

    // === @Query 自定义 JPQL 查询 ===
    @Query("SELECT u FROM User u WHERE u.email like %:domain")
    List<User> findByEmailDomain(@Param("domain") String domain);

    // === @Query 原生 SQL ===
    @Query(value = "SELECT * FROM users WHERE age > :minAge ORDER BY created_at DESC",
           nativeQuery = true)
    List<User> findAdults(@Param("minAge") int minAge);

    // === @Modifying 更新操作 ===
    @Modifying
    @Query("UPDATE User u SET u.status = :status WHERE u.id IN :ids")
    int batchUpdateStatus(@Param("ids") List<Long> ids, @Param("status") UserStatus status);
}
```

方法命名约定的核心关键词：`findBy`、`readBy`、`queryBy`（查询）、`countBy`（计数）、`existsBy`（存在判断）、`deleteBy`（删除）。条件用 `And`、`Or`、`Between`、`LessThan`、`GreaterThan`、`Like`、`In`、`OrderBy`、`IgnoreCase` 连接。

## 分页与排序

```java
public class UserPagingDemo {

    @Autowired
    private UserRepository userRepository;

    public Page<User> findUsersByPage(int page, int size) {
        // PageRequest 第一个参数是 0-based 页码
        PageRequest pageable = PageRequest.of(page, size,
            Sort.by(Sort.Direction.DESC, "createdAt"));

        return userRepository.findAll(pageable);
        // Page 对象包含：getContent()、getTotalElements()、getTotalPages()
    }

    // 带条件的分页
    public Page<User> searchUsers(String keyword, Pageable pageable) {
        return userRepository.findByNameLike("%" + keyword + "%", pageable);
    }
}
```

## Spring Data JPA Auditing（审计）

自动记录 "谁在何时创建 / 修改了记录"，结合 `@CreatedBy` + `@LastModifiedBy`：

```java
// 1. 在启动类上加 @EnableJpaAuditing
@SpringBootApplication
@EnableJpaAuditing
public class Application { ... }

// 2. BaseEntity 抽象基类
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {

    @CreatedDate
    @Column(updatable = false)
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;

    @CreatedBy
    @Column(updatable = false)
    private String createdBy;

    @LastModifiedBy
    private String updatedBy;
}

// 3. 实现 AuditorAware
@Component
public class AuditorAwareImpl implements AuditorAware<String> {
    @Override
    public Optional<String> getCurrentAuditor() {
        // 从 SecurityContext 获取当前登录用户
        return Optional.of(SecurityContextHolder.getContext()
            .getAuthentication().getName());
    }
}
```

## Flyway 数据库迁移

Flyway 通过 SQL 版本脚本管理数据库 schema 变更，实现版本化、可追溯、可复现的数据库演进。

```yaml
# application.yml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true  # 对已有数据库执行 baseline
```

迁移脚本命名：`V1__init_tables.sql`、`V2__add_age_column.sql`，放在 `src/main/resources/db/migration/` 下：

```sql
-- V1__init_tables.sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);
```

应用启动时 Flyway 自动扫描并执行未应用的脚本，版本信息存在 `flyway_schema_history` 表中。

> 最佳实践：`FetchType.LAZY` 是默认推荐，避免 N+1 查询；`@EntityGraph` 可覆盖懒加载按需 join；生产环境 `ddl-auto` 设 `none` 或 `validate`，通过 Flyway 管理 DDL。
