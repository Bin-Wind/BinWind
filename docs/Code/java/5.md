# 面向对象基础

Java 是纯粹的面向对象语言（除了基本类型）。掌握类、对象、封装三大概念，是理解 Java 设计哲学的关键。

## 类与对象（Class & Object）

类（Class）是对象的蓝图，对象（Object）是类的实例。用 `new` 关键字创建对象：

```java
public class Person {
    // 字段（Field）
    private String name;
    private int age;

    // 构造方法（Constructor）
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 方法（Method）
    public void introduce() {
        System.out.println("我叫 " + name + "，今年 " + age + " 岁");
    }
}

// 使用
Person p = new Person("张三", 25);
p.introduce();  // 我叫 张三，今年 25 岁
```

每个 `.java` 文件只能有一个 `public class`，且类名必须与文件名相同。

## 构造方法（Constructor）

构造方法与类同名、无返回类型，负责对象初始化：

```java
public class Student {
    private String name;
    private int grade;

    // 无参构造（默认也不写时编译器会自动生成）
    public Student() {
        this("未知", 0);  // 调用本类另一个构造方法
    }

    // 有参构造
    public Student(String name, int grade) {
        this.name = name;
        this.grade = grade;
    }

    // 一旦显式写了构造方法，默认无参构造就消失了。
    // 若需要无参构造，必须手动写出。
}
```

## this 关键字

`this` 代表当前对象的引用，有三个主要用途：

```java
public class Rectangle {
    private double width;
    private double height;

    // 1. 区分字段和局部变量
    public Rectangle(double width, double height) {
        this.width = width;   // this.width 是字段
        this.height = height; // height 是参数
    }

    // 2. 调用本类其他构造方法（必须在第一行）
    public Rectangle(double side) {
        this(side, side);  // 调用上面的构造，创建正方形
    }

    // 3. 返回当前对象（链式调用）
    public Rectangle scale(double factor) {
        this.width *= factor;
        this.height *= factor;
        return this;
    }
}

// 链式调用
Rectangle r = new Rectangle(10.0).scale(1.5);
```

## 封装（Encapsulation）

封装是将数据（字段）和操作数据的方法绑定在一起，并隐藏内部实现：

```java
public class BankAccount {
    private String accountNumber;   // private：外部不可直接访问
    private double balance;

    public BankAccount(String accountNumber, double initialDeposit) {
        if (initialDeposit < 0) {
            throw new IllegalArgumentException("初始余额不能为负");
        }
        this.accountNumber = accountNumber;
        this.balance = initialDeposit;
    }

    // Getter：外部读取
    public double getBalance() { return balance; }

    // 不提供 setter，余额只能通过业务方法修改
    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("存款金额必须为正");
        }
        this.balance += amount;
    }

    public void withdraw(double amount) {
        if (amount > balance) {
            throw new IllegalArgumentException("余额不足");
        }
        this.balance -= amount;
    }
}
```

封装的好处：可以在 getter/setter 中加入校验逻辑，字段变化不影响外部代码。

## 访问修饰符（Access Modifiers）

| 修饰符      | 同类 | 同包 | 子类 | 全局 |
|-------------|------|------|------|------|
| `private`   | ✔    | ✘    | ✘    | ✘    |
| (default)   | ✔    | ✔    | ✘    | ✘    |
| `protected` | ✔    | ✔    | ✔    | ✘    |
| `public`    | ✔    | ✔    | ✔    | ✔    |

## toString、equals 和 hashCode

```java
public class Product {
    private String name;
    private double price;

    public Product(String name, double price) {
        this.name = name;
        this.price = price;
    }

    @Override
    public String toString() {
        return "Product{name='" + name + "', price=" + price + "}";
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Product)) return false;
        Product other = (Product) o;
        return Double.compare(other.price, price) == 0
            && name.equals(other.name);
    }

    @Override
    public int hashCode() {
        return java.util.Objects.hash(name, price);
    }
}
```

**重要规则**：重写 `equals` 必须重写 `hashCode`，否则 HashMap/HashSet 中将无法正确工作。
