# 继承与多态

继承（Inheritance）和多态（Polymorphism）是面向对象编程的两大支柱。继承实现代码复用，多态则让同一接口表现出不同行为。

## extends 继承

用 `extends` 关键字建立父子类关系。Java 只支持单继承（一个类只能有一个直接父类）：

```java
// 父类（基类）
public class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " 在吃东西");
    }
}

// 子类（派生类）
public class Dog extends Animal {
    private String breed;

    public Dog(String name, String breed) {
        super(name);        // 必须调用父类构造方法
        this.breed = breed;
    }

    public void bark() {
        System.out.println(name + " 汪汪叫");
    }
}

Dog dog = new Dog("旺财", "金毛");
dog.eat();    // 继承自 Animal：旺财 在吃东西
dog.bark();   // Dog 特有：旺财 汪汪叫
```

**调用父类构造**：`super()` 必须在子类构造第一行。若无显式调用，编译器自动插入 `super()`（无参）。

## 方法重写（@Override）

子类提供父类方法的具体实现，用 `@Override` 注解让编译器协助检查：

```java
public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    public void eat() {
        System.out.println(name + " 优雅地吃猫粮");
        // super.eat();  可调用父类版本
    }
}

Animal a1 = new Dog("旺财", "金毛");
Animal a2 = new Cat("咪咪");
a1.eat();  // 旺财 在吃东西（未重写）
a2.eat();  // 咪咪 优雅地吃猫粮（重写后）
```

重写规则：参数列表一致、返回类型相同或协变、访问级别不能更低、不能抛出更宽泛的异常。

## 多态（Polymorphism）

父类引用指向子类对象，运行时动态绑定实际方法。这是 Java 多态的核心：

```java
Animal[] pets = {
    new Dog("大黄", "土狗"),
    new Cat("小花"),
    new Dog("小白", "萨摩耶")
};

// 同一个方法调用，表现不同行为
for (Animal a : pets) {
    a.eat();  // 实际调用的方法是运行时的类型决定
}
// 大黄 在吃东西
// 小花 优雅地吃猫粮
// 小白 在吃东西
```

## abstract 抽象类

不能实例化、可包含抽象方法（无方法体）和具体方法：

```java
public abstract class Shape {
    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    // 抽象方法：子类必须实现
    public abstract double area();

    // 具体方法：子类可直接继承使用
    public String description() {
        return "这是一个 " + color + " 形状，面积 " + area();
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```

## interface 接口

接口定义行为契约，JDK 8+ 支持 `default` 方法和 `static` 方法：

```java
public interface Flyable {
    // 抽象方法（public abstract 可省略）
    void fly();

    // default 方法（JDK 8+，带默认实现）
    default void land() {
        System.out.println("降落中...");
    }

    // static 方法（JDK 8+）
    static String flyCategory() {
        return "飞行";
    }
}

public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟儿拍打翅膀飞行");
    }
}
```

**接口 vs 抽象类**：接口定义"能做什么"（is-a capability），抽象类定义"是什么"（is-a）。一个类可以实现多个接口，但只能继承一个类。

## instanceof 类型检查

```java
Animal a = new Dog("小黑", "拉布拉多");

if (a instanceof Dog dog) {           // JDK 16 模式匹配
    dog.bark();                        // 自动转换为 Dog 类型
}

// 旧式写法
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
}

// 检查子类型关系
System.out.println(a instanceof Animal);  // true
System.out.println(a instanceof Object);  // true（所有类的根）
```

继承与多态让代码更灵活、更易扩展 —— 对接口编程而非对实现编程，是优秀 Java 代码的标志。
