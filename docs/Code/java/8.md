# 异常处理机制

异常（Exception）是程序运行时发生的不正常情况。Java 提供了强大的异常处理机制，让代码在异常情况下仍能优雅地恢复或退出。

## 异常分类

Java 异常体系以 `Throwable` 为根，分三大类：

```
Throwable
├── Error              — 严重系统错误（OOM、StackOverflow），不应捕获
└── Exception
    ├── RuntimeException（unchecked）— 编程错误，如 NPE、数组越界
    └── 其他 Exception（checked） — 编译器强制要求处理，如 IOException
```

- **Checked Exception**：编译器强制你 `try-catch` 或 `throws`，代表可恢复的外部错误
- **Unchecked Exception**（RuntimeException）：逻辑错误，不强制处理

## try-catch-finally

```java
try {
    int result = 10 / 0;                    // 可能抛出异常的代码
    System.out.println(result);
} catch (ArithmeticException e) {            // 捕获具体异常
    System.err.println("除零错误：" + e.getMessage());
} catch (Exception e) {                      // 捕获更宽泛的异常（父类放后面）
    System.err.println("未知错误：" + e.getClass().getSimpleName());
} finally {
    System.out.println("finally 块总是执行"); // 清理资源
}
```

**多 catch 注意**：子类异常必须写在父类前面，否则编译错误（unreachable）。

## finally 陷阱

```java
public static int test() {
    try {
        return 1;
    } finally {
        return 2;  // 最终返回 2！finally 中的 return 会覆盖 try 中的
    }
}
```

**最佳实践**：`finally` 块不应包含 `return` 语句，只做资源释放。

## throw 与 throws

```java
// throws：方法签名声明可能抛出的异常
public static int divide(int a, int b) throws ArithmeticException {
    if (b == 0) {
        // throw：手动抛出异常
        throw new ArithmeticException("除数不能为零");
    }
    return a / b;
}

// 调用方处理
try {
    int result = divide(10, 0);
} catch (ArithmeticException e) {
    System.err.println("计算失败: " + e.getMessage());
}
```

## 自定义异常

```java
// 业务异常（建议继承 RuntimeException）
public class InsufficientFundsException extends RuntimeException {
    private final double deficit;

    public InsufficientFundsException(double balance, double requested) {
        super(String.format("余额 %.2f 不足，请求 %.2f", balance, requested));
        this.deficit = requested - balance;
    }

    public double getDeficit() { return deficit; }
}

// 使用
public void withdraw(double amount) {
    if (amount > balance) {
        throw new InsufficientFundsException(balance, amount);
    }
    balance -= amount;
}
```

## try-with-resources（自动资源管理）

JDK 7 引入，实现了 `AutoCloseable` 接口的资源会自动关闭，无需显式 `finally`：

```java
// 旧式写法（繁琐）
BufferedReader reader = null;
try {
    reader = new BufferedReader(new FileReader("data.txt"));
    String line = reader.readLine();
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (reader != null) {
        try { reader.close(); } catch (IOException ignored) { }
    }
}

// try-with-resources（简洁安全）
try (BufferedReader reader = new BufferedReader(new FileReader("data.txt"));
     BufferedWriter writer = new BufferedWriter(new FileWriter("out.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line.toUpperCase());
        writer.newLine();
    }
} catch (IOException e) {
    e.printStackTrace();
}
// reader 和 writer 自动关闭，顺序与打开顺序相反
```

JDK 9 起，try-with-resources 支持使用已有的 final 或 effectively final 变量：

```java
BufferedReader r = new BufferedReader(new FileReader("data.txt"));
try (r) {   // 直接引用外部变量
    r.readLine();
}
```

## 异常处理的最佳实践

1. **只捕获能处理的异常** — 不要写空 `catch` 块吞掉异常
2. **尽量精确** — 捕获最具体的异常类型，别一上来就 `catch (Exception e)`
3. **保留原始信息** — 封装异常时传入原异常：`throw new ServiceException("xxx", e)`
4. **不要用异常控制业务流程** — 异常有性能开销，正常流程用 `if` 判断
5. **日志记录** — catch 中至少记录日志，方便排查问题

```java
// 好：用 if 处理正常业务逻辑
if (map.containsKey(key)) {
    return map.get(key);
}

// 坏：用异常控制流程
try {
    return map.get(key);
} catch (NullPointerException e) {
    return null;
}
```
