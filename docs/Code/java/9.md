# 泛型（Generics）详解

泛型（Generics）是 JDK 5 引入的最重要特性之一，让类型参数化，从而实现编译时类型安全检查。没有泛型，集合框架将充满 `Object` 转换和 `ClassCastException`。

## 类型参数（Type Parameter）

```java
// 泛型类
public class Box<T> {
    private T content;

    public void set(T content) { this.content = content; }
    public T get() { return content; }
}

Box<String> stringBox = new Box<>();
stringBox.set("Hello");
String s = stringBox.get();   // 无需强制转型，类型安全

Box<Integer> intBox = new Box<>();
intBox.set(42);
int value = intBox.get();     // 自动拆箱
```

常见的类型参数命名惯例：`T`（Type）、`E`（Element）、`K/V`（Key/Value）、`N`（Number）、`?`（通配符）。

## 有界类型参数（Bounded Type Parameter）

限制类型参数必须是某类的子类或实现某接口：

```java
// <T extends Comparable<T>> 表示 T 必须实现了 Comparable
public class SortedBox<T extends Comparable<T>> {
    private T first;
    private T second;

    public SortedBox(T first, T second) {
        this.first = first;
        this.second = second;
    }

    public T max() {
        return first.compareTo(second) >= 0 ? first : second;
    }
}

SortedBox<Integer> box = new SortedBox<>(3, 7);
System.out.println(box.max());  // 7
```

多重边界用 `&` 连接：`T extends Number & Comparable<T>`。类必须放前面，接口放后面。

## 通配符（Wildcard）

通配符用于在泛型方法或集合使用时表达更灵活的类型约束：

```java
// 上界通配符 ? extends T — 只读（生产者）
public static double sumOfList(List<? extends Number> list) {
    double sum = 0;
    for (Number n : list) {
        sum += n.doubleValue();
    }
    // list.add(10);  编译错误！不知道具体是哪种 Number 子类型
    return sum;
}

List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(1.5, 2.5, 3.5);
System.out.println(sumOfList(ints));     // 6.0
System.out.println(sumOfList(doubles));  // 7.5

// 下界通配符 ? super T — 只写（消费者）
public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}

List<Number> numbers = new ArrayList<>();
addNumbers(numbers);    // Number 是 Integer 的父类，可以接收

List<Object> objects = new ArrayList<>();
addNumbers(objects);    // Object 也是 Integer 的父类
```

**PECS 口诀**：Producer Extends, Consumer Super — `? extends` 用于读取（生产者），`? super` 用于写入（消费者）。

## 泛型方法

```java
// 在返回值前声明类型参数
public static <T> T firstOrElse(List<T> list, T fallback) {
    return list.isEmpty() ? fallback : list.get(0);
}

// 多类型参数
public static <K, V> Map<K, V> singletonMap(K key, V value) {
    Map<K, V> map = new HashMap<>();
    map.put(key, value);
    return map;
}

// 调用时编译器自动推断
String f = firstOrElse(List.of("a", "b", "c"), "default");
Map<String, Integer> m = singletonMap("age", 30);
```

## 类型擦除（Type Erasure）

这是 Java 泛型最重要的底层机制 —— 泛型信息只存在于编译期，编译后会被擦除：

```java
// 编译前
List<String> strings = new ArrayList<>();
strings.add("hello");
String s = strings.get(0);

// 编译后等效代码（类型信息被擦除）
List strings = new ArrayList();
strings.add("hello");
String s = (String) strings.get(0);   // 编译器自动插入强制转型

// 因此以下代码返回 true（类型被擦除）
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass());  // true
```

类型擦除的后果：
1. 无法在运行时检查泛型类型（`instanceof` 不能带类型参数）
2. 无法创建泛型数组（`new T[10]` 非法）
3. 无法创建参数化类型的实例（`new T()` 非法）

## 实战：泛型 DAO 层

```java
public interface BaseRepository<T, ID> {
    Optional<T> findById(ID id);
    List<T> findAll();
    void save(T entity);
    void deleteById(ID id);
}

public class UserRepository implements BaseRepository<User, Long> {
    private Map<Long, User> store = new HashMap<>();

    @Override
    public Optional<User> findById(Long id) {
        return Optional.ofNullable(store.get(id));
    }

    @Override
    public List<User> findAll() {
        return new ArrayList<>(store.values());
    }

    @Override
    public void save(User entity) {
        store.put(entity.getId(), entity);
    }

    @Override
    public void deleteById(Long id) {
        store.remove(id);
    }
}
```

泛型让 Java 代码在保持类型安全的同时获得良好复用性，是构建可扩展框架的基础能力。
