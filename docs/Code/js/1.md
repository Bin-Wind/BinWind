# JavaScript 运行环境与开发工具

JavaScript 最初只能在浏览器中运行，如今已扩展至服务端、桌面应用甚至嵌入式设备。了解运行环境和开发工具是掌握 JS 的第一步。

## 浏览器控制台（Browser Console）

每个现代浏览器都内置了开发者工具（DevTools），控制台是最直接的 JS 交互环境。

打开方式：`F12` 或 `Ctrl+Shift+I`（Windows）/ `Cmd+Option+I`（Mac）。

```javascript
// 在 Console 中直接执行 JS 代码
console.log("Hello, World!");
console.table([{ name: "Alice", age: 25 }, { name: "Bob", age: 30 }]);
console.time("loop");
for (let i = 0; i < 1000000; i++) {}
console.timeEnd("loop"); // loop: ~2ms
```

常用 Console API：
- `console.log()` / `console.warn()` / `console.error()` — 分级日志
- `console.table()` — 表格化展示数组/对象
- `console.group()` / `console.groupEnd()` — 分组日志
- `console.time()` / `console.timeEnd()` — 性能计时

## Node.js 环境

Node.js 让 JavaScript 脱离了浏览器，基于 Chrome V8 引擎构建。安装后通过终端使用：

```bash
node -v          # 查看版本
node script.js   # 运行脚本
node             # 进入 REPL 交互模式
```

```javascript
// Node.js 中的模块系统
const fs = require("fs");
const path = require("path");

// 读取文件
fs.readFile("data.json", "utf-8", (err, data) => {
  if (err) throw err;
  console.log(JSON.parse(data));
});
```

Node.js 核心能力：文件系统（fs）、HTTP 服务（http）、路径处理（path）、流（stream）、事件（events）。

## VS Code 配置

Visual Studio Code 是 JS 开发的首选编辑器。推荐扩展：

- **ESLint** — 代码规范检查
- **Prettier** — 代码格式化
- **JavaScript (ES6) code snippets** — 代码片段
- **Quokka.js** — 实时反馈 JS 执行结果

VS Code 内置了优秀的调试功能。创建 `.vscode/launch.json` 配置调试：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "program": "${workspaceFolder}/index.js"
    }
  ]
}
```

设置断点后按 `F5` 启动调试，可查看变量值、调用堆栈、单步执行。

## Quokka.js 实时反馈

Quokka 是一款"刮刮卡"式工具，在编辑器中直接显示代码执行结果，无需运行整个文件。

```javascript
const arr = [1, 2, 3];
const doubled = arr.map(x => x * 2);
doubled // 按 Ctrl+Shift+Q 启动 Quokka，注释处会显示 [2, 4, 6]
```

## 调试方法（Debugging Techniques）

**1. Console 调试**

```javascript
function calculate(a, b) {
  console.log("输入参数:", { a, b }); // 对象简写打印
  const result = a + b;
  console.log("计算结果:", result);
  return result;
}
```

**2. debugger 语句**

```javascript
function buggyFunction(data) {
  debugger; // 浏览器/Node 会在此行暂停
  return data.map(item => item.value * 2);
}
```

**3. Node.js 调试**

```bash
node --inspect-brk script.js  # 启动时暂停，等待调试器连接
node inspect script.js        # 命令行调试器
```

然后在 Chrome 地址栏输入 `chrome://inspect` 连接调试。

## 总结

- 浏览器 Console 适合快速实验和前端调试
- Node.js 让 JS 具备服务端能力
- VS Code + Quokka 提升开发效率
- 掌握多种调试手段，能迅速定位问题

选择适合场景的工具组合，是高效开发的基础。
