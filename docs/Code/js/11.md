# 异步编程

JavaScript 是单线程语言，但通过异步机制可以高效处理 I/O、网络请求等耗时操作。从回调到 Promise 再到 Async/Await，异步编程模式日趋优雅。

## 回调函数（Callback）— 最早的异步方案

```javascript
// 典型的回调模式
function fetchData(url, callback) {
  const xhr = new XMLHttpRequest();
  xhr.open("GET", url);
  xhr.onload = () => callback(null, xhr.responseText);
  xhr.onerror = () => callback(new Error("请求失败"));
  xhr.send();
}

fetchData("/api/user", (err, data) => {
  if (err) return console.error(err);
  console.log(JSON.parse(data));
});
```

**回调地狱（Callback Hell）** — 多层嵌套导致代码难以维护：

```javascript
// ❌ 回调地狱 — 层层嵌套
getUser(userId, (err, user) => {
  if (err) return handleError(err);
  getOrders(user.id, (err, orders) => {
    if (err) return handleError(err);
    getOrderDetails(orders[0].id, (err, details) => {
      if (err) return handleError(err);
      console.log(details);
    });
  });
});
```

## Promise — 链式调用的优雅方案

Promise 代表一个异步操作的最终结果，有三种状态：**pending**（进行中）、**fulfilled**（成功）、**rejected**（失败）。

```javascript
// 创建 Promise
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = Math.random() > 0.5;
    if (success) {
      resolve("操作成功");
    } else {
      reject(new Error("操作失败"));
    }
  }, 1000);
});

// 消费 Promise
promise
  .then(result => {
    console.log(result);        // "操作成功"
    return result.toUpperCase(); // 返回值会被包装为 Promise
  })
  .then(upper => console.log(upper))
  .catch(err => console.error(err.message)) // 捕获链上任意错误
  .finally(() => console.log("操作完成"));  // 无论成败都会执行

// 常用 Promise 静态方法
Promise.resolve("立即成功");   // 创建已解决的 Promise
Promise.reject(new Error());  // 创建已拒绝的 Promise
```

**Promise 链式调用** — 解决回调地狱：

```javascript
// ✅ Promise 链 — 扁平化
getUser(userId)
  .then(user => getOrders(user.id))
  .then(orders => getOrderDetails(orders[0].id))
  .then(details => console.log(details))
  .catch(err => console.error("出错了:", err));
```

## Promise 并发控制

```javascript
const p1 = fetch("/api/user").then(r => r.json());
const p2 = fetch("/api/posts").then(r => r.json());
const p3 = fetch("/api/comments").then(r => r.json());

// Promise.all — 全部成功才成功，一个失败就失败
const [user, posts, comments] = await Promise.all([p1, p2, p3]);

// Promise.allSettled — 等待全部完成（无论成败），返回每项结果状态
const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => {
  if (r.status === "fulfilled") {
    console.log("成功:", r.value);
  } else {
    console.log("失败:", r.reason);
  }
});

// Promise.race — 返回第一个完成的（无论成败）
const timeout = new Promise((_, reject) =>
  setTimeout(() => reject(new Error("超时")), 5000)
);
const data = await Promise.race([fetch("/api/data"), timeout]);

// Promise.any — 返回第一个成功的，全部失败才 reject
const firstSuccess = await Promise.any([p1, p2, p3]);
```

## Async / Await — 同步风格的异步代码

Async/Await 是 Promise 的语法糖，让异步代码看起来像同步代码。

```javascript
// async 函数始终返回 Promise
async function greet() {
  return "hello";  // 等价于 return Promise.resolve("hello")
}
greet().then(console.log); // "hello"

// await 暂停函数执行直到 Promise 完成
async function loadUser(id) {
  try {
    const response = await fetch(`/api/users/${id}`);

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }

    const user = await response.json();
    return user;
  } catch (err) {
    console.error("加载用户失败:", err.message);
    throw err; // 重新抛出，让调用方处理
  }
}

// 使用
const user = await loadUser(123);

// 并发 vs 串行
async function loadData() {
  // ❌ 串行 — 总耗时累加
  const user = await fetchUser();
  const posts = await fetchPosts();
  const comments = await fetchComments();

  // ✅ 并发 — 总耗时取决于最慢的那个
  const [user2, posts2, comments2] = await Promise.all([
    fetchUser(),
    fetchPosts(),
    fetchComments(),
  ]);
}
```

## fetch API

`fetch` 是现代浏览器内置的 HTTP 请求 API，返回 Promise。

```javascript
// GET 请求
const response = await fetch("/api/users");
const data = await response.json(); // .json() 也返回 Promise

// POST 请求
const newUser = await fetch("/api/users", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ name: "Alice", age: 25 }),
});

// 带超时的 fetch
async function fetchWithTimeout(url, options = {}, timeout = 5000) {
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);

  try {
    const response = await fetch(url, {
      ...options,
      signal: controller.signal,
    });
    clearTimeout(timeoutId);
    return response;
  } catch (err) {
    clearTimeout(timeoutId);
    if (err.name === "AbortError") {
      throw new Error("请求超时");
    }
    throw err;
  }
}

// 错误处理 — fetch 只在网络错误时 reject，HTTP 错误状态码不会 reject
async function safeFetch(url) {
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }
  return response.json();
}
```

## 实际应用：带重试的请求

```javascript
async function fetchWithRetry(url, options = {}, retries = 3, delay = 1000) {
  for (let i = 0; i <= retries; i++) {
    try {
      const response = await fetch(url, options);
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}`);
      }
      return await response.json();
    } catch (err) {
      if (i === retries) throw err;
      console.log(`第 ${i + 1} 次重试，${delay}ms 后重试...`);
      await new Promise(resolve => setTimeout(resolve, delay * (i + 1)));
    }
  }
}
```

## 总结

| 阶段 | 写法 | 优点 | 缺点 |
|------|------|------|------|
| Callback | 嵌套回调 | 简单直接 | 回调地狱 |
| Promise | `.then().catch()` | 链式调用、错误统一 | 仍有一定复杂度 |
| Async/Await | `await` | 同步风格、try/catch | 需注意并发优化 |

- 默认使用 `async/await` + `try/catch`
- 独立请求用 `Promise.all` 并发，而非串行 `await`
- `Promise.allSettled` 适合需要全部结果（含失败）的场景
- `Promise.race` / `Promise.any` 处理超时和竞速
- `fetch` 不会因 HTTP 错误码 reject，记得检查 `response.ok`
