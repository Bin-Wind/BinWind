# JavaScript Event Loop 详解

Event Loop（事件循环）是 JavaScript 运行时的核心机制，它协调调用栈、任务队列和微任务队列，使得单线程的 JS 能够高效处理异步操作。

## JavaScript 的单线程模型

JavaScript 在单个主线程上执行，一次只能做一件事。但通过 Event Loop，它将耗时任务交给浏览器/Node 的其他线程（如网络 I/O、定时器），完成后将回调推入队列等待执行。

```javascript
console.log("1: 同步代码");

setTimeout(() => {
  console.log("2: setTimeout 回调");
}, 0);

Promise.resolve().then(() => {
  console.log("3: Promise 微任务");
});

console.log("4: 同步代码");

// 输出顺序：1 → 4 → 3 → 2
```

## 核心组件

Event Loop 由四个核心部分组成：

1. **Call Stack（调用栈）** — 追踪当前正在执行的函数。LIFO（后进先出）。
2. **Task Queue（宏任务队列/Macrotask Queue）** — 存放回调任务（setTimeout、I/O、事件等）。
3. **Microtask Queue（微任务队列）** — 存放高优先级任务（Promise.then、MutationObserver、queueMicrotask）。
4. **Event Loop** — 不断检查调用栈是否为空，为空时优先清空微任务队列，再从宏任务队列取一个任务。

```javascript
// 可视化调用栈
function multiply(a, b) { return a * b; }
function square(n) { return multiply(n, n); }
function printSquare(n) {
  const result = square(n);
  console.log(result);
}
printSquare(5);
// 调用栈：printSquare → square → multiply → 返回 → 返回 → console.log → 返回
```

## 微任务 vs 宏任务

**关键规则**：每个宏任务执行完毕后，在取下一个宏任务之前，**必须清空整个微任务队列**。

```javascript
console.log("start");

setTimeout(() => {
  console.log("宏任务 1");
  Promise.resolve().then(() => console.log("微任务 1-1"));
}, 0);

setTimeout(() => {
  console.log("宏任务 2");
}, 0);

Promise.resolve().then(() => {
  console.log("微任务 init-1");
}).then(() => {
  console.log("微任务 init-2");
});

console.log("end");

// 输出：
// start
// end
// 微任务 init-1
// 微任务 init-2
// 宏任务 1
// 微任务 1-1
// 宏任务 2
```

**图解执行步骤**：

1. 执行同步代码：`start` → `end`。调用栈清空。
2. 清空微任务队列：`微任务 init-1` → `微任务 init-2`。
3. 取出一个宏任务（setTimeout 1）：`宏任务 1`。
4. 宏任务 1 创建了微任务 `微任务 1-1`，调用栈清空后立即清空微任务。
5. 取出下一个宏任务（setTimeout 2）：`宏任务 2`。

## 各类任务的归属

| 类型 | 示例 |
|------|------|
| **宏任务** | `setTimeout`、`setInterval`、`setImmediate`(Node)、I/O 回调、UI 渲染回调 |
| **微任务** | `Promise.then/catch/finally`、`queueMicrotask()`、`MutationObserver`、`process.nextTick`(Node) |

```javascript
// 微任务中创建微任务 — 小心无限循环
function microLoop() {
  Promise.resolve().then(() => {
    console.log("又一个微任务");
    microLoop(); // 微任务不断产生新微任务，阻塞宏任务和渲染
  });
}
// microLoop(); // 不要这样做！页面会卡死
```

## setTimeout 与 Promise 的经典题

```javascript
console.log("1");

setTimeout(() => console.log("2"), 0);

Promise.resolve().then(() => console.log("3"));

console.log("4");

setTimeout(() => console.log("5"), 0);

Promise.resolve().then(() => {
  console.log("6");
  return Promise.resolve("7");
}).then(val => console.log(val));

console.log("8");

// 输出：1 → 4 → 8 → 3 → 6 → 7 → 2 → 5
```

**分析**：同步代码（1, 4, 8）→ 清空微任务（3, 6, 7）→ 取第一个宏任务（2）→ 清空微任务（无）→ 取第二个宏任务（5）。

## requestAnimationFrame

`requestAnimationFrame`（rAF）在浏览器**每帧渲染之前**执行，位于宏任务之后、渲染之前。它的执行时机和屏幕刷新率绑定（通常 60fps，即约 16.6ms/帧）。

```javascript
// 事件循环中一帧的流程：
// 1. 执行一个宏任务
// 2. 清空微任务队列
// 3. 执行 requestAnimationFrame 回调（如有）
// 4. 浏览器渲染（样式计算、布局、绘制）
// 5. 如有空闲时间执行 requestIdleCallback

setTimeout(() => {
  console.log("宏任务");
  requestAnimationFrame(() => console.log("rAF"));
}, 0);

Promise.resolve().then(() => console.log("微任务"));

// 输出：微任务 → 宏任务 → rAF
```

```javascript
// requestAnimationFrame 常用于动画
function animate() {
  box.style.transform = `translateX(${x}px)`;
  x += 2;
  if (x < 200) {
    requestAnimationFrame(animate);
  }
}
let x = 0;
requestAnimationFrame(animate);
```

## Node.js 的 Event Loop

Node.js 的 Event Loop 分为多个阶段，每阶段有独立的宏任务队列：

```
   ┌───────────────────────────┐
┌─>│  timers (setTimeout/setInterval)  │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │  pending callbacks (I/O)  │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │  idle, prepare            │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │  poll (I/O, 新连接)        │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │  check (setImmediate)     │
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
└──┤  close callbacks          │
   └───────────────────────────┘
```

```javascript
// Node.js 中 setTimeout vs setImmediate
setTimeout(() => console.log("timeout"), 0);
setImmediate(() => console.log("immediate"));

// 输出不确定（取决于 Event Loop 启动时间）
// 如果在 poll 阶段之外：timeout → immediate
// 如果在 poll 阶段之内：immediate → timeout

// process.nextTick — Node 特有，优先级高于 Promise
console.log("1");
process.nextTick(() => console.log("nextTick"));
Promise.resolve().then(() => console.log("promise"));
console.log("2");
// 输出：1 → 2 → nextTick → promise
```

## 常见陷阱与最佳实践

```javascript
// ❌ 阻塞主线程 — 长循环应拆分
function heavyTask() {
  for (let i = 0; i < 10000000000; i++) {} // 页面冻结
}

// ✅ 使用 setTimeout 拆分
function chunkedTask(items, processFn, done) {
  let index = 0;
  function processChunk() {
    const chunkSize = 100;
    const end = Math.min(index + chunkSize, items.length);
    while (index < end) {
      processFn(items[index], index);
      index++;
    }
    if (index < items.length) {
      setTimeout(processChunk, 0); // 让出主线程
    } else {
      done();
    }
  }
  setTimeout(processChunk, 0);
}

// ✅ 或使用 requestIdleCallback 利用空闲时间
function idleTask(items) {
  let i = 0;
  function processChunk(deadline) {
    while (i < items.length && deadline.timeRemaining() > 1) {
      processItem(items[i]);
      i++;
    }
    if (i < items.length) {
      requestIdleCallback(processChunk);
    }
  }
  requestIdleCallback(processChunk);
}
```

## 总结

- Event Loop 的核心：**一个宏任务 → 清空所有微任务 → 下一个宏任务**
- Promise 回调是微任务，setTimeout 是宏任务
- 微任务可以无限产生新微任务，小心阻塞渲染
- `requestAnimationFrame` 适合动画，在渲染前执行
- Node.js 的 Event Loop 有多阶段宏任务队列，`process.nextTick` 优先级最高
- 长任务应拆分为小块，避免阻塞 Event Loop
