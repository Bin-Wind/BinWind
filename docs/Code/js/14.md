# 模块化（ESM）详解

ES Modules（ESM）是 JavaScript 官方模块化方案，自 ES2015 起成为语言标准。它从根本上改变了代码组织方式，让大型应用拆分、按需加载、依赖管理变得自然。本文从 named/default export 讲到动态导入和树摇（tree shaking）原理。

## 命名导出与默认导出

ESM 提供两种导出方式：`export` 命名导出可输出多个值，`export default` 只能输出一个。

```javascript
// math.js — 命名导出（named export）
export const PI = 3.14159;
export function add(a, b) { return a + b; }
export class Calculator { /* ... */ }

// 也可以一次性导出
const multiply = (a, b) => a * b;
const divide = (a, b) => a / b;
export { multiply, divide };

// user.js — 默认导出（default export）
export default class User {
  constructor(name) { this.name = name; }
}
```

导入时的语法对应关系：

```javascript
// 命名导入 — 必须用花括号，名字必须匹配
import { PI, add, Calculator } from './math.js';

// 可以起别名
import { multiply as mul } from './math.js';

// 使用 namespace import 全部导入
import * as math from './math.js';
console.log(math.PI, math.add(1, 2));

// 默认导入 — 不用花括号，名字任意
import User from './user.js';

// 混合导入
import User, { PI } from './module.js';
```

## 动态导入（Dynamic Import）

`import()` 函数返回 Promise，实现按需加载。这对首屏优化和条件加载至关重要。

```javascript
// 条件加载：只在需要时加载组件
async function loadChart(type) {
  if (type === 'bar') {
    const { default: BarChart } = await import('./charts/BarChart.js');
    return new BarChart();
  }
  if (type === 'line') {
    const { default: LineChart } = await import('./charts/LineChart.js');
    return new LineChart();
  }
}

// 路由懒加载（Vue Router / React.lazy 底层原理）
const routes = [
  {
    path: '/dashboard',
    component: () => import('./pages/Dashboard.js')
  }
];

// 借助 Promise.all 并行加载
const [moduleA, moduleB] = await Promise.all([
  import('./moduleA.js'),
  import('./moduleB.js')
]);
```

## import.meta 元信息

`import.meta` 提供当前模块上下文信息，最常见的是 `import.meta.url`。

```javascript
// import.meta.url — 当前文件 URL
console.log(import.meta.url);
// Browser: "https://example.com/js/app.js"
// Node:    "file:///project/src/app.js"

// Vite 拓展的 import.meta.env
if (import.meta.env?.MODE === 'development') {
  console.log('开发模式');
}
```

## ESM vs CommonJS 核心差异

理解二者差异有助于避免依赖引入报错和打包配置问题。

```javascript
// ========= CommonJS（同步加载，Node.js 默认）==========
const fs = require('fs');
const lodash = require('lodash');
module.exports = function greet(name) {
  return `Hello, ${name}`;
};

// ========= ES Modules（异步加载，浏览器 + Node 都支持）==========
import fs from 'node:fs';
import { debounce } from 'lodash-es';

// 导出绑定是实时引用（live binding）
export let count = 0;
export function increment() { count++; }
// 导入方会看到 count 的实时变化
```

关键区别总结：

| 特性 | CommonJS | ES Modules |
|------|----------|------------|
| 语法 | `require` / `module.exports` | `import` / `export` |
| 加载时机 | 运行时同步 | 编译时静态分析 |
| 导出复制 vs 引用 | 值拷贝 | 实时绑定（live binding） |
| 树摇支持 | 不支持 | 原生支持 |
| 顶层 await | 不支持 | 支持 |
| 使用场景 | Node 老项目 | 现代 Web + Node |

## ESM 在 Node.js 中的使用

Node 通过 `package.json` 的 `"type"` 字段或 `.mjs` 扩展名启用 ESM。

```json
{ "type": "module" }
```

## 打包与树摇（Tree Shaking）

ESM 静态分析特性使打包工具能在编译阶段识别未使用代码并移除：

```javascript
// utils.js
export function used() { return 'used'; }
export function unused() { return 'unused'; }
// main.js 只导入 used → unused 永远不会出现在最终打包结果中
```

要点回顾：默认导出适合"一个模块一件事"，命名导出适合工具函数集合；动态导入实现按需加载；ESM 的静态结构是 tree shaking 的前提。掌握 ESM，就是掌握了现代 JS 工程的基石。
