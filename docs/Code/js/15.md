# 错误处理与调试

错误是程序的一部分，优雅地处理它们是区分新手和熟手的标志。JavaScript 提供了一套完整的错误处理机制——从 `try/catch` 到自定义错误类型，配合浏览器 DevTools 和结构化日志，让调试更高效。

## try/catch/finally 基础

`try` 包裹可能出错的代码，`catch` 捕获并处理，`finally` 无论是否出错都会执行。

```javascript
function parseUserData(jsonStr) {
  try {
    const user = JSON.parse(jsonStr);
    if (!user.name) throw new Error('缺少 name 字段');
    return user;
  } catch (err) {
    console.error('解析失败:', err.message);
    return { name: 'Guest' };  // 提供降级方案
  } finally {
    hideLoadingSpinner(); // 无论如何都关闭 loading
  }
}
```

`catch` 的错误对象在 ES2022 后不再自动绑定：

```javascript
try { /* ... */ } catch { /* 不关心错误内容时直接省略参数 */ }
```

## throw 与自定义 Error 类型

`throw` 可以抛出任意值，但继承 `Error` 类能让调用方统一处理。

```javascript
// 自定义错误类型 — 提供语义化的错误分类
class NetworkError extends Error {
  constructor(message, statusCode) {
    super(message);
    this.name = 'NetworkError';
    this.statusCode = statusCode;
  }
}

class ValidationError extends Error {
  constructor(field, message) {
    super(message);
    this.name = 'ValidationError';
    this.field = field;
  }
}

// 使用
async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`);
  if (!res.ok) throw new NetworkError('请求失败', res.status);
  return res.json();
}

// 调用方可以按类型区分处理
try {
  const user = await fetchUser(123);
} catch (err) {
  if (err instanceof NetworkError) {
    showToast('网络异常，请稍后重试');
  } else if (err instanceof ValidationError) {
    highlightField(err.field);
  } else {
    console.error('未知错误', err);
  }
}
```

## JavaScript 内置 Error 类型

| 类型 | 触发场景 |
|------|---------|
| `Error` | 通用错误 |
| `SyntaxError` | JSON.parse 非法字符串 |
| `TypeError` | 对非函数调用、访问 undefined 属性 |
| `ReferenceError` | 引用未定义变量 |
| `RangeError` | 数字越界，如 `new Array(-1)` |
| `URIError` | `decodeURIComponent` 接收非法 URI |

## 全局错误捕获

未处理的错误需要全局兜底，避免页面白屏。

```javascript
// 同步错误 & 未 catch 的 Promise
window.addEventListener('error', (event) => {
  console.error('全局错误:', event.error?.message);
  reportError({ message: event.message, filename: event.filename, lineno: event.lineno });
});

// Promise 未处理的 rejection
window.addEventListener('unhandledrejection', (event) => {
  console.error('未捕获的 Promise 错误:', event.reason);
  reportError({ type: 'unhandledRejection', message: event.reason?.message });
  event.preventDefault();  // 阻止控制台默认警告
});
```

## Console 方法全览

`console` 远不止 `log`，合理使用能大幅提升调试效率。

```javascript
console.log('普通日志');
console.info('信息提示');
console.warn('警告', { reason: 'deprecated parameter' });
console.error('错误堆栈', new Error('出错了'));

// 结构化输出
console.table([
  { name: 'Alice', score: 95 },
  { name: 'Bob', score: 87 }
]);

// 分组折叠
console.group('用户操作日志');
console.log('点击按钮');
console.log('发送请求');
console.groupEnd();

// 性能计量
console.time('dataTransform');
heavyComputation();
console.timeEnd('dataTransform');  // → dataTransform: 42.5ms

// 断言（只在条件为 false 时打印）
console.assert(value > 0, 'value 必须为正数', value);
```

## debugger 语句

在代码中加 `debugger`，浏览器 DevTools 开启时会自动在此处暂停。

```javascript
function calculateDiscount(price, level) {
  debugger;  // 执行到这里时自动暂停，可检查 price 和 level
  const rates = { normal: 1, silver: 0.9, gold: 0.8 };
  return price * (rates[level] ?? 1);
}
```

## Chrome DevTools 核心调试技巧

**Breakpoints（断点）**：Sources 面板中点击行号添加断点，右键可设置条件断点（`price > 1000`）或日志断点（不暂停只输出）。

**Watch 与 Call Stack**：右侧 Watch 面板可添加任意表达式实时求值；Call Stack 显示完整调用链，点击任意帧可跳转到对应代码。

**Network 面板**：看到红色状态码请求直接点开看 Request/Response payload，确认是参数问题还是后端返回异常。

**Performance 面板**：录制一段操作，分析火焰图定位长任务（long task > 50ms）。

**Sources Overrides**：在 Sources > Overrides 中指定本地文件夹，可直接在 DevTools 中修改线上代码并保存到本地验证修复。

## async/await 错误处理模式

```javascript
// 方案一：try/catch 包裹
async function loadData() {
  try {
    return await fetchData();
  } catch (err) {
    return defaultData; // 降级
  }
}

// 方案二：包装为 Result 类型 — 避免 try/catch 嵌套
async function to(promise) {
  try { return [null, await promise]; }
  catch (err) { return [err, null]; }
}
const [err, data] = await to(fetchData());
if (err) return handleError(err);
```

掌握错误处理，不是为了写出永不报错的程序，而是让错误发生时系统依然可控。
