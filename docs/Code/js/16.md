# 浏览器存储方案

浏览器提供了多种客户端存储手段——从最简单的 `localStorage` 到功能强大的 IndexedDB。选择哪种方案，取决于数据大小、持久化需求、安全考量。本文梳理四种主流方案及其使用场景。

## localStorage — 持久化键值存储

5MB 容量上限，数据永久存储（用户手动清除才消失），同源共享。

```javascript
// 基本 CRUD
localStorage.setItem('theme', 'dark');
const theme = localStorage.getItem('theme');          // 'dark'
localStorage.removeItem('theme');
localStorage.clear();                                // 清空所有

// 存储对象需要 JSON 序列化
const userSettings = { lang: 'zh', fontSize: 16 };
localStorage.setItem('settings', JSON.stringify(userSettings));
const restored = JSON.parse(localStorage.getItem('settings'));

// 监听存储变化（跨标签页同步）
window.addEventListener('storage', (e) => {
  console.log(`${e.key} 从 ${e.oldValue} 变为 ${e.newValue}`);
  // 注意：触发事件的标签页不会收到此事件，只有其他同源标签页会
});
```

常见用途：主题偏好、用户语言、草稿暂存。

## sessionStorage — 仅会话期间有效

API 与 `localStorage` 完全一样，区别是：关闭标签页后数据自动清除，且不同标签页之间隔离。

```javascript
sessionStorage.setItem('formDraft', JSON.stringify(formData));
// 刷新页面数据还在，关闭标签页后消失
// 适用于：表单临时草稿、多步骤向导中间状态、页面间临时传参
```

## Cookie — 服务端可读的小片段

每个 cookie 最大 4KB，每次 HTTP 请求自动携带，支持过期时间和安全标记。

```javascript
// 读取 cookie
function getCookie(name) {
  const match = document.cookie.match(new RegExp(`(?:^|; )${name}=([^;]*)`));
  return match ? decodeURIComponent(match[1]) : null;
}

// 设置 cookie（带安全标记）
function setCookie(name, value, days = 7) {
  const expires = new Date(Date.now() + days * 864e5).toUTCString();
  document.cookie = `${name}=${encodeURIComponent(value)}; expires=${expires}; path=/; SameSite=Lax; Secure`;
}
```

安全标记说明：

| 标记 | 作用 |
|------|------|
| `HttpOnly` | JS 无法读取，防 XSS 窃取 |
| `Secure` | 仅 HTTPS 传输 |
| `SameSite=Strict/Lax` | 防 CSRF — Lax 允许从链接跳转时携带 |

## IndexedDB — 客户端数据库

IndexedDB 提供结构化数据存储能力，异步 API，支持索引查询和事务，容量可达数百 MB。

```javascript
// 打开/创建数据库
function openDB() {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('MyApp', 1);

    request.onupgradeneeded = (e) => {
      const db = e.target.result;
      const store = db.createObjectStore('users', { keyPath: 'id' });
      store.createIndex('email', 'email', { unique: true });
    };

    request.onsuccess = (e) => resolve(e.target.result);
    request.onerror = (e) => reject(e.target.error);
  });
}

// 事务读写
async function saveUser(user) {
  const db = await openDB();
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  store.put(user);
  return new Promise((resolve, reject) => {
    tx.oncomplete = () => resolve();
    tx.onerror = (e) => reject(e.target.error);
  });
}

// 索引查询
async function findByEmail(email) {
  const db = await openDB();
  const tx = db.transaction('users', 'readonly');
  const index = tx.objectStore('users').index('email');
  const request = index.get(email);
  return new Promise((resolve, reject) => {
    request.onsuccess = () => resolve(request.result);
    request.onerror = () => reject(request.error);
  });
}
```

## 存储方案对比与选型

| 特性 | Cookie | localStorage | sessionStorage | IndexedDB |
|------|--------|-------------|----------------|-----------|
| 容量 | ~4KB | ~5MB | ~5MB | >100MB |
| 生命周期 | 可设过期 | 永久 | 标签页关闭 | 永久 |
| 请求携带 | 自动 | 否 | 否 | 否 |
| API 风格 | 字符串 | 同步键值 | 同步键值 | 异步事务 |
| 查询能力 | 无 | 仅 key | 仅 key | 索引查询 |

## 存储容量检测与安全

```javascript
// 安全写入（捕获配额超限）
function safeSetItem(key, value) {
  try {
    localStorage.setItem(key, value);
  } catch (e) {
    if (e.name === 'QuotaExceededError') {
      console.warn('存储空间已满，尝试清理旧数据');
    }
  }
}
```

浏览器存储是前端架构的基础设施。小数据用 `localStorage`，敏感数据上 `HttpOnly Cookie`，结构化大数据选 IndexedDB。没有银弹，只有合适的组合。
