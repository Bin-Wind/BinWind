# 事件循环与任务优先级

事件循环（Event Loop）是 JavaScript 的单线程异步模型核心。理解宏任务（macro task）和微任务（micro task）的执行顺序，才能写出可预测的异步代码，避免"为什么这行先执行"的困惑。

## 事件循环基础模型

JavaScript 引擎只有一个调用栈（Call Stack），异步操作通过任务队列实现非阻塞：

```
调用栈 → 微任务队列 → 宏任务队列 → 下一帧渲染 → 循环...
```

```javascript
console.log('1'); // 同步
setTimeout(() => console.log('2'), 0);  // 宏任务
Promise.resolve().then(() => console.log('3')); // 微任务
console.log('4');
// 输出: 1 → 4 → 3 → 2
```

执行过程：先跑完所有同步代码 → 清空微任务队列 → 取一个宏任务执行 → 再次清空微任务 → 渲染。

## 宏任务 vs 微任务

**宏任务（Macro Task）**：`setTimeout`、`setInterval`、`postMessage`、I/O、UI 渲染、`setImmediate`（Node）。

**微任务（Micro Task）**：`Promise.then/catch/finally`、`queueMicrotask`、`MutationObserver`。

```javascript
setTimeout(() => console.log('timeout'));
Promise.resolve().then(() => console.log('promise'));
queueMicrotask(() => console.log('microtask'));
// 输出: promise → microtask → timeout
// 原因：同一轮循环中，微任务队列全部执行完后，才取下一个宏任务
```

## queueMicrotask — 显式添加微任务

比 `Promise.resolve().then()` 更语意明确，开销也更小。

```javascript
// 确保回调在当前任务完成后、下一宏任务前执行
function batchUpdate(fn) {
  let pending = false;
  return (...args) => {
    if (!pending) {
      pending = true;
      queueMicrotask(() => {
        fn(...args);
        pending = false;
      });
    }
  };
}

// 批量 DOM 更新：同一轮多次调用只执行一次
const updateView = batchUpdate(() => console.log('只执行一次 DOM 更新'));
updateView();
updateView();
updateView(); // 微任务队列中只有一次调用
```

## MutationObserver — 监听 DOM 变化的微任务

`MutationObserver` 的回调以微任务形式执行，在当前宏任务结束前批量触发。

```javascript
const observer = new MutationObserver((mutations) => {
  mutations.forEach(mutation => {
    console.log('变化类型:', mutation.type);
    if (mutation.type === 'childList') {
      console.log('新增节点:', mutation.addedNodes.length,
                  '移除节点:', mutation.removedNodes.length);
    }
    if (mutation.type === 'attributes') {
      console.log('属性变化:', mutation.attributeName);
    }
  });
});

observer.observe(document.body, {
  childList: true,
  attributes: true,
  subtree: true,
  characterData: true
});

// 即使连续添加 3 个子元素，回调也只触发一次
list.appendChild(li1);
list.appendChild(li2);
list.appendChild(li3);
// MutationObserver 回调触发一次，mutations 数组包含全部 3 条记录
```

## requestAnimationFrame 与 requestIdleCallback

`requestAnimationFrame`（rAF）在渲染前执行，适合动画；`requestIdleCallback`（rIC）在浏览器空闲时执行，适合非关键任务。

```javascript
// rAF — 每秒约 60 帧
function animate() {
  element.style.transform = `translateX(${x}px)`;
  x += 1;
  requestAnimationFrame(animate);
}
requestAnimationFrame(animate);

// rIC — 闲置时执行低优先级工作
function processQueue(deadline) {
  while (tasks.length > 0 && deadline.timeRemaining() > 1) {
    const task = tasks.shift();
    task();
  }
  if (tasks.length > 0) {
    requestIdleCallback(processQueue);
  }
}
requestIdleCallback(processQueue);

// rIC 超时兜底：确保任务最终执行
requestIdleCallback(processQueue, { timeout: 2000 });
```

## 完整执行时序图

```javascript
function testOrder() {
  console.log('1. 同步代码');
  requestIdleCallback(() => console.log('6. 空闲回调'));
  requestAnimationFrame(() => console.log('5. 渲染前 rAF'));
  setTimeout(() => console.log('4. 宏任务'), 0);
  queueMicrotask(() => console.log('3. 微任务'));
  Promise.resolve().then(() => console.log('2. Promise 微任务'));
}
testOrder();
// 输出: 1 → 2 → 3 → 4 → 5 → 6
```

## 实战：避免长任务阻塞

耗时计算不应该直接放在同步栈中，会阻塞 UI 渲染：

```javascript
// ❌ 不好的做法 — 阻塞主线程 2 秒
function processHeavyArray(items) {
  return items.map(item => expensiveCalculation(item));
}

// ✅ 好做法 — 分片让出主线程
async function processChunked(items, chunkSize = 100) {
  const results = [];
  for (let i = 0; i < items.length; i += chunkSize) {
    const chunk = items.slice(i, i + chunkSize);
    results.push(...chunk.map(expensiveCalculation));
    // 将控制权交回事件循环，允许渲染和用户交互
    await new Promise(resolve => setTimeout(resolve, 0));
  }
  return results;
}
```

理解事件循环不是为了炫技面试，而是为了写出流畅的用户体验——让关键渲染不被阻塞，让用户交互立即响应。
