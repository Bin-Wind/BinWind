# 变量声明与作用域

JavaScript 提供了三种变量声明方式：`var`、`let` 和 `const`。它们的作用域规则和提升（Hoisting）行为截然不同，理解这些差异是写好 JS 的基础。

## var — 函数作用域

`var` 是 ES5 时代的声明方式，具有函数作用域（function scope），且存在变量提升（hoisting）。

```javascript
function demoVar() {
  console.log(x); // undefined，不会报错（变量提升）
  var x = 10;
  console.log(x); // 10

  if (true) {
    var y = 20; // var 没有块级作用域
  }
  console.log(y); // 20，在 if 块外也能访问
}
demoVar();
```

上面的代码等价于：

```javascript
function demoVar() {
  var x = undefined; // 提升到函数顶部
  var y = undefined;
  console.log(x);    // undefined
  x = 10;
  console.log(x);    // 10
  if (true) { y = 20; }
  console.log(y);    // 20
}
```

## let 与 const — 块级作用域

ES6 引入了 `let` 和 `const`，它们具有块级作用域（block scope），且存在暂时性死区（Temporal Dead Zone, TDZ）。

```javascript
{
  let a = 1;
  const b = 2;
  console.log(a, b); // 1 2
}
// console.log(a);   // ReferenceError: a is not defined
```

```javascript
// TDZ 演示
console.log(temp);   // ReferenceError: Cannot access 'temp' before initialization
let temp = "hello";
```

与 `var` 不同，`let` 和 `const` 声明的变量不会被提升到块顶部，从块开始到声明语句之间的区域就是 **TDZ**，访问该变量会抛出 `ReferenceError`。

## const 的特性

`const` 声明的变量必须立即初始化，且不能重新赋值。但对于引用类型（对象、数组），其内部属性是可以修改的。

```javascript
const PI = 3.14159;
// PI = 3; // TypeError: Assignment to constant variable

const user = { name: "Alice" };
user.name = "Bob";     // 允许：修改对象属性
user.age = 25;         // 允许：添加新属性
// user = {};          // TypeError：不能重新赋值

const arr = [1, 2, 3];
arr.push(4);           // 允许
// arr = [5, 6];       // TypeError
```

## 作用域链（Scope Chain）

JavaScript 采用词法作用域（lexical scoping），函数在定义时确定作用域，而非调用时。

```javascript
const globalVar = "global";

function outer() {
  const outerVar = "outer";

  function inner() {
    const innerVar = "inner";
    console.log(innerVar); // 当前作用域
    console.log(outerVar); // 上级作用域
    console.log(globalVar);// 全局作用域
  }

  inner();
}
outer();
```

访问变量时，JS 引擎从当前作用域逐级向外查找，直到全局作用域。如果全局都没有，抛出 `ReferenceError`。

## 最佳实践

```javascript
// ✅ 推荐：默认使用 const
const MAX_SIZE = 100;
const config = { debug: true };

// ✅ 需要重新赋值时使用 let
let counter = 0;
for (let i = 0; i < 10; i++) {
  counter += i;
}

// ❌ 避免使用 var
// var 的变量提升和函数作用域容易引发难以追踪的 bug

// ❌ 避免在循环中用 var
for (var j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100); // 输出 3, 3, 3
}
// ✅ 使用 let 修复
for (let k = 0; k < 3; k++) {
  setTimeout(() => console.log(k), 100); // 输出 0, 1, 2
}
```

## 总结

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函数 | 块 | 块 |
| 提升 | 初始化为 undefined | TDZ | TDZ |
| 可重复声明 | 允许 | 不允许 | 不允许 |
| 可重新赋值 | 允许 | 允许 | 不允许 |
| 全局声明时挂载到 window | 是 | 否 | 否 |

默认使用 `const`，需要重新赋值时用 `let`，不再使用 `var`。
