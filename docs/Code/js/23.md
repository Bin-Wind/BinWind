# 性能优化技巧

前端性能直接影响用户体验和转化率。本文是可落地的优化技巧集合，涵盖防抖节流、资源加载、代码分割、Web Worker 和内存排查。

## 防抖（Debounce）与节流（Throttle）

```javascript
// 防抖 — 最后一次操作后才执行（搜索输入、窗口 resize）
function debounce(fn, delay = 300) {
  let timer;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流 — 固定时间间隔最多执行一次（滚动、鼠标移动）
function throttle(fn, interval = 100) {
  let lastTime = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

input.addEventListener('input', debounce(onSearch, 400));
window.addEventListener('scroll', throttle(onScroll, 50));
```

## 代码分割（Code Splitting）

动态 `import()` 使构建工具将模块打包为独立 chunk：

```javascript
// 路由懒加载（React/Vue）
const Dashboard = () => import('./pages/Dashboard.vue');

// 条件加载大型依赖
async function exportExcel() {
  const { utils, writeFile } = await import('xlsx');
  const wb = utils.book_new();
  utils.book_append_sheet(wb, utils.json_to_sheet(data), 'Sheet1');
  writeFile(wb, 'report.xlsx');
}

// Magic comment 控制 chunk 名称（Webpack）
import(/* webpackChunkName: "charts" */ './charts.js');
```

## Web Worker — 把重计算移出主线程

主线程阻塞 50ms 以上用户就能感知。

```javascript
// worker.js
self.onmessage = (e) => {
  const { data, pattern } = e.data;
  const matches = [];
  const regex = new RegExp(pattern, 'g');
  let match;
  while ((match = regex.exec(data)) !== null) {
    matches.push({ index: match.index, text: match[0] });
  }
  self.postMessage({ matches });
};

// main.js
const worker = new Worker(new URL('./worker.js', import.meta.url));
worker.postMessage({ data: hugeText, pattern: '\\b\\w+' });
worker.onmessage = (e) => {
  console.log(`找到 ${e.data.matches.length} 个匹配`);
};
worker.terminate(); // 用完后释放
```

## 虚拟滚动（Virtual Scrolling）

万级列表不渲染全部 DOM，只渲染可视区：

```javascript
class VirtualList {
  constructor(container, items, rowHeight) {
    this.container = container;
    this.items = items;
    this.rowHeight = rowHeight;
    this.render();
  }

  render() {
    const visibleCount = Math.ceil(this.container.clientHeight / this.rowHeight) + 2;
    const scrollTop = this.container.scrollTop;
    const startIndex = Math.floor(scrollTop / this.rowHeight);
    const visibleItems = this.items.slice(startIndex, startIndex + visibleCount);

    this.container.innerHTML = visibleItems
      .map((item, i) => {
        const top = (startIndex + i) * this.rowHeight;
        return `<div style="position:absolute;top:${top}px;height:${this.rowHeight}px">${item}</div>`;
      })
      .join('');
    this.container.style.height = `${this.items.length * this.rowHeight}px`;
  }
}
```

## 内存泄漏排查

常见 JS 内存泄漏类型：

```javascript
// 1. 忘记清除定时器
class Component {
  startPolling() {
    this.timer = setInterval(() => this.fetch(), 5000);
  }
  destroy() { clearInterval(this.timer); }
}

// 2. 事件监听未移除
document.addEventListener('scroll', this.onScroll);
this.destroy = () => document.removeEventListener('scroll', this.onScroll);

// 3. 闭包意外持有大对象引用
function processData() {
  const bigData = new Array(10000000);
  return function handler() {
    console.log(bigData[0]); // 整个 bigData 都被闭包保留
  };
}

// 4. 游离 DOM（Detached DOM）
let button = document.getElementById('myBtn');
button.remove();
button = null; // 解除引用让 GC 回收
```

### Chrome DevTools 内存排查流程

1. Memory 面板 → Heap snapshot → 操作前拍快照
2. 执行疑似泄漏操作 → 再拍快照
3. Comparison 视图 → 按 Delta 降序 → 增加最多的对象即疑似泄漏源

## Performance API 与 Core Web Vitals

```javascript
// 测量自定义任务耗时
performance.mark('task-start');
doSomethingExpensive();
performance.mark('task-end');
const measure = performance.measure('task', 'task-start', 'task-end');
console.log(`耗时: ${measure.duration.toFixed(2)}ms`);

// LCP（Largest Contentful Paint）
new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(`LCP: ${entry.renderTime || entry.loadTime}ms`);
  }
}).observe({ type: 'largest-contentful-paint', buffered: true });

// CLS（Cumulative Layout Shift）
new PerformanceObserver((list) => {
  let cls = 0;
  for (const entry of list.getEntries()) {
    if (!entry.hadRecentInput) cls += entry.value;
  }
  console.log(`CLS: ${cls}`);
}).observe({ type: 'layout-shift', buffered: true });
```

优化的核心是建立监控 → 定位瓶颈 → 针对性优化 → 验证效果的循环。
