# 设计模式在 JavaScript 中的应用

设计模式是经过验证的代码组织方案。JavaScript 的动态特性让某些经典模式实现格外简洁，也让一些模式有更地道的写法。本文选择 6 个在前端开发中最实用的模式，给出可直接使用的 JS 实现。

## 单例模式（Singleton）

确保一个类只有一个实例，并提供全局访问点。

```javascript
// ES Module 天然单例 — 模块只执行一次
// config.js
export const config = { apiBase: '/api', theme: 'dark' };
// 所有 import config from './config.js' 引用同一个对象

// 类单例 — 懒初始化（private static field）
class Database {
  static #instance = null;
  constructor() {
    if (Database.#instance) return Database.#instance;
    this.connection = this.connect();
    Database.#instance = this;
  }
  connect() { return { /* 数据库连接对象 */ }; }
  query(sql) { return this.connection.execute(sql); }
}
```

## 发布-订阅模式（Pub-Sub）

通过事件总线解耦组件间通信。

```javascript
class EventBus {
  #handlers = new Map();

  on(event, handler) {
    if (!this.#handlers.has(event)) this.#handlers.set(event, new Set());
    this.#handlers.get(event).add(handler);
    return () => this.off(event, handler); // 返回取消订阅函数
  }

  off(event, handler) {
    this.#handlers.get(event)?.delete(handler);
  }

  emit(event, data) {
    this.#handlers.get(event)?.forEach(handler => handler(data));
  }

  once(event, handler) {
    const wrapper = (data) => {
      handler(data);
      this.off(event, wrapper);
    };
    this.on(event, wrapper);
  }
}

// 使用 — 组件间解耦通信
const bus = new EventBus();
bus.on('user:login', (user) => updateHeader(user));
bus.on('user:login', (user) => loadCart(user.id));
bus.emit('user:login', { id: 1, name: 'Alice' });
```

## 工厂模式（Factory）

封装对象创建逻辑。

```javascript
// 根据类型创建不同 UI 组件
class ButtonFactory {
  static create(type, options) {
    const types = {
      primary: { class: 'btn-primary', icon: null },
      danger:  { class: 'btn-danger',  icon: '⚠' },
      ghost:   { class: 'btn-ghost',   icon: null },
    };
    const config = types[type] || types.primary;
    const btn = document.createElement('button');
    btn.className = `btn ${config.class}`;
    btn.textContent = (config.icon ? config.icon + ' ' : '') + options.label;
    btn.onclick = options.onClick;
    return btn;
  }
}

// 验证器工厂
function createValidator(rules) {
  return (values) => {
    const errors = {};
    for (const [field, validators] of Object.entries(rules)) {
      for (const v of validators) {
        const error = v(values[field], values);
        if (error) { errors[field] = error; break; }
      }
    }
    return Object.keys(errors).length === 0 ? null : errors;
  };
}
```

## 策略模式（Strategy）

定义一组可互换的算法，运行时动态选择。

```javascript
// 运费计算策略
const shippingStrategies = {
  standard: (weight) => weight * 0.5 + 5,
  express:  (weight) => weight * 1.2 + 12,
  overnight: (weight) => weight * 2.5 + 25,
  free:     () => 0,
};

function calculateShipping(weight, method) {
  const strategy = shippingStrategies[method];
  if (!strategy) throw new Error(`不支持的发货方式: ${method}`);
  return strategy(weight);
}

// 表单提交策略
const submitStrategies = {
  json: (form) => JSON.stringify(Object.fromEntries(new FormData(form))),
  multipart: (form) => new FormData(form),
  urlencoded: (form) => new URLSearchParams(new FormData(form)).toString(),
};
```

## 装饰器模式（Decorator）

不修改原有函数，动态添加功能。

```javascript
function withRetry(fn, maxRetries = 3) {
  return async (...args) => {
    for (let i = 0; i <= maxRetries; i++) {
      try { return await fn(...args); } catch (err) {
        if (i === maxRetries) throw err;
        await new Promise(r => setTimeout(r, 1000 * Math.pow(2, i)));
      }
    }
  };
}

function withCache(fn, ttl = 60000) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    const cached = cache.get(key);
    if (cached && Date.now() - cached.time < ttl) return cached.value;
    const value = fn(...args);
    cache.set(key, { value, time: Date.now() });
    return value;
  };
}

// 组合多个装饰器
const fetchUser = withCache(withRetry(async (id) => {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
}), 30000);
```

## 适配器模式（Adapter）

转换接口，让不兼容的 API 可以协作。

```javascript
// 将不同支付 SDK 统一为同一个接口
class WechatPayAdapter {
  async pay(order) {
    return wx.requestPayment({
      out_trade_no: order.id,
      total_fee: order.amount, // 微信单位为分
      body: order.description,
    });
  }
}

class AliPayAdapter {
  async pay(order) {
    return my.tradePay({
      out_trade_no: order.id,
      total_amount: order.amount / 100, // 支付宝单位为元
      subject: order.description,
    });
  }
}

class PaymentService {
  #adapter;
  constructor(channel) {
    this.#adapter = channel === 'wechat'
      ? new WechatPayAdapter() : new AliPayAdapter();
  }
  async checkout(order) { return this.#adapter.pay(order); }
}
```

## 模式选择指南

| 场景 | 推荐模式 |
|------|---------|
| 全局配置/数据库连接 | 单例 |
| 跨组件通信 | 发布-订阅 |
| 条件创建对象 | 工厂 |
| 多条件分支逻辑 | 策略 |
| 非侵入式增强 | 装饰器 |
| 对接不同第三方 | 适配器 |

设计模式不是拿来硬套的。当你发现代码中有大量 if-else 分支、重复的创建逻辑、紧耦合的通信——那就是模式该登场的时刻。
