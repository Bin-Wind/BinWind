# 数据类型与类型转换

JavaScript 是动态类型语言，变量可以随时改变类型。理解 JS 的数据类型和隐式转换规则，是避免诡异 bug 的关键。

## 原始类型（Primitive Types）

JS 有 7 种原始类型：

```javascript
// string — 字符串
const name = "Alice";
const template = `Hello, ${name}`;

// number — 数字（整数和浮点数统一为 64 位双精度）
const age = 25;
const price = 19.99;
const notANumber = NaN;       // typeof NaN === "number"
const infinity = Infinity;

// boolean — 布尔值
const isActive = true;

// null — 空值（故意的"无"）
const empty = null;           // typeof null === "object" — 历史遗留 bug

// undefined — 未定义（非故意的"无"）
let notAssigned;              // undefined

// symbol — 唯一标识符（ES6）
const id = Symbol("id");
const id2 = Symbol("id");
console.log(id === id2); // false，每个 Symbol 都是唯一的

// bigint — 大整数（ES2020）
const bigNumber = 9007199254740991n;
const fromNumber = BigInt(100);
```

## 引用类型（Reference Types）

引用类型只有一种：`Object`。数组、函数、Date、RegExp 等本质上都是对象。

```javascript
// 对象
const obj = { key: "value" };

// 数组
const arr = [1, 2, 3];       // typeof arr === "object"

// 函数
function greet() {}           // typeof greet === "function"（特殊的对象）

// 包装对象（Wrapper Objects）
const str = "hello";
console.log(str.length);      // 5 — 原始值自动装箱为 String 对象
console.log(str.toUpperCase()); // "HELLO"
```

## typeof 操作符

```javascript
typeof "hello"      // "string"
typeof 42           // "number"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof Symbol()     // "symbol"
typeof 123n         // "bigint"
typeof null         // "object" — 著名的 bug
typeof {}           // "object"
typeof []           // "object"
typeof function(){} // "function"
```

准确检测数组和 null 需要额外手段：

```javascript
Array.isArray([]);          // true
Array.isArray({});          // false
Object.prototype.toString.call([]);   // "[object Array]"
Object.prototype.toString.call(null); // "[object Null]"
```

## == vs ===

`===`（strict equality）比较值和类型，两者都相同才返回 true。`==`（loose equality）会触发**隐式类型转换**。

```javascript
// === 严格相等
1 === 1          // true
1 === "1"        // false
null === null    // true
NaN === NaN      // false — 用 Number.isNaN() 或 Object.is(NaN, NaN)

// == 抽象相等（不推荐）
1 == "1"         // true — 字符串转数字
true == 1        // true — 布尔转数字
null == undefined // true — 特例
"" == false      // true — 都转为 0
"   " == 0       // true — 空白字符串转 0
```

## 隐式类型转换（Type Coercion）

**转字符串（To String）**

```javascript
// + 运算符中有一方是字符串
"5" + 3         // "53"
"5" + true      // "5true"

// 模板字符串自动转字符串
`${5}`          // "5"
`${null}`       // "null"
```

**转数字（To Number）**

```javascript
// 算术运算符（+ 除外）
"5" - 3         // 2
"5" * "2"       // 10
"abc" - 1       // NaN

// 一元 + 运算符
+"5"            // 5
+true           // 1
+false          // 0
+null           // 0
+undefined      // NaN
```

**转布尔（To Boolean）**

Falsy 值只有 6 个：`false`、`0`、`""`（空字符串）、`null`、`undefined`、`NaN`。其余都是 truthy。

```javascript
// Falsy 值
Boolean(0)       // false
Boolean("")      // false
Boolean(null)    // false
Boolean(undefined) // false
Boolean(NaN)     // false
Boolean(false)   // false

// Truthy 值（注意这些容易误判）
Boolean(" ")     // true — 包含空格的字符串
Boolean("0")     // true — 非空字符串
Boolean([])      // true — 空数组
Boolean({})      // true — 空对象
```

## 显式类型转换

```javascript
// 转字符串
String(123);           // "123"
(123).toString();      // "123"
123 + "";              // "123"（利用隐式转换）

// 转数字
Number("123");         // 123
parseInt("123px");     // 123 — 解析到第一个非数字字符
parseFloat("3.14px");  // 3.14
+"123";                // 123

// 转布尔
Boolean(1);            // true
!!1;                   // true — 双重否定
```

## 总结

- 7 种原始类型 + Object。注意 `typeof null === "object"` 这个遗留 bug
- 始终使用 `===` 代替 `==`，除非明确需要 `null == undefined` 这种特例
- 理解 falsy 值列表，避免在判断中踩坑（`[]` 和 `{}` 是 truthy）
- `parseInt` 和 `Number()` 行为不同，前者截断非数字字符，后者严格转换
