# 运算符与表达式

运算符是 JavaScript 表达式的构建基石。掌握运算规则和优先级，才能写出简洁正确的代码。

## 算术运算符（Arithmetic Operators）

```javascript
const a = 10, b = 3;

const sum = a + b;        // 13 — 加法
const diff = a - b;       // 7  — 减法
const product = a * b;    // 30 — 乘法
const quotient = a / b;   // 3.333... — 除法
const remainder = a % b;  // 1  — 取余
const power = a ** b;     // 1000 — 指数（ES2016）

// 自增/自减（前缀 vs 后缀）
let x = 5;
const preIncrement = ++x;  // x=6, 返回 6 — 先加后用
const postIncrement = x++; // 返回 6, x=7 — 先用后加
```

## 赋值运算符（Assignment Operators）

```javascript
let n = 10;
n += 5;   // n = 15  — 等同于 n = n + 5
n -= 3;   // n = 12
n *= 2;   // n = 24
n /= 4;   // n = 6
n %= 4;   // n = 2
n **= 3;  // n = 8

// 逻辑赋值（ES2021）
let config = { timeout: 0 };
config.timeout ||= 3000;   // 没有用，0 是 falsy
config.retries ??= 3;      // retries 不存在，赋值为 3
```

## 比较运算符（Comparison Operators）

```javascript
// 标准比较
3 > 2        // true
3 >= 3       // true
"b" > "a"    // true — 按 Unicode 逐位比较

// 相等比较（永远用 ===）
0 === false  // false
0 == false   // true — 类型转换后相等

// NaN 特判
NaN === NaN  // false
Number.isNaN(NaN); // true
Object.is(NaN, NaN); // true

// Object.is 与 === 的区别
Object.is(+0, -0);    // false — +0 和 -0 不同
Object.is(NaN, NaN);  // true  — 与 === 相反
```

## 逻辑运算符（Logical Operators）

```javascript
// && 逻辑与 — 短路求值：遇到第一个 falsy 值就返回
true && "hello"      // "hello"
false && "hello"     // false（短路，不执行右侧）
0 && anything        // 0

// || 逻辑或 — 短路求值：遇到第一个 truthy 值就返回
false || "default"   // "default"
"hello" || "world"   // "hello"（短路）
"" || 0 || "ok"      // "ok" — 用于提供默认值（但注意 falsy 陷阱）

// ! 逻辑非
!true               // false
!!"hello"           // true — 双重否定转换为布尔值
```

## 空值合并运算符 ??（Nullish Coalescing）

`??` 仅在左侧为 `null` 或 `undefined` 时返回右侧值，解决了 `||` 将 `0`、`""`、`false` 视为 falsy 的问题。

```javascript
// || 的陷阱
const count = 0;
const result = count || 10;   // 10 — 0 被当作 falsy！

// ?? 的正确行为
const result2 = count ?? 10;  // 0 — 只有 null/undefined 才用默认值

const name = "";
const displayName = name ?? "Anonymous"; // ""

// 实际应用
function getUser(user) {
  return {
    name: user.name ?? "Unknown",
    age: user.age ?? 18,
    bio: user.bio ?? "No bio yet"
  };
}
```

## 可选链操作符 ?.（Optional Chaining）

安全访问深层嵌套的属性，中间任何一环为 `null` 或 `undefined` 时短路返回 `undefined`，而不是抛出错误。

```javascript
const user = {
  profile: {
    address: { city: "Beijing" }
  }
};

// 传统写法 — 又臭又长
const city = user && user.profile && user.profile.address && user.profile.address.city;

// 可选链写法 — 简洁优雅
const city2 = user?.profile?.address?.city; // "Beijing"
const district = user?.profile?.address?.district; // undefined，不会报错

// 也可以用于方法调用
const result = obj?.method?.();  // 如果 method 不存在，返回 undefined

// 数组索引
const first = users?.[0]?.name;

// 注意：可选链不能用于赋值
// user?.name = "Bob";  // SyntaxError
```

## 扩展运算符（Spread Syntax）

```javascript
// 数组展开
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combined = [0, ...arr1, ...arr2, 7]; // [0, 1, 2, 3, 4, 5, 6, 7]
const copy = [...arr1];                     // 浅拷贝

// 对象展开（ES2018）
const defaults = { theme: "light", lang: "zh" };
const userSettings = { lang: "en", fontSize: 14 };
const merged = { ...defaults, ...userSettings }; // { theme: "light", lang: "en", fontSize: 14 }

// 字符串展开
const chars = [..."hello"]; // ["h", "e", "l", "l", "o"]
```

## 运算符优先级

高到低：成员访问 `.` / `[]` > `new` > 函数调用 `()` > `?.` > 后置 `++` / `--` > 前置 `++` / `--` > `!` > `**` > `*` `/` `%` > `+` `-` > 比较运算符 > `&&` > `||` > `??` > `=`。

```javascript
// 复杂表达式 — 加括号提高可读性
const score = (base + bonus) * multiplier / (count ?? 1);

// 不依赖记忆，用括号明确意图
const isAdmin = (user.role === "admin" && user.isActive) || user.isSuper;
```

## 总结

- `??` 替代 `||` 处理默认值，避免 falsy 陷阱
- `?.` 安全访问深层属性，告别冗长的 `&&` 链
- `...` 展开运算符让数组/对象合并更加简洁
- 使用括号明确运算优先级，代码是写给人看的
