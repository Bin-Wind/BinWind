# 对象与原型链

JavaScript 是基于原型（prototype）的面向对象语言。ES6 引入的 `class` 语法只是原型继承的糖衣，底层机制仍然是原型链。

## 对象字面量（Object Literal）

```javascript
// 基本写法
const user = {
  name: "Alice",
  age: 25,

  // 方法简写（ES6）
  greet() {
    return `Hello, I'm ${this.name}`;
  },

  // 计算属性名（ES6）
  ["role_" + "admin"]: true,
};

// 属性简写（ES6）
const name = "Bob";
const age = 30;
const user2 = { name, age }; // { name: "Bob", age: 30 }
```

## 属性描述符与访问器

```javascript
const product = {
  _price: 100,

  // getter — 访问属性时调用
  get price() {
    return this._price;
  },

  // setter — 赋值时调用
  set price(value) {
    if (value < 0) throw new Error("价格不能为负");
    this._price = value;
  },

  // 普通属性也可以配置描述符
};

// Object.defineProperty — 精确控制属性行为
const obj = {};
Object.defineProperty(obj, "id", {
  value: 123,
  writable: false,     // 不可修改
  enumerable: true,    // 可被 for...in 遍历
  configurable: false, // 不可删除、不可重新配置
});
// obj.id = 456; // TypeError (strict mode) / 静默失败
```

## 原型与原型链

每个 JS 对象都有一个内部属性 `[[Prototype]]`，指向它的原型对象。通过 `__proto__` 或 `Object.getPrototypeOf()` 访问。

```javascript
const animal = {
  eat() {
    console.log("eating...");
  }
};

const dog = {
  bark() {
    console.log("woof!");
  }
};

// 设置 dog 的原型为 animal
Object.setPrototypeOf(dog, animal);
// 或创建时设置
// const dog = Object.create(animal);

dog.bark();  // "woof!" — 自有方法
dog.eat();   // "eating..." — 从原型继承

console.log(dog.__proto__ === animal);                         // true
console.log(Object.getPrototypeOf(dog) === animal);            // true
console.log(Object.getPrototypeOf(animal) === Object.prototype); // true
console.log(Object.getPrototypeOf(Object.prototype));          // null — 原型链终点
```

**属性查找过程**：

```javascript
// 访问 dog.eat 时：
// 1. 查找 dog 自身 → 没有
// 2. 查找 dog.__proto__ (animal) → 找到 eat()
// 3. 如果还没有，继续沿 __proto__ 向上查找直到 null
```

## 构造函数与 new

```javascript
function Person(name, age) {
  this.name = name;   // new 调用时 this 指向新对象
  this.age = age;
}

// 在原型上添加方法（共享，而非每个实例一份）
Person.prototype.greet = function () {
  return `Hi, I'm ${this.name}`;
};

const alice = new Person("Alice", 25);
const bob = new Person("Bob", 30);

console.log(alice.greet()); // "Hi, I'm Alice"
console.log(bob.greet());   // "Hi, I'm Bob"

// alice 和 bob 共享同一个 greet 方法
console.log(alice.greet === bob.greet); // true

// new 背后做了什么：
// 1. 创建一个空对象 {}
// 2. 将该对象的 __proto__ 指向 Person.prototype
// 3. 以该对象为 this 调用 Person 构造函数
// 4. 返回该对象（如果构造函数不返回其他对象）
```

## class 语法（ES6）

```javascript
class Animal {
  // 私有字段（ES2022）— 以 # 开头
  #id;

  constructor(name) {
    this.name = name;
    this.#id = Symbol(name);
  }

  // 实例方法 → 存储在 Animal.prototype 上
  speak() {
    return `${this.name} makes a sound.`;
  }

  // getter
  get id() {
    return this.#id;
  }

  // 静态方法 → 存储在 Animal 本身上
  static isAnimal(obj) {
    return obj instanceof Animal;
  }
}

// extends 继承
class Dog extends Animal {
  constructor(name, breed) {
    super(name);     // 必须调用 super() 才能使用 this
    this.breed = breed;
  }

  // 方法重写
  speak() {
    return `${super.speak()} And it's a ${this.breed}.`;
  }
}

const dog = new Dog("Buddy", "Golden Retriever");
console.log(dog.speak());           // "Buddy makes a sound. And it's a Golden Retriever."
console.log(dog instanceof Dog);    // true
console.log(dog instanceof Animal); // true
console.log(Animal.isAnimal(dog));  // true
```

## this 绑定规则

`this` 的值取决于**函数的调用方式**，而非定义方式。四条规则，优先级从高到低：

```javascript
// 1. new 绑定（优先级最高）
function Foo() { this.a = 1; }
const bar = new Foo();
console.log(bar.a); // 1

// 2. 显式绑定 — call / apply / bind
function greet() { console.log(this.name); }
const user = { name: "Alice" };
greet.call(user);  // "Alice" — call 逐个传参
greet.apply(user); // "Alice" — apply 数组传参
const boundGreet = greet.bind(user); // bind 返回新函数，永久绑定

// 3. 隐式绑定 — 作为对象方法调用
const obj = {
  name: "Bob",
  greet() { console.log(this.name); }
};
obj.greet(); // "Bob" — this 指向 obj

// 4. 默认绑定 — 严格模式下为 undefined，否则为 window/global
function showThis() { console.log(this); }
showThis(); // window（非严格）/ undefined（严格模式）

// 箭头函数的 this 不遵循上述规则，继承定义时的外层 this
const arrowObj = {
  name: "Carol",
  greet: () => console.log(this.name) // this 指向全局
};
arrowObj.greet(); // undefined（浏览器）或 undefined（Node）
```

## 常用对象方法

```javascript
const target = { a: 1, b: 2 };
const source = { b: 3, c: 4 };

// Object.assign — 浅拷贝，合并到 target
const merged = Object.assign(target, source);
console.log(merged); // { a: 1, b: 3, c: 4 }
console.log(target); // { a: 1, b: 3, c: 4 } — target 被修改

// 展开运算符 — 创建新对象（推荐）
const newObj = { ...target, ...source }; // { a: 1, b: 3, c: 4 }

// Object.keys / values / entries
const user2 = { name: "Dave", age: 35 };
console.log(Object.keys(user2));   // ["name", "age"]
console.log(Object.values(user2)); // ["Dave", 35]
console.log(Object.entries(user2)); // [["name", "Dave"], ["age", 35]]

// Object.fromEntries — entries 逆操作
const entries = [["x", 10], ["y", 20]];
console.log(Object.fromEntries(entries)); // { x: 10, y: 20 }

// Object.freeze / Object.seal
const frozen = Object.freeze({ a: 1 });
frozen.a = 2; // 静默失败（严格模式报错）
console.log(frozen.a); // 1

const sealed = Object.seal({ a: 1 });
sealed.a = 2;  // 可以修改现有属性
sealed.b = 3;  // 不能添加新属性
```

## 总结

- JS 通过原型链实现继承，`class` 是语法糖
- 私有字段 `#field` 实现真正的封装（ES2022）
- `this` 由调用方式决定：`new` > `call/apply/bind` > 隐式绑定 > 默认绑定
- 箭头函数不绑定自己的 `this`，继承外层作用域
- `Object.create()`、`Object.assign()`、`Object.entries()` 是日常高频 API
