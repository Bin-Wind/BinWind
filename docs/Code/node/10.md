# 数据库操作：Prisma ORM 实战

Prisma 是新一代 Node.js ORM，提供类型安全的数据库访问、声明式 Schema 和自动迁移。本文以 PostgreSQL 为例，演示从初始化到 CRUD 的完整流程。

## 安装与初始化

```bash
npm install prisma @prisma/client
npx prisma init
```

执行后会生成 `prisma/schema.prisma` 和 `.env` 文件（含 `DATABASE_URL`）。

## Schema 定义

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int       @id @default(autoincrement())
  email     String    @unique
  username  String
  password  String
  bio       String?
  role      Role      @default(USER)
  isActive  Boolean   @default(true)
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  posts     Post[]
  comments  Comment[]
}

model Post {
  id        Int       @id @default(autoincrement())
  title     String
  content   String    @db.Text
  published Boolean   @default(false)
  viewCount Int       @default(0)
  authorId  Int
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  author    User      @relation(fields: [authorId], references: [id])
  comments  Comment[]
  tags      PostTag[]
}

model Comment {
  id        Int      @id @default(autoincrement())
  content   String   @db.Text
  authorId  Int
  postId    Int
  createdAt DateTime @default(now())

  author    User     @relation(fields: [authorId], references: [id])
  post      Post     @relation(fields: [postId], references: [id], onDelete: Cascade)
}

model Tag {
  id    Int       @id @default(autoincrement())
  name  String    @unique

  posts PostTag[]
}

model PostTag {
  postId Int
  tagId  Int

  post   Post     @relation(fields: [postId], references: [id], onDelete: Cascade)
  tag    Tag      @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([postId, tagId])
  @@index([tagId])
}

enum Role {
  USER
  ADMIN
  MODERATOR
}
```

## 数据库迁移

```bash
# 开发环境：创建迁移文件并应用
npx prisma migrate dev --name init

# 生产环境：应用已有迁移
npx prisma migrate deploy

# 查看迁移状态
npx prisma migrate status
```

每次修改 schema 后，运行 `prisma migrate dev --name <描述>` 生成新的迁移文件。

## CRUD 操作

### 创建

```js
const prisma = require('./lib/prisma');

// 创建单个
const user = await prisma.user.create({
  data: {
    email: 'john@example.com',
    username: 'john',
    password: hashedPassword,
    role: 'USER',
  },
});

// 创建并关联
const post = await prisma.post.create({
  data: {
    title: '我的第一篇文章',
    content: '这是文章内容...',
    author: { connect: { id: 1 } },       // 关联已有 user
    tags: {
      connectOrCreate: ['Node.js', 'ORM'].map((name) => ({
        where: { name },
        create: { name },
      })),
    },
  },
});

// 批量创建
const users = await prisma.user.createMany({
  data: [
    { email: 'alice@example.com', username: 'alice', password: '...' },
    { email: 'bob@example.com', username: 'bob', password: '...' },
  ],
  skipDuplicates: true,
});
```

### 查询

```js
// 查询全部
const users = await prisma.user.findMany();

// 按主键
const user = await prisma.user.findUnique({ where: { id: 1 } });

// 单条条件查询
const user = await prisma.user.findFirst({
  where: { email: 'john@example.com' },
});

// 分页与排序
const posts = await prisma.post.findMany({
  where: { published: true },
  orderBy: { createdAt: 'desc' },
  skip: 0,
  take: 10, // 相当于 LIMIT
});

// 条件过滤
const admins = await prisma.user.findMany({
  where: {
    role: 'ADMIN',
    isActive: true,
    createdAt: { gte: new Date('2024-01-01') },
  },
});

// 模糊搜索
const results = await prisma.post.findMany({
  where: {
    OR: [
      { title: { contains: 'Node.js' } },
      { content: { contains: 'Node.js' } },
    ],
  },
});

// 聚合
const count = await prisma.user.count({ where: { role: 'ADMIN' } });
const stats = await prisma.post.aggregate({
  _avg: { viewCount: true },
  _sum: { viewCount: true },
  _max: { viewCount: true },
});
```

### 关联查询 (include / select)

```js
// include — 包含关联数据（全字段）
const userWithPosts = await prisma.user.findUnique({
  where: { id: 1 },
  include: {
    posts: {
      where: { published: true },
      orderBy: { createdAt: 'desc' },
    },
  },
});

// select — 选择返回字段
const user = await prisma.user.findUnique({
  where: { id: 1 },
  select: {
    id: true,
    username: true,
    email: true,
    _count: { select: { posts: true } },
  },
});
// { id: 1, username: 'john', email: '...', _count: { posts: 5 } }
```

### 更新

```js
// 更新单条
const updated = await prisma.user.update({
  where: { id: 1 },
  data: { bio: '这是我的个人简介', role: 'ADMIN' },
});

// 批量更新
const result = await prisma.user.updateMany({
  where: { isActive: false },
  data: { isActive: true },
});
console.log(`恢复了 ${result.count} 个用户`);

// upsert — 存在则更新，不存在则创建
const user = await prisma.user.upsert({
  where: { email: 'john@example.com' },
  update: { username: 'johnny' },
  create: { email: 'john@example.com', username: 'john', password: '...' },
});
```

### 删除

```js
// 单条删除
await prisma.user.delete({ where: { id: 1 } });

// 批量删除
const result = await prisma.user.deleteMany({
  where: { lastLoginAt: { lt: new Date('2023-01-01') } },
});
console.log(`删除了 ${result.count} 个用户`);
```

## 事务

### 交互式事务

```js
const [post, notification] = await prisma.$transaction(async (tx) => {
  const post = await tx.post.create({
    data: { title: '新文章', content: '...', authorId: 1 },
  });

  const notification = await tx.notification.create({
    data: { userId: 1, message: '文章发布成功', postId: post.id },
  });

  return [post, notification];
});
```

### 批量事务

```js
const results = await prisma.$transaction([
  prisma.user.update({ where: { id: 1 }, data: { points: { decrement: 100 } } }),
  prisma.user.update({ where: { id: 2 }, data: { points: { increment: 100 } } }),
]);
```

## Prisma vs Sequelize 对比

| 维度          | Prisma                        | Sequelize                      |
|---------------|-------------------------------|--------------------------------|
| Schema        | 独立的 .prisma 文件，声明式      | JS 代码定义模型                  |
| 类型安全       | 自动生成类型，开箱即用           | 需额外安装 @types                |
| 迁移          | 内置 prisma migrate，版本控制   | 需要 sequelize-cli + 手动写迁移  |
| 关联查询       | include/select 清晰直观        | include + attributes 配置较繁琐  |
| 复杂查询       | where 简洁，支持嵌套过滤         | Op 操作符丰富，功能更全           |
| 性能          | 查询优化好，无 N+1 问题          | include 可能产生大 JOIN          |
| 生态成熟度     | 较新，发展快                    | 老牌 ORM，社区资源丰富            |

## 使用建议

- **新项目优先选 Prisma**：类型安全、Schema 直观、迁移方便。
- **已有 Sequelize 项目不必迁移**：重写成本高，功能足够用。
- **复杂原始查询多时**：Sequelize 的 raw query 和事务嵌套支持更灵活。

## 总结

Prisma 的核心优势在于类型安全和声明式 Schema。从 `prisma init` 开始，定义 Schema -> 迁移 -> 使用 Client CRUD，工作流清晰。`include`/`select` 让关联查询和字段控制变得直观。
