# 日志系统搭建 (Winston / Pino)

日志是生产环境排障的第一手资料。本文介绍 Winston 和 Pino 主流日志库的使用，以及结构化日志的最佳实践。

## 为什么需要结构化日志

```js
// ❌ console.log — 难以解析，没有时间戳和级别
console.log('user 123 logged in');

// ✅ 结构化日志 — JSON 格式，可被日志系统自动解析
logger.info('用户登录', { userId: 123, ip: '192.168.1.5', userAgent: '...' });
// {"level":"info","message":"用户登录","userId":123,"timestamp":"2024-07-15T10:30:00.000Z"}
```

## Winston 入门

### 安装

```bash
npm install winston
```

### 基础配置

```js
const winston = require('winston');

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp({ format: 'YYYY-MM-DD HH:mm:ss' }),
    winston.format.errors({ stack: true }),
    winston.format.json(),
  ),
  transports: [
    new winston.transports.Console({
      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.printf(({ level, message, timestamp, ...meta }) => {
          const metaStr = Object.keys(meta).length ? JSON.stringify(meta) : '';
          return `${timestamp} [${level}]: ${message} ${metaStr}`;
        }),
      ),
    }),
  ],
});

module.exports = logger;
```

### Transport 配置

```js
// 控制台（开发环境彩色输出）
new winston.transports.Console(),

// 文件 — 所有日志
new winston.transports.File({
  filename: 'logs/app.log',
  maxsize: 10 * 1024 * 1024, // 10MB
  maxFiles: 5,
}),

// 文件 — 只记录错误
new winston.transports.File({
  filename: 'logs/error.log',
  level: 'error',
}),

// HTTP — 发送到远程日志服务
new winston.transports.Http({
  host: 'log-collector.example.com',
  port: 9200,
  path: '/logs',
}),
```

### 日志级别

```js
logger.error('数据库连接失败', { db: 'postgres', error: err.message });
logger.warn('Redis 连接池接近上限', { poolUsed: 18, poolMax: 20 });
logger.info('服务启动成功', { port: 3000, env: 'production' });
logger.http('GET /api/users 200 45ms');
logger.verbose('查询参数已解析', { params: req.query });
logger.debug('认证中间件：Token 已通过验证', { userId: 123 });
logger.silly('请求体原始数据', { body: req.body });
```

## 日志文件轮转

```bash
npm install winston-daily-rotate-file
```

```js
const DailyRotateFile = require('winston-daily-rotate-file');

const transport = new DailyRotateFile({
  filename: 'logs/app-%DATE%.log',
  datePattern: 'YYYY-MM-DD',
  maxSize: '20m',
  maxFiles: '30d', // 保留最近 30 天
  zippedArchive: true,
});

logger.add(transport);
```

每天生成一个日志文件，自动压缩旧文件，过期自动删除。

## Pino — 极致性能的 JSON Logger

Pino 是 Node.js 生态中最快的日志库，适合高吞吐场景。

### 安装

```bash
npm install pino pino-http pino-pretty
```

### 基础使用

```js
const pino = require('pino');

const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  transport: process.env.NODE_ENV !== 'production'
    ? {
        target: 'pino-pretty',
        options: { colorize: true, translateTime: 'SYS:standard' },
      }
    : undefined, // 生产环境输出纯 JSON
});

logger.info({ userId: 123 }, '用户登录');
logger.error({ err, userId: 123 }, '订单创建失败');
```

### Express 中间件集成

```js
const pinoHttp = require('pino-http');

const httpLogger = pinoHttp({
  logger,
  customLogLevel(res, err) {
    if (res.statusCode >= 500 || err) return 'error';
    if (res.statusCode >= 400) return 'warn';
    return 'info';
  },
  customSuccessMessage(res) {
    return `${res.statusCode} - 请求完成`;
  },
  customErrorMessage(err, res) {
    return `${res.statusCode} - ${err.message}`;
  },
  serializers: {
    req(req) {
      return { method: req.method, url: req.url, ip: req.ip };
    },
  },
});

app.use(httpLogger);
```

## 请求日志中间件模式

无论使用哪个库，请求日志中间件都应记录：

```js
// middleware/requestLogger.js
const logger = require('../lib/logger');

function requestLogger(req, res, next) {
  const requestId = req.headers['x-request-id'] || crypto.randomUUID();
  req.requestId = requestId;

  const start = Date.now();

  res.on('finish', () => {
    const duration = Date.now() - start;
    const logData = {
      requestId,
      method: req.method,
      url: req.originalUrl,
      status: res.statusCode,
      duration,
      ip: req.ip,
      userAgent: req.get('User-Agent'),
    };

    if (res.statusCode >= 500) {
      logger.error('请求处理异常', logData);
    } else if (res.statusCode >= 400) {
      logger.warn('请求被拒绝', logData);
    } else {
      logger.info('请求处理完成', logData);
    }
  });

  next();
}
```

## 完整的日志模块封装

```js
// lib/logger.js
const winston = require('winston');
const DailyRotateFile = require('winston-daily-rotate-file');

const isDev = process.env.NODE_ENV === 'development';

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || (isDev ? 'debug' : 'info'),
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json(),
  ),
  defaultMeta: {
    service: process.env.SERVICE_NAME || 'my-app',
    env: process.env.NODE_ENV || 'development',
  },
  transports: [],
});

// 开发环境：彩色控制台
if (isDev) {
  logger.add(new winston.transports.Console({
    format: winston.format.combine(
      winston.format.colorize(),
      winston.format.simple(),
    ),
  }));
} else {
  // 生产环境：文件轮转
  logger.add(new DailyRotateFile({
    filename: 'logs/app-%DATE%.log',
    datePattern: 'YYYY-MM-DD',
    maxFiles: '30d',
    zippedArchive: true,
  }));

  // 错误单独存一份
  logger.add(new DailyRotateFile({
    filename: 'logs/error-%DATE%.log',
    level: 'error',
    datePattern: 'YYYY-MM-DD',
    maxFiles: '90d',
  }));
}

// 未捕获异常
process.on('uncaughtException', (err) => {
  logger.error('未捕获异常', { error: err.message, stack: err.stack });
  process.exit(1);
});

process.on('unhandledRejection', (reason) => {
  logger.error('未处理的 Promise Rejection', { reason });
});

module.exports = logger;
```

## 什么应该记录

每条日志应包含：

| 字段        | 说明                               | 示例                                 |
|------------|------------------------------------|--------------------------------------|
| timestamp  | 时间戳（ISO 8601）                   | `2024-07-15T10:30:00.000Z`           |
| level      | 日志级别                            | `info`, `error`, `warn`              |
| message    | 描述消息                            | `用户登录成功`                        |
| requestId  | 请求追踪 ID（全链路）                | `a1b2c3d4-e5f6-...`                  |
| userId     | 当前用户（如有）                     | `123`                                |
| context    | 上下文数据                           | `{ orderId: 456, amount: 99 }`       |

不应记录的内容：密码、Token、身份证号、手机号等敏感信息。

## 日志聚合与可视化

生产环境多实例运行时，需要将日志集中管理：

- **ELK (Elasticsearch + Logstash + Kibana)**：经典方案，用 Filebeat 采集日志
- **Loki + Grafana**：轻量方案，Grafana 原生集成
- **云服务**：阿里云 SLS、AWS CloudWatch、腾讯云 CLS

```bash
# Docker Compose 快速搭建 Loki + Grafana
docker-compose up loki grafana promtail
```

## 总结

核心原则：**结构化输出（JSON）**，区分环境（开发彩色/生产 JSON），文件轮转防磁盘爆满，请求日志带 requestId 便于追踪，生产环境使用日志聚合系统集中管理。小项目 Winston 足够，高吞吐场景选 Pino。
