# 进程管理 (PM2) 生产部署

PM2 是 Node.js 生态中最成熟的生产级进程管理器。它提供了进程守护、集群模式、零停机重启、日志管理和开机自启等关键特性。无论是单机部署还是多机集群，PM2 都能大幅简化运维工作。

## 安装 PM2

```bash
npm install -g pm2
```

安装后验证：

```bash
pm2 --version
```

## 基础命令速查

### 启动应用

```bash
# 基本启动
pm2 start app.js --name my-app

# 指定实例数（集群模式）
pm2 start app.js -i max --name my-app

# 带环境变量
pm2 start app.js --env production

# 监听文件变化自动重启
pm2 start app.js --watch
```

### 进程管理

```bash
pm2 list              # 查看所有进程
pm2 show my-app       # 查看单个进程详情
pm2 monit             # 实时监控面板（CPU、内存、请求）
pm2 logs              # 查看所有日志
pm2 logs my-app       # 查看指定应用日志
pm2 logs --lines 100  # 最近100行
pm2 flush             # 清空所有日志
```

### 停止与重启

```bash
pm2 stop my-app       # 停止进程
pm2 restart my-app    # 重启进程
pm2 delete my-app     # 删除进程（从 PM2 列表中移除）
pm2 reload my-app     # 零停机重载（仅集群模式）
pm2 reset my-app      # 重置计数器（重启次数、运行时长）
```

## Cluster 模式

Node.js 是单线程的，默认只能利用一个 CPU 核心。Cluster 模式可以 fork 多个进程来利用所有核心。

```bash
# 使用所有 CPU 核心
pm2 start app.js -i max

# 指定核心数
pm2 start app.js -i 4

# 查看各进程负载
pm2 monit
```

**Express 应用示例：**

```javascript
// app.js
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  // process.env.pm_id 可以区分不同 worker
  res.json({ worker: process.env.pm_id, pid: process.pid });
});

app.listen(3000, () => {
  console.log(`Worker ${process.env.pm_id || 0} 启动，PID: ${process.pid}`);
});
```

## Ecosystem File 声明式配置

对于复杂应用，推荐使用 `ecosystem.config.js` 统一管理配置。

```javascript
// ecosystem.config.js
module.exports = {
  apps: [
    {
      name: 'my-api',
      script: './src/app.js',
      instances: 'max',
      exec_mode: 'cluster',
      env: {
        NODE_ENV: 'development',
        PORT: 3000,
      },
      env_production: {
        NODE_ENV: 'production',
        PORT: 8080,
      },
      watch: ['src'],
      ignore_watch: ['node_modules', 'logs'],
      max_memory_restart: '500M',
      error_file: './logs/err.log',
      out_file: './logs/out.log',
      merge_logs: true,
      log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
      kill_timeout: 5000,          // 发 SIGKILL 前的等待时间
      listen_timeout: 3000,        // 等待应用 ready 的时间
    },
  ],
};
```

使用 ecosystem file：

```bash
pm2 start ecosystem.config.js --env production
```

## 自动重启与开机自启

### Crash 自动重启

PM2 默认在进程异常退出时自动重启，无需额外配置。

```bash
# 查看重启次数
pm2 list | grep "restart"
```

### 开机自启动

```bash
# 生成启动脚本
pm2 startup

# 执行上面命令输出的命令（需要 sudo）
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u your_user --hp /home/your_user

# 保存当前进程列表，重启时自动恢复
pm2 save
```

## 日志管理

生产环境中日志会快速增长，需要轮转控制。

```bash
# 安装日志轮转模块
pm2 install pm2-logrotate

# 配置轮转参数
pm2 set pm2-logrotate:max_size 10M        # 单文件最大10MB
pm2 set pm2-logrotate:retain 30           # 保留30个日志文件
pm2 set pm2-logrotate:compress true       # 压缩旧日志
pm2 set pm2-logrotate:dateFormat YYYY-MM-DD_HH-mm-ss
pm2 set pm2-logrotate:rotateInterval '0 0 * * *' # 每天轮转
```

## Zero-Downtime Reload

这是 PM2 cluster 模式的核心优势。`pm2 reload` 会逐个重启 worker，始终保持至少一个 worker 存活。

```bash
# 代码更新后，零停机重载
git pull
npm install
pm2 reload my-app
```

`reload` 的工作原理：

1. 启动一个新的 worker
2. 等待新 worker ready（`listen_timeout` 内）
3. 拿掉一个旧 worker
4. 重复步骤 1-3，直到所有旧 worker 被替换

## 部署工作流示例

```bash
# 首次部署
pm2 start ecosystem.config.js --env production
pm2 save

# 日常更新
git pull && npm install && npm run build
pm2 reload my-app

# 健康检查
pm2 list
pm2 logs --lines 50
curl http://localhost:3000/health
```

## 总结

PM2 是单机部署 Node.js 的最佳实践工具。它解决了进程守护、多核利用、日志管理和零停机更新四大核心问题。对于更复杂的场景（如多机集群），可以结合 Docker 编排（Kubernetes）使用，但对于大多数中小型项目，PM2 足够强大且易于上手。
