# Buffer 与二进制数据处理

Buffer 是 Node.js 处理二进制数据的核心类。在 JavaScript 诞生之初，语言层面没有二进制数据类型，Buffer 就是为了弥补这一空白。它在 V8 堆外分配固定大小的内存，专门用于处理文件 I/O、网络传输和加密等场景。

## 什么是 Buffer

Buffer 本质上是一块固定大小的原始内存，位于 V8 堆之外。它类似于整数数组，但每个元素只占一个字节（0-255）。Buffer 是 `Uint8Array` 的子类，继承了 TypedArray 的所有方法。

```javascript
const buf = Buffer.from('Hello');
console.log(buf);        // <Buffer 48 65 6c 6c 6f>
console.log(buf.length);  // 5
console.log(buf[0]);      // 72 (字符 'H' 的 ASCII 码)
console.log(buf.toString()); // Hello
```

## 创建 Buffer

```javascript
// 从字符串创建（最常用）
const buf1 = Buffer.from('Hello World', 'utf8');

// 从数组创建
const buf2 = Buffer.from([0x48, 0x65, 0x6c, 0x6c, 0x6f]);

// 分配指定大小的零填充内存（安全，但稍慢）
const buf3 = Buffer.alloc(1024); // 1KB，全部初始化为 0

// 分配未初始化的内存（快，但可能含旧数据）
const buf4 = Buffer.allocUnsafe(1024);
// ⚠️ 使用前需要手动填充或覆盖
buf4.fill(0);
```

**安全提醒：** `Buffer.allocUnsafe()` 返回的内存区域可能包含之前进程遗留的敏感数据（密码、密钥等）。只在确定会立即覆盖全部内容时使用它。

## 编码转换

Buffer 支持多种字符编码：

```javascript
// utf8（默认）
const utf8Buf = Buffer.from('你好', 'utf8');
console.log(utf8Buf.toString('utf8')); // 你好

// base64
const base64Buf = Buffer.from('Hello World');
console.log(base64Buf.toString('base64')); // SGVsbG8gV29ybGQ=

const decoded = Buffer.from('SGVsbG8gV29ybGQ=', 'base64');
console.log(decoded.toString()); // Hello World

// hex
const hexBuf = Buffer.from('abc', 'utf8');
console.log(hexBuf.toString('hex')); // 616263

// binary (latin1)
const latinBuf = Buffer.from('café', 'latin1');
console.log(latinBuf.toString('latin1')); // café
```

## 文件读写与二进制处理

```javascript
const fs = require('fs');

// 以 Buffer 方式读取文件（不指定 encoding）
const data = fs.readFileSync('photo.jpg');
console.log(`文件大小: ${data.length} 字节`);

// 读取部分内容
const fd = fs.openSync('photo.jpg', 'r');
const header = Buffer.alloc(8);
fs.readSync(fd, header, 0, 8, 0); // 读取前8字节
fs.closeSync(fd);
```

## 实用案例：通过 Magic Bytes 检测图片格式

```javascript
const fs = require('fs');

function detectImageFormat(filePath) {
  const buf = fs.readFileSync(filePath);
  const signature = buf.slice(0, 4).toString('hex').toUpperCase();

  const formats = {
    FFD8FF: 'JPEG',
    '89504E47': 'PNG',
    '47494638': 'GIF',
    '52494646': 'WEBP',  // RIFF...WEBP
    '424D': 'BMP',        // BM
  };

  for (const [sig, format] of Object.entries(formats)) {
    if (signature.startsWith(sig)) return format;
  }
  return '未知格式';
}

console.log(detectImageFormat('photo.jpg')); // JPEG
console.log(detectImageFormat('icon.png'));  // PNG
```

## Buffer.concat() 组装数据块

在处理网络请求分片或流式数据时频繁使用。

```javascript
// 手动收集 chunk 并拼接
const chunks = [];

// 模拟从流中接收数据
chunks.push(Buffer.from('Hello '));
chunks.push(Buffer.from('World'));
chunks.push(Buffer.from('!'));

const result = Buffer.concat(chunks);
console.log(result.toString()); // Hello World!
console.log(`总长度: ${result.length}`); // 12

// 典型用法：Express 中解析原始 body
app.post('/upload', (req, res) => {
  const chunks = [];
  req.on('data', (chunk) => chunks.push(chunk));
  req.on('end', () => {
    const body = Buffer.concat(chunks);
    console.log(`收到 ${body.length} 字节`);
    res.send('ok');
  });
});
```

## TypedArray 与 Buffer 的关系

Buffer 是 `Uint8Array` 的子类（Node.js 8+），两者可以互相操作。

```javascript
// Buffer → TypedArray
const buf = Buffer.from([1, 2, 3, 4, 5, 6, 7, 8]);
const uint32 = new Uint32Array(buf.buffer, buf.byteOffset, 2);
console.log(uint32); // Uint32Array [ 67305985, 134678021 ]

// TypedArray → Buffer
const arr = new Uint8Array([65, 66, 67]);
const buf2 = Buffer.from(arr);
console.log(buf2.toString()); // ABC

// 共享内存（修改一方会影响另一方）
const shared = new Uint8Array(4);
const sharedBuf = Buffer.from(shared.buffer);
shared[0] = 72;
console.log(sharedBuf.toString()); // H
```

## Base64 编解码实战

```javascript
// 编码：字符串 → Base64
function encodeBase64(text) {
  return Buffer.from(text, 'utf8').toString('base64');
}

// 解码：Base64 → 字符串
function decodeBase64(base64Str) {
  return Buffer.from(base64Str, 'base64').toString('utf8');
}

console.log(encodeBase64('admin:password123'));
// YWRtaW46cGFzc3dvcmQxMjM=

console.log(decodeBase64('SGVsbG8gV29ybGQ='));
// Hello World

// Data URL 解析：提取 base64 图片数据
function parseDataUrl(dataUrl) {
  const match = dataUrl.match(/^data:(.+);base64,(.+)$/);
  if (!match) throw new Error('无效的 Data URL');
  return {
    mimeType: match[1],
    buffer: Buffer.from(match[2], 'base64'),
  };
}

const { buffer, mimeType } = parseDataUrl(
  'data:image/png;base64,iVBORw0KGgoAAAANS...'
);
require('fs').writeFileSync('output.png', buffer);
```

## 性能优化建议

1. **复用 Buffer。** 在循环中频繁操作时，预分配 Buffer 并复用，避免反复分配释放。
2. **选择合适的 Buffer.alloc。** 确定每个字节都会被写入时用 `allocUnsafe`，否则用 `alloc`。
3. **避免 Buffer → String → Buffer 的来回转换。** 每次转换都有开销，尽量在 Buffer 层面操作。
4. **Stream 默认就是 Buffer。** 除非显式设置了 encoding，否则 Stream 的 chunk 就是 Buffer。

## 总结

Buffer 是 Node.js 处理二进制数据的基石。掌握 `Buffer.from()`、`Buffer.alloc()`、编码转换、`Buffer.concat()` 以及 TypedArray 关系，能让你从容处理文件 I/O、加密解密、图片处理和网络传输中的二进制数据。注意 `allocUnsafe` 的安全隐患，在生产代码中谨慎使用。
