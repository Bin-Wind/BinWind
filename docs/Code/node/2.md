# Node.js 模块系统详解 (CJS vs ESM)

Node.js 有两套模块系统：CommonJS (CJS) 和 ES Modules (ESM)。理解它们的区别和互操作方式是每个 Node.js 开发者的必备知识。

## CommonJS 模块

CommonJS 是 Node.js 默认的模块系统，使用 `require()` 和 `module.exports`。

### 导出模块

```js
// math.js
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

const PI = 3.14159;

// 逐个导出
module.exports = {
  add,
  multiply,
  PI,
};

// 或者单独赋值
// exports.add = add;
// exports.multiply = multiply;
```

### 导入模块

```js
// app.js
const math = require('./math');
// 解构导入
const { add, multiply } = require('./math');

console.log(add(2, 3));      // 5
console.log(multiply(4, 5)); // 20
```

### 模块缓存机制

CommonJS 模块在首次 `require` 时执行并缓存结果，后续 `require` 直接返回缓存副本。

```js
// counter.js
let count = 0;
module.exports = {
  increment() { count++; },
  getCount() { return count; },
};

// app.js
const c1 = require('./counter');
const c2 = require('./counter');
c1.increment();
console.log(c2.getCount()); // 1 — 同一个缓存对象
```

### __dirname 和 __filename

```js
console.log(__dirname);  // 当前模块所在目录的绝对路径
console.log(__filename); // 当前模块文件的绝对路径
```

### module.exports vs exports

```js
// 两者初始指向同一对象，但只有 module.exports 是最终导出
exports.foo = 'bar';           // ✅ 有效
module.exports = { foo: 'bar' }; // ✅ 覆盖整个导出
exports = { foo: 'bar' };      // ❌ 无效，断开了引用链
```

## ES Modules (ESM)

ESM 是 JavaScript 标准模块系统，TypeScript 和现代浏览器均原生支持。

### 导出模块

```js
// math.mjs
export function add(a, b) {
  return a + b;
}

export const PI = 3.14159;

// 默认导出
export default function multiply(a, b) {
  return a * b;
}
```

### 导入模块

```js
// app.mjs
import multiply, { add, PI } from './math.mjs';

console.log(add(1, 2));      // 3
console.log(multiply(3, 4)); // 12
console.log(PI);              // 3.14159
```

### 动态 import()

```js
// 按需加载，返回 Promise
async function loadModule() {
  if (condition) {
    const { flatten } = await import('lodash-es');
    return flatten;
  }
}
```

### import.meta.url

```js
// 等价于 CJS 的 __filename
console.log(import.meta.url);
// file:///Users/example/project/src/app.mjs

import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);
```

## 如何选择模块系统

### package.json "type" 字段

```json
{
  "type": "module"   // .js 文件视为 ESM
}
```

```json
{
  "type": "commonjs" // .js 文件视为 CJS（默认）
}
```

### .mjs 和 .cjs 扩展名

| 文件扩展名 | 行为（无视 package.json type） |
|-----------|-------------------------------|
| `.mjs`    | 始终作为 ESM                   |
| `.cjs`    | 始终作为 CommonJS              |
| `.js`     | 由 package.json type 决定      |

## CJS 与 ESM 互操作

### ESM 文件中导入 CJS 模块

```js
// utils.cjs (CommonJS)
module.exports = { greet: (name) => `Hello, ${name}` };

// app.mjs (ESM)
import utils from './utils.cjs';
console.log(utils.greet('World')); // Hello, World
```

### CJS 文件中导入 ESM 模块

```js
// math.mjs (ESM)
export const add = (a, b) => a + b;

// app.cjs (CommonJS) — 只能用动态 import()
(async () => {
  const math = await import('./math.mjs');
  console.log(math.add(1, 2)); // 3
})();
```

CJS 不能使用静态 `import` 导入 ESM，必须用 `await import()`。

## 何时使用哪种

| 场景                         | 推荐         |
|------------------------------|-------------|
| 新项目                        | ESM         |
| 需要 Tree Shaking             | ESM         |
| TypeScript 项目               | ESM         |
| 维护旧项目                     | 保持 CJS     |
| 使用 __dirname / require 的动态特性 | CJS |
| 浏览器和 Node 共享代码          | ESM         |

## 总结

CJS 是 Node.js 的传统模块系统，同步加载、简单直接。ESM 是 JavaScript 标准，支持静态分析、Tree Shaking 和异步加载。理解两者差异和互操作方式，有助于在不同项目间灵活切换。
