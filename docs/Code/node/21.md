# Cluster 模块实现多核利用

Node.js 默认是单线程的，一个进程只能跑在一个 CPU 核心上。如果你的服务器有 8 核 16 线程，不经处理的话，Node 只能用满 1 个核心，其余全部闲置。Cluster 模块正是为了解决这个问题而生的 —— 它让你轻松创建多个 worker 进程，每个进程运行在独立的核心上，共享同一个服务端口。

## 为什么需要 Cluster

```bash
# 查看服务器 CPU 核心数
node -e "console.log(require('os').cpus().length)"
```

- Node.js 单线程 → 单个 CPU 核心
- I/O 操作是异步的不阻塞 → 但 CPU 计算会阻塞
- 多进程 = 多核心 = 真正的并行处理

## Master-Worker 模型

Cluster 模块内部有一个 master（主进程）和多个 worker（工作进程）。Master 负责管理 worker 的生命周期，Worker 负责处理实际的 HTTP 请求。

```
客户端请求 → Master（端口监听）
                  │
                  ├── Worker 1（CPU 核心 1）
                  ├── Worker 2（CPU 核心 2）
                  ├── Worker 3（CPU 核心 3）
                  └── Worker N（CPU 核心 N）
```

## 基础用法

```javascript
// cluster-app.js
const cluster = require('cluster');
const http = require('http');
const os = require('os');

const numCPUs = os.cpus().length;

if (cluster.isPrimary) {
  console.log(`主进程 PID: ${process.pid}`);

  // Fork 与 CPU 核心数相同的 worker
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  // 监听 worker 退出并自动重启
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} 退出 (code: ${code})`);
    console.log('重新启动一个新 worker...');
    cluster.fork();
  });

} else {
  // Worker 进程：创建 HTTP 服务器
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end(`由 Worker PID: ${process.pid} 处理\n`);
  }).listen(8000);

  console.log(`Worker ${process.pid} 已启动`);
}
```

所有 worker 共同监听 `8000` 端口 —— 操作系统内核负责将进来的连接分发给不同的 worker。

## 负载分配策略

### Round-Robin（默认，除 Windows 外）

操作系统将连接轮流分发给各 worker。Windows 上由 master 手动分发。

```javascript
// 显式设置调度策略（Windows 上可改为 round-robin）
cluster.schedulingPolicy = cluster.SCHED_RR;      // 轮询
cluster.schedulingPolicy = cluster.SCHED_NONE;    // 由 OS 决定
```

### 手动分配（需要自定义逻辑）

```javascript
if (cluster.isPrimary) {
  const workers = [];

  for (let i = 0; i < numCPUs; i++) {
    workers.push(cluster.fork());
  }

  // 自定义负载均衡 —— 选择负载最低的 worker
  let currentIndex = 0;
  function getLeastLoadedWorker() {
    // 简单轮询，实际可基于 worker 连接数
    const worker = workers[currentIndex];
    currentIndex = (currentIndex + 1) % workers.length;
    return worker;
  }
}
```

## Worker 间通信 (IPC)

Master 和 Worker 之间通过内置 IPC 通道传递消息：

```javascript
// Master 侧
if (cluster.isPrimary) {
  const worker = cluster.fork();

  // 向 worker 发送消息
  worker.send({ cmd: 'updateConfig', data: { port: 8080 } });

  // 接收 worker 消息
  worker.on('message', (msg) => {
    console.log(`Master 收到: ${msg}`);
  });

} else {
  // Worker 侧接收
  process.on('message', (msg) => {
    if (msg.cmd === 'updateConfig') {
      console.log('Worker 收到配置更新:', msg.data);
    }
    // 回复消息
    process.send('配置已更新');
  });
}
```

## 优雅重启（零停机更新）

```javascript
// graceful-restart.js
if (cluster.isPrimary) {
  const workers = new Map(); // worker.id → worker

  function createWorker() {
    const worker = cluster.fork();
    workers.set(worker.id, worker);

    worker.on('exit', () => {
      workers.delete(worker.id);
      // 异常退出时自动创建新 worker
      createWorker();
    });

    return worker;
  }

  // 初始创建
  for (let i = 0; i < numCPUs; i++) {
    createWorker();
  }

  // 监听 SIGHUP 信号执行零停机重启
  process.on('SIGHUP', () => {
    const workerIds = Array.from(workers.keys());
    let i = 0;

    function restartNext() {
      if (i >= workerIds.length) {
        console.log('所有 worker 已重启完成');
        return;
      }

      const oldWorker = workers.get(workerIds[i]);
      const newWorker = createWorker();

      // 新 worker 就绪后，断开并终止旧 worker
      newWorker.on('listening', () => {
        oldWorker.disconnect();
        setTimeout(() => {
          oldWorker.kill();
          i++;
          restartNext();
        }, 5000); // 给旧 worker 5 秒处理剩余请求
      });
    }

    restartNext();
  });

  console.log(`Master ${process.pid} 运行中，kill -HUP ${process.pid} 触发重启`);
}
```

测试优雅重启：

```bash
# 启动
node graceful-restart.js

# 在另一个终端触发重启
kill -HUP <master-pid>
```

## worker.disconnect() 与优雅关闭

```javascript
// Worker 侧处理优雅关闭
process.on('disconnect', () => {
  console.log('Master 已断开 IPC，准备关闭...');
  server.close(() => {
    console.log('所有连接已关闭，退出');
    process.exit(0);
  });
});

// 给当前正在处理的请求留出时间
process.on('SIGTERM', () => {
  console.log('收到 SIGTERM，5秒后退出...');
  setTimeout(() => process.exit(0), 5000);
});
```

## Cluster 与 PM2 的对比

| 特性 | Cluster 模块 | PM2 |
|------|-------------|-----|
| 零配置 | 需要手写代码 | 命令行一行启动 |
| 进程守护 | 需手动处理 | 内置 |
| 日志管理 | 无 | 内置 + 日志轮转 |
| 部署管理 | 无 | 生态系统文件 |
| 性能监控 | 无 | `pm2 monit` |
| 灵活性 | 完全可控 | 封装好了 |
| 建议 | 学习原理 | 生产环境首选 |

**实际开发建议：** 了解 Cluster 原理（本文），生产中用 PM2（参考文章 15）。

## 总结

Cluster 模块是 Node.js 原生多核方案，核心模式是 master fork 多个 worker，worker 共享同一端口。记住三个关键点：`cluster.isPrimary` 判断角色、`cluster.fork()` 创建 worker、IPC 通道 `worker.send()` / `process.on('message')` 通信。对于零停机重启，模式是 disconnect → new worker → kill old，逐个替换。生产环境推荐用 PM2 来简化这套流程。
