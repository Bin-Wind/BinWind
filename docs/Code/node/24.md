# GraphQL API 搭建 (Apollo Server)

GraphQL 是 Facebook 开发的数据查询语言。与 REST 不同，GraphQL 允许客户端精确指定需要的数据字段，避免了 Over-fetching 和 Under-fetching 的问题。Apollo Server 是 Node.js 生态中最成熟的 GraphQL 服务端框架。

## GraphQL vs REST

| 特性 | REST | GraphQL |
|------|------|---------|
| 请求端点 | 多个 (`/users`, `/posts`) | 单个 (`/graphql`) |
| 数据控制 | 服务端决定返回字段 | 客户端决定返回字段 |
| 取数效率 | 可能 Over-fetch 或需多次请求 | 一次请求拿所有需要的数据 |
| 学习曲线 | 低 | 中 |
| 缓存 | HTTP 缓存天然 | 需额外处理 |
| 适用场景 | 简单 CRUD | 复杂关联数据、多端异构 |

**GraphQL 最佳场景：** 移动端 + Web端共用 API、数据关联复杂（用户→文章→评论）、前端需要灵活查询。

## 安装与基础配置

## 定义 Schema

```javascript
// schema.js
const typeDefs = `#graphql
  type User {
    id: ID!
    name: String!
    email: String!
    posts: [Post!]!
  }

  type Post {
    id: ID!
    title: String!
    content: String!
    author: User!
    publishedAt: String
  }

  # 输入类型（用于 Mutation 参数）
  input CreateUserInput {
    name: String!
    email: String!
  }

  input CreatePostInput {
    title: String!
    content: String!
    authorId: ID!
  }

  # 查询入口
  type Query {
    users: [User!]!
    user(id: ID!): User
    posts(authorId: ID): [Post!]!
    post(id: ID!): Post
  }

  # 修改入口
  type Mutation {
    createUser(input: CreateUserInput!): User!
    createPost(input: CreatePostInput!): Post!
    deletePost(id: ID!): Boolean!
  }
`;

module.exports = typeDefs;
```

## 编写 Resolvers

```javascript
// resolvers.js — 模拟数据
const users = [
  { id: '1', name: 'Alice', email: 'alice@example.com' },
  { id: '2', name: 'Bob', email: 'bob@example.com' },
];

const posts = [
  { id: '1', title: 'GraphQL 入门', content: '这是一篇入门文章', authorId: '1' },
  { id: '2', title: 'Apollo Server', content: '如何使用 Apollo', authorId: '1' },
  { id: '3', title: 'Node.js 性能', content: '性能优化技巧', authorId: '2' },
];

const resolvers = {
  Query: {
    users: () => users,
    user: (parent, args) => users.find(u => u.id === args.id),
    posts: (parent, args) => args.authorId
      ? posts.filter(p => p.authorId === args.authorId)
      : posts,
    post: (parent, args) => posts.find(p => p.id === args.id),
  },

  Mutation: {
    createUser: (parent, args) => {
      const user = { id: String(users.length + 1), ...args.input };
      users.push(user);
      return user;
    },
    createPost: (parent, args) => {
      const post = { id: String(posts.length + 1), ...args.input };
      posts.push(post);
      return post;
    },
    deletePost: (parent, args) => {
      const idx = posts.findIndex(p => p.id === args.id);
      if (idx === -1) return false;
      posts.splice(idx, 1);
      return true;
    },
  },

  // 字段级 Resolver：自动关联
  User: {
    posts: (parent) => posts.filter(p => p.authorId === parent.id),
  },
  Post: {
    author: (parent) => users.find(u => u.id === parent.authorId),
  },
};

module.exports = resolvers;
```

关键点：`User.posts` 和 `Post.author` 是字段级 Resolver，当客户端查询到这些关联字段时才执行，体现了 GraphQL "按需加载" 的设计。

## Apollo Server + Express 集成

```javascript
// server.js
const express = require('express');
const cors = require('cors');
const { ApolloServer } = require('@apollo/server');
const { expressMiddleware } = require('@apollo/server/express4');
const typeDefs = require('./schema');
const resolvers = require('./resolvers');

async function startServer() {
  const app = express();

  const server = new ApolloServer({
    typeDefs,
    resolvers,
    // 共享 context
    context: async ({ req }) => {
      const token = req.headers.authorization || '';
      // 模拟从 token 解析用户
      return { authUser: { id: '1', role: 'user' } };
    },
  });

  await server.start();

  app.use(cors());
  app.use(express.json());
  app.use('/graphql', expressMiddleware(server, {
    context: async ({ req }) => ({
      authUser: { id: '1', role: 'user' },
    }),
  }));

  app.listen(4000, () => {
    console.log('GraphQL 服务运行在 http://localhost:4000/graphql');
  });
}

startServer();
```

## 查询语法：参数、别名与 Fragment

```graphql
# 1. 基础查询 —— 只取需要的字段
query GetUsers {
  users {
    id
    name
    email
  }
}

# 2. 带参数查询
query GetUser {
  user(id: "1") {
    name
    posts {
      title
    }
  }
}

# 3. 别名 —— 同一个字段查不同参数
query GetTwoUsers {
  alice: user(id: "1") { name }
  bob: user(id: "2") { name }
}

# 4. Fragment —— 复用字段集
query GetUsersWithPosts {
  users {
    ...UserFields
    posts { title }
  }
}

fragment UserFields on User {
  id
  name
  email
}
```

## DataLoader 解决 N+1 问题

N+1 问题是 GraphQL 的经典性能陷阱：查询 N 个用户时，每个用户的 posts 字段都触发一次数据库查询。DataLoader 可以批量合并和缓存这些查询。

```bash
npm install dataloader
```

```javascript
const DataLoader = require('dataloader');

// 创建批量加载函数
const postLoader = new DataLoader(async (authorIds) => {
  // authorIds 是所有待查的作者 ID 列表
  // 批量查询，一次 SQL 解决
  const allPosts = await db.query(
    'SELECT * FROM posts WHERE author_id = ANY($1)',
    [authorIds]
  );
  // 按 authorId 分组返回
  return authorIds.map(id => allPosts.filter(p => p.authorId === id));
});

// 在 Resolver 中使用
const resolvers = {
  User: {
    posts: (parent) => postLoader.load(parent.id),
  },
};
```

DataLoader 在每个请求内缓存结果，同一请求中对同一 key 的多次查询只会访问一次数据源。

## Context 高级用法

```javascript
// 共享数据库连接和认证信息
const server = new ApolloServer({
  typeDefs,
  resolvers,
});

app.use('/graphql', expressMiddleware(server, {
  context: async ({ req }) => {
    const token = req.headers.authorization?.replace('Bearer ', '');
    let user = null;
    if (token) {
      user = await verifyToken(token); // JWT 验证
    }
    return {
      db: getDbConnection(),
      user,
      DataLoader: {
        posts: new DataLoader(batchGetPosts),
      },
    };
  },
}));

// Resolver 中使用 context
const resolvers = {
  Mutation: {
    createPost: async (parent, { input }, context) => {
      if (!context.user) {
        throw new GraphQLError('未授权', {
          extensions: { code: 'UNAUTHORIZED' },
        });
      }
      return context.db.posts.create({ ...input, authorId: context.user.id });
    },
  },
};
```

## 总结

GraphQL 的核心优势是让客户端按需获取数据，尤其适合多客户端、复杂数据关联的场景。Apollo Server 提供了 Schema 定义、Resolvers、Context、DataLoader 等完备的工具链。关键记忆点：Schema 定义数据形状 → Resolvers 实现数据获取 → Context 共享 DB/认证 → DataLoader 解决 N+1。如果你的 API 需要同时服务 Web、iOS、Android 多个前端，GraphQL 是比 REST 更高效的选择。
