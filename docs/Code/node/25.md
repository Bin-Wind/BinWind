# Docker 容器化 Node.js 应用

Docker 能将你的 Node.js 应用及其所有依赖打包成一个轻量级、可移植的容器。无论是在本地开发、CI 测试还是云服务器部署，容器都能保证环境一致性。本文覆盖从写 Dockerfile 到多服务编排的完整体验。

## Dockerfile 基础

### 最简 Dockerfile

```dockerfile
# Dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./

RUN npm ci --production

COPY . .

EXPOSE 3000

CMD ["node", "src/app.js"]
```

关键指令解读：

- `FROM`：基础镜像，`alpine` 版本体积极小（约 50MB），适合生产
- `WORKDIR`：设置容器内工作目录，不存在则自动创建
- `COPY`：复制文件到容器
- `RUN`：在构建阶段执行命令
- `EXPOSE`：声明容器监听的端口（文档作用，不实际映射）
- `CMD`：容器启动时的默认命令

### .dockerignore

```dockerignore
node_modules
npm-debug.log
.git
.gitignore
.env
.env.*
*.md
.vscode
.idea
coverage
test
tests
dist
```

`.dockerignore` 能显著加快构建速度，削减镜像体积。最重要的规则是排 `node_modules`，因为会通过 `npm ci` 在容器内重新安装。

## 构建与运行

```bash
# 构建镜像
docker build -t my-node-app .

# 查看镜像
docker images | grep my-node-app

# 运行容器
docker run -p 3000:3000 --name my-app my-node-app

# 后台运行 + 环境变量
docker run -d \
  -p 3000:3000 \
  --name my-app \
  -e NODE_ENV=production \
  -e DATABASE_URL=postgresql://host/db \
  my-node-app

# 常用管理命令
docker logs my-app          # 查看日志
docker logs -f my-app       # 跟踪日志
docker exec -it my-app sh   # 进入容器 shell
docker stop my-app          # 停止
docker rm my-app            # 删除
docker restart my-app       # 重启
```

## 多阶段构建 (Multi-stage Build)

多阶段构建将编译环境和运行环境分离，最终镜像只含必要的生产文件，体积可减少 60%-80%。

```dockerfile
# Dockerfile — 多阶段构建
# ============ 阶段一：构建 ============
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./

# 安装全部依赖（含 devDependencies）
RUN npm ci

COPY . .

# TypeScript 编译或其他构建
RUN npm run build

# 删除 devDependencies
RUN npm prune --production

# ============ 阶段二：生产运行 ============
FROM node:18-alpine

WORKDIR /app

# 从构建阶段复制精简后的 node_modules
COPY --from=builder /app/node_modules ./node_modules

# 复制编译产物
COPY --from=builder /app/dist ./dist

# 复制 package.json（运行时可能需要 scripts）
COPY --from=builder /app/package.json ./

EXPOSE 3000

CMD ["node", "dist/app.js"]
```

## 非 Root 用户运行

默认容器以 root 运行，若应用被攻破，攻击者可能获得容器 root 权限。

```dockerfile
FROM node:18-alpine

# 创建非 root 用户
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup

WORKDIR /app

COPY --chown=appuser:appgroup package*.json ./
RUN npm ci --production

COPY --chown=appuser:appgroup . .

# 切换到非 root 用户
USER appuser

EXPOSE 3000

CMD ["node", "src/app.js"]
```

## 信号处理 (SIGTERM)

Docker 停止容器时会向 PID 1 发送 SIGTERM，如果应用不处理该信号，10 秒后被 SIGKILL 强制杀死。Express 默认不处理 SIGTERM，需手动实现优雅关闭。

```javascript
// app.js
const app = require('./server');

const server = app.listen(3000);

// 优雅关闭
function gracefulShutdown(signal) {
  console.log(`收到 ${signal} 信号，开始优雅关闭...`);

  server.close(() => {
    console.log('HTTP 服务已关闭');
    // 在这里关闭数据库连接、Redis 等
    // await db.disconnect();
    process.exit(0);
  });

  // 强制退出兜底
  setTimeout(() => {
    console.error('强制退出');
    process.exit(1);
  }, 10000);
}

process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
process.on('SIGINT', () => gracefulShutdown('SIGINT'));
```

在 Dockerfile 中确保 Node 作为 PID 1 运行：

```dockerfile
CMD ["node", "src/app.js"]  # Node 是 PID 1，直接接收信号
# 不要写：
# CMD npm start  # npm 作为 PID 1，可能不转发信号给 Node
```

## HEALTHCHECK

```dockerfile
# Dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1
```

对应的 Express 端点：

```javascript
app.get('/health', (req, res) => {
  res.json({
    status: 'ok',
    uptime: process.uptime(),
    timestamp: new Date().toISOString(),
  });
});
```

## docker-compose.yml — 多服务编排

```yaml
# docker-compose.yml
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: node-app
    ports:
      - '3000:3000'
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgres://postgres:password@db:5432/myapp
      - REDIS_URL=redis://redis:6379
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    restart: unless-stopped
    volumes:
      - ./uploads:/app/uploads
    networks:
      - app-network

  db:
    image: postgres:16-alpine
    container_name: node-db
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
      POSTGRES_DB: myapp
    volumes:
      - pgdata:/var/lib/postgresql/data
    networks:
      - app-network
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U postgres']
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: node-redis
    networks:
      - app-network

volumes:
  pgdata:

networks:
  app-network:
    driver: bridge
```

使用：

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f app

# 重新构建并启动
docker-compose up -d --build

# 停止所有服务
docker-compose down

# 停止并清除数据卷（⚠️ 小心）
docker-compose down -v
```

## 环境变量管理

```bash
# 方式一：命令行
docker run -e NODE_ENV=production my-app

# 方式二：env 文件
docker run --env-file .env.production my-app

# 方式三：Dockerfile 默认值
ENV NODE_ENV=production
```

```yaml
# docker-compose 中使用 env_file
services:
  app:
    env_file:
      - .env.production
```

## 生产镜像最佳实践 Checklist

| 项目 | 做法 |
|------|------|
| 基础镜像 | `node:18-alpine`（体积小） |
| .dockerignore | 排除 node_modules, .git, .env |
| 安装命令 | `npm ci --production`（更快更确定） |
| 多阶段构建 | 构建 + 运行分离 |
| 非 root 用户 | `USER appuser` |
| 信号处理 | 应用监听 SIGTERM + CMD 让 Node 作为 PID 1 |
| 健康检查 | HEALTHCHECK 指令 |
| 镜像扫描 | `docker scan my-app`（Snyk 漏洞扫描） |
| 日志管理 | 输出到 stdout/stderr，由 Docker 收集 |

## 完整生产 Dockerfile 模板

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build
RUN npm prune --production

FROM node:18-alpine
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup
WORKDIR /app
COPY --chown=appuser:appgroup --from=builder /app/node_modules ./node_modules
COPY --chown=appuser:appgroup --from=builder /app/dist ./dist
COPY --chown=appuser:appgroup --from=builder /app/package.json ./

USER appuser
EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

CMD ["node", "dist/app.js"]
```

## 总结

Docker 化 Node.js 应用的核心四步：写 `.dockerignore` 排除无用文件、用多阶段构建缩小镜像、以非 root 用户运行、处理 SIGTERM 优雅关闭。`docker-compose` 让多服务（Node + Postgres + Redis）一键启动。掌握这些，你的应用就能在任何地方以一致的环境运行。
