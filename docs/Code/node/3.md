# 异步编程入门：Callback、Promise 与 Async/Await

Node.js 基于事件驱动和非阻塞 I/O，异步编程是其核心。本文梳理从 Callback 到 Async/Await 的演进路径。

## Callback 模式

回调是 Node.js 最早的异步处理方式，遵循 error-first 约定。

### Error-first Callback

```js
const fs = require('fs');

fs.readFile('/path/to/file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('读取失败:', err.message);
    return;
  }
  console.log(data);
});
```

### 回调地狱 (Callback Hell)

当多个异步操作串行时，代码会层层嵌套：

```js
fs.readFile('config.json', 'utf8', (err, config) => {
  if (err) return console.error(err);
  db.connect(JSON.parse(config).dbUrl, (err, connection) => {
    if (err) return console.error(err);
    connection.query('SELECT * FROM users', (err, users) => {
      if (err) return console.error(err);
      connection.query('SELECT * FROM orders', (err, orders) => {
        if (err) return console.error(err);
        console.log({ users, orders });
        connection.close();
      });
    });
  });
});
```

每一层嵌套都增加缩进，错误处理散落各处，代码难以维护。

## Promise

Promise 代表一个异步操作的最终结果，有三种状态：pending、fulfilled、rejected。

### 创建 Promise

```js
function readFilePromise(path) {
  return new Promise((resolve, reject) => {
    fs.readFile(path, 'utf8', (err, data) => {
      if (err) reject(err);
      else resolve(data);
    });
  });
}
```

### then / catch / finally

```js
readFilePromise('data.json')
  .then((data) => JSON.parse(data))
  .then((json) => {
    console.log('解析结果:', json);
  })
  .catch((err) => {
    console.error('出错:', err.message);
  })
  .finally(() => {
    console.log('操作结束');
  });
```

### Promise 组合方法

```js
// Promise.all — 全部成功才成功，一个失败即失败
const [users, orders] = await Promise.all([
  fetch('/api/users'),
  fetch('/api/orders'),
]);

// Promise.allSettled — 等待全部完成，不管成败
const results = await Promise.allSettled([
  fetch('/api/users'),
  fetch('/api/orders'),
]);
results.forEach((r) => {
  if (r.status === 'fulfilled') console.log(r.value);
  else console.error(r.reason);
});

// Promise.race — 第一个完成即返回
const result = await Promise.race([
  fetch('/api/fast'),
  new Promise((_, reject) => setTimeout(() => reject(new Error('超时')), 3000)),
]);

// Promise.any — 第一个成功即返回，全失败才 reject
const first = await Promise.any([
  fetch('https://cdn1.example.com/data'),
  fetch('https://cdn2.example.com/data'),
  fetch('https://cdn3.example.com/data'),
]);
```

## Async / Await

Async/Await 是 Promise 的语法糖，让异步代码看起来像同步代码。

### 基本用法

```js
async function loadUserData(userId) {
  try {
    const user = await db.query('SELECT * FROM users WHERE id = $1', [userId]);
    const orders = await db.query('SELECT * FROM orders WHERE user_id = $1', [userId]);
    return { user, orders };
  } catch (err) {
    console.error('查询失败:', err.message);
    throw err;
  }
}
```

### 串行 vs 并行执行

```js
// ❌ 串行 — 两个无关联的查询依次执行，浪费时间
async function loadData() {
  const users = await fetch('/api/users');   // 等待 200ms
  const orders = await fetch('/api/orders'); // 再等待 200ms
  // 总耗时 ~400ms
}

// ✅ 并行 — 同时发起，总耗时取最长
async function loadData() {
  const [users, orders] = await Promise.all([
    fetch('/api/users'),   // 200ms
    fetch('/api/orders'),  // 200ms
  ]);
  // 总耗时 ~200ms
}
```

## 三种方式错误处理对比

```js
// Callback
function callbackStyle(cb) {
  doAsync((err, result) => {
    if (err) return cb(err);
    cb(null, processResult(result));
  });
}

// Promise
function promiseStyle() {
  return doAsync()
    .then(processResult)
    .catch(handleError);
}

// Async/Await
async function asyncStyle() {
  try {
    const result = await doAsync();
    return processResult(result);
  } catch (err) {
    handleError(err);
  }
}
```

## 常见错误

### 1. 忘记 await

```js
// ❌ data 是 Promise 对象，不是实际值
const data = fetch('/api/users');
console.log(data); // Promise { <pending> }

// ✅
const data = await fetch('/api/users');
```

### 2. 未处理的 Promise rejection

```js
// ❌ 没有任何 catch，进程会收到 unhandledRejection 警告
async function main() {
  await fetch('/api/might-fail');
}
main();

// ✅
main().catch((err) => console.error('启动失败:', err));
```

### 3. 循环中滥用 await

```js
// ❌ 串行执行，每次等待上一个完成
for (const id of ids) {
  await processItem(id);
}

// ✅ 并行执行，同时处理所有
await Promise.all(ids.map((id) => processItem(id)));
```

## 实用示例：综合场景

```js
async function buildUserReport(userId) {
  try {
    // 并行获取不相关的数据
    const [user, orders, settings] = await Promise.all([
      fetch(`/api/users/${userId}`).then((r) => r.json()),
      fetch(`/api/users/${userId}/orders`).then((r) => r.json()),
      fetch(`/api/settings`).then((r) => r.json()),
    ]);

    // 对 orders 做串行处理（每条需要上一条的上下文）
    let total = 0;
    for (const order of orders) {
      const detail = await fetch(`/api/orders/${order.id}/detail`).then((r) => r.json());
      total += detail.amount;
    }

    return { user, total, currency: settings.currency };
  } catch (err) {
    console.error('构建报告失败:', err);
    return null;
  }
}
```

## 总结

- **Callback**: 最基础，适合简单场景，复杂逻辑易陷入回调地狱。
- **Promise**: 链式调用解决嵌套问题，提供 all/race 等组合能力。
- **Async/Await**: 语法最清晰，配合 try/catch 错误处理自然，推荐作为默认选择。
