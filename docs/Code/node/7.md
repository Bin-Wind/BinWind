# 环境变量管理 (dotenv)

环境变量是管理不同环境（开发、测试、生产）配置的标准做法。dotenv 让 Node.js 应用从 `.env` 文件加载配置变得简单。

## 安装与基础使用

```bash
npm install dotenv
```

```js
// 在项目入口文件最开头加载
require('dotenv').config();

console.log(process.env.DATABASE_URL);
console.log(process.env.PORT);
```

ESM 中：

```js
import 'dotenv/config';
```

## .env 文件格式

```bash
# 服务器配置
PORT=3000
NODE_ENV=development
HOST=localhost

# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
DATABASE_POOL_MAX=20

# JWT 密钥
JWT_SECRET=your-super-secret-key-change-in-production
JWT_EXPIRES_IN=7d

# 第三方 API
REDIS_URL=redis://localhost:6379
AWS_REGION=us-east-1
S3_BUCKET=my-app-uploads

# 功能开关
ENABLE_CACHE=true
ENABLE_EMAIL=false
```

规则：`KEY=VALUE`，不需要引号，`#` 表示注释，等号两侧不要有空格。

## 读取环境变量

```js
const port = process.env.PORT || 3000;
const dbUrl = process.env.DATABASE_URL;

if (!dbUrl) {
  console.error('DATABASE_URL 环境变量未设置');
  process.exit(1);
}
```

## 多环境配置

```
.env                  # 默认，所有环境共享
.env.development      # 开发环境
.env.production       # 生产环境
.env.test             # 测试环境
.env.local            # 本地覆盖，添加到 .gitignore
```

### 根据 NODE_ENV 加载不同文件

```js
const envFile = `.env.${process.env.NODE_ENV || 'development'}`;
require('dotenv').config({ path: envFile });

console.log(`从 ${envFile} 加载环境变量`);
```

### 多文件叠加加载

```js
// 先加载默认 .env，再用环境特定的覆盖
require('dotenv').config();                          // .env
require('dotenv').config({ path: `.env.${process.env.NODE_ENV}` }); // .env.production
require('dotenv').config({ path: '.env.local' });   // .env.local 最高优先级
```

## 安全最佳实践

### .gitignore

```bash
# .gitignore
.env
.env.*
!.env.example
```

只有 `.env.example` 提交到仓库，作为模板：

```bash
# .env.example — 提交到 Git
PORT=3000
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
JWT_SECRET=your-secret-key
REDIS_URL=redis://localhost:6379
```

### 生产环境管理

生产环境不应使用 `.env` 文件，而是通过部署平台注入：

```bash
# Docker
docker run -e DATABASE_URL=... -e JWT_SECRET=... my-app

# Docker Compose
environment:
  - DATABASE_URL=${DATABASE_URL}
  - JWT_SECRET=${JWT_SECRET}

# K8s Secret
env:
  - name: DATABASE_URL
    valueFrom:
      secretKeyRef:
        name: db-secret
        key: url
```

## Config 模块模式

将所有环境变量集中在 config 文件中管理，提供默认值和类型转换：

```js
// config/index.js
require('dotenv').config();

module.exports = {
  // 服务器
  port: parseInt(process.env.PORT, 10) || 3000,
  env: process.env.NODE_ENV || 'development',
  isDev: process.env.NODE_ENV === 'development',
  isProd: process.env.NODE_ENV === 'production',

  // 数据库
  db: {
    url: process.env.DATABASE_URL || 'postgresql://localhost:5432/mydb',
    poolMax: parseInt(process.env.DATABASE_POOL_MAX, 10) || 10,
    ssl: process.env.DATABASE_SSL === 'true',
  },

  // JWT
  jwt: {
    secret: process.env.JWT_SECRET || 'dev-secret-do-not-use-in-prod',
    expiresIn: process.env.JWT_EXPIRES_IN || '7d',
  },

  // Redis
  redis: {
    url: process.env.REDIS_URL || 'redis://localhost:6379',
    password: process.env.REDIS_PASSWORD,
  },

  // 功能开关
  features: {
    cache: process.env.ENABLE_CACHE !== 'false',
    email: process.env.ENABLE_EMAIL === 'true',
  },

  // 上传
  upload: {
    maxSize: parseInt(process.env.UPLOAD_MAX_SIZE, 10) || 5 * 1024 * 1024,
    allowedTypes: (process.env.UPLOAD_ALLOWED_TYPES || 'jpg,png,gif').split(','),
  },
};
```

使用示例：

```js
const config = require('./config');

const app = express();
app.listen(config.port, () => {
  console.log(`Server running on port ${config.port} (${config.env})`);
});

// 数据库连接
const pool = new Pool({
  connectionString: config.db.url,
  max: config.db.poolMax,
  ssl: config.db.ssl ? { rejectUnauthorized: false } : false,
});
```

## 环境变量验证

使用 envalid 确保必需的环境变量存在且格式正确：

```bash
npm install envalid
```

```js
const { cleanEnv, str, num, bool, url } = require('envalid');

const env = cleanEnv(process.env, {
  PORT:           num({ default: 3000 }),
  NODE_ENV:       str({ choices: ['development', 'production', 'test'], default: 'development' }),
  DATABASE_URL:   url(),
  JWT_SECRET:     str({ default: 'dev-secret' }),
  REDIS_URL:      url({ default: 'redis://localhost:6379' }),
  ENABLE_CACHE:   bool({ default: true }),
});

module.exports = env;
```

应用启动时验证，如果缺少必要的环境变量会立即抛出明确错误，避免运行到一半才发现问题。

## 常见问题

**1. dotenv 为什么要在最开头加载？** dotenv 在 `config()` 时把 `.env` 内容注入 `process.env`。如果其他模块在之前就读取了环境变量（如 Sequelize 初始化），会拿到 undefined。把 `require('dotenv').config()` 放在入口文件第一行即可避免。

**2. .env 文件中的值为什么全是字符串？** 环境变量始终是字符串。数字 `3000` 实际是 `'3000'`。用 `parseInt` 或 envalid 转换类型。

**3. NODE_ENV 的值有哪些？** 约定俗成的值：`development`、`production`、`test`。Express 会根据 NODE_ENV 调整缓存策略和错误输出。

## 总结

dotenv 是配置管理的起点：`.env` 文件存储敏感和非公开配置，`.env.example` 提交到仓库作为模板，config.js 集中管理并提供默认值，生产环境通过平台注入真实配置。
