# Python 环境安装与配置

搭建一个干净、可复用的 Python 开发环境是高效编程的第一步。本文将带你从零开始，完成 Python 版本管理、虚拟环境和依赖管理的完整配置。

## 安装 pyenv - 多版本管理

pyenv 让你在同一台机器上安装并切换多个 Python 版本，互不干扰。

```bash
# macOS 安装
brew install pyenv

# 查看可安装的 Python 版本
pyenv install --list | grep "^\s*3\.[89]\|^3\.1[0-2]"

# 安装 Python 3.11
pyenv install 3.11.7

# 设置全局默认版本
pyenv global 3.11.7

# 检查当前版本
python --version
```

配置 shell 环境（添加到 `~/.zshrc` 或 `~/.bashrc`）：

```bash
export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init -)"
```

## 虚拟环境 venv - 项目隔离

每个项目应该有独立的依赖空间，避免版本冲突。

```bash
# 创建虚拟环境
python -m venv myproject-env

# 激活虚拟环境
source myproject-env/bin/activate   # macOS / Linux
# myproject-env\Scripts\activate    # Windows

# 确认当前环境
which python

# 退出虚拟环境
deactivate
```

虚拟环境激活后，`pip install` 安装的包仅作用于当前项目。

## pip - Python 包管理器

```bash
# 安装包
pip install requests
pip install flask==2.3.0           # 指定版本
pip install "django>=4.2,<5.0"    # 版本范围

# 查看已安装的包
pip list
pip show requests                   # 查看某个包的详细信息

# 卸载
pip uninstall requests

# 升级 pip 自身
pip install --upgrade pip
```

## requirements.txt - 依赖清单

```bash
# 冻结当前环境的所有依赖
pip freeze > requirements.txt

# 从清单安装依赖
pip install -r requirements.txt
```

典型的 requirements.txt 内容：

```
Django==4.2.7
requests==2.31.0
gunicorn==21.2.0
```

## 开发环境推荐结构

```
myproject/
├── .python-version       # pyenv 使用，指定 Python 版本
├── venv/                 # 虚拟环境目录（加入 .gitignore）
├── requirements.txt      # 生产依赖
├── requirements-dev.txt  # 开发依赖（pytest, black 等）
├── src/                  # 源代码
└── tests/                # 测试代码
```

## 常见问题

**Q: pip install 太慢？** 使用国内镜像源：

```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests
# 或永久配置
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

**Q: 虚拟环境和 pyenv 的关系？** pyenv 管理"用哪个 Python 版本"，venv 管理"这个项目装哪些包"。两者配合使用：pyenv 选版本，venv 建隔离环境。
