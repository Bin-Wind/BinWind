# Python 并发编程入门

Python 的并发编程因其 GIL（Global Interpreter Lock，全局解释器锁）而独具特色。理解不同的并发模型及其适用场景，是写出高性能 Python 程序的关键。

## GIL：Python 并发的核心约束

GIL 是 CPython 解释器中的一个互斥锁，确保同一时刻只有一个线程执行 Python 字节码。这意味着即使在多核 CPU 上，多线程 Python 程序也无法真正并行执行 CPU 密集型任务。GIL 的设计简化了 CPython 的内存管理，但也成为并发性能的瓶颈。

## threading：多线程

多线程适用于 I/O 密集型任务（网络请求、文件读写、数据库查询），因为线程在等待 I/O 时会释放 GIL。

```python
import threading
import time
from concurrent.futures import ThreadPoolExecutor

def download(url: str) -> str:
    """模拟下载任务"""
    print(f"开始下载: {url}")
    time.sleep(2)  # 模拟 I/O 等待
    print(f"下载完成: {url}")
    return f"{url} content"

urls = ["https://a.com", "https://b.com", "https://c.com"]

# 方式一：手动创建线程
threads = []
for url in urls:
    t = threading.Thread(target=download, args=(url,))
    t.start()
    threads.append(t)

for t in threads:
    t.join()

# 方式二：使用 ThreadPoolExecutor（推荐）
with ThreadPoolExecutor(max_workers=3) as executor:
    results = list(executor.map(download, urls))
```

## multiprocessing：多进程

多进程绕过 GIL 限制，每个进程有独立的 Python 解释器和内存空间，适用于 CPU 密集型任务（数值计算、图像处理、数据转换）。

```python
import multiprocessing
from concurrent.futures import ProcessPoolExecutor

def cpu_intensive(n: int) -> int:
    """模拟 CPU 密集计算"""
    total = 0
    for i in range(10**7):
        total += i * n
    return total

# 方式一：手动创建进程池
with multiprocessing.Pool(processes=4) as pool:
    results = pool.map(cpu_intensive, [1, 2, 3, 4])

# 方式二：使用 ProcessPoolExecutor（推荐）
with ProcessPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(cpu_intensive, [1, 2, 3, 4]))
```

进程间不共享内存，数据传递需要序列化（pickle），较大的数据会有额外开销。

## concurrent.futures：统一的并发接口

`concurrent.futures` 模块提供了高层次的异步执行接口，屏蔽了线程和进程的底层细节。

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import requests

def fetch_status(url: str) -> tuple[str, int]:
    resp = requests.get(url, timeout=10)
    return url, resp.status_code

urls = ["https://httpbin.org/get", "https://httpbin.org/delay/2"]

with ThreadPoolExecutor(max_workers=3) as executor:
    # 提交任务并获取 Future 对象
    futures = {executor.submit(fetch_status, url): url for url in urls}

    # 按完成顺序处理结果
    for future in as_completed(futures):
        url, status = future.result()
        print(f"{url} -> {status}")
```

`map` 保持顺序，`as_completed` 按完成顺序迭代，选择取决于是否关心结果顺序。

## asyncio：协程与异步编程

asyncio 是 Python 官方提供的异步 I/O 框架，使用单线程事件循环来处理并发 I/O。它是最高效的 I/O 密集型任务方案。

```python
import asyncio
import aiohttp

async def fetch(session: aiohttp.ClientSession, url: str) -> str:
    async with session.get(url) as response:
        return await response.text()

async def main():
    urls = [
        "https://httpbin.org/get",
        "https://httpbin.org/uuid",
        "https://httpbin.org/ip",
    ]
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, url) for url in urls]
        results = await asyncio.gather(*tasks)
        for url, content in zip(urls, results):
            print(f"{url}: {len(content)} bytes")

asyncio.run(main())
```

协程用 `async def` 定义，`await` 挂起当前协程让出控制权给事件循环。注意 asyncio 生态需要使用支持异步的库（如 `aiohttp` 替代 `requests`），否则阻塞调用会卡住整个事件循环。

## 如何选择并发模型

| 任务类型 | 推荐方案 | 原因 |
|----------|----------|------|
| I/O 密集型（网络请求） | asyncio + aiohttp | 最高效，单线程即可处理上千并发连接 |
| I/O 密集型（文件操作） | threading | 文件 I/O 通常阻塞，多线程简单够用 |
| CPU 密集型（数值计算） | multiprocessing | 绕过 GIL，利用多核 |
| 混合型任务 | multiprocessing + asyncio | 进程内跑事件循环 |
| 简单并发 | concurrent.futures | 统一 API，代码简洁 |

## 线程安全与共享数据

多线程共享数据时需要同步机制：

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    for _ in range(100_000):
        with lock:          # 临界区保护
            counter += 1

threads = [threading.Thread(target=increment) for _ in range(4)]
for t in threads:
    t.start()
for t in threads:
    t.join()

print(f"counter = {counter}")  # 400000（加锁正确）
```

理解 Python 的并发模型，核心是记住：**I/O 用 asyncio，CPU 用 multiprocessing，简单场景用 concurrent.futures，线程是最后考虑的默认选项**。
