# 类型注解与静态检查

Python 是动态类型语言，但自 PEP 484 引入 Type Hints 以来，类型注解逐渐成为 Python 开发的最佳实践。类型注解不改变运行时行为，但配合静态检查工具能提前发现大量 bug，并提供更好的 IDE 支持。

## 基础类型注解

从最简单的基本类型开始：

```python
# 基本类型
name: str = "Alice"
age: int = 30
price: float = 19.99
is_active: bool = True

# 复合类型
from typing import Optional, Union

def greet(name: str) -> str:
    return f"Hello, {name}!"

def find_user(user_id: int) -> Optional[dict]:
    """可能返回 None 的函数"""
    users = {1: {"name": "Alice"}}
    return users.get(user_id)

# Union 类型（Python 3.10+ 可用 | 语法）
def process(data: str | int | None) -> str:
    if data is None:
        return "no data"
    return str(data)
```

## 容器类型注解

`typing` 模块为内置容器提供了泛型版本：

```python
from typing import List, Dict, Set, Tuple, Sequence

# Python 3.9+ 可直接使用内置类型
names: list[str] = ["Alice", "Bob"]
scores: dict[str, float] = {"Alice": 95.5, "Bob": 87.0}
unique_ids: set[int] = {1, 2, 3}
point: tuple[float, float] = (3.0, 4.0)

# 兼容老版本的写法
from typing import List, Dict
names: List[str] = ["Alice", "Bob"]
scores: Dict[str, float] = {"Alice": 95.5}
```

## TypedDict：结构化的字典类型

当字典有固定的 key 结构时，TypedDict 比 dict 更精确：

```python
from typing import TypedDict, NotRequired

class User(TypedDict):
    name: str
    age: int
    email: str
    verified: NotRequired[bool]  # 可选字段

def create_user(data: User) -> str:
    return f"User {data['name']} created"

user: User = {
    "name": "Alice",
    "age": 30,
    "email": "alice@example.com",
}
```

## Protocol：结构化子类型

Protocol 让你定义接口而无需显式继承，实现"鸭子类型"的静态检查：

```python
from typing import Protocol

class Drawable(Protocol):
    def draw(self) -> None: ...
    def resize(self, width: float, height: float) -> None: ...

class Circle:
    def draw(self) -> None:
        print("Drawing circle")
    def resize(self, width: float, height: float) -> None:
        print(f"Resized to {width}x{height}")

class Button:
    def draw(self) -> None:
        print("Drawing button")
    def resize(self, width: float, height: float) -> None:
        print(f"Button {width}x{height}")

def render(shape: Drawable) -> None:
    shape.draw()

render(Circle())   # OK — Circle 隐式满足 Drawable 协议
render(Button())   # OK — Button 也满足
```

## dataclass：减少样板代码的数据类

`dataclass` 是定义数据容器的利器，自动生成 `__init__`、`__repr__`、`__eq__` 等方法：

```python
from dataclasses import dataclass, field
from typing import ClassVar

@dataclass
class Product:
    name: str
    price: float
    stock: int = 0
    tags: list[str] = field(default_factory=list)

    # 实例变量（不在 __init__ 中）
    _cache: str = field(default="", repr=False)
    # 类变量（不在 __init__ 中）
    category: ClassVar[str] = "general"

    @property
    def is_available(self) -> bool:
        return self.stock > 0

p1 = Product(name="Laptop", price=999.99, stock=10)
p2 = Product(name="Laptop", price=999.99, stock=10)
print(p1 == p2)  # True
print(p1)        # Product(name='Laptop', price=999.99, stock=10)
```

## mypy 静态类型检查

`mypy` 是目前最成熟的 Python 静态类型检查器：

```bash
# 安装
pip install mypy

# 检查单个文件
mypy app.py

# 检查整个项目
mypy src/

# 常用配置（pyproject.toml）
[tool.mypy]
strict = true
disallow_untyped_defs = true
warn_return_any = true
ignore_missing_imports = false
```

示例：`mypy` 能发现类型不匹配的 bug：

```python
# app.py
def add(x: int, y: int) -> int:
    return x + y

result = add("hello", 5)  # mypy 报错: error: Argument 1 has incompatible type "str"
```

## 高级类型技巧

```python
from typing import Callable, TypeVar, Generic, Literal

# 泛型函数（TypeVar）
T = TypeVar("T")

def first(items: list[T], default: T | None = None) -> T | None:
    return items[0] if items else default

print(first([1, 2, 3]))       # 类型推断为 int | None
print(first(["a", "b", "c"])) # 类型推断为 str | None

# Literal 类型（限定值的范围）
def set_status(status: Literal["active", "inactive", "pending"]) -> None:
    print(f"Status set to {status}")

set_status("active")   # OK
# set_status("unknown") # mypy 报错

# Callable 类型
Handler = Callable[[int, str], bool]  # 接受 int 和 str，返回 bool

def register(callback: Handler) -> None:
    result = callback(42, "test")
```

## 渐进式采用策略

向已有项目添加类型注解不需要一次完成：

1. **先加函数签名** — 给公开 API 的函数加上参数和返回值类型。
2. **配置 mypy 宽松模式** — 初期只检查修改过的文件，逐步收紧规则。
3. **使用 `Any` 作为过渡** — 不确定的类型先用 `Any`，后续再精确化。
4. **CI 集成** — 把 `mypy` 加入 CI 流程，新代码必须有类型注解。

类型注解是现代 Python 工程的标配。配合 `mypy` + IDE（PyCharm / VSCode），你能在编码时就获得实时类型检查和智能补全，大幅减少运行时类型错误。
