# 单元测试

单元测试是保证代码质量的基础防线。Python 标准库自带了 `unittest`，而第三方框架 `pytest` 已经事实上成为社区标准。本文以 pytest 为主线，介绍测试的核心实践。

## pytest 快速开始

```bash
pip install pytest
```

编写第一个测试文件 `test_calculator.py`：

```python
# src/calculator.py
def add(a: float, b: float) -> float:
    return a + b

def divide(a: float, b: float) -> float:
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

```python
# tests/test_calculator.py
from calculator import add, divide
import pytest

def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0.1, 0.2) == pytest.approx(0.3)

def test_divide():
    assert divide(10, 2) == 5

def test_divide_by_zero():
    with pytest.raises(ValueError, match="Cannot divide by zero"):
        divide(10, 0)
```

```bash
# 运行测试
pytest                    # 自动发现 test_*.py / *_test.py
pytest -v                 # 详细输出
pytest -s                 # 打印 stdout/stderr
pytest tests/test_calculator.py::test_add  # 运行单个测试
```

## Test Discovery 规则

pytest 默认按以下规则查找测试文件：

- 文件名匹配 `test_*.py` 或 `*_test.py`
- 以 `Test` 开头的类（不含 `__init__` 方法）
- 以 `test_*` 开头的函数

推荐的目录结构：

```
project/
├── src/
│   └── calculator.py
├── tests/
│   ├── __init__.py
│   ├── conftest.py         # 共享 fixture
│   ├── test_calculator.py
│   └── test_integration.py
└── pyproject.toml
```

## Fixture：测试的依赖注入

Fixture 是 pytest 最强大的特性之一，用于管理测试的前置条件和清理工作。

```python
import pytest
import sqlite3
from typing import Generator

# 简单 fixture
@pytest.fixture
def sample_data() -> dict:
    return {"name": "Alice", "age": 30}

def test_with_fixture(sample_data: dict):
    assert sample_data["name"] == "Alice"

# 带清理的 fixture
@pytest.fixture
def db_connection() -> Generator[sqlite3.Connection, None, None]:
    conn = sqlite3.connect(":memory:")
    conn.execute("CREATE TABLE users (id INTEGER, name TEXT)")
    yield conn          # 测试使用这个值
    conn.close()        # 测试完成后执行清理

def test_db_insert(db_connection: sqlite3.Connection):
    db_connection.execute("INSERT INTO users VALUES (1, 'Alice')")
    result = db_connection.execute("SELECT name FROM users").fetchone()
    assert result[0] == "Alice"

# fixture 作用域
@pytest.fixture(scope="module")  # function / class / module / session
def expensive_resource():
    """整个测试模块只创建一次"""
    return {"data": "expensive setup"}
```

## parametrize：参数化测试

用一组参数跑同一个测试逻辑，避免重复代码：

```python
@pytest.mark.parametrize("a,b,expected", [
    (2, 3, 5),
    (-1, 1, 0),
    (100, 200, 300),
    (0, 0, 0),
])
def test_add_parametrized(a: float, b: float, expected: float):
    assert add(a, b) == expected

# 组合参数化
@pytest.mark.parametrize("dividend", [10, 0, -10])
@pytest.mark.parametrize("divisor", [2, 5])
def test_divide_combinations(dividend: int, divisor: int):
    result = divide(dividend, divisor)
    assert result == dividend / divisor
```

## Mock 与 Patch：隔离外部依赖

测试不应该依赖真实的外部服务（数据库、API、文件系统）。`unittest.mock` 提供强大的模拟能力：

```python
from unittest.mock import Mock, patch, MagicMock
import requests

def fetch_user_data(api_url: str) -> dict:
    resp = requests.get(f"{api_url}/users/1", timeout=5)
    resp.raise_for_status()
    return resp.json()

# 方式一：手动创建 Mock 对象
def test_fetch_user_data():
    mock_response = Mock()
    mock_response.json.return_value = {"id": 1, "name": "Alice"}
    mock_response.raise_for_status = Mock()

    with patch("requests.get", return_value=mock_response) as mock_get:
        result = fetch_user_data("https://api.example.com")
        assert result["name"] == "Alice"
        mock_get.assert_called_once_with("https://api.example.com/users/1", timeout=5)

# 方式二：使用装饰器
@patch("requests.get")
def test_fetch_error(mock_get: Mock):
    mock_get.return_value.raise_for_status.side_effect = requests.HTTPError("404")

    with pytest.raises(requests.HTTPError):
        fetch_user_data("https://api.example.com")
```

## Coverage：代码覆盖率

`pytest-cov` 统计哪些代码行被测试覆盖了：

```bash
pip install pytest-cov

# 生成终端报告
pytest --cov=src --cov-report=term-missing

# 生成 HTML 报告
pytest --cov=src --cov-report=html
```

在 `pyproject.toml` 中配置：

```toml
[tool.coverage.run]
source = ["src"]
omit = ["*/tests/*", "*/migrations/*"]

[tool.coverage.report]
fail_under = 80
exclude_lines = [
    "pragma: no cover",
    "if __name__ == .__main__.:",
]
```

## 测试驱动开发（TDD）实践

TDD 的三步循环：**Red**（写一个失败的测试）→ **Green**（写最少代码让测试通过）→ **Refactor**（重构优化）。

```python
# Step 1: Red — 先写测试
def test_validate_email():
    assert validate_email("a@b.com") == True
    assert validate_email("invalid") == False

# Step 2: Green — 实现最简单版本
def validate_email(email: str) -> bool:
    return "@" in email and "." in email.split("@")[-1]

# Step 3: Refactor — 扩展测试然后改进实现
def test_validate_email_empty():
    assert validate_email("") == False

def validate_email(email: str) -> bool:
    if not email:
        return False
    parts = email.split("@")
    if len(parts) != 2 or not parts[0] or not parts[1]:
        return False
    return "." in parts[1]
```

## 编写好测试的原则

- **独立** — 每个测试不依赖其他测试的运行顺序。
- **快速** — 单元测试应该毫秒级完成，CI 中跑全部测试不超过几分钟。
- **可读** — 测试即文档，清晰的测试名称和断言胜过注释。
- **每个测试只测一件事** — 一个测试函数验证一个行为。
- **FIRST 原则** — Fast, Independent, Repeatable, Self-validating, Timely。

掌握 pytest 的核心用法（fixture、parametrize、mock），你就能写出可靠、可维护的测试套件。测试不是负担，而是你修改代码时的安全网。
