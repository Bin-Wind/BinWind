# requests 库：HTTP 请求实战

`requests` 是 Python 最流行的 HTTP 客户端库，"HTTP for Humans" 的设计哲学让它比标准库 `urllib` 易用得多。本文覆盖日常开发中最常见的 HTTP 请求模式。

## 安装与基本用法

```bash
pip install requests
```

```python
import requests

# GET 请求
resp = requests.get("https://httpbin.org/get")
print(resp.status_code)      # 200
print(resp.headers["Content-Type"])  # application/json
print(resp.json())           # 将 JSON 响应转为 dict
print(resp.text)             # 响应体（字符串）
print(resp.content)          # 响应体（bytes）
```

## 查询参数与请求头

```python
# URL 查询参数
params = {"q": "python", "page": 2, "size": 20}
resp = requests.get("https://httpbin.org/get", params=params)
print(resp.url)  # https://httpbin.org/get?q=python&page=2&size=20

# 自定义请求头
headers = {
    "User-Agent": "MyApp/1.0",
    "Authorization": "Bearer token_abc123",
    "Accept": "application/json",
}
resp = requests.get("https://httpbin.org/headers", headers=headers)
```

## POST 请求与请求体

```python
# JSON 格式请求体（最常用）
data = {"name": "Alice", "email": "alice@example.com"}
resp = requests.post("https://httpbin.org/post", json=data)

# 表单格式请求体
form_data = {"username": "alice", "password": "secret"}
resp = requests.post("https://httpbin.org/post", data=form_data)

# 直接发送 JSON 字符串
import json
resp = requests.post("https://httpbin.org/post", data=json.dumps(data),
                     headers={"Content-Type": "application/json"})
```

## Session：连接复用与状态保持

Session 能在多个请求之间保持 cookie 和 TCP 连接，适合需要登录态的场景：

```python
# 创建持久会话
session = requests.Session()

# 设置会话级默认值
session.headers.update({"User-Agent": "MyApp/1.0"})
session.auth = ("username", "password")  # Basic Auth

# 登录并保持 cookie
login_resp = session.post("https://httpbin.org/post",
                          json={"user": "alice", "pass": "secret"})
# 后续请求自动携带 cookie
profile_resp = session.get("https://httpbin.org/cookies")

session.close()  # 释放连接
```

也可以用上下文管理器：

```python
with requests.Session() as session:
    session.headers.update({"User-Agent": "MyApp/1.0"})
    resp = session.get("https://httpbin.org/get")
```

## Timeout 与错误处理

**永远不要发送不带 timeout 的 request**。没有 timeout 的请求可能永远阻塞：

```python
from requests.exceptions import (
    Timeout, ConnectionError, HTTPError, RequestException
)

try:
    # timeout=(连接超时, 读取超时)
    resp = requests.get("https://httpbin.org/delay/5",
                        timeout=(3.0, 10.0))
    resp.raise_for_status()  # 4xx/5xx 时抛出 HTTPError
except Timeout:
    print("请求超时")
except ConnectionError:
    print("无法连接到服务器")
except HTTPError as e:
    print(f"HTTP 错误: {e.response.status_code}")
except RequestException as e:
    print(f"请求异常: {e}")
```

## 文件上传与下载

```python
# 上传文件
with open("report.pdf", "rb") as f:
    files = {"file": ("report.pdf", f, "application/pdf")}
    resp = requests.post("https://httpbin.org/post", files=files)

# 上传多个文件
files = {
    "image": ("photo.jpg", open("photo.jpg", "rb"), "image/jpeg"),
    "data": ("info.json", open("info.json", "rb"), "application/json"),
}
resp = requests.post("https://httpbin.org/post", files=files)

# 流式下载大文件
def download_file(url: str, dest: str):
    with requests.get(url, stream=True, timeout=30) as resp:
        resp.raise_for_status()
        with open(dest, "wb") as f:
            for chunk in resp.iter_content(chunk_size=8192):
                f.write(chunk)

download_file("https://example.com/large_file.zip", "large_file.zip")
```

## 重试机制

配合 `urllib3` 的 Retry 或自定义装饰器实现重试：

```python
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

session = requests.Session()

retry_strategy = Retry(
    total=3,                    # 最多重试 3 次
    backoff_factor=1,           # 重试间隔: 1s, 2s, 4s
    status_forcelist=[429, 500, 502, 503, 504],
    allowed_methods=["GET", "POST"],
)
adapter = HTTPAdapter(max_retries=retry_strategy)
session.mount("https://", adapter)
session.mount("http://", adapter)

resp = session.get("https://httpbin.org/status/500")  # 会自动重试 3 次
```

## 实用技巧

```python
# 获取重定向历史
resp = requests.get("https://httpbin.org/redirect/3")
print([r.status_code for r in resp.history])  # [302, 302, 302]

# 禁止重定向
resp = requests.get("https://httpbin.org/redirect/1", allow_redirects=False)
print(resp.status_code)  # 302

# 代理
proxies = {"http": "http://127.0.0.1:7890", "https": "http://127.0.0.1:7890"}
resp = requests.get("https://httpbin.org/get", proxies=proxies)

# SSL 证书验证
resp = requests.get("https://example.com", verify="/path/to/cert.pem")
# 禁止验证（仅调试用，生产环境禁用）
resp = requests.get("https://example.com", verify=False)
```

## 最佳实践总结

1. **始终设置 timeout** — 防止请求无限等待。
2. **使用 Session 复用连接** — 多次请求同一主机时显著提升性能。
3. **调用 `raise_for_status()`** — 不要假设所有 200 以外状态都是可忽略的。
4. **流式下载大文件** — 避免将整个文件加载到内存。
5. **实现重试逻辑** — 网络不总是可靠的，临时故障应该自动重试。
6. **管理好连接** — 使用 `with` 语句或 `Session.close()` 释放资源。

`requests` 的 API 设计简洁直觉，掌握以上模式后，你就能应对绝大多数 HTTP 交互场景。对于需要高并发的场景（成千上万请求），可以考虑 `aiohttp` 或 `httpx`（支持 async）。
