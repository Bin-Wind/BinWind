# 网页爬虫入门

网页爬虫（Web Crawler / Web Scraper）是从网页中自动提取结构化数据的技术。本文从零开始，用 `requests` + `BeautifulSoup` 构建一个实用的爬虫。

## 环境准备

```bash
pip install requests beautifulsoup4 lxml
```

- `requests`：发送 HTTP 请求，获取网页 HTML。
- `beautifulsoup4`：解析 HTML，提取数据。
- `lxml`：高性能 HTML 解析器（BS4 的底层引擎推荐选择）。

## 第一个爬虫：获取网页标题

```python
import requests
from bs4 import BeautifulSoup

url = "https://books.toscrape.com/"
resp = requests.get(url, timeout=10)
resp.raise_for_status()

soup = BeautifulSoup(resp.text, "lxml")

# 获取网页标题
print(soup.title.string)  # All products | Books to Scrape - Sandbox

# 获取所有图书标题
for book in soup.select("article.product_pod h3 a"):
    print(book.get("title"))
```

## BeautifulSoup 选择器完全指南

BS4 提供多种定位元素的方式，按推荐度排序：

```python
html = """
<html>
  <body>
    <div class="product">
      <h2 class="title">Laptop</h2>
      <span class="price">$999</span>
      <a href="/detail/1">Detail</a>
    </div>
    <div class="product">
      <h2 class="title">Mouse</h2>
      <span class="price">$29</span>
      <a href="/detail/2">Detail</a>
    </div>
  </body>
</html>
"""

soup = BeautifulSoup(html, "lxml")

# --- CSS 选择器（最推荐） ---
titles = soup.select(".product .title")
print([t.text for t in titles])  # ['Laptop', 'Mouse']

first_price = soup.select_one(".product .price")
print(first_price.text)  # $999

# 属性选择
links = soup.select("a[href^='/detail']")  # href 以 /detail 开头
for link in links:
    print(link["href"])  # /detail/1, /detail/2

# --- find / find_all ---
divs = soup.find_all("div", class_="product")
for div in divs:
    title = div.find("h2", class_="title").text
    price = div.find("span", class_="price").text
    print(f"{title}: {price}")

# --- 导航遍历 ---
div = soup.find("div", class_="product")
print(div.h2.text)                     # 直接访问子标签
print(div.find_next_sibling("div"))    # 下一个兄弟节点
```

## 实战：爬取图书信息（含分页）

```python
import requests
from bs4 import BeautifulSoup
import time
from typing import NamedTuple

class Book(NamedTuple):
    title: str
    price: float
    availability: str

BASE_URL = "https://books.toscrape.com/catalogue/"
HEADERS = {"User-Agent": "Mozilla/5.0 (compatible; LearningBot/1.0)"}

def scrape_page(url: str) -> list[Book]:
    """爬取单个分页的所有图书"""
    print(f"Scraping: {url}")
    resp = requests.get(url, headers=HEADERS, timeout=15)
    resp.raise_for_status()
    soup = BeautifulSoup(resp.text, "lxml")

    books = []
    for article in soup.select("article.product_pod"):
        title = article.select_one("h3 a").get("title")
        price_text = article.select_one("p.price_color").text
        price = float(price_text.replace("£", "").replace("Â", ""))
        availability = article.select_one("p.instock.availability").text.strip()
        books.append(Book(title=title, price=price, availability=availability))
    return books

def scrape_all_pages() -> list[Book]:
    """爬取所有分页"""
    all_books = []
    page_num = 1

    while True:
        if page_num == 1:
            page_url = f"{BASE_URL}page-1.html"
        else:
            page_url = f"{BASE_URL}page-{page_num}.html"

        resp = requests.get(page_url, headers=HEADERS, timeout=15)
        if resp.status_code == 404:
            break  # 没有更多页面

        all_books.extend(scrape_page(page_url))
        page_num += 1
        time.sleep(1)  # 礼貌性延迟，避免给服务器压力

    return all_books

if __name__ == "__main__":
    books = scrape_all_pages()
    print(f"\n爬取完成，共 {len(books)} 本书")
    for book in books[:5]:
        print(f"  {book.title} - £{book.price} ({book.availability})")
```

## robots.txt 与爬虫伦理

每个网站通常有 `robots.txt` 声明哪些路径允许爬取。**始终尊重它**：

```python
import requests

def check_robots(base_url: str) -> bool:
    """检查 robots.txt 是否允许爬取指定路径"""
    resp = requests.get(f"{base_url}/robots.txt", timeout=10)
    print(resp.text)

check_robots("https://books.toscrape.com")
```

案例：GitHub 的 `robots.txt` 禁止爬取大量路径，违反可能导致 IP 被封。

## 爬虫伦理准则

1. **遵守 robots.txt** — 不爬取被禁止的路径。
2. **控制请求频率** — 添加 `time.sleep()` 延迟（通常 1~3 秒），避免对目标服务器造成压力。
3. **设置 User-Agent** — 标识自己的身份，提供联系方式，方便网站管理员联系。
4. **尊重版权与条款** — 不爬取受版权保护或 TOS 明确禁止的内容。
5. **使用公开 API 优先** — 如果网站提供 API，优先使用 API 而不是爬虫。
6. **不绕过反爬机制** — 不过度使用代理池、验证码破解等技术。
7. **合理使用数据** — 不将爬取的数据用于商业用途（除非获得授权）。

## 常见反爬策略对抗

```python
# 1. 设置合理的 User-Agent
HEADERS = {
    "User-Agent": "Mozilla/5.0 (compatible; MyBot/1.0; +https://my.site/bot)"
}

# 2. 随机延迟
import random
time.sleep(random.uniform(1.0, 3.0))

# 3. 使用 Session 保持 cookie
session = requests.Session()
session.headers.update(HEADERS)
resp = session.get(url, timeout=15)

# 4. 处理编码问题
resp.encoding = resp.apparent_encoding  # 自动检测编码

# 5. 重试机制（网络波动处理）
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

retry = Retry(total=3, backoff_factor=1)
session.mount("https://", HTTPAdapter(max_retries=retry))
```

## BeautifulSoup 常用操作速查

| 方法 | 用途 | 示例 |
|------|------|------|
| `select()` | CSS 选择器查找所有 | `soup.select(".price")` |
| `select_one()` | CSS 选择器查找一个 | `soup.select_one("#main")` |
| `find()` | 查找第一个匹配 | `soup.find("div", class_="item")` |
| `find_all()` | 查找所有匹配 | `soup.find_all("a", href=True)` |
| `get("attr")` | 获取属性值 | `tag.get("href")` |
| `text` / `string` | 获取文本内容 | `tag.text.strip()` |

网页爬虫是获取公开数据的有力工具，但务必以负责任的态度使用。对简单静态页面，`requests + BeautifulSoup` 是最简洁高效的组合。如果目标网站大量使用 JavaScript 动态渲染，下一章的 Selenium 将是不二之选。
