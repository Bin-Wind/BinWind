# 模块与包管理

当代码量增长时，组织代码成为关键。Python 的模块和包系统让你可以拆分代码、复用逻辑、管理依赖，形成清晰的项目结构。

## import 导入模块

```python
# 基本导入
import math
print(math.sqrt(16))                           # 4.0
print(math.pi)                                 # 3.14159...

# 从模块导入特定内容
from math import sqrt, pi
print(sqrt(25))                                # 5.0 - 无需 math. 前缀

# 导入时起别名
import numpy as np
from collections import defaultdict as dd

# 导入整个模块内容（不推荐，会污染命名空间）
# from math import *
```

Python 导入时会按以下顺序搜索模块：
1. 当前目录
2. `PYTHONPATH` 环境变量中的路径
3. 标准库目录
4. site-packages（第三方包安装位置）

## 创建自己的模块

创建文件 `calculator.py`：

```python
# calculator.py
PI = 3.14159

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def _internal_helper():       # 以下划线开头 -> 约定为"私有"
    return "内部函数"

print("calculator 模块被加载了")   # import 时会执行一次
```

在另一个文件中使用：

```python
import calculator

print(calculator.add(10, 5))            # 15
print(calculator.PI)                    # 3.14159
# calculator._internal_helper()         # 可调用但不建议
```

## if __name__ == "__main__" 模式

```python
# calculator.py（改进版）
def main():
    """模块的入口点，只在直接运行时执行"""
    print("计算器运行中...")
    result = add(10, 20)
    print(f"10 + 20 = {result}")

if __name__ == "__main__":
    main()
```

- 直接运行 `python calculator.py` → `__name__` 为 `"__main__"`，执行 `main()`
- 被 import 时 → `__name__` 为 `"calculator"`，不执行 `main()`

这个模式让同一个文件既能作为脚本运行，又能作为模块被导入——是 Python 工程化的必备技巧。

## 包 Package 结构

包是包含 `__init__.py` 文件的目录（Python 3.3+ 可省略，但建议保留）。

```
mypackage/
├── __init__.py          # 包的初始化文件
├── core.py              # mypackage.core
├── utils.py             # mypackage.utils
└── subpackage/          # 子包
    ├── __init__.py
    └── extra.py         # mypackage.subpackage.extra
```

`__init__.py` 示例：

```python
# mypackage/__init__.py
from .core import main_function     # 从子模块导入到包命名空间
from .utils import helper

__all__ = ["main_function", "helper"]   # 控制 from mypackage import * 的行为
__version__ = "1.0.0"
```

使用时：

```python
import mypackage
mypackage.main_function()

# 或
from mypackage import main_function
main_function()

# 子包
from mypackage.subpackage import extra
```

## 相对导入

在包内部，使用相对导入引用兄弟模块：

```python
# mypackage/core.py
from .utils import helper          # . 表示当前包
from .subpackage.extra import load # 进入子包
from .. import another_package     # .. 表示上级包（不常用）
```

相对导入只能在包内部使用，脚本直接运行时不能用。

## 虚拟环境回顾与 pip

```bash
# 创建并激活虚拟环境
python -m venv env
source env/bin/activate

# 安装第三方包
pip install requests pandas flask

# 查看已安装包
pip list
pip freeze > requirements.txt

# 从文件恢复
pip install -r requirements.txt
```

## Poetry 简介 - 现代依赖管理

Poetry 统一了依赖管理和打包，比 pip + requirements.txt 更强大。

```bash
# 安装 Poetry
curl -sSL https://install.python-poetry.org | python3 -

# 创建新项目
poetry new myproject
cd myproject

# 添加依赖
poetry add requests flask
poetry add --group dev pytest black   # 开发依赖

# 安装所有依赖
poetry install

# 运行脚本
poetry run python main.py

# 进入虚拟环境
poetry shell
```

pyproject.toml 示例：

```toml
[tool.poetry]
name = "myproject"
version = "0.1.0"
description = "我的项目"

[tool.poetry.dependencies]
python = "^3.10"
requests = "^2.31"
flask = "^3.0"

[tool.poetry.group.dev.dependencies]
pytest = "^8.0"
black = "^24.0"
```

## 项目结构最佳实践

```
myproject/
├── src/
│   └── myproject/          # 包根目录
│       ├── __init__.py
│       ├── core.py
│       └── utils.py
├── tests/
│   ├── __init__.py
│   ├── test_core.py
│   └── test_utils.py
├── scripts/
│   └── run.py
├── pyproject.toml
├── README.md
└── .gitignore
```

> 模块化不只是代码拆分——它是复用、隔离和测试的基础。小项目用 `requirements.txt` 足够，中大型项目推荐 Poetry 或 PDM 管理依赖。
