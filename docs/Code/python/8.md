# 异常处理与调试

程序运行中难免遇到错误。Python 的异常机制让你优雅地处理意外情况，而 logging 和 pdb 则是定位问题的利器。

## try / except / else / finally

```python
# 基本异常处理
try:
    num = int(input("请输入数字："))
    result = 100 / num
    print(f"100 / {num} = {result}")
except ValueError:
    print("输入的不是有效数字")
except ZeroDivisionError:
    print("除数不能为0")

# 捕获多个异常
try:
    data = {"name": "Alice"}
    print(data["age"])
except (KeyError, TypeError) as e:
    print(f"出了点问题：{e}")

# else：没有异常时执行
try:
    config = open("config.json", "r")
except FileNotFoundError:
    print("配置文件不存在，使用默认配置")
    config = {}
else:
    print("配置文件加载成功")

# finally：无论是否异常都执行（常用于资源清理）
f = None
try:
    f = open("data.txt", "r")
    content = f.read()
except FileNotFoundError:
    print("文件不存在")
finally:
    if f:
        f.close()       # 确保文件被关闭
    print("清理完成")
```

with 语句实际上就是 `try/finally` 的语法糖，因此不需要手动 close。

## 异常层级结构

Python 异常都继承自 `BaseException`，常见层级：

```
BaseException
├── KeyboardInterrupt     # 用户 Ctrl+C
├── Exception             # 常规异常的基类
│   ├── ValueError
│   ├── TypeError
│   ├── KeyError
│   ├── IndexError
│   ├── FileNotFoundError
│   ├── ZeroDivisionError
│   ├── AttributeError
│   └── ImportError
└── SystemExit            # sys.exit() 触发
```

捕获 `Exception` 可以兜底大部分错误，但不要捕获 `BaseException`（会拦截 `KeyboardInterrupt` 等系统信号）。

## 抛出异常 raise

```python
def validate_age(age):
    if not isinstance(age, int):
        raise TypeError(f"age 必须是 int 类型，但收到了 {type(age).__name__}")
    if age < 0 or age > 150:
        raise ValueError(f"年龄必须介于 0-150 之间，但收到了 {age}")
    return age

# validate_age("abc")    # 抛出 TypeError
# validate_age(200)      # 抛出 ValueError

# raise from：异常链，保留原始上下文
def load_data(filename):
    try:
        import json
        with open(filename) as f:
            return json.load(f)
    except FileNotFoundError as e:
        raise ValueError(f"无法读取数据文件：{filename}") from e
```

## 自定义异常

```python
class ValidationError(Exception):
    """自定义验证异常"""

class InsufficientBalanceError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"余额不足：{balance} < {amount}")

class BankAccount:
    def __init__(self, balance):
        self.balance = balance

    def withdraw(self, amount):
        if amount <= 0:
            raise ValidationError("取款金额必须为正数")
        if amount > self.balance:
            raise InsufficientBalanceError(self.balance, amount)
        self.balance -= amount
        return self.balance
```

## logging 日志模块

```python
import logging

# 基本配置
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S",
    handlers=[
        logging.FileHandler("app.log", encoding="utf-8"),
        logging.StreamHandler()          # 同时输出到控制台
    ]
)

logger = logging.getLogger(__name__)

logger.debug("调试信息")
logger.info("程序启动")
logger.warning("磁盘空间不足")
logger.error("连接数据库失败")
logger.critical("系统崩溃！")

# 典型用法
def divide(a, b):
    logger.debug(f"divide({a}, {b}) 被调用")
    try:
        result = a / b
    except ZeroDivisionError:
        logger.error(f"除零错误：a={a}, b={b}")
        raise
    else:
        logger.info(f"计算结果：{result}")
        return result
```

日志级别：`DEBUG < INFO < WARNING < ERROR < CRITICAL`。生产环境建议设为 `WARNING` 或 `INFO`。

## 使用 pdb 调试

```python
# 方法一：程序内插入断点
def complex_function(x, y):
    z = x * 2
    import pdb; pdb.set_trace()      # 程序在此暂停
    result = z + y * 3
    return result

# 方法二：Python 3.7+ 内置 breakpoint()
def another_function(data):
    if not data:
        breakpoint()                  # 更简洁的写法
    return sum(data)
```

pdb 常用命令：

| 命令 | 缩写 | 说明 |
|------|------|------|
| `list` | `l` | 显示当前代码位置 |
| `next` | `n` | 执行下一行（不进入函数内部） |
| `step` | `s` | 单步执行（进入函数内部） |
| `continue` | `c` | 继续执行直到下一个断点 |
| `print(x)` | `p x` | 打印变量值 |
| `where` | `w` | 显示调用栈 |
| `quit` | `q` | 退出调试 |

## 调试黄金法则

```python
# 1. 使用 assert 快速验证假设
def calculate_discount(price, rate):
    assert 0 <= rate <= 1, f"折扣率必须在 0-1 之间，但收到了 {rate}"
    assert price >= 0, "价格不能为负"
    return price * (1 - rate)

# 2. 打印 vs 日志 - 后者更专业
# 使用 logging 而不是 print() 加 TODO 注释

# 3. 使用 traceback 获得完整异常信息
import traceback
try:
    1 / 0
except Exception:
    error_detail = traceback.format_exc()
    print(error_detail)
```

> 异常处理不是"错误检查"，而是正常控制流的一部分（Look Before You Leap 风格可向 Easier to Ask Forgiveness than Permission 风格转变）。先 `try` 再 `except`，比先检查条件再做事，通常是更 Pythonic 的选择。
