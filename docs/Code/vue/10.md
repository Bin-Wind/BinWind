# Composables 组合式函数

Composables（组合式函数）是 Vue 3 中复用**有状态逻辑**的核心模式。它利用 Composition API 将组件逻辑抽取为独立的可复用函数，类似于 React Hooks 但更加灵活。

## 什么是 Composable

一个 Composable 就是一个以 `use` 开头命名的函数，内部使用响应式 API 封装业务逻辑：

```javascript
// composables/useCounter.js
import { ref, computed } from 'vue'

export function useCounter(initialValue = 0) {
  const count = ref(initialValue)

  const doubled = computed(() => count.value * 2)

  function increment() {
    count.value++
  }

  function decrement() {
    count.value--
  }

  function reset() {
    count.value = initialValue
  }

  return { count, doubled, increment, decrement, reset }
}
```

在组件中使用：

```vue
<script setup>
import { useCounter } from '@/composables/useCounter'

const { count, doubled, increment, decrement, reset } = useCounter(10)
</script>

<template>
  <p>{{ count }} (x2 = {{ doubled }})</p>
  <button @click="increment">+1</button>
  <button @click="decrement">-1</button>
  <button @click="reset">重置</button>
</template>
```

## 示例一：useMouse — 追踪鼠标位置

```javascript
// composables/useMouse.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useMouse() {
  const x = ref(0)
  const y = ref(0)

  function onMouseMove(event) {
    x.value = event.clientX
    y.value = event.clientY
  }

  onMounted(() => {
    window.addEventListener('mousemove', onMouseMove)
  })

  onUnmounted(() => {
    window.removeEventListener('mousemove', onMouseMove)
  })

  return { x, y }
}
```

## 示例二：useFetch — 通用数据请求

```javascript
// composables/useFetch.js
import { ref, watchEffect, toValue } from 'vue'

export function useFetch(url) {
  const data = ref(null)
  const error = ref(null)
  const loading = ref(false)

  async function execute() {
    loading.value = true
    error.value = null
    try {
      const resolvedUrl = toValue(url)
      const response = await fetch(resolvedUrl)
      if (!response.ok) throw new Error(`HTTP ${response.status}`)
      data.value = await response.json()
    } catch (err) {
      error.value = err.message
    } finally {
      loading.value = false
    }
  }

  // url 可能是 ref 或 computed，变化时自动重新请求
  watchEffect(() => {
    toValue(url)  // 追踪依赖
    execute()
  })

  return { data, error, loading, execute }
}
```

使用方式：

```vue
<script setup>
import { ref } from 'vue'
import { useFetch } from '@/composables/useFetch'

const userId = ref(1)
const url = computed(() => `/api/user/${userId.value}`)
const { data: user, loading, error } = useFetch(url)
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else-if="error">错误：{{ error }}</div>
  <div v-else>
    <h2>{{ user?.name }}</h2>
    <p>{{ user?.email }}</p>
  </div>
  <button @click="userId++">下一个用户</button>
</template>
```

## 示例三：useLocalStorage — 响应式本地存储

```javascript
// composables/useLocalStorage.js
import { ref, watch } from 'vue'

export function useLocalStorage(key, defaultValue) {
  const storedValue = localStorage.getItem(key)
  const data = ref(storedValue ? JSON.parse(storedValue) : defaultValue)

  watch(data, (newVal) => {
    if (newVal === null || newVal === undefined) {
      localStorage.removeItem(key)
    } else {
      localStorage.setItem(key, JSON.stringify(newVal))
    }
  }, { deep: true })

  function remove() {
    data.value = null
    localStorage.removeItem(key)
  }

  return { data, remove }
}
```

```vue
<script setup>
import { useLocalStorage } from '@/composables/useLocalStorage'

const { data: theme, remove: clearTheme } = useLocalStorage('app-theme', 'light')

function toggleTheme() {
  theme.value = theme.value === 'light' ? 'dark' : 'light'
}
</script>

<template>
  <p>当前主题：{{ theme }}</p>
  <button @click="toggleTheme">切换</button>
  <button @click="clearTheme">清除设置</button>
</template>
```

## 示例四：useDebounce — 防抖

```javascript
// composables/useDebounce.js
import { ref, watch } from 'vue'

export function useDebounce(value, delay = 300) {
  const debouncedValue = ref(value.value)

  let timer = null
  watch(value, (newVal) => {
    clearTimeout(timer)
    timer = setTimeout(() => {
      debouncedValue.value = newVal
    }, delay)
  })

  return { debouncedValue }
}
```

结合搜索输入框使用：

```vue
<script setup>
import { ref } from 'vue'
import { useDebounce } from '@/composables/useDebounce'

const keyword = ref('')
const { debouncedValue } = useDebounce(keyword, 500)

// 当 debouncedValue 变化时才发送请求
watch(debouncedValue, (val) => {
  if (val) fetchSearchResults(val)
})
</script>

<template>
  <input v-model="keyword" placeholder="搜索..." />
  <p>实际搜索词：{{ debouncedValue }}</p>
</template>
```

## 示例五：useEventListener — 事件监听器封装

```javascript
// composables/useEventListener.js
import { onMounted, onUnmounted } from 'vue'

export function useEventListener(target, event, handler, options) {
  onMounted(() => {
    target.addEventListener(event, handler, options)
  })

  onUnmounted(() => {
    target.removeEventListener(event, handler, options)
  })
}
```

## Composables 最佳实践

### 1. 命名规范

- 函数名以 `use` 开头（约定俗成，IDE 也能识别）
- 文件名与函数名保持一致：`useMouse.js`
- 返回对象使用具名属性，方便组件按需解构

### 2. 输入参数使用 toValue / toRef

让 Composable 接收 `ref`、`computed` 或普通值：

```javascript
import { toValue } from 'vue'

export function useTitle(newTitle) {
  const title = toValue(newTitle)  // 统一处理
  document.title = title
}
```

### 3. 返回值使用 toRefs 保持响应式

```javascript
import { reactive, toRefs } from 'vue'

export function useFeature() {
  const state = reactive({ x: 0, y: 0, isActive: false })
  // ...
  return { ...toRefs(state) }  // 调用方可自由解构
}
```

### 4. 副作用清理

```javascript
export function useTimer(callback, interval) {
  let timer = null

  onMounted(() => {
    timer = setInterval(callback, interval)
  })

  onUnmounted(() => {
    clearInterval(timer)  // 确保清理
  })
}
```

### 5. 单一职责

一个 Composable 只做一件事。如果 `useSearch` 同时处理搜索、分页、筛选，应该拆分为 `useSearch`、`usePagination`、`useFilter`。

## 何时提取为 Composable

| 信号 | 说明 |
|------|------|
| 多个组件共享相同的响应式逻辑 | 提取为 Composable |
| 组件 `<script setup>` 超过 100 行 | 按功能拆分 Composable |
| 逻辑包含生命周期钩子 + 响应式状态 | 适合 Composable 模式 |
| 纯工具函数（无响应式） | 提取为普通 utils，不需要 Composable |

## 对比：Pinia vs Composables

| 维度 | Composables | Pinia |
|------|-------------|-------|
| 状态共享范围 | 调用方组件树内共享 | 全局单例 |
| 适用场景 | 组件级逻辑复用 | 跨组件/跨页面共享状态 |
| 实例数量 | 每个调用创建新实例 | 单例（一个 store 一个实例） |
| DevTools | 需要自己处理 | 原生支持 |

**规则**：如果状态需要在多个不相关的组件间共享且需要全局一致，用 Pinia。如果是特定功能的逻辑封装（如鼠标追踪），用 Composable。

## 小结

- Composables 是复用**有状态逻辑**的核心模式
- 以 `use` 开头命名，返回响应式状态和方法
- 利用 `onMounted` / `onUnmounted` 管理副作用生命周期
- 参数使用 `toValue` 兼容 ref/computed/普通值
- 返回值使用 `toRefs` 让调用方能解构

Composables 让 Vue 代码真正实现"高内聚、低耦合"，是通往高阶 Vue 开发的必修课。下一节深入插槽（Slots）的各类用法。
