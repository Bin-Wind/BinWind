# 插槽（Slots）详解

插槽是 Vue 组件的内容分发机制，允许父组件向子组件注入模板内容，实现高度灵活的可复用组件。Vue 提供了三种插槽：默认插槽、具名插槽和作用域插槽。

## 默认插槽

最简单的插槽形式，子组件用 `<slot />` 占位，父组件传入的内容会渲染到该位置：

```vue
<!-- AlertBox.vue -->
<template>
  <div class="alert" :class="type">
    <slot>默认提示信息</slot>
  </div>
</template>

<script setup>
defineProps({ type: { type: String, default: 'info' } })
</script>
```

父组件使用：

```vue
<AlertBox type="success">
  <strong>操作成功！</strong> 数据已保存。
</AlertBox>

<AlertBox type="warning" />
<!-- 未传内容，渲染默认的"默认提示信息" -->
```

## 具名插槽

子组件定义多个 `<slot name="">`，父组件用 `<template #name>` 指定内容分发到哪个插槽：

```vue
<!-- Layout.vue -->
<template>
  <div class="layout">
    <header>
      <slot name="header">
        <h1>默认标题</h1>
      </slot>
    </header>

    <aside>
      <slot name="sidebar" />
    </aside>

    <main>
      <slot />
      <!-- 没有 name 即为默认插槽，等价于 name="default" -->
    </main>

    <footer>
      <slot name="footer">
        <p>&copy; 2026 My App</p>
      </slot>
    </footer>
  </div>
</template>
```

父组件使用：

```vue
<Layout>
  <template #header>
    <h1>用户管理中心</h1>
  </template>

  <template #sidebar>
    <nav>
      <a href="/profile">个人信息</a>
      <a href="/orders">我的订单</a>
    </nav>
  </template>

  <!-- 未用 <template> 包裹的内容自动进入默认插槽 -->
  <p>欢迎回来，这是你的控制台。</p>

  <template #footer>
    <p>联系客服：service@example.com</p>
  </template>
</Layout>
```

`#header` 是 `v-slot:header` 的简写。`#default` 可省略直接用裸内容。

### 动态插槽名

```vue
<template #[dynamicSlotName]>
  动态分配内容
</template>
```

## 作用域插槽（Scoped Slots）

当父组件需要**访问子组件内部数据**来渲染插槽内容时，使用作用域插槽。子组件在 `<slot>` 上绑定属性，父组件接收这些属性。

```vue
<!-- TodoList.vue -->
<script setup>
import { ref } from 'vue'

const todos = ref([
  { id: 1, text: '学习 Vue', done: false, priority: 'high' },
  { id: 2, text: '写项目', done: true, priority: 'low' }
])
</script>

<template>
  <ul>
    <!-- 将 item, index 暴露给父组件 -->
    <li v-for="(item, index) in todos" :key="item.id">
      <slot name="item" :item="item" :index="index">
        <!-- 默认渲染 -->
        {{ item.text }}
      </slot>
    </li>
  </ul>
</template>
```

父组件通过 `v-slot` 或 `#slotName="slotProps"` 接收数据：

```vue
<TodoList>
  <template #item="{ item, index }">
    <!-- 解构获取数据，自由决定如何渲染 -->
    <span :class="{ done: item.done }">
      {{ index + 1 }}. {{ item.text }}
    </span>
    <span class="badge">{{ item.priority }}</span>
  </template>
</TodoList>
```

### 默认插槽的作用域写法

当作用域数据在默认插槽上时：

```vue
<!-- 子组件 -->
<slot :user="user" :role="role" />

<!-- 父组件 — 直接写在组件标签上 -->
<ChildComponent v-slot="{ user, role }">
  <p>{{ user.name }} - {{ role }}</p>
</ChildComponent>

<!-- 或者用 template（具名 + 默认并用时推荐） -->
<ChildComponent>
  <template #default="{ user, role }">
    <p>{{ user.name }}</p>
  </template>
</ChildComponent>
```

## Renderless Components（无渲染组件）

利用作用域插槽，可以创建不渲染任何自身 DOM、只提供数据和逻辑的组件：

```vue
<!-- FetchData.vue — 无渲染数据获取组件 -->
<script setup>
import { ref, watch } from 'vue'

const props = defineProps({ url: String })
const data = ref(null)
const loading = ref(false)
const error = ref(null)

async function fetchData() {
  loading.value = true
  error.value = null
  try {
    const res = await fetch(props.url)
    if (!res.ok) throw new Error(`HTTP ${res.status}`)
    data.value = await res.json()
  } catch (e) {
    error.value = e.message
  } finally {
    loading.value = false
  }
}

watch(() => props.url, fetchData, { immediate: true })
</script>

<template>
  <slot :data="data" :loading="loading" :error="error" :retry="fetchData" />
</template>
```

使用 Renderless Component：

```vue
<FetchData url="/api/users" v-slot="{ data, loading, error, retry }">
  <div v-if="loading">加载中...</div>
  <div v-else-if="error">
    <p>出错了：{{ error }}</p>
    <button @click="retry">重试</button>
  </div>
  <ul v-else>
    <li v-for="user in data" :key="user.id">{{ user.name }}</li>
  </ul>
</FetchData>
```

**优势**：完全分离数据获取与 UI 渲染逻辑，UI 端完全控制展示方式。

## 综合示例：可复用的 Table 组件

```vue
<!-- DataTable.vue -->
<script setup>
defineProps({
  data: { type: Array, required: true },
  loading: { type: Boolean, default: false }
})
</script>

<template>
  <table class="data-table">
    <thead>
      <tr>
        <slot name="header" :columns="data[0] ? Object.keys(data[0]) : []">
          <th v-for="col in data[0] ? Object.keys(data[0]) : []" :key="col">
            {{ col }}
          </th>
        </slot>
      </tr>
    </thead>
    <tbody>
      <tr v-if="loading">
        <td>
          <slot name="loading">加载中...</slot>
        </td>
      </tr>
      <tr v-else-if="!data.length">
        <td>
          <slot name="empty">暂无数据</slot>
        </td>
      </tr>
      <template v-else>
        <tr v-for="(row, rowIndex) in data" :key="row.id ?? rowIndex">
          <slot name="row" :row="row" :index="rowIndex">
            <td v-for="value in Object.values(row)" :key="value">
              {{ value }}
            </td>
          </slot>
        </tr>
      </template>
    </tbody>
  </table>
</template>
```

父组件自定义渲染：

```vue
<DataTable :data="users" :loading="loading">
  <template #header>
    <th>姓名</th>
    <th>邮箱</th>
    <th>状态</th>
    <th>操作</th>
  </template>

  <template #row="{ row }">
    <td>{{ row.name }}</td>
    <td>{{ row.email }}</td>
    <td>
      <span :class="{ active: row.active }">
        {{ row.active ? '激活' : '禁用' }}
      </span>
    </td>
    <td>
      <button @click="editUser(row.id)">编辑</button>
    </td>
  </template>

  <template #empty>
    <p>还没有用户数据，<a href="/invite">去邀请</a></p>
  </template>

  <template #loading>
    <div class="skeleton">数据加载中...</div>
  </template>
</DataTable>
```

## 插槽 vs Props 传递内容的对比

| 场景 | 推荐 | 原因 |
|------|------|------|
| 传入纯数据（字符串、数字） | Props | 简单直接 |
| 传入 HTML 结构 / 组件 | Slots | Props 传 HTML 字符串不安全且难维护 |
| 需要访问子组件数据来渲染 | 作用域插槽 | 唯一选择 |
| 完全自定义子组件内部结构 | 作用域插槽 | 组件库 / UI 框架常用模式 |

## 小结

| 插槽类型 | 语法 | 用途 |
|----------|------|------|
| 默认插槽 | `<slot />` | 单一内容分发 |
| 具名插槽 | `<slot name="x">` + `<template #x>` | 多区域内容分发 |
| 作用域插槽 | `<slot :data="x">` + `<template #default="props">` | 父组件访问子组件数据来渲染 |
| Renderless | 纯逻辑组件 + 作用域插槽 | 逻辑与 UI 彻底分离 |

插槽是 Vue 组件设计的精髓之一，几乎所有高质量的组件库（Element Plus、Vuetify、Naive UI）都大量使用作用域插槽来提供灵活的定制能力。下一节学习动态组件与 Teleport。
