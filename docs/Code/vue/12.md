# 组件注册与动态组件

Vue 组件需要注册后才能使用。除了常规的静态组件，Vue 还提供了 `<component :is="">` 动态切换组件、`<KeepAlive>` 缓存状态、`<Teleport>` 传送 DOM，以及 `<Suspense>` 处理异步依赖。

## 全局注册 vs 局部注册

### 全局注册

在 `main.js` 中使用 `app.component()` 注册，之后可在任何组件中直接使用，无需导入：

```javascript
// main.js
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

// 全局注册
import BaseButton from './components/BaseButton.vue'
import BaseInput from './components/BaseInput.vue'
import BaseModal from './components/BaseModal.vue'

app.component('BaseButton', BaseButton)
app.component('BaseInput', BaseInput)
app.component('BaseModal', BaseModal)

app.mount('#app')
```

**全局注册适用场景**：基础 UI 组件、高频使用的通用组件。

**缺点**：即使未使用也会打包进最终代码（Tree Shaking 无法移除），增加包体积。

### 局部注册

在使用处直接 `import` 并注册，只在导入它的组件中可用：

```vue
<script setup>
// <script setup> 中导入即可直接使用，无需显式注册
import UserCard from './UserCard.vue'
import UserProfile from './UserProfile.vue'
</script>

<template>
  <UserCard />
  <UserProfile />
</template>
```

**局部注册是推荐做法**，配合 Tree Shaking 确保只打包实际使用的组件。

### 批量注册（自动化导入）

使用 `import.meta.glob` (Vite) 批量导入组件：

```javascript
// components/index.js
const components = import.meta.glob('./*.vue', { eager: true })

export default {
  install(app) {
    Object.entries(components).forEach(([path, module]) => {
      // 从路径提取文件名作为组件名
      const name = path
        .split('/').pop()
        .replace('.vue', '')
      app.component(name, module.default)
    })
  }
}
```

```javascript
// main.js
import BaseComponents from './components'
app.use(BaseComponents)
```

## 动态组件 — component :is

`<component :is="">` 根据绑定值动态切换渲染的组件：

```vue
<script setup>
import { ref } from 'vue'
import HomeTab from './HomeTab.vue'
import PostsTab from './PostsTab.vue'
import SettingsTab from './SettingsTab.vue'

const tabs = {
  home: HomeTab,
  posts: PostsTab,
  settings: SettingsTab
}

const currentTab = ref('home')
</script>

<template>
  <nav>
    <button
      v-for="(comp, name) in tabs"
      :key="name"
      @click="currentTab = name"
      :class="{ active: currentTab === name }"
    >
      {{ name }}
    </button>
  </nav>

  <!-- 动态渲染当前选中的组件 -->
  <component :is="tabs[currentTab]" />
</template>
```

`:is` 也可以直接传组件对象：

```vue
<component :is="someComponentObject" />
```

### 使用 keep-alive 时传递组件名

```vue
<component :is="currentComponent" />
<!-- 等价于以下写法（当组件已全局/局部注册时） -->
<component :is="'ComponentName'" />
```

## KeepAlive — 缓存组件状态

`<KeepAlive>` 包裹动态组件，在切换时缓存不活跃组件的状态，避免重复创建销毁：

```vue
<template>
  <nav>
    <button @click="tab = 'List'">列表</button>
    <button @click="tab = 'Detail'">详情</button>
  </nav>

  <KeepAlive>
    <component :is="tab" />
  </KeepAlive>
</template>
```

### include / exclude — 选择性缓存

```vue
<!-- 仅缓存 UserList 和 UserDetail -->
<KeepAlive :include="['UserList', 'UserDetail']">
  <component :is="tab" />
</KeepAlive>

<!-- 排除 Header 组件，其他都缓存 -->
<KeepAlive :exclude="['Header']">
  <component :is="tab" />
</KeepAlive>

<!-- 正则匹配 -->
<KeepAlive :include="/User/">
  <component :is="tab" />
</KeepAlive>
```

### max — 最大缓存实例数

```vue
<KeepAlive :max="5">
  <component :is="tab" />
</KeepAlive>
```

当缓存数量超过 `max` 时，最久未访问的实例会被销毁。

### 缓存生命周期钩子

被 `<KeepAlive>` 缓存的组件会多两个生命周期钩子：

```vue
<script setup>
import { onActivated, onDeactivated } from 'vue'

// 组件被激活（从缓存中恢复）时触发
onActivated(() => {
  console.log('组件激活')
  refreshData()  // 重新拉取最新数据
})

// 组件被缓存（从 DOM 移除但保留状态）时触发
onDeactivated(() => {
  console.log('组件缓存')
  clearTimer()   // 暂停定时器等
})
</script>
```

## Teleport — 传送门

`<Teleport>` 将组件内的一部分模板"传送"到 DOM 中的指定位置，常用于 Modal、Dialog、Toast 等需要脱离组件层级渲染的场景：

```vue
<!-- Modal.vue -->
<script setup>
defineProps({ visible: Boolean })
const emit = defineEmits(['close'])
</script>

<template>
  <Teleport to="body">
    <div v-if="visible" class="modal-overlay" @click.self="emit('close')">
      <div class="modal-content">
        <slot />
        <button @click="emit('close')">关闭</button>
      </div>
    </div>
  </Teleport>
</template>
```

```vue
<!-- 父组件中使用 -->
<template>
  <div class="app">
    <button @click="showModal = true">打开弹窗</button>
    <!-- Modal 的 DOM 仍然在 app 内，实际渲染到了 body -->
    <Modal :visible="showModal" @close="showModal = false">
      <h2>提示</h2>
      <p>确定删除此项目吗？</p>
    </Modal>
  </div>
</template>
```

最终 DOM 结构：

```html
<body>
  <div id="app">
    <!-- 父组件模板在此 -->
  </div>
  <!-- Teleport 把 Modal 移到了这里 -->
  <div class="modal-overlay">
    <div class="modal-content">
      <h2>提示</h2>
      <p>确定删除此项目吗？</p>
      <button>关闭</button>
    </div>
  </div>
</body>
```

### disabled 属性

```vue
<!-- 当 disabled 为 true 时不传送，正常在组件位置渲染 -->
<Teleport to="body" :disabled="isMobile">
  <Modal />
</Teleport>
```

### 多个 Teleport 到同一目标

多个 `<Teleport>` 传送到同一个 `to` 目标时，按顺序追加：

```vue
<Teleport to="#notifications">
  <p>通知 A</p>
</Teleport>
<Teleport to="#notifications">
  <p>通知 B</p>
</Teleport>

<!-- 结果 -->
<div id="notifications">
  <p>通知 A</p>
  <p>通知 B</p>
</div>
```

## Suspense — 异步组件加载

`<Suspense>` 用于协调异步依赖，在组件树加载完成前显示 fallback 内容。适合异步组件 + async setup：

```vue
<!-- 异步子组件 Dashboard.vue -->
<script setup>
// setup 中 await 让组件成为异步依赖
const data = await fetch('/api/dashboard').then(r => r.json())
</script>

<template>
  <div>{{ data.summary }}</div>
</template>
```

```vue
<!-- 父组件 -->
<template>
  <Suspense>
    <!-- 加载完成后渲染 -->
    <Dashboard />

    <!-- 加载中的 fallback -->
    <template #fallback>
      <div class="skeleton">加载中，请稍候...</div>
    </template>
  </Suspense>
</template>
```

### onErrorCaptured — 处理异步错误

```vue
<script setup>
import { onErrorCaptured, ref } from 'vue'

const error = ref(null)

onErrorCaptured((err) => {
  error.value = err.message
  return false  // 阻止错误继续传播
})
</script>

<template>
  <div v-if="error">出错了：{{ error }}</div>
  <Suspense v-else>
    <Dashboard />
    <template #fallback>加载中...</template>
  </Suspense>
</template>
```

## 小结

| 特性 | 用途 | 关键语法 |
|------|------|----------|
| 全局注册 | 基础 UI 组件高频复用 | `app.component()` |
| 局部注册 | 推荐方式，配合 Tree Shaking | `import` 后直接使用 |
| 动态组件 | Tab 切换、多态渲染 | `<component :is="">` |
| KeepAlive | 缓存组件状态，避免重复渲染 | `<KeepAlive :include="[]">` |
| Teleport | Modal/Toast 脱离层级渲染 | `<Teleport to="body">` |
| Suspense | 异步组件加载状态管理 | `<Suspense>` + `#fallback` |
