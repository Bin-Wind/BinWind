# 组件测试

Vue 3 官方测试方案是 **Vitest + Vue Test Utils**。Vitest 兼容 Vite，Vue Test Utils 提供组件挂载、查询、交互和断言 API。

## 安装与配置

```bash
npm install -D vitest @vue/test-utils happy-dom
```

```typescript
// vitest.config.ts
import { defineConfig } from 'vitest/config'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  test: {
    environment: 'happy-dom',
    globals: true,
    include: ['src/**/*.{test,spec}.ts']
  }
})
```

## 挂载、查询与交互

```typescript
import { mount } from '@vue/test-utils'

// 挂载
const wrapper = mount(Counter, { props: { initial: 10 } })
expect(wrapper.text()).toContain('10')

// 查询
const btn = wrapper.find('[data-test="increment"]')
const items = wrapper.findAll('.list-item')
expect(items).toHaveLength(3)

// 触发事件
await btn.trigger('click')
expect(wrapper.text()).toContain('11')

// setValue
await wrapper.find('input').setValue('hello')
```

## 测试 Emits

```typescript
await wrapper.find('input[name="name"]').setValue('Tom')
await wrapper.find('form').trigger('submit.prevent')

expect(wrapper.emitted().submit).toBeTruthy()
expect(wrapper.emitted().submit[0]).toEqual([{ name: 'Tom' }])
```

## Stub 与 Mock

```typescript
const wrapper = mount(UserPage, {
  global: {
    stubs: { RouterLink: { template: '<a><slot /></a>' } },
    mocks: { $route: { params: { id: '123' } } },
    provide: { theme: 'dark' }
  }
})

// Mock API
vi.mock('@/api/user', () => ({
  fetchUser: vi.fn().mockResolvedValue({ id: 1, name: 'Mock' })
}))
```

## Composable 测试

```typescript
import { useCounter } from './useCounter'

it('increments and decrements', () => {
  const { count, increment, decrement } = useCounter(5)
  expect(count.value).toBe(5)
  increment()
  expect(count.value).toBe(6)
})
```

依赖生命周期的 Composable 需额外容器：

```typescript
import { createApp } from 'vue'

export function withSetup<T>(composable: () => T): T {
  let result!: T
  const app = createApp({
    setup() { result = composable(); return () => null }
  })
  app.mount(document.createElement('div'))
  return result
}
```

## data-test 属性策略

用 `data-test` 隔离测试选择器与样式/结构变化：

```vue
<button data-test="submit-btn" @click="onSubmit">提交</button>
```

```typescript
wrapper.find('[data-test="submit-btn"]').trigger('click')
```

## 总结

| 测试类型 | 工具 | 要点 |
|----------|------|------|
| 挂载/查询 | mount, find | 断言 DOM 结构 |
| 交互 | trigger, setValue | await 后断言 |
| Emit | wrapper.emitted() | 验证事件和负载 |
| Mock | global.stubs, vi.mock | 隔离依赖 |
| Composable | 直接调用 / withSetup | 维持响应式 |
