# 条件渲染与列表渲染

Vue 提供了 `v-if`、`v-show` 控制元素的显示隐藏，以及 `v-for` 遍历数组和对象渲染列表。合理使用这些指令是构建动态界面的基础。

## v-if / v-else-if / v-else

`v-if` 根据表达式的真假值决定是否渲染元素。为假时元素完全从 DOM 中移除：

```vue
<template>
  <div v-if="status === 'loading'">加载中...</div>
  <div v-else-if="status === 'error'">加载失败，请重试</div>
  <div v-else-if="status === 'empty'">暂无数据</div>
  <div v-else>数据加载完成，共 {{ total }} 条</div>
</template>

<script setup>
import { ref } from 'vue'
const status = ref('loading') // 'loading' | 'error' | 'empty' | 'success'
const total = ref(0)
</script>
```

## v-show — 频繁切换的选择

`v-show` 同样控制显示隐藏，但通过 `display: none` 实现，元素始终存在于 DOM 中：

```vue
<template>
  <div v-show="isVisible">这段文字通过 CSS 控制显示</div>
  <button @click="isVisible = !isVisible">切换</button>
</template>

<script setup>
import { ref } from 'vue'
const isVisible = ref(true)
</script>
```

## v-if vs v-show 选择指南

| 场景 | 推荐 | 原因 |
|------|------|------|
| 运行时条件很少改变 | `v-if` | 惰性渲染，初始开销小 |
| 频繁切换显示/隐藏 | `v-show` | 无销毁重建开销，切换更快 |
| 初始条件为 false 且大概率不变 | `v-if` | 完全不渲染，节省初始开销 |
| 需要配合 `<template>` 包裹多元素 | `v-if` | `v-show` 不支持 `<template>` |

## v-for 列表渲染

使用 `v-for` 遍历数组或对象，基础语法为 `item in items`：

```vue
<template>
  <ul>
    <li v-for="item in fruits" :key="item.id">
      {{ item.name }} - ¥{{ item.price }}
    </li>
  </ul>
</template>

<script setup>
const fruits = [
  { id: 1, name: '苹果', price: 8 },
  { id: 2, name: '香蕉', price: 5 },
  { id: 3, name: '橘子', price: 6 }
]
</script>
```

### v-for 支持第二个参数获取索引

```vue
<li v-for="(item, index) in items" :key="item.id">
  {{ index + 1 }}. {{ item.name }}
</li>
```

### v-for 遍历对象

```vue
<li v-for="(value, key, index) in userInfo" :key="key">
  {{ key }}: {{ value }} (第 {{ index + 1 }} 个属性)
</li>
```

## :key 的重要性

`:key` 是 `v-for` 的必需属性，作为每个节点的唯一标识，帮助 Vue 的虚拟 DOM diff 算法高效追踪节点变化：

```vue
<!-- 错误：使用 index 作为 key，数据顺序变化时产生问题 -->
<li v-for="(item, index) in list" :key="index">{{ item }}</li>

<!-- 正确：使用唯一且稳定的标识 -->
<li v-for="item in list" :key="item.id">{{ item.name }}</li>
```

**为什么不要用 index 作为 key？** 当列表重新排序、在中间插入或删除元素时，index 对应的元素内容变了但 key 没变，导致 Vue 错误地复用 DOM，可能引发状态错乱。

## 数组响应式注意事项

Vue 3 使用 Proxy 实现响应式，数组操作基本都能被正确追踪。但以下写法需要特别注意：

```vue
<script setup>
import { ref } from 'vue'

const list = ref([1, 2, 3])

// ✅ 这些操作是响应式的
list.value.push(4)           // 追加
list.value.splice(1, 1)      // 删除
list.value[0] = 99            // 直接下标赋值 — Vue 3 支持！
list.value = [...list.value, 5] // 整体替换

// ⚠️ 注意：ref 包装的数组需要 .value
const newList = list.value.filter(n => n > 2) // 返回新数组
list.value = newList                           // 重新赋值
</script>
```

## v-if 与 v-for 不要同时使用

当 `v-if` 和 `v-for` 在同一元素上时，`v-if` 优先级更高，但它拿不到 `v-for` 的变量。推荐的替代方案：

```vue
<!-- ❌ 不推荐 -->
<li v-for="item in items" v-if="item.active" :key="item.id">

<!-- ✅ 方案一：用 computed 先过滤 -->
<template>
  <li v-for="item in activeItems" :key="item.id">
    {{ item.name }}
  </li>
</template>

<script setup>
import { computed } from 'vue'
const activeItems = computed(() => items.value.filter(i => i.active))
</script>

<!-- ✅ 方案二：用 <template> 包装 v-if 在外层 -->
<template v-for="item in items" :key="item.id">
  <li v-if="item.active">{{ item.name }}</li>
</template>
```

## n in N 快速生成数字序列

```vue
<!-- 生成 1 到 10 -->
<span v-for="n in 10" :key="n">{{ n }}</span>
```

## 综合示例：Todo 列表

```vue
<script setup>
import { ref, computed } from 'vue'

const newTodo = ref('')
const todos = ref([
  { id: 1, text: '学习 Vue', done: false },
  { id: 2, text: '写文章', done: true }
])

const activeTodos = computed(() => todos.value.filter(t => !t.done))
const doneTodos = computed(() => todos.value.filter(t => t.done))

function addTodo() {
  if (!newTodo.value.trim()) return
  todos.value.push({
    id: Date.now(),
    text: newTodo.value,
    done: false
  })
  newTodo.value = ''
}
</script>

<template>
  <input v-model="newTodo" @keyup.enter="addTodo" placeholder="添加待办" />

  <p v-if="todos.length === 0">暂无待办事项</p>

  <ul v-else>
    <li v-for="todo in todos" :key="todo.id"
        :class="{ done: todo.done }">
      {{ todo.text }}
    </li>
  </ul>

  <p>未完成 {{ activeTodos.length }} / 已完成 {{ doneTodos.length }}</p>
</template>
```

## 小结

- `v-if` 控制 DOM 的存在与否，适合条件较少变化
- `v-show` 控制 CSS 显示隐藏，适合频繁切换
- `v-for` 遍历渲染列表，**必须绑定唯一的 `:key`**
- 避免 `v-if` 和 `v-for` 同时使用，用 `computed` 或 `<template>` 拆分
- Vue 3 的 Proxy 响应式支持直接下标修改数组
