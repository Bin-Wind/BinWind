# 组件基础 — Props、Emits 与 Slots 入门

组件是 Vue 应用的核心构建单元。每个 `.vue` 文件就是一个独立组件，封装了模板、逻辑和样式。组件之间通过 Props（父传子）和 Emits（子传父）进行通信。

## 定义一个组件

使用 `<script setup>` 语法糖编写的单文件组件（SFC）：

```vue
<!-- UserCard.vue -->
<script setup>
defineProps({
  name: String,
  avatar: String,
  role: { type: String, default: 'User' }
})
</script>

<template>
  <div class="user-card">
    <img :src="avatar" :alt="name" />
    <h3>{{ name }}</h3>
    <span>{{ role }}</span>
  </div>
</template>

<style scoped>
.user-card {
  border: 1px solid #eee;
  border-radius: 8px;
  padding: 16px;
}
</style>
```

## Props — 父组件向子组件传递数据

### defineProps 声明 Props

子组件使用 `defineProps` 声明接收的属性。`defineProps` 是编译宏，无需导入：

```vue
<!-- Child.vue -->
<script setup>
// 方式一：数组简写（仅声明名称）
const props = defineProps(['title', 'count'])

// 方式二：对象方式（带类型和默认值）—— 推荐
const props = defineProps({
  title: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    default: 0
  },
  disabled: {
    type: Boolean,
    default: false
  },
  items: {
    type: Array,
    default: () => []   // 对象/数组默认值必须用工厂函数
  },
  callback: {
    type: Function,
    default: null
  }
})
</script>
```

### TypeScript 类型声明

```vue
<script setup lang="ts">
interface Props {
  title: string
  count?: number
  tags?: string[]
}

const props = withDefaults(defineProps<Props>(), {
  count: 0,
  tags: () => []
})
</script>
```

### 父组件传递 Props

```vue
<template>
  <Child title="欢迎" :count="10" :items="list" />
  <!-- 传 boolean 简写：disabled 等同于 :disabled="true" -->
  <Child disabled />
</template>
```

## Emits — 子组件向父组件通信

子组件通过 `defineEmits` 声明事件，父组件通过 `@event` 监听：

```vue
<!-- Counter.vue -->
<script setup>
const emit = defineEmits(['update', 'reset'])

// 传单个值
function increment() {
  emit('update', 1)
}

// 传多个值
function reset(value) {
  emit('reset', value, Date.now())
}
</script>

<template>
  <button @click="increment">+1</button>
  <button @click="reset(0)">重置</button>
</template>
```

### TypeScript 类型声明

```vue
<script setup lang="ts">
const emit = defineEmits<{
  update: [delta: number]
  reset: [value: number, timestamp: number]
}>()
</script>
```

### 父组件接收事件

```vue
<template>
  <Counter @update="handleUpdate" @reset="handleReset" />
</template>

<script setup>
function handleUpdate(delta) {
  console.log('增加', delta)
}
function handleReset(value, timestamp) {
  console.log('重置为', value, timestamp)
}
</script>
```

## v-model 在组件上的使用

`v-model` 可以绑定到组件上，实现对 Props + Emits 的封装：

```vue
<!-- CustomInput.vue -->
<script setup>
const props = defineProps({ modelValue: String })
const emit = defineEmits(['update:modelValue'])

function onInput(e) {
  emit('update:modelValue', e.target.value)
}
</script>

<template>
  <input :value="modelValue" @input="onInput" />
</template>
```

父组件使用：

```vue
<CustomInput v-model="username" />
<!-- 等价于 -->
<CustomInput :model-value="username" @update:model-value="username = $event" />
```

## Slot 基础 — 内容分发

Slot（插槽）允许父组件向子组件传递模板内容：

```vue
<!-- Card.vue -->
<template>
  <div class="card">
    <div class="card-header">
      <slot name="header">默认标题</slot>
    </div>
    <div class="card-body">
      <slot>默认内容</slot>
    </div>
    <div class="card-footer">
      <slot name="footer">
        <button>确定</button>
      </slot>
    </div>
  </div>
</template>
```

父组件使用 Card：

```vue
<Card>
  <template #header>
    <h2>用户信息</h2>
  </template>

  <p>这里是卡片正文内容，会渲染到默认插槽。</p>

  <template #footer>
    <button>取消</button>
    <button>保存</button>
  </template>
</Card>
```

`#header` 是 `v-slot:header` 的简写，`#default` 可省略直接用默认内容。

## 父子组件通信完整示例

一个搜索 + 列表的经典场景：

```vue
<!-- SearchBox.vue -->
<script setup>
import { ref } from 'vue'

const emit = defineEmits(['search'])
const keyword = ref('')

function onSearch() {
  emit('search', keyword.value)
}
</script>

<template>
  <div>
    <input v-model="keyword" @keyup.enter="onSearch" />
    <button @click="onSearch">搜索</button>
  </div>
</template>
```

```vue
<!-- ResultList.vue -->
<script setup>
defineProps({
  results: { type: Array, default: () => [] },
  loading: { type: Boolean, default: false }
})
</script>

<template>
  <div v-if="loading">搜索中...</div>
  <ul v-else-if="results.length">
    <li v-for="item in results" :key="item.id">{{ item.title }}</li>
  </ul>
  <p v-else>暂无结果</p>
</template>
```

```vue
<!-- 父组件使用 -->
<script setup>
import { ref } from 'vue'
import SearchBox from './SearchBox.vue'
import ResultList from './ResultList.vue'

const results = ref([])
const loading = ref(false)

async function handleSearch(keyword) {
  loading.value = true
  results.value = await fetch(`/api/search?q=${keyword}`)
    .then(r => r.json())
  loading.value = false
}
</script>

<template>
  <SearchBox @search="handleSearch" />
  <ResultList :results="results" :loading="loading" />
</template>
```

## 小结

| 通信方向 | 机制 | 关键字 |
|----------|------|--------|
| 父 → 子 | Props | `defineProps` |
| 子 → 父 | Emits | `defineEmits` |
| 双向 | v-model | `modelValue` + `update:modelValue` |
| 内容分发 | Slot | `<slot>` / `<slot name="">` |

组件化是 Vue 的精髓所在。Props down、Events up 是单向数据流的核心理念。下一节我们将深入学习 Composition API，探索更灵活的逻辑组织方式。
