# Composition API 深入

Composition API（组合式 API）是 Vue 3 的核心特性，通过 `setup` 函数或 `<script setup>` 在组件内组织逻辑。它与 Options API 不同，不再按 data、methods、computed 等选项拆分，而是按功能关注点聚合代码。

## ref 深入

`ref` 创建响应式引用，适用于基本类型和需要整体替换的场景：

```vue
<script setup>
import { ref } from 'vue'

// 基本类型
const count = ref(0)
const name = ref('Vue')

// 对象和数组 —— ref 内部会自动用 reactive 包装
const user = ref({ name: 'Alice', age: 25 })
const list = ref([1, 2, 3])

// 修改方式
count.value++                   // 基本类型需要 .value
user.value.name = 'Bob'         // 对象修改属性
list.value.push(4)              // 数组操作
list.value = [4, 5, 6]          // 整体替换
</script>
```

### ref 的解包规则

```vue
<script setup>
import { ref } from 'vue'
const msg = ref('hello')

// 在 <script> 中必须 .value
console.log(msg.value) // 'hello'

// 在 <template> 中自动解包
// {{ msg }} 直接使用，无需 .value

// 在 reactive 对象中自动解包
const state = reactive({ message: msg })
console.log(state.message)      // 'hello' — 自动解包
state.message = 'world'
console.log(msg.value)           // 'world' — 双向同步

// 在数组 / Map 中不会自动解包
const arr = reactive([msg])
console.log(arr[0].value)        // 需要 .value
</script>
```

## reactive 深入

`reactive` 返回对象的响应式代理，直接操作属性：

```vue
<script setup>
import { reactive } from 'vue'

const state = reactive({
  user: { name: 'Alice', age: 25 },
  settings: { theme: 'dark', lang: 'zh' }
})

// 直接修改属性，无需 .value
state.user.name = 'Bob'
state.settings.theme = 'light'
</script>
```

### reactive 的限制

```javascript
// ❌ 不能整体替换
let state = reactive({ count: 0 })
state = reactive({ count: 1 })  // 失去响应式

// ❌ 解构会失去响应式
const { count } = reactive({ count: 0 }) // count 不再是响应式

// ❌ 不能赋值给基本类型
const n = reactive(0)  // 无效
```

## toRefs — 解构保持响应式

`toRefs` 将 reactive 对象的每个属性转为 ref，解构后仍然保持响应式：

```vue
<script setup>
import { reactive, toRefs } from 'vue'

const state = reactive({
  name: 'Alice',
  age: 25
})

// 直接解构会丢失响应式
// const { name, age } = state  ❌

// 使用 toRefs 解构
const { name, age } = toRefs(state)

// 现在修改这些 ref 等同于修改原对象
name.value = 'Bob'
console.log(state.name) // 'Bob'
</script>
```

**最佳实践**：在组合式函数（composables）中返回数据时使用 `toRefs`，让调用方可以解构使用。

## readonly — 防止意外修改

`readonly` 创建只读代理，常用于暴露状态但不允许外部修改：

```vue
<script setup>
import { reactive, readonly } from 'vue'

const state = reactive({ count: 0 })

// 暴露只读版本
const readonlyState = readonly(state)

// readonlyState.count = 10  ❌ 开发模式下会警告

function increment() {
  state.count++   // ✅ 内部可以修改
}
</script>
```

## computed 深入

计算属性除了返回派生值，也可以定义可写的计算属性：

```vue
<script setup>
import { ref, computed } from 'vue'

const firstName = ref('Zhang')
const lastName = ref('San')

// 只读计算属性
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

// 可写计算属性
const fullNameWritable = computed({
  get() {
    return `${firstName.value} ${lastName.value}`
  },
  set(val) {
    const [first, last] = val.split(' ')
    firstName.value = first
    lastName.value = last
  }
})

// 使用可写计算属性
fullNameWritable.value = 'Li Si'
console.log(firstName.value)  // 'Li'
console.log(lastName.value)   // 'Si'
</script>
```

## watchEffect — 自动追踪依赖

`watchEffect` 与 `watch` 不同，它会自动追踪回调中使用的响应式依赖，无需显式指定监听源：

```vue
<script setup>
import { ref, watchEffect } from 'vue'

const userId = ref(1)
const userData = ref(null)

// watchEffect 自动追踪 userId.value
const stop = watchEffect(async () => {
  // 当 userId 变化时，自动重新执行
  userData.value = await fetch(`/api/user/${userId.value}`)
    .then(r => r.json())
})

// 手动停止监听
// stop()
</script>
```

### watchEffect 的配置

```javascript
watchEffect(
  () => { /* 副作用 */ },
  {
    flush: 'post',  // 在 DOM 更新后触发（默认 'pre'）
    onTrack(e) {},  // 依赖被追踪时
    onTrigger(e) {} // 依赖变化时
  }
)
```

## watch vs watchEffect 对比

| 特性 | watch | watchEffect |
|------|-------|-------------|
| 监听源 | 手动指定 | 自动追踪 |
| 访问旧值 | 支持 `(newVal, oldVal)` | 不支持 |
| 惰性执行 | 默认不立即执行 | 立即执行一次 |
| 适用场景 | 精确控制监听 | 副作用自动追踪依赖 |

## 生命周期钩子

Composition API 中，生命周期钩子以 `onXxx` 形式存在：

```vue
<script setup>
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted
} from 'vue'

onBeforeMount(() => {
  console.log('组件即将挂载')
})

onMounted(() => {
  console.log('组件已挂载，可以访问 DOM')
  // 初始化数据、绑定事件、启动定时器
})

onBeforeUpdate(() => {
  console.log('DOM 更新前')
})

onUpdated(() => {
  console.log('DOM 更新后')
})

onBeforeUnmount(() => {
  console.log('组件卸载前，清理工作')
  // 清除定时器、取消订阅、移除事件监听
})

onUnmounted(() => {
  console.log('组件已卸载')
})
</script>
```

### 与 Options API 对照

| Options API | Composition API |
|-------------|-----------------|
| `beforeCreate` | `setup()` 本身 |
| `created` | `setup()` 本身 |
| `beforeMount` | `onBeforeMount` |
| `mounted` | `onMounted` |
| `beforeUpdate` | `onBeforeUpdate` |
| `updated` | `onUpdated` |
| `beforeUnmount` | `onBeforeUnmount` |
| `unmounted` | `onUnmounted` |

## 综合示例：计时器组件

```vue
<script setup>
import { ref, computed, onMounted, onUnmounted } from 'vue'

const seconds = ref(0)
const running = ref(false)
let timer = null

const display = computed(() => {
  const m = Math.floor(seconds.value / 60)
  const s = seconds.value % 60
  return `${String(m).padStart(2, '0')}:${String(s).padStart(2, '0')}`
})

function start() {
  running.value = true
  timer = setInterval(() => seconds.value++, 1000)
}

function pause() {
  running.value = false
  clearInterval(timer)
}

function reset() {
  pause()
  seconds.value = 0
}

onUnmounted(() => clearInterval(timer))
</script>

<template>
  <h1>{{ display }}</h1>
  <button @click="start" :disabled="running">开始</button>
  <button @click="pause" :disabled="!running">暂停</button>
  <button @click="reset">重置</button>
</template>
```

## 小结

- `ref` 适合基本类型和整体替换，`reactive` 适合复杂对象
- `toRefs` 解构 reactive 保持响应式，`readonly` 防止外部修改
- `computed` 可读可写，`watchEffect` 自动追踪依赖
- 生命周期钩子在 `<script setup>` 中以 `onXxx` 形式使用
- `onUnmounted` 中务必清理定时器、订阅和事件监听，避免内存泄漏
