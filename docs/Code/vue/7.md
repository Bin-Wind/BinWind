# 组件通信大全

Vue 组件间通信方式多种多样，从简单的父子通信到跨层级注入再到全局状态管理。选择正确的通信方式，能让组件关系清晰、代码可维护。

## 通信方式概览

```
父子组件      → Props down / Events up + v-model
祖先后代      → provide / inject
兄弟组件      → 共同父组件提升状态 / Pinia / 事件总线
任意组件      → Pinia / 事件总线 / 全局状态
```

## 1. Props + Emits（父子通信）

这是最基本、最推荐的通信方式。前文已详细介绍，这里仅作速查：

```vue
<!-- 父组件 -->
<Child
  :title="pageTitle"
  :count="10"
  @update="handleUpdate"
  v-model:visible="showModal"
/>

<!-- 子组件 -->
<script setup>
const props = defineProps({ title: String, count: Number, visible: Boolean })
const emit = defineEmits(['update', 'update:visible'])
// emit('update:visible', false) — 配合 v-model
</script>
```

## 2. provide / inject（跨层级注入）

当祖先组件需要向深层后代传递数据时，逐层传递 Props 繁琐且不灵活（prop drilling）。`provide` / `inject` 解决了这个问题：

```vue
<!-- Ancestor.vue — 祖先组件 -->
<script setup>
import { ref, provide, readonly } from 'vue'

const theme = ref('light')
const user = ref({ name: 'Alice', role: 'admin' })

// 提供数据
provide('theme', theme)
provide('user', readonly(user))    // 只读方式提供
provide('changeTheme', (val) => {  // 同时提供修改方法
  theme.value = val
})
</script>
```

```vue
<!-- DeepChild.vue — 深层后代组件 -->
<script setup>
import { inject } from 'vue'

const theme = inject('theme', 'light')            // 第二个参数为默认值
const user = inject('user')
const changeTheme = inject('changeTheme')

function toggleTheme() {
  changeTheme(theme.value === 'light' ? 'dark' : 'light')
}
</script>

<template>
  <div :class="theme">
    <p>当前用户：{{ user.name }}</p>
    <button @click="toggleTheme">切换主题</button>
  </div>
</template>
```

### provide / inject 最佳实践

- **在提供方尽可能用 `readonly` 包裹**，保证单向数据流
- **提供修改方法**，让数据变更可追溯
- **使用 Symbol 作为注入键**，避免命名冲突

```javascript
// keys.js
export const ThemeKey = Symbol('theme')
export const UserKey = Symbol('user')

// 提供方
provide(ThemeKey, theme)

// 注入方
const theme = inject(ThemeKey)
```

## 3. 状态提升（兄弟组件通信）

当两个兄弟组件需要共享状态时，将状态提升到它们共同的父组件中：

```vue
<!-- Parent.vue -->
<script setup>
import { ref } from 'vue'
import SiblingA from './SiblingA.vue'
import SiblingB from './SiblingB.vue'

const sharedData = ref('')

function handleUpdate(val) {
  sharedData.value = val
}
</script>

<template>
  <SiblingA @update="handleUpdate" />
  <SiblingB :data="sharedData" />
</template>
```

## 4. 事件总线（轻量全局通信）

对于简单的跨组件通信，可以使用 `mitt` 库实现事件总线。`$on`、`$off`、`$emit` 在 Vue 3 中已移除：

```bash
pnpm add mitt
```

```javascript
// eventBus.js
import mitt from 'mitt'
export const emitter = mitt()
```

```vue
<!-- ComponentA.vue — 发送事件 -->
<script setup>
import { emitter } from './eventBus'

function sendMessage() {
  emitter.emit('global-message', { text: 'Hello from A', time: Date.now() })
}
</script>
```

```vue
<!-- ComponentB.vue — 接收事件 -->
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import { emitter } from './eventBus'

const messages = ref([])

onMounted(() => {
  emitter.on('global-message', (msg) => {
    messages.value.push(msg)
  })
})

// 务必在卸载时取消监听，防止内存泄漏
onUnmounted(() => {
  emitter.off('global-message')
})
</script>
```

**注意**：事件总线适合小型应用或特定场景（如通知系统）。项目中超过 3-5 处使用时，建议升级到 Pinia。

## 5. Pinia 共享状态（推荐）

Pinia 是 Vue 3 官方推荐的状态管理库，适用于中大型应用（详见第 9 节）：

```javascript
// stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', () => {
  const count = ref(0)

  function increment() {
    count.value++
  }

  return { count, increment }
})
```

```vue
<!-- 任意组件 -->
<script setup>
import { useCounterStore } from '@/stores/counter'
const counter = useCounterStore()
</script>

<template>
  <p>{{ counter.count }}</p>
  <button @click="counter.increment">+1</button>
</template>
```

## 6. attrs — 透传属性

`$attrs` 包含父组件传递的、未被声明为 Props 的属性，常用于包装原生元素：

```vue
<!-- BaseInput.vue — 透传包装组件 -->
<script setup>
defineProps({ label: String })
</script>

<template>
  <label>
    <span>{{ label }}</span>
    <!-- $attrs 自动透传到内部 input -->
    <input v-bind="$attrs" />
  </label>
</template>

<!-- 使用 -->
<BaseInput label="用户名" type="text" placeholder="请输入" maxlength="20" />
```

在 `<script setup>` 中，使用 `useAttrs` 获取：

```vue
<script setup>
import { useAttrs } from 'vue'
const attrs = useAttrs()
console.log(attrs.placeholder)  // '请输入'
</script>
```

## 通信方式的选型决策树

```
需要通信的两个组件关系如何？
├── 父子关系
│   ├── 父传子 → Props
│   ├── 子传父 → Emits
│   └── 双向绑定 → v-model
├── 祖先后代（跨多层级）
│   └── 不希望逐级传递 → provide / inject
├── 兄弟或任意关系
│   ├── 共享少量状态 → 状态提升到共同父组件
│   ├── 简单事件通知（< 5 处） → mitt 事件总线
│   └── 全局状态 / 中大型应用 → Pinia
└── 需要透传属性到原生元素
    └── useAttrs / $attrs
```

## 综合示例：多组件协作

以一个简单的通知系统展示事件总线的使用：

```javascript
// eventBus.js
import mitt from 'mitt'
export const emitter = mitt()
```

```vue
<!-- NotificationProvider.vue -->
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import { emitter } from './eventBus'

interface Notification {
  id: number
  message: string
  type: 'info' | 'success' | 'error'
}

const notifications = ref<Notification[]>([])

function addNotification(msg) {
  const id = Date.now()
  notifications.value.push({ id, ...msg })
  setTimeout(() => {
    notifications.value = notifications.value.filter(n => n.id !== id)
  }, 3000)
}

onMounted(() => emitter.on('notify', addNotification))
onUnmounted(() => emitter.off('notify', addNotification))
</script>

<template>
  <div class="notification-container">
    <div v-for="n in notifications" :key="n.id" :class="['toast', n.type]">
      {{ n.message }}
    </div>
  </div>
</template>
```

```vue
<!-- 任意业务组件发送通知 -->
<script setup>
import { emitter } from './eventBus'

async function saveData() {
  try {
    await api.save()
    emitter.emit('notify', { message: '保存成功', type: 'success' })
  } catch {
    emitter.emit('notify', { message: '保存失败', type: 'error' })
  }
}
</script>
```

## 小结

| 通信方式 | 适用场景 | 复杂度 |
|----------|----------|--------|
| Props + Emits | 父子组件 | 低 |
| v-model | 父子双向绑定 | 低 |
| provide / inject | 跨层级传递 | 中 |
| 状态提升 | 兄弟组件 | 低 |
| mitt 事件总线 | 轻量全局事件 | 中 |
| Pinia | 全局状态管理 | 中 |
| $attrs / useAttrs | 属性透传 | 低 |

选型原则：能 Props/Emits 解决的不要上 Pinia，能状态提升的不要用事件总线。简单优于复杂。
