# Vue Router 路由实战

Vue Router 是 Vue.js 的官方路由管理器，用于构建单页应用（SPA）。它通过 URL 路径映射到不同组件，实现页面无刷新切换。

## 安装与初始化

使用 `create-vue` 创建项目时选择 Router，或手动安装：

```bash
pnpm add vue-router@4
```

基础路由配置：

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/',
    name: 'home',
    component: () => import('@/views/Home.vue')
  },
  {
    path: '/about',
    name: 'about',
    component: () => import('@/views/About.vue')
  }
]

const router = createRouter({
  history: createWebHistory(),   // HTML5 History 模式
  routes
})

export default router
```

在 `main.js` 中注册：

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

createApp(App).use(router).mount('#app')
```

## RouterView 与 RouterLink

`<RouterView>` 是路由匹配组件的渲染出口，`<RouterLink>` 替代 `<a>` 标签实现导航：

```vue
<!-- App.vue -->
<template>
  <nav>
    <RouterLink to="/">首页</RouterLink>
    <RouterLink to="/about">关于</RouterLink>
  </nav>
  <RouterView />
</template>
```

`RouterLink` 的常用属性：

```vue
<!-- 路径拼接 -->
<RouterLink :to="`/user/${userId}`">用户</RouterLink>
<!-- 对象写法 -->
<RouterLink :to="{ name: 'user', params: { id: 1 } }">用户</RouterLink>
<!-- 带查询参数 -->
<RouterLink :to="{ path: '/search', query: { q: 'vue' } }">搜索</RouterLink>
<!-- replace 模式 — 不产生历史记录 -->
<RouterLink to="/login" replace>登录</RouterLink>
```

## 动态路由

路径中带 `:param` 即为动态路由，用于匹配不同资源的同一页面模板：

```javascript
const routes = [
  {
    path: '/user/:userId',
    name: 'user',
    component: () => import('@/views/User.vue')
  }
]
```

在组件中获取路由参数：

```vue
<!-- User.vue -->
<script setup>
import { useRoute } from 'vue-router'
import { watch } from 'vue'

const route = useRoute()

// 获取动态参数
console.log(route.params.userId)
console.log(route.query.page)     // 查询参数 ?page=1

// 监听参数变化（同一组件复用时触发）
watch(() => route.params.userId, (newId) => {
  fetchUser(newId)
})
</script>

<template>
  <h1>用户 ID：{{ $route.params.userId }}</h1>
</template>
```

## 编程式导航

在 JavaScript 中进行页面跳转：

```vue
<script setup>
import { useRouter } from 'vue-router'

const router = useRouter()

function goToUser(id) {
  router.push({ name: 'user', params: { userId: id } })
}

function goBack() {
  router.back()          // 返回上一页
}

function goForward() {
  router.forward()       // 前进一页
}

function replaceLogin() {
  router.replace('/login')  // 替换当前历史记录，不可返回
}
</script>
```

## 路由导航守卫

导航守卫在路由切换前后执行逻辑，常用于权限校验和页面标题设置。

### 全局前置守卫

```javascript
// router/index.js
router.beforeEach((to, from, next) => {
  const isAuthenticated = localStorage.getItem('token')

  // 需要登录的页面，未登录重定向
  if (to.meta.requiresAuth && !isAuthenticated) {
    next({ name: 'login', query: { redirect: to.fullPath } })
  } else {
    next()
  }
})
```

### 全局后置钩子

```javascript
router.afterEach((to) => {
  document.title = to.meta.title || '默认标题'
})
```

### 路由独享守卫

```javascript
const routes = [
  {
    path: '/admin',
    component: AdminPage,
    beforeEnter: (to, from, next) => {
      // 仅该路由触发
      if (isAdmin()) next()
      else next('/403')
    }
  }
]
```

### 组件内守卫

```vue
<script setup>
import { onBeforeRouteLeave, onBeforeRouteUpdate } from 'vue-router'

// 离开当前页面前 — 常用于提示未保存
onBeforeRouteLeave((to, from) => {
  if (hasUnsavedChanges.value) {
    const answer = window.confirm('有未保存的更改，确定离开？')
    if (!answer) return false
  }
})

// 路由参数变化但组件复用时
onBeforeRouteUpdate((to, from) => {
  fetchData(to.params.id)
})
</script>
```

## 路由元信息 meta

在路由配置中携带额外信息：

```javascript
const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    meta: {
      title: '控制台',
      requiresAuth: true,
      roles: ['admin', 'editor']
    }
  }
]

// 守卫中使用
router.beforeEach((to, from) => {
  const { roles } = to.meta
  if (roles && !roles.includes(currentUser.role)) {
    return '/403'
  }
})
```

## 嵌套路由

页面中有子路由视口时使用嵌套路由：

```javascript
const routes = [
  {
    path: '/user/:id',
    component: UserLayout,
    children: [
      {
        path: '',              // 默认子路由
        component: UserProfile
      },
      {
        path: 'posts',         // 匹配 /user/:id/posts
        component: UserPosts
      },
      {
        path: 'settings',
        component: UserSettings
      }
    ]
  }
]
```

`UserLayout.vue` 中需要放置 `<RouterView />` 渲染子路由组件：

```vue
<template>
  <div class="user-layout">
    <aside>
      <RouterLink :to="`/user/${id}`">个人信息</RouterLink>
      <RouterLink :to="`/user/${id}/posts`">文章</RouterLink>
      <RouterLink :to="`/user/${id}/settings`">设置</RouterLink>
    </aside>
    <main>
      <RouterView />
    </main>
  </div>
</template>
```

## 懒加载（路由级代码分割）

所有路由组件使用动态 `import()` 实现按需加载：

```javascript
// 静态导入 — 全部打入主包
// import Home from '@/views/Home.vue'

// 动态导入 — 独立 chunk，按需加载
const Home = () => import('@/views/Home.vue')

// 分组合并 chunk — 同组被打包到同一文件
const UserProfile = () => import(/* webpackChunkName: "user" */ '@/views/UserProfile.vue')
const UserPosts   = () => import(/* webpackChunkName: "user" */ '@/views/UserPosts.vue')
```

Vite 环境下同样支持 Magic Comment 分包：

```javascript
const Admin = () => import(/* chunkName: "admin" */ '@/views/Admin.vue')
```

## scrollBehavior — 滚动行为控制

配置路由切换时的滚动位置：

```javascript
const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) {
      return savedPosition       // 浏览器前进/后退恢复位置
    }
    if (to.hash) {
      return { el: to.hash }     // 锚点定位
    }
    return { top: 0 }            // 始终回到顶部
  }
})
```

## 404 页面与重定向

```javascript
const routes = [
  // 重定向
  { path: '/', redirect: '/home' },
  { path: '/old-path', redirect: { name: 'home' } },

  // 别名
  { path: '/about', component: About, alias: '/about-us' },

  // 404 捕获所有未匹配路径（必须放在最后）
  {
    path: '/:pathMatch(.*)*',
    name: 'notFound',
    component: () => import('@/views/NotFound.vue')
  }
]
```

## 小结

| 概念 | 说明 |
|------|------|
| `createRouter` | 创建路由实例 |
| `createWebHistory` | HTML5 History 模式（推荐） |
| `createWebHashHistory` | Hash 模式 (`/#/path`) |
| `<RouterView>` | 路由组件渲染出口 |
| `<RouterLink>` | 声明式导航 |
| `useRouter().push()` | 编程式导航 |
| `useRoute()` | 获取当前路由信息 |
| `beforeEach` | 全局前置守卫，权限拦截 |
| 动态 `import()` | 路由级代码分割 / 懒加载 |

路由是 SPA 的骨架。合理的路由设计加上懒加载，能让应用即使用户页面多也保持快速的首屏加载。下一节学习 Pinia 状态管理。
