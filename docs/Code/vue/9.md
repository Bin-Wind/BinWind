# Pinia 状态管理入门

Pinia 是 Vue.js 官方推荐的状态管理库，也是 Vuex 5 的精神续作。它提供类型安全的 Store、Devtools 支持、热更新等特性，API 设计简洁直观。

## 安装与初始化

```bash
pnpm add pinia
```

```javascript
// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const pinia = createPinia()
const app = createApp(App)

app.use(pinia)
app.mount('#app')
```

## 定义一个 Store

Pinia 推荐使用 Setup Store 语法（类似 Composition API），也可以使用 Options Store 语法。

### Setup Store 风格（推荐）

```javascript
// stores/counter.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useCounterStore = defineStore('counter', () => {
  // 状态 — 相当于 data
  const count = ref(0)
  const history = ref([])

  // 计算属性 — 相当于 getters
  const doubleCount = computed(() => count.value * 2)
  const lastAction = computed(() => history.value.at(-1) ?? '无')

  // 方法 — 相当于 actions
  function increment() {
    count.value++
    history.value.push(`+1 → ${count.value}`)
  }

  function decrement() {
    count.value--
    history.value.push(`-1 → ${count.value}`)
  }

  function reset() {
    count.value = 0
    history.value = []
  }

  // 返回需要暴露的内容
  return { count, history, doubleCount, lastAction, increment, decrement, reset }
})
```

### Options Store 风格

```javascript
export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0,
    history: []
  }),
  getters: {
    doubleCount: (state) => state.count * 2,
    lastAction: (state) => state.history.at(-1) ?? '无'
  },
  actions: {
    increment() {
      this.count++
      this.history.push(`+1 → ${this.count}`)
    },
    decrement() {
      this.count--
      this.history.push(`-1 → ${this.count}`)
    },
    reset() {
      this.count = 0
      this.history = []
    }
  }
})
```

## 在组件中使用 Store

```vue
<script setup>
import { useCounterStore } from '@/stores/counter'
import { storeToRefs } from 'pinia'

const counter = useCounterStore()

// ❌ 直接解构会丢失响应式
// const { count, doubleCount } = counter

// ✅ 使用 storeToRefs 保持响应式
const { count, doubleCount, lastAction } = storeToRefs(counter)

// ✅ actions 可以直接解构（它们本身就是函数）
const { increment, decrement, reset } = counter
</script>

<template>
  <p>计数：{{ count }}</p>
  <p>双倍：{{ doubleCount }}</p>
  <p>最近操作：{{ lastAction }}</p>
  <button @click="increment">+1</button>
  <button @click="decrement">-1</button>
  <button @click="reset">重置</button>
</template>
```

**关键规则**：`storeToRefs` 仅用于解构 state 和 getters，actions 直接解构即可。

## $patch — 批量更新

```javascript
const store = useUserStore()

// 对象形式 — 批量修改多个属性
store.$patch({
  name: 'Bob',
  age: 30,
  email: 'bob@example.com'
})

// 函数形式 — 适合复杂逻辑
store.$patch((state) => {
  state.items.push(newItem)
  state.total += newItem.price
  state.lastUpdated = Date.now()
})
```

## $reset — 重置 Store

```javascript
// 将 state 重置为初始值（仅 Options Store 默认支持）
store.$reset()

// Setup Store 需要手动实现
const useMyStore = defineStore('my', () => {
  const count = ref(0)
  function $reset() {
    count.value = 0
  }
  return { count, $reset }
})
```

## $subscribe — 订阅状态变化

```javascript
const store = useCounterStore()

const unsubscribe = store.$subscribe((mutation, state) => {
  console.log('变更类型：', mutation.type)  // 'direct' | 'patch object' | 'patch function'
  console.log('Store ID：', mutation.storeId)
  console.log('新状态：', state.count)

  // 持久化到 localStorage
  localStorage.setItem('counter', JSON.stringify(state))
})

// 组件卸载时取消订阅
onUnmounted(() => unsubscribe())
```

## 异步 Actions

Actions 支持 async/await，无需像 Vuex 那样区分 action 和 mutation：

```javascript
// stores/user.js
export const useUserStore = defineStore('user', () => {
  const user = ref(null)
  const loading = ref(false)
  const error = ref(null)

  async function fetchUser(id) {
    loading.value = true
    error.value = null
    try {
      const response = await fetch(`/api/user/${id}`)
      if (!response.ok) throw new Error('请求失败')
      user.value = await response.json()
    } catch (err) {
      error.value = err.message
    } finally {
      loading.value = false
    }
  }

  async function updateProfile(data) {
    const response = await fetch(`/api/user/${user.value.id}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    })
    user.value = await response.json()
  }

  return { user, loading, error, fetchUser, updateProfile }
})
```

## Store 之间的相互引用

一个 Store 可以使用另一个 Store：

```javascript
// stores/cart.js
import { useUserStore } from './user'

export const useCartStore = defineStore('cart', () => {
  const userStore = useUserStore()
  const items = ref([])

  const totalPrice = computed(() => {
    const discount = userStore.user?.vip ? 0.8 : 1
    return items.value.reduce((sum, i) => sum + i.price, 0) * discount
  })

  return { items, totalPrice }
})
```

**注意**：不要在 Store 顶层使用 `useXxxStore()`，应该放在函数内部，否则可能导致循环依赖问题。如果担心性能，可将 `useXxxStore()` 放在需要用到的 getter/action 中。

## Pinia DevTools

Pinia 内置支持 Vue DevTools，无需额外配置。打开浏览器 Vue DevTools 可以看到：

- **Timeline** — 状态变更时间线
- **Pinia 面板** — 查看所有 Store、实时编辑 state
- **Time Travel** — 回溯状态到之前的快照

## Pinia vs Vuex 对比

| 特性 | Pinia | Vuex 4 |
|------|-------|--------|
| 类型推断 | 完善（TypeScript 首选） | 需要额外类型声明 |
| API 风格 | Composition / Options 双模式 | 仅 Options |
| mutations | 不再需要，直接修改 state | 必须通过 mutation |
| 模块嵌套 | 扁平化，无需嵌套 | 支持嵌套模块 |
| 代码分割 | 天然支持 | 需要动态模块注册 |
| 体积 | ~1.5KB | ~10KB |
| DevTools | 原生支持 | 支持 |

## 持久化插件

借助 `pinia-plugin-persistedstate` 轻松实现状态持久化：

```bash
pnpm add pinia-plugin-persistedstate
```

```javascript
// main.js
import { createPinia } from 'pinia'
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

const pinia = createPinia()
pinia.use(piniaPluginPersistedstate)
```

```javascript
// stores/auth.js
export const useAuthStore = defineStore('auth', () => {
  const token = ref('')

  return { token }
}, {
  persist: {
    key: 'auth-token',         // localStorage 键名
    storage: localStorage,     // 存储方式，也可用 sessionStorage
    pick: ['token'],           // 仅持久化指定字段
  }
})
```

## 小结

- Pinia 以 `defineStore` 创建 Store，Setup Store 风格更贴近 Composition API
- `storeToRefs` 解构 state/getters 保持响应式，actions 直接解构
- `$patch` 批量修改，`$subscribe` 订阅变化，`$reset` 重置
- Actions 直接支持 async/await，无需额外的 mutation 概念
- 相比 Vuex，Pinia 更轻量、类型友好、API 更简洁
