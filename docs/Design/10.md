# 图标设计原理与工具

图标是 UI 中的微型视觉语言。一个好的图标集能让用户在不阅读文字的情况下快速识别功能和操作——搜索放大镜、购物车、心形收藏、齿轮设置，这些符号已经形成了跨文化、跨年龄段的通用视觉词汇。设计图标可能看起来简单（不就是画个小房子吗），但要做好一套专业级别的图标集，需要遵循一系列经过图形科学验证的原则。

## 图标设计的五项核心原则

### 1. 简洁（Simple）

图标应只包含传达核心含义所必需的元素。一个好的检验方法是：画完草图后，尝试再删掉一个元素，如果含义仍然清晰，那个元素就是多余的。商业图标的复杂度上限通常是"三笔以内"或 3-5 个核心形状。

### 2. 可辨识（Recognizable）

图标必须让用户在 0.1 秒内认出它代表什么。这意味着优先使用用户已经熟悉的通用隐喻（信封 = 邮件，而非"信鸽"），避免过于抽象或文化特定的图形。如果在内部评审中需要向同事解释这个图标代表什么，你就需要重新画。

### 3. 统一描边（Consistent Stroke）

在一套图标系统中，所有图标使用相同的描边宽度（1.5px 或 2px 是 UI 图标最常用的选择）。这套系统中的图标要么全部使用描边风格（Stroke Style），要么全部使用填充风格（Fill Style），不允许混用。描边风格更适合功能型 UI（工具图标、导航图标），填充风格更适合状态型元素（选中态、品牌标识）。

### 4. 视觉均衡（Optical Balance）

数学居中不等于视觉居中。一个三角形在数学校准的方框中看起来会偏下，需要人工向上微调；一个圆形在方框中会在垂直方向视觉上偏短，需要略微超出边界。这种"看起来对"的校准称为**光学修正**，是区分业余和专业图标的关键标志。

### 5. 一致的细节水平

整套图标的细节量应保持在同一水平。如果一个图标用了 7 个描边段来表达，另一个图标不应仅有 2 段。细节密度的一致性让图标集看起 来像一个家族，而非来自不同设计师的拼凑集合。

## 图标网格系统

专业图标通常在一个 24x24dp 的基准画布上设计，其中：

- **有效绘制区域（Live Area）**：20x20dp 的内区，图标的主体图形应该在此范围内绘制。
- **修剪区域（Trim Area）**：2dp 的外围缓冲区，仅图形的末端延伸部分可以进入此区域（用于视觉校准）。
- **关键线形状（Keylines）**：正方形图形（如复选框、停止符号）通常撑满到 18x18dp；圆形图形（如用户头像、录制按钮）撑满到 20x20dp（因为圆在方形中的视觉面积感知较小，需要稍微扩大）；横纵长的矩形（如文件夹、文档）宽度到 20dp，高度到 16-18dp。

这套网格体系来源于 Google 的 Material Design Icon Guidelines，是目前业界最广泛采用的图标基线标准。

## 描边 vs 填充：何时使用哪种

描边图标（Outline/Stroke style）占据 UI 场景的绝大多数——导航栏图标、工具栏图标、列表项操作图标。它们视觉重量轻，不喧宾夺主，适合在密集的信息环境中作为功能标识。

填充图标（Filled/Solid style）用于需要强调视觉权重的场景：当前选中的导航标签、成功/错误状态指示器、带有数值的徽标。在 iOS 的 Tab Bar 和 Material Design 的 Bottom Navigation 中，标准做法是：选中项使用填充图标、未选中项使用描边图标——用风格差异而非颜色来传达选中态，符合无障碍设计的最佳实践。

## 不同尺寸的适配

图标库通常需要覆盖多个尺寸，适应不同场景：

- **16px**：密集 UI（表格内操作、标签内嵌图标、工具栏）。16px 的图标描边不宜超过 1.5px，细节需要大幅简化。
- **20px**：中等密度 UI，部分应用的导航和列表场景。
- **24px**：标准尺寸，Material Design 和大多数 Web UI 的默认图标尺寸。
- **32px+**：大尺寸用途（空状态插图辅助、功能入口图标）。32px 的图标可以包含更多细节，可能需要使用 2.5-3px 的描边。

关键原则是：**图标不能简单地等比缩放**。将 24px 图标缩小到 16px 时，描边会变细到几乎不可见；放大到 32px 时，描边会显得粗糙。每种主要尺寸的图标都需要一条专门的、针对该尺寸调整了细节和描边粗细的变体路径。

## Figma 中的图标绘制工具

在 Figma 中绘制图标的工具组合包括：

### 钢笔工具（Pen Tool，P）

用于贝塞尔曲线绘制，是图标设计最核心的工具。绘制时注意锚点（Anchor Points）的数量和位置——尽量使用最少的锚点来完成图形，多余的锚点会让曲线不够平滑，后期修改也困难。

### 布尔运算（Boolean Operations）

Figma 的布尔运算可以将多个简单形状组合为复杂图形：Union（合并，用于组合多个形状为一个图形）、Subtract（减去，用一个形状从另一个中切掉一块）、Intersect（相交，只保留重叠区域）、Exclude（排除，保留非重叠区域）。例如，云朵图标可以由多个圆形 Union 后再用矩形 Subtact 底部来创建。

### 矢量网格对齐

在 Figma Preference 中启用 Snap to Pixel Grid，确保图标路径对齐到像素网格，避免导出后的模糊边缘。绘制图标时 View → Pixel Grid（Ctrl+'）可以显示像素网格覆盖层，有针对性地检查关键边缘是否对齐。

## 导出与优化

图标的标准导出格式是 **SVG（可缩放矢量图形）**。SVG 本质上是 XML 代码，体积小、任意缩放不失真、可用 CSS 控制颜色，是最理想的 Web 图标交付格式。但 Figma 默认导出的 SVG 包含大量冗余信息（Figma 专有属性、多余的图层组等），在生产环境使用前，建议通过 **SVGO**（SVG Optimizer）工具进行压缩优化，一般可减少 30-60% 的文件体积。Figma 中有一个名为 "SVG Export" 的社区插件可以直接在 Figma 中完成 SVGO 压缩。

## 图标字体 vs SVG：如何选择

图标字体（如 Font Awesome、Material Icons）通过字体文件加载，使用 CSS class 调用。优点是一个文件包含所有图标、通过 `font-size` 和 `color` 即可调整大小和颜色、兼容旧浏览器。缺点是图标只有单一颜色、在部分浏览器中渲染不一致、每个图标需要被单独映射。

SVG 的优势是多色支持、独立的 DOM 节点、更好的屏幕阅读器支持、可以被精细控制和动画化。在现代前端开发中，SVG 已经成为主流选择，图标字体仅在一些旧项目或对兼容性要求极高的场景中使用。
