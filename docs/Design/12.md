# Figma 原型 (Prototype) 交互连线

原型是连接静态界面和真实用户体验之间的桥梁。一份精美的视觉稿只能展示"界面看起来如何"，而交互原型能展示"界面用起来如何"——点击按钮后会发生什么、页面之间如何过渡、滚动时页面有什么变化。Figma 的原型功能让设计师无需离开设计工具就能创建可交互的演示，并分享给团队成员或用户进行测试。

## 切换到原型模式

Figma 编辑器顶部右上角有三个标签页：Design（默认）、Prototype、Dev Mode。点击 Prototype 标签或使用快捷键 **Shift+E** 进入原型模式。在原型模式下，画布上的 Frame 周围会出现交互锚点（小白圆点），你可以从这些锚点拖出连线（俗称"noodle"）来定义交互关系。

## 创建第一个交互连接

交互连接的基本操作流程如下：

1. 在画布上确保你有至少两个 Frame（比如"列表页"和"详情页"）。
2. 进入 Prototype 模式，在"列表页"Frame 中，点击你想作为交互热区的元素（比如一个"了解更多"按钮或一张卡片图）。
3. 从该元素边缘的锚点拖出一条连线，指向目标 Frame（"详情页"）。
4. 松开鼠标后，Figma 会弹出交互设置面板，让你配置触发方式、动作类型和动画效果。

连线箭头指向的方向就是交互的前进方向。一个元素可以有多条连线（点击去 A 页，长按去 B 页）。

## 触发器类型（Trigger）

Figma 支持丰富的触发条件，适配桌面端和移动端的不同交互场景：

- **On Click（点击）**：最常用的交互触发器。用户点击或轻触元素时触发。
- **On Hover（悬停）**：鼠标移入时触发。仅在桌面端有效，移动端不适用。
- **While Pressing（按下期间）**：用户按住元素时保持目标状态，松手后恢复。适合展示按钮的按下态。
- **While Hovering（悬停期间）**：鼠标悬停期间保持目标状态，移开后恢复。适合 Tooltip（提示框）信息展示。
- **Mouse Enter / Mouse Leave**：鼠标进入和离开时分别触发，适合需要进入显 示、离开隐藏的复杂 Tooltip。
- **After Delay（延迟触发）**：页面打开后自动在指定时间（毫秒）后触发。适合自动播放的引导页、闪屏到主页的自动跳转。
- **On Drag（拖拽）**：用户在元素上执行拖拽手势时触发，配合 Smart Animate 可以构建拖拽交互原型。

## 动作类型（Action）

触发后可以执行的动作包括：

- **Navigate to（导航至）**：最常用的动作，从当前 Frame 切换到目标 Frame。这是页面间跳转的标准选择。
- **Open Overlay（打开叠加层）**：在目标 Frame 当前页面上以覆盖层形式打开目标 Frame，适合弹窗、底部弹出菜单、下拉菜单等"不离开当前页"的轻量内容。
- **Swap Overlay（交换叠加层）**：替换当前已打开的 Overlay 为另一个 Frame。适合在 Overlay 场景内部的"下一步"操作。
- **Back（返回）**：关闭当前 Frame 并返回上一个浏览过的 Frame，模拟浏览器/系统级的返回手势。
- **Close Overlay（关闭叠加层）**：关闭当前 Overlay。弹窗的关闭按钮应绑定此动作。
- **Open Link（打开链接）**：在用户浏览器中打开 URL。适合原型中跳转到外部网页。
- **Set Variable（设置变量）**：Figma 的高级功能，在原型中修改变量值，适合需要记录状态的原型（购物车计数、暗黑模式切换）。
- **Conditional（条件判断）**：根据变量值的不同条件决定执行什么动作，用于构建有状态的交互逻辑流。

## Smart Animate：让过渡更自然

Smart Animate 是 Figma 原型中最强大的动画功能。它的工作方式是：当两个 Frame 之间存在同名的图层时，Figma 自动识别这些图层在位置、大小、旋转角度、透明度等方面的差异，并生成平滑的过渡动画。

例如，你想做一个"点击卡片后卡片放大展开为详情页"的效果：

1. 在列表页 Frame 中，给产品图片图层命名为 "product-image"。
2. 在详情页 Frame 中，将对应的产品图片图层也命名为 "product-image"（**图层名必须完全一致**）。
3. 在列表页设置交互连线：On Click → Navigate to 详情页 → Smart Animate，Duration 300ms，Easing 设为 Ease Out。

Figma 会在用户点击后自动识别两个 "product-image" 图层的差异（位置从列表位置移动到详情页顶部、尺寸从缩略图变为大图），并平滑过渡。

Smart Animate 的关键规则：同名的图层匹配；不同名的图层按简单淡入淡出处理（Dis dissolve）；Frame 背景色变化也会被平滑过渡；不透明度、旋转角度、位置、尺寸差异都能被动画化。

## 缓动曲线与动画设置

在交互设置面板的 Animation 下拉菜单中，你可以选择过渡动画类型和参数：

- **Instant（即时）**：无过渡，立即切换。用于"立即响应"的交互如 Tab 切换。
- **Dissolve（溶解）**：简单的透明度过渡。用于图层名不匹配的 Smart Animate 场景。
- **Smart Animate**：如上所述，智能过渡。
- **Move In / Move Out（推入/推出）**：目标 Frame 从指定方向滑动进入（或当前页面滑出），带有 Push（推动）或 On Top（覆盖）两种模式。
- **Slide In / Slide Out（滑入/滑出）**：类似 Move，但方向感和力度感不同。

Easing 曲线的选择直接影响过渡的质感：
- **Ease Out**：适合进入动画（内容滑入页面），让进入在最后阶段自然减速。
- **Ease In**：适合退出动画（内容离开页面）。
- **Ease In and Out**：中性过渡，在页面转场中最为常用。
- **Spring**：带有弹性的过渡，适合需要愉悦感或品牌调性的交互。
- **Custom（自定义）**：通过贝塞尔曲线控制点手动定义缓动曲线，适合对动效有精确要求的设计师。

## 滚动与固定定位

在原型模式中，你还可以配置 Frame 的滚动行为：

- **Scroll Position**：设置 Overflow scrolling（设置 Frame 内内容高度超出其可见区域时的滚动行为），选项包括 Vertical、Horizontal、Both、None。
- **Fix Position When Scrolling**：勾选后，该元素固定在其屏幕位置，不随 Frame 内容滚动。典型用途是固定顶部导航栏、固定底部操作按钮、固定"回到顶部"浮动按钮。

## 预览原型 & 分享

点击顶部工具栏右侧的播放按钮（Present 图标）或使用快捷键 **Cmd/Ctrl+Option+Enter**，在一个新窗口中启动原型预览。在预览模式下，你可以真实地点击、悬停、拖拽，体验完整的交互流。

要分享原型给其他人，点击右上角的 **Share** 按钮，生成一个链接。打开链接的人在浏览器中就能体验原型，无需 Figma 账号或登录（取决于你的分享权限设置）。这对于向客户演示设计方案、向用户进行远程可用性测试或收集利益相关者反馈都非常实用。

## 实战：构建一个三屏引导页流程

下面通过一个完整示例，用 Figma 原型构建一个 App 欢迎引导页：

1. **创建三个引导页 Frame**：均为 iPhone 14 尺寸（390x844）。分别命名为 Onboarding-1、Onboarding-2、Onboarding-3。
2. **设计引导页内容**：每页包含一张居中的插画（layer 名称均设为 "illustration"）、标题文字（"欢迎使用 XX" / "简单三步" / "开始探索"）、三个指示点（当前页为实心，其余为空心）。
3. **创建 CTA 按钮**：前两页的按钮文字为 "下一步"，第三页为 "开始使用"。将按钮图层嵌套在一个名为 "cta-button" 的 Frame 内。
4. **设置交互连线**：
   - Onboarding-1 的"下一步"按钮 → On Click → Navigate to → Onboarding-2 → Smart Animate，Ease Out，300ms。
   - Onboarding-2 的"下一步"按钮 → On Click → Navigate to → Onboarding-3 → Smart Animate，Ease Out，300ms。
   - Onboarding-3 的"开始使用"按钮 → On Click → Navigate to → 首页 Frame → Smart Animate，Ease Out，300ms。
5. **添加跳过按钮**：在每个引导页右上角添加"跳过"文字链接，On Click → Navigate to → 首页 Frame，Instant（立即跳转）。
6. **预览测试**：按播放按钮进入预览模式，依次点击"下一步"，确认动画流畅过渡；点击"跳过"，确认立即跳转到首页。
