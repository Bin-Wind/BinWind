# UI 中的动效设计原则

动效不是装饰。在 UI 设计中，动效承担着引导注意力、建立空间关系、提供操作反馈和塑造品牌个性的功能。好的动效用户不会注意到，但能感受到产品更流畅、更有生命力。差的动效则让人烦躁、迷失，甚至阻碍任务的完成。

## 动效的四大目的

每一个动效都应该服务于以下至少一个目的。**引导注意力 (Guide Attention)**：新元素出现时用微动效引导视线，比如通知徽章的红点弹跳、表单错误时输入框的轻微抖动。动效利用人类视觉系统对运动的天然敏感性——我们的周边视觉对运动极其敏感，这是进化塑造的生存本能——快速将用户注意力带到变化处。你不需要让用户读文字才知道哪里变了，动效直接引导视线。

**展示空间关系 (Show Spatial Relationships)**：当界面元素从一个位置移动到另一个位置，平滑的过渡帮助用户理解"这是同一个东西，只是换了位置"。列表项的添加和删除动画、模态窗口从触发按钮处展开——这些都是"空间锚定"的典型应用。没有过渡动画，用户看到一个元素瞬间出现在新位置，认知负担是判定"这是刚才那个东西吗？"；有过渡动画，这个判定由视觉系统自动完成。

**提供操作反馈 (Provide Feedback)**：按钮点击的下沉效果、下拉刷新的加载动画、表单提交成功的对勾动画——这些动效让用户知道"你的操作已被系统接收"。没有反馈的界面让人不安——用户不确定操作是否生效，于是反复点击、重复提交。动效反馈的成本极低，但消除的焦虑极大。

**塑造品牌个性 (Add Personality)**：Logo 动画、页面加载时的趣味动效、空状态的插图动画。这是动效最具创造性的应用，但要控制频率和时长。品牌动效是调味料，不是主菜——第二次见到时应该是"嗯，挺有意思的"，而不是"又来？"。同样的品牌动效每次加载都播放一遍，忠诚用户在第 50 次登录时就不是觉得有趣而是厌烦了。

## 动效时长的黄金法则

时长是关键——太快显得突兀，太慢让人烦躁。**微交互 (Micro-interactions)**：100-200ms，适用于按钮 hover、图标切换、开关 toggle。在这个时间范围内，用户感知不到延迟，但变化足以被察觉。低于 100ms 的变化人眼几乎无法分辨是否为动效还是瞬间切换。

**小幅度动效 (Small animations)**：200-500ms，适用于菜单展开或收起、tooltip 出现、下拉选项展开、卡片翻转。这是最常见也最通用的动效类型。绝大多数 UI 动效都在这个范围内。如果拿不准，默认选 300ms——这是一个已经被广泛验证的舒适值。

**大幅面动效 (Large transitions)**：500-1000ms，适用于页面切换、路由过渡、模态窗口打开、onboarding 引导。大幅面动效涉及更多元素和更大距离的位移，需要更长的时间来保持视觉舒适。但 1000ms 是上限——超过 1 秒的等待会让用户开始焦虑。

**超过 1 秒的动效**：谨慎使用，通常仅用于复杂的品牌动画或数据可视化中的重大变化。超过 3 秒需要提供跳过按钮或进度指示。经验法则：动效不是为了被"看到"，而是为了被"感受"。如果用户注意到了你的动效而不是内容，动效就太慢或太花哨了。

## 缓动曲线：动效的灵魂

缓动 (Easing) 让运动感觉自然而非机械。**ease-out (减速)**：快速开始然后缓慢结束，适用于元素进入屏幕（淡入、滑入）。物理世界中物体不会瞬间停止——推一个箱子，放手后它有惯性但最终因摩擦力而减速到零。ease-out 模拟了这个自然减速过程。几乎所有进入型动效都建议使用 ease-out，因为它让元素"优雅地停住"而不是"撞墙停下"。

**ease-in (加速)**：缓慢开始然后快速结束，适用于元素离开屏幕（淡出、滑出）。物理类比：物体落下时加速。视觉心理：物体离你而去时，你不需要跟踪它的轨迹，干净利落地加速离开是最好的。退出动效用 ease-in，元素会感觉像是"决然地离开"而非"犹豫地走"。

**ease-in-out (先加速后减速)**：适用于元素在屏幕内的位置变换（如拖拽排序后各元素归位、列表重排）。起止两端平滑过渡，中间加速。严格来说，ease-in-out 不适用于进入和退出动效——进入用 ease-out，退出用 ease-in，不要混用。线性动效 (linear) 在 UI 动效中几乎不应该使用——自然界中不存在匀速运动，线性动效给人机械感。

## Material Design 缓动标准

Google 定义了四种标准缓动曲线，覆盖绝大多数 UI 动效场景。**Standard (标准缓动)**：`cubic-bezier(0.4, 0.0, 0.2, 1)`，最通用的缓动，适用于大部分 UI 元素的动画。特征是快速加速、大幅减速，让动画感觉自然且响应迅速。如果你不确定用哪种缓动，就用 Standard。

**Emphasized (强调缓动)**：`cubic-bezier(0.0, 0.0, 0.2, 1)`，比标准缓动更显著的减速阶段，让运动结束时更加柔和。适用于需要用户特别注意的动画——如 FAB 按钮展开为多个操作按钮、新页面从右侧滑入。强调缓动给元素一种"郑重地抵达"的感觉。

**Decelerated (减速缓动)**：`cubic-bezier(0.0, 0.0, 0.0, 1)`，全程减速，没有初始加速。适用于元素从屏幕外进入——如底部滑入的 bottom sheet、从屏幕外飞入的通知。因为它没有初始加速，元素仿佛以全速从屏幕外冲进来然后优雅地停下。

**Accelerated (加速缓动)**：`cubic-bezier(0.4, 0.0, 1.0, 1.0)`，全程加速，没有减速。适用于元素退出屏幕——元素加速离开，仿佛被"吸走"。配合 Decelerated 进入，形成"快速出、优雅入"的节奏。

在 CSS 中，`transition-timing-function` 和 `animation-timing-function` 可以使用 cubic-bezier 自定义曲线。在 Figma 中，Smart Animate 也支持自定义缓动曲线调节——设计阶段定义的缓动参数可以直接传递给开发。

## 编排与节奏

当多个元素同时运动时，编排决定观感。**交错动画 (Stagger)**：子元素依次出现，每个延迟 30-80ms，比所有元素同时出现更有层次感和方向性。列表项的渐入是经典场景——每个 item 间隔约 50ms，用户的视线自然跟随动画流向从上到下扫描列表。**父子关系**：父容器先展开/移动，子元素紧随其后（延迟 50-100ms）出现。顺序为：容器先到 → 内容在我里面。这帮助用户自动理解"这个标题属于这个卡片"的层级关系。

**优先级**：重要的元素先动，次要的后动。不要让装饰元素抢在内容前面动起来——用户等待看核心信息，结果先看到一个背景色逐渐展开的动画然后才是内容，这种编排顺序是错误的。**组动画**：当多个元素作为一个逻辑整体变化时，它们应该同步运动而非交错。例如，卡片内部的标题和描述应该作为一个组整体淡入——如果标题先出现、描述后出现，用户会认为它们是两个独立的信息单元。

## 何时使用动效 / 何时不用

动效不是越多越好。**应该用动效的场景**：页面过渡（让用户理解从 A 到了 B）、导航变化（告知用户在空间中的位置转移）、加载状态（减少感知等待时间，骨架屏动画已被证明能让等待感觉缩短 20-30%）、数据可视化变化（帮助用户跟踪数据点的移动和变换）、新手引导（吸引注意力到关键功能或首次使用的核心操作）。

**不应该用动效的场景**：每次滚动都触发的视差效果（性能杀手，滚动帧率从 60fps 跌到 30fps，且动画持续触发会让部分用户眩晕）、纯装饰性的循环动画（持续分散注意力，用户想读内容但旁边有个东西一直在动）、拖慢任务完成的复杂过渡（用户想快速操作时动效就是阻力，比如设置页的每一项之间都有 200ms 的动画延迟，用户连续点击时会累积出明显的卡顿感）、信息密集页面中的持续动画（如数据大屏中的非关键装饰动画）。

**尊重用户偏好**：使用 `prefers-reduced-motion` 媒体查询，当用户系统设置了"减少动画"时，禁用或简化所有非必要的动效。这不是可选优化，而是无障碍要求。实现方法：`@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }`。

核心原则：如果你不确定某个动效是否有必要，就问自己——这个动效帮助用户理解了什么？如果回答不出一句完整的话，大概率这个动效是多余的装饰。
