# Figma 组件 (Components) 与变体 (Variants) 实战

组件（Component）是 Figma 实现设计系统复用的核心机制。当你需要在整个产品中保持按钮、输入框、导航栏等元素的一致性时，直接在每一页复制粘贴图层会让你陷入维护噩梦——修改一个按钮样式需要手动改几十处。组件的出现正是为了解决这个问题：一次定义，全局复用，一处修改，处处更新。

## 理解组件与实例的关系

组件在 Figma 中遵循**主从（Master-Master/Instance）**模式。你首先创建一个主组件（Master Component），它就像一个蓝本。当你从主组件复制出新副本时，这些副本被称为实例（Instance）。实例与主组件之间保持着链接关系：修改主组件的属性（如背景色、文字样式、圆角），所有实例会同步更新；但在实例上你可以单独覆盖文字内容或局部样式，而不影响其他实例。

这种机制让你同时获得了"全局一致性"和"局部灵活性"——按钮的圆角和阴影统一由主组件控制，但每个按钮的文字可以不同。

## 创建组件

选中你希望做成组件的图层（可以是一个简单的形状，也可以是包含多个图层的组合），然后执行以下任一操作：

- 右键点击选中的图层，选择 **Create Component**
- 使用快捷键 **Cmd/Ctrl+Alt+K**
- 点击顶部工具栏中间位置的 **Create Component** 图标（四个小菱形）

创建后，在图层面板中，组件会以实心紫色菱形图标标示。主组件通常存放在专门的页面（Page）中，比如创建一个名为 "Components" 的页面集中管理。

## 实例的覆盖与重置

将主组件拖入 Frame 后，它就是一个实例。你可以在实例上覆盖以下属性：文字内容、颜色、可见性、尺寸等。如果覆盖后想恢复为主组件的默认设置，右键实例选择 **Reset Instance** 或在属性面板中点击对应属性旁的还原按钮。

## 推送变更

当你修改主组件后，所有实例会自动更新，无需手动操作。这是组件系统最大的价值——比如你可以将全局按钮的圆角从 8px 改为 12px，所有页面上的按钮实例立即响应。

## 变体（Variants）：组件的进阶用法

在实际设计中，一个"按钮"远不止一种状态。你可能需要默认态、悬停态（hover）、按下态（pressed）、禁用态（disabled）等等。它们属于同一个组件家族，只是在某些属性上有所区别。Figma 的变体功能让你将这些不同状态组织在一起，作为一个统一的组件集使用。

### 创建变体

将多个同类组件（比如默认按钮、hover 按钮、disabled 按钮）同时选中，右键选择 **Combine as Variants**。Figma 会将它们自动打包成一个组件集（Component Set），以虚线边框标示。切换变体时，只需在实例的属性面板中切换 Variant 属性值即可。

### 变体属性

Figma 支持四种变体属性类型：

- **文本属性（Text Property）**：定义组件内文字内容为可变量，实例使用时可修改。
- **布尔属性（Boolean Property）**：控制某个图层的显示/隐藏，适合图标是否需要显示等场景。
- **实例交换（Instance Swap）**：允许在实例中将嵌套的子组件替换为其他组件，常用于图标切换。
- **变体属性（Variant Property）**：定义变体之间的区别属性，如 State、Size、Type 等。

### 命名规约

变体属性的值由 Figma 根据图层命名自动推断。命名的标准格式为 `ComponentName/Property1=Value1, Property2=Value2`。举例：

```
Button/Primary/Default
Button/Primary/Hover
Button/Primary/Disabled
Button/Secondary/Default
Button/Secondary/Hover
Button/Secondary/Disabled
```

Figma 会将 `/` 和 `=` 解析为属性分组，自动生成 Variant 和 Size 两个属性维度。

## 组件库：跨文件复用

当你的组件积累到一定数量后，你可能希望在不同项目文件中复用它们。在团队版或企业版 Figma 中，你可以通过 Assets 面板上的 **Publish** 按钮将当前文件发布为团队库。发布后，其他团队成员在任意文件中打开 Assets 面板，就能看到并拖入你发布的组件。主组件更新后，使用方会收到更新通知，可以选择接受更新。

## 实战示例：构建一个完整的按钮组件集

以下步骤带你从零构建一套生产级的按钮组件：

1. **创建基础按钮**：画一个 200x48pt 的矩形，圆角设为 24。添加文字 "按钮"，字号 16，字重 Medium。选中两者，按 Cmd+Alt+K 创建基础主组件。
2. **制作变体副本**：复制该组件四次，分别修改为 Default / Hover / Pressed / Disabled 四种状态。Hover 态调亮 10%，Pressed 态调暗 10%，Disabled 态设置 40% 透明度。
3. **合并为变体集**：选中四个组件，右键 Combine as Variants。Figma 自动识别 State 属性。
4. **添加 Size 维度**：复制整个变体集，分别创建 Large（200x56pt）、Medium（160x40pt）、Small（120x32pt）三个尺寸。调整字体大小和圆角。修改命名加入 Size=Large 等标记，再次组合为统一变体集。
5. **添加图标属性**：在按钮文字左侧添加一个图标组件（一个小箭头或心形），将其设为 Boolean Property——勾选时图标显示，取消时隐藏。
6. **测试使用**：新建 Frame，从 Assets 面板拖入按钮变体集，通过属性面板切换 State、Size 和图标显示，验证所有变体功能正常。

## 最佳实践建议

组件命名要清晰直观，避免随意命名如 "Rectangle 143"。组件结构尽量扁平，减少不必要的嵌套层级。为组件添加描述（在属性面板的 Description 字段），帮助团队成员理解用途。定期审查和清理未使用的组件，保持组件库的精简和高质量。
