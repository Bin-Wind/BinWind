# 设计趋势 2025-2026

趋势不是用来追的，是用来理解的。理解趋势背后的驱动力——技术变革、用户期待的变化、行业成熟度的提升——才能在你自己的设计实践中做出明智的判断：哪些趋势值得引入、哪些只是过眼云烟。

## AI-Native 界面：交互范式的转变

2025 年是 AI 真正改变 UI 范式的一年:

- **对话式 UI (Conversational UI)**: ChatGPT 开创的聊天式交互正在渗入各种产品。聊天不仅是对话——它成为了一种新的信息架构。用户不再通过菜单层级寻找功能；他们直接描述需求，AI 理解意图并执行。这对传统的信息架构和导航模式是根本性挑战。
- **Copilot 式交互**: 不是替代用户操作，而是在用户操作的同时提供 AI 辅助。GitHub Copilot 的"你写代码，AI 补全"模式已经迁移到 Microsoft 365 Copilot（Word、Excel、PPT 中的边栏 AI 助手）、Figma AI、Notion AI。核心理念: AI 在工作流中不是中心，而是背景中的协作伙伴。
- **Prompt-as-Interface**: 输入框成为主要交互入口。传统的表单、筛选器、设置面板被一个灵活的 prompt 输入框替代——"帮我找出上周销售额超过 10 万的门店，按地区分组"。这对信息密集型 B 端产品的影响尤其深远。
- **挑战**: 可发现性问题——没有菜单，用户如何知道系统能做什么？AI 能力需要被"教授"给用户（onboarding AI features）。透明度和信任——系统为什么给出这个结果？错误处理——AI 误解用户意图时怎么办？

## Glassmorphism 的进化

毛玻璃效果走向成熟:

- 2020-2022 年的 Glassmorphism 更多是视觉层的探索（半透明卡片 + 背景模糊）。2025-2026 的进化在于: 它不再是纯装饰，而是服务于空间层次和信息分层。
- **分层信息**: 浮动层（菜单、弹窗、提示）使用玻璃效果，与主内容区形成物理层次感。你不是在"看一个模糊的背景"，而是在感知"前面的东西离你更近"。
- **动态玻璃**: 玻璃层的模糊度和颜色随滚动、鼠标位置或系统状态动态变化。适用于仪表盘和音乐播放器等场景。
- **搭配鲜艳背景**: 玻璃效果需要有"内容"来穿透。静态的纯色背景上玻璃效果很乏味。鲜艳的渐变、动态光效、视频背景让 glass 效果出彩。
- **可访问性注意**: 玻璃效果对对比度的影响不可忽略。玻璃层上的文字必须有足够的可读性保证。在 WCAG 合规性方面，玻璃效果叠加在各种背景上的一致性对比度难以保证——需要仔细测试。

## Variable Fonts：一个字体的无数可能

一个字体文件承载无限变化:

- Variable fonts 将字体家族的所有变体（weight, width, slant, optical size）合并到一个文件中。文件体积比多字体文件方案小很多——显著的性能收益，尤其对 Web 设计。
- **设计表现力**: 可以用微小的字宽调整适应不同的容器宽度（响应式排版的新维度）。可以在 hover 动效中平滑过渡字重（`font-weight: 300` → `font-weight: 700` 的平滑动画）。可以对标题做精细的字重微调以达到像素级的视觉完美。
- **主流支持**: 所有现代浏览器均已支持 variable fonts。Google Fonts 提供大量 variable font 选项。Adobe Fonts 也增加了 variable font 库。
- **设计系统中的应用**: 用 variable font 的 weight axis 替代多个独立的字体文件，在 Design Tokens 中用数值而非名称管理字重（`font-weight-regular: 400`, `font-weight-bold: 687`——对，variable font 可以是 687 而不必须是 700）。

## 空间设计：新平台催生新范式

Apple Vision Pro 和 Meta Quest 推动空间设计进入主流:

- **深度作为信息层**: 在空间界面中，Z 轴不是装饰——是功能。更重要的内容离用户更近。通知出现在远处 vs 近处的含义不同。
- **从平面网格到空间布局**: 传统 UI 的 X-Y 网格在空间中被 X-Y-Z 体积取代。元素可以排列在用户周围——左右、上下、前后。这不仅是视觉变化，是交互模型的根本改变。
- **手势 + 眼动追踪**: Vision Pro 的眼动 + 手指捏合交互模式定义了空间计算的基础交互范式。看什么就选择什么，无需鼠标、触控、手柄。
- **对传统屏幕设计的启示**: 空间设计探索——如用半透明和景深表达层级、用空间位置而非仅用颜色表达状态——开始反馈影响传统的 2D 屏幕设计。深度和透明度的运用在 2D 屏幕上也变得更加大胆和精细化。

## 微交互无处不在

微交互从锦上添花变为基本要求:

- 2025-2026 的趋势不是"做微交互"，而是"微交互是默认项"。就像 10 年前的响应式设计从"额外功能"变成了"基本要求"，微交互正在经历同样的轨迹。
- **表现**: 按钮的 hover/active 状态动画、卡片的 hover 悬浮、滚动触发的渐进显示、页面切换过渡、数据更新的平滑变化——这些不再被认为是锦上添花，而是"合格 UI 的基本组成部分"。
- **驱动因素**: CSS 动画和 Web 动画 API 的成熟、Lottie 动画的广泛采用、Figma 的 Smart Animate 让设计师可以精确设计动效参数（而非口头描述给开发者"这里加个过渡"）、用户对静态界面的容忍度降低。
- **原则不变**: 动效应服务于功能，而非纯装饰。非必要的动效不仅浪费开发资源，还会触发 prefers-reduced-motion 用户的不适。

## Accessibility-First：从事后补救到系统内置

设计文化中最重要的转变:

- 2025 年，将无障碍视为"最后的检查清单项"的公司正在快速落后。Accessibility-first 意味着从设计的第一天起——用户研究、信息架构、线框图、视觉设计——a11y 就是不可分割的一部分。
- **法律推动**: 欧盟 European Accessibility Act (EAA) 2025 年 6 月生效，对在欧盟运营的数字产品提出强制性的无障碍要求。美国 ADA 数字无障碍诉讼持续增长。法律不再是遥远的威胁，而是当下的合规要求。
- **工具的成熟**: Figma 的无障碍插件（Stark、A11y Focus Order）、axe DevTools 自动化检测的准确率提升、CI/CD 中嵌入可访问性检查——工具链的完善让 a11y 不再是"额外工作"而是"标准流程"。
- **商业价值**: 无障碍设计改善所有人的体验——curb-cut effect 的反复验证。为视障用户设计的语音交互也方便了正在开车的用户。为键盘用户设计的完整键盘导航也让高级用户效率大增。
- **设计系统层面的 a11y**: 不再是在每个组件中手动确保可访问性。而是将 a11y 标准和测试写入 Design System 的组件定义中——"这个 Button 组件已经内置了 focus visible 样式，48px 最小触摸目标，语义化 role='button'"——设计师使用组件时自动继承可访问性。

## Bento Grids：模块化卡片布局

受 Apple 发布会启发，被互联网广泛采用:

- Bento 布局: 不同大小、比例的卡片模块像日本的便当盒（Bento Box）一样，划分出大小不一的格子空间
- **特征**: 没有严格对齐的网格。大模块和小模块共存——一个占据 2x2 空间的主功能卡片，周围环绕着 1x1 的小卡片。视觉上有节奏感和层次感。
- **适用场景**: Dashboard、功能概览页、产品展示页（Apple 官网的产品介绍是 Bento 布局的典范）、个人主页。不适合数据密度高、需要快速扫描对比的表格场景。
- **设计要点**: 大小对比产生节奏——如果一个 Bento 布局中所有卡片大小相同，它就不是 Bento。在大小不同的模块之间建立视觉群组关系——内容相关的小卡片应该视觉上邻近。在响应式场景中，Bento 网格会坍塌为线性布局，需要仔细规划移动端的卡片顺序。

## 复古未来主义：像素与霓虹的复兴

一个小众但有影响力的反潮流:

- 在 AI 生成的"完美、光滑、渐变卡片"同质化美学泛滥的背景下，一些产品故意走向反方向: 粗糙、手工感、低技术感的视觉风格。
- **表现形式**: 像素字体（Pixel Font）回潮、高饱和霓虹色在深色背景上的使用、Brutalism（野兽派）Web 设计的持续小众流行——故意暴露"丑陋"的系统默认控件、单色背景、大号无样式字体。
- **吸引力**: 在千篇一律的圆角卡片和 SF Pro 字体海洋中，这种风格立刻变得"独特"。对于小众创意工具、独立游戏、音乐人网站、艺术家作品集——它传达了"我们不是另一个 SaaS 创业公司"的信号。
- **风险**: 这种风格的主要风险是可读性和可用性。如果形式压倒功能，让用户找不到他们想要的内容，那就走火入魔了。复古未来主义在"小众酷"和"真难用"之间的界限很薄。

## 趋势判断框架：用，还是不用？

如何评估趋势是否适合你的产品:

- **是否解决了真实的用户问题？** 如果 AI-native 界面让你的产品更好用，这是一个应该采纳的趋势。如果只是因为"别人都这么做"——再等等。
- **是否符合你的品牌调性？** Bento 布局适合展示型产品（消费者 App、官网），不适合工具型产品（项目管理、代码编辑器）。趋势需要适配品牌，不是品牌适配趋势。
- **团队和能力是否支持？** Micro-interactions 需要开发团队对 CSS 动画和 Web 动画 API 有把握。如果团队在赶 deadline，宁可把不需要动效的基础交互做到位，也不要勉强做一堆 bug 动画。
- **可访问性影响？** Glassmorphism 和霓虹色风格在对比度上可能不达标。采纳视觉趋势前，确认你的产品对所有用户仍然可用。
- **趋势的生命周期判断**: 微交互和 a11y-first 是结构性变化——长期存在。Bento grids 和 retro-futurism 可能更多是流行周期——来得快也可能去得快。对前者，认真投入。对后者，保持关注但慎重采用到核心产品。

## 最重要的趋势

对用户需求的理解永不过时。工具在变、审美在变、平台在变——但理解用户、解决问题、创造价值，是永恒不变的设计核心。追逐每个新趋势不会让你成为更好的设计师；深入理解用户、做出有影响力的设计决策，才会。
