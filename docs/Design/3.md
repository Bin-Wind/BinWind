# Auto Layout 自动布局完全指南

Auto Layout 是 Figma 最具变革意义的功能之一，它将"画"界面的思维方式转变为"构建"界面。传统设计师习惯于逐一摆放元素，当内容变化（比如文字变长、列表项增加）时，需要手动一一调整位置。Auto Layout 让 Frame 具备了类似 CSS Flexbox 的动态布局能力——元素会随着内容变化自动调整大小和位置。

## 什么是 Auto Layout

简单来说，Auto Layout 是一个可以添加到任意 Frame 上的动态布局属性。启用后，Frame 内的所有子元素会按照设定的方向（水平或垂直）、间距和内边距自动排列。当你修改文字内容、增删图层或改变元素尺寸时，Frame 的宽高会自动重新计算，无需手动调整。

## 创建 Auto Layout Frame

选中一个 Frame 或一组图层后，有以下几种方式添 加 Auto Layout：

- 右键选择 **Add Auto Layout**
- 使用快捷键 **Shift+A**
- 在右侧属性面板中点击 Auto Layout 区域旁的 **+** 按钮

添加后你会注意到几个明显变化：图层面板中 Frame 图标从常规方形变为带有两条横杠的特殊图标；画布中的 Frame 会出现粉色实线（实线间距和内边距）；属性面板中出现了 Auto Layout 专属控制区。

## 关键属性详解

### 方向（Direction）

决定了子元素是沿着水平轴（Horizontal）还是垂直轴（Vertical）排列。常见的用法是：导航栏用水平方向，卡片列表用垂直方向。

### 间距（Gap）

子元素之间的间隔距离，单位为 px。这个数值可以输入负值（但通常不建议），也可以通过直接在画布上拖拽粉红色间距手柄来调整。

### 内边距（Padding）

Frame 边缘到最近子元素的距离，分上下左右四个方向。点击内边距数值旁的"对齐"图标，可以独立设置四个方向的值（类似于 CSS 的 padding-top、padding-right、padding-bottom、padding-left）。

### 尺寸模式（Resizing）

这是 Auto Layout 最关键也最容易被误解的部分。每个子元素都有宽度和高度两个维度的尺寸模式：

- **Hug contents（自适应内容）**：Frame 根据内部子元素的尺寸自动调整自身尺寸。当你修改子元素内容时，Frame 会撑大或缩小以适应。
- **Fill container（填充容器）**：子元素填满父 Frame 的可用空间。如果多个子元素同时设为 Fill，它们会等分剩余空间。
- **Fixed（固定尺寸）**：指定固定像素值，不随内容变化。

一个常见的卡片设计模式是：卡片的宽度 Fixed、高度 Hug contents（随内容增减而扩展），内部的标题和描述文字宽度 Fill container、高度 Hug contents。

### 对齐（Alignment）

当 Frame 尺寸大于子元素时，子元素在主轴方向和交叉轴方向上的对齐方式。水平和垂直各提供三个选项：起始（Start）、居中（Center）、结束（End）。对比 CSS Flexbox：主轴对齐对应 justify-content，交叉轴对齐对应 align-items。

## 嵌套 Auto Layout

复杂的 UI 组件几乎总是由多层 Auto Layout 嵌套构建。以一张产品卡片为例，最外层是一个垂直排布的 Auto Layout Frame，从上到下依次为：图片 → 文字区域 → 操作按钮。文字区域内部又是一个垂直 Auto Layout，包含标题（水平居左）、描述（水平居左）、标签行（水平 Auto Layout，内含多个标签）。

嵌套 Auto Layout 的设计思路是：**每一次你看到"一个元素内部有一组相关元素"，就应当使用一个 Auto Layout 来管理它们的排列关系。**

## 绝对定位（Absolute Position）

如何在 Auto Layout 内部让某个元素脱离自动排列流？选中子元素，在右侧属性面板的 Auto Layout 区域点击 **Absolute Position** 按钮。该元素将脱离排列流，你可以自由拖动到任意位置，在属性面板中通过约束（Constraints）设置它相对于父 Frame 的定位。绝对定位最典型的使用场景包括：卡片右上角的收藏图标、头像上的在线状态小圆点、购物车图标上的数量角标。

## 实战：构建一张响应式卡片

以下步骤带你通过嵌套 Auto Layout 构建一张响应式产品卡片：

1. **创建外层 Frame**：画一个 320x400pt 的 Frame，添加 Auto Layout（Shift+A），Direction 设为 Vertical，Padding 全部 0，Gap 设为 0。
2. **添加图片区域**：在 Frame 内插入一个 320x200pt 的矩形作为图片占位，Resizing 宽度设为 Fill container，高度 Fixed 200。圆角 12 仅作用于上半部分（在 Corner Radius 中取消链接，独立设置顶部圆角）。
3. **添加文字区域**：在图片下方插入一个 Frame，添加 Auto Layout，Direction Vertical，Gap 8，Padding 16。Resizing 宽度 Fill container，高度 Hug contents。
4. **填充文字内容**：在文字区域的 Frame 内依次添加标题文字（字号 18，字重 Bold）、描述文字（字号 14，颜色 #666666，适合多行）、价格文字（字号 20，字重 Bold，颜色主色）。
5. **添加底部按钮**：在文字区域下方插入一个 Frame，添加 Auto Layout，Direction Vertical，Padding 16。在其中放置一个按钮——用 Auto Layout 构建：矩形（填充主色，宽度 Fill container，圆角 8）+ 文字 "加入购物车"（字号 16，颜色 #FFFFFF）。按钮的 Padding 设为 Vertical 12，Horizontal 0，文字水平居中。
6. **验证响应性**：选中整个卡片，修改描述文字为更长内容，观察卡片高度是否自动扩展；拖拽卡片宽度从 320 改为 400，检查图片和按钮是否自适应填充。

## 常见陷阱与解决方案

Auto Layout 的常见问题包括：忘记设置 Resizing 导致元素尺寸异常、嵌套层级过深影响性能、绝对定位元素在不同尺寸下错位。建议每次添加 Auto Layout 后立即检查 Resizing 设置，命名 Auto Layout 相关 Frame 时尽量体现其布局功能（如 "Card Body"、"Button Row"），方便后续维护。
