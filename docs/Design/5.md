# 色彩理论在 UI 设计中的应用

色彩是用户界面中最先被感知的视觉属性，它在用户看到任何文字或图标之前就已经传达了品牌调性、功能层次和情绪指引。掌握色彩理论并不意味着你要成为艺术家，而是要学会一套可复用的方法论，让你在任何项目中都能构建出协调、可访问、语义化的色彩系统。

## 色轮基础

色轮是理解色彩关系的基础工具。标准的 12 色轮包含三类颜色：

- **原色（Primary）**：红、黄、蓝，三种无法通过混合其他颜色获得的颜色。
- **间色（Secondary）**：橙、绿、紫，由两种原色等量混合而成。
- **复色（Tertiary）**：红橙、黄橙、黄绿、蓝绿、蓝紫、红紫，由原色与邻近间色混合而成。

在 UI 设计中，我们不会直接使用色轮的纯色，而是调整饱和度（Saturation）和亮度（Lightness）来获得适合界面的柔和色调。

## 四种核心配色和谐类型

### 互补色（Complementary）

色轮上相对的两个颜色，如蓝+橙、紫+黄。这种搭配视觉冲击力极强，适合需要强烈对比的场景。在 UI 中，互补色常用于主按钮（蓝色）搭配 CTA 按钮（橙色），或亮色系统与中性背景的对比。

### 相似色（Analogous）

色轮上相邻的 2-3 个颜色，如蓝+蓝绿+绿。相似色搭配自然柔和，适合用于背景层次区分、图表数据可视化、品牌辅助色系统。

### 三元配色（Triadic）

色轮上均匀分布的三个颜色（间隔 120 度），如红+黄+蓝。三元配色生动但不失平衡，适合需要色彩丰富的产品，如儿童应用或创意工具。

### 单色配色（Monochromatic）

单一色相，通过调整饱和度和亮度创造出明暗层次的变化。单色方案简洁专业，适合企业级产品、数据仪表盘等追求信息清晰度的场景。

## 60-30-10 法则

这是一条来自室内设计的经典配色比例，在 UI 设计中同样适用：

- **60% 主导色**：页面中最主要的颜色，通常是中性背景色（白色、浅灰）或品牌主色的浅色变体。覆盖区域最广，营造整体氛围。
- **30% 次要色**：与主导色形成对比，用于区分内容区块——侧边栏背景、卡片背景、导航栏等。
- **10% 点缀色**：最鲜艳、最引人注目的颜色，用于 CTA 按钮、链接、选中状态、通知角标。这是用户视线最先被吸引到的颜色。

举例来说，一个典型的 SaaS 管理后台可能是：60% 浅灰背景（#F5F7FA），30% 白色卡片和侧边栏（#FFFFFF），10% 品牌蓝色点缀（#3B82F6）。

## 构建 UI 调色板

一套完整的 UI 调色板至少包含以下语义化色系：

- **Primary（主色）**：品牌识别色，用于主按钮、链接、选中态。通常包含 5-10 个明度层次（50/100/200 到 900）。
- **Secondary（辅色）**：对主色的补充或替代，用于次要按钮、标签。
- **Success（成功）**：绿色系，用于成功提示、完成状态图标。
- **Warning（警告）**：橙黄色系，用于警告提示、需要用户注意的信息。
- **Error（错误）**：红色系，用于错误提示、删除操作、校验失败字段。
- **Neutral（中性）**：灰色系，用于文字层级、边框、分割线、禁用态背景。

### 使用 HSL 创建色彩层级

相比直接写 HEX 值，用 HSL 模式构建调色板更加直观。HSL 的三个参数分别是：Hue（色相，0-360）、Saturation（饱和度，0-100%）、Lightness（亮度，0-100%）。保持 Hue 和 Saturation 不变，逐步调整 Lightness 就能得到同色系的不同层次：

- **50（最浅）**：L=95%，用于浅色背景
- **500（标准）**：L=50%，作为标准主色
- **900（最深）**：L=15%，用于深色模式背景或信息层级最高的文字

Figma 的颜色选择器支持 HSL 模式，你可以在颜色面板的下拉菜单中切换。

## 无障碍色彩对比

WCAG（Web Content Accessibility Guidelines）2.1 为文字与背景之间的颜色对比度制定了量化标准：

- **正常文字（小于 18px 加粗或小于 24px 常规）**：最低对比度 4.5:1（AA 级），增强级 7:1（AAA 级）。
- **大文字（大于等于 18px 加粗或大于等于 24px 常规）**：最低对比度 3:1（AA 级），增强级 4.5:1（AAA 级）。
- **非文字 UI 组件（按钮边框、输入框边框、图标等）**：最低对比度 3:1。

在 Figma 中，你可以安装 **Stark** 或 **Able** 插件来实时检查选中文字的对比度。也可以使用免费的 WebAIM Contrast Checker 工具快速验证颜色搭配。

## 语义化色彩 Token

避免在设计文件和代码中直接出现具体的颜色值（如 #3B82F6），而是使用语义化命名。比如不叫 `blue-500`，而叫 `color-bg-primary`、`color-text-secondary`、`color-border-default`。语义化命名的价值在于：当将来切换到暗黑模式或品牌换色时，只需要修改 Token 的映射值，而不需要找到每一个使用该颜色的组件逐一更改。

## 暗黑模式的色彩考量

设计暗黑模式不是简单地把白色背景变成黑色。关键原则：避免纯黑背景（#000000），使用深灰（#121212 或 #1E1E1E）更舒适；亮度层级反转，浅色背景中提升层级用阴影（变得更亮），暗色中提升层级用亮度（表面比背景亮 1-2 个层级）；文字饱和度降低——纯白文字在深色背景上过于刺眼，使用略带灰色的白色更舒适；品牌色在深色背景下需要提高亮度以保持相同的视觉权重感。
