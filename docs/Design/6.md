# 字体排版 (Typography) 设计原则

字体排版是 UI 设计中信息传递的核心载体。再精美的界面，如果文字难以阅读——行距太挤、字号太小、层次混乱——用户的注意力会全部消耗在"辨认文字"上，而没有多余的认知资源去理解内容。好的排版设计是隐形的：它让用户轻松获取信息，而不会注意到排版本身的存在。

## 字体层级（Type Scale）

字体层级是一组有规律的字号序列，用于在界面中建立视觉信息架构。最常见的构建方式是使用**模块化比例（Modular Scale）**——后一个字号等于前一个乘以一个固定比例。两个最经典的比例如下：

- **1.25（大三度）**：在同一个层级内变化温和，适合信息密集的后台系统、数据仪表盘。得到序列：12 / 15 / 19 / 24 / 30 / 37 / 47。
- **1.333（完全四度）**：层级变化更明显，适合需要强烈视觉层次的内容型产品、营销落地页。得到序列：12 / 16 / 21 / 28 / 37 / 50 / 67。

实践中还有一个常用的四舍五入序列直接在多数项目中使用：**12 / 14 / 16 / 20 / 24 / 32 / 40 / 48**。记住一条原则：一个界面的字号种类通常不应超过 5-6 种层级，否则层次感会变得混乱。

## 建立文字层级（Hierarchy）

视觉层级通过四个维度的组合变化来传达信息的重要性顺序：

- **字号（Size）**：最直接的层级信号。标题越大，信息层级越高。
- **字重（Weight）**：常规（Regular/400）、中等（Medium/500）、半粗（SemiBold/600）、粗体（Bold/700）。字重变化搭配同字号，可以在不改变布局尺寸的情况下区分层级。
- **颜色（Color）**：深色文字引人注意，浅色文字次之。主文字使用接近黑色的深灰（#1A1A1A），次要说明使用中灰（#666666），最弱的辅助信息使用浅灰（#999999）。
- **间距（Spacing）**：标题前留有较多留白、标题与正文之间距离小于正文段落间距——用留白关系暗示内容群组。

## 行高（Line-height）黄金数值

行高的设置直接影响大段文字的阅读舒适度。过小的行高会让读者在换行时"串行"，过大的行高会让文字显得松散，影响阅读连贯性。

- **正文**：行高 1.5（即 150% 的字号），是公认的最适合长文阅读的比例。比如 16px 字号的正文，行高设为 24px。
- **标题**：行高 1.2，标题较短，不需要过大的行高来辅助换行，紧凑的比例让标题更有分量感。
- **中文特殊考虑**：中文字形方正，结构比拉丁字母更密，建议行高提高到 1.7-2.0。最小可读中文正文字号为 14px（低于 14px 的中文阅读感严重下降）。

## 行长（Line-length）与可读性

每行文字的字符数直接决定了阅读节奏。研究表明，每行 45-75 个字符（含空格和标点）是最佳的可读范围。太短（读者频繁换行，眼球运动不流畅），太长（读者追踪下一行的起始位置困难，容易串行）。在实际设计中，这通常对应约 600-700px 宽度的正文区域（以 16px 字号计算）。

## 字体搭配

设计师通常面临两种字体选择：衬线体（Serif）和无衬线体（Sans-serif）。衬线体在笔画末端带有装饰性小勾（如 Times New Roman、思源宋体），气质传统、正式，适合品牌标题或长篇艺术类阅读。无衬线体笔画干净利落（如 Inter、苹方），现代、清晰，是当下 UI 设计的主流选择。

常见的搭配遵循一条简单规则：**对比产生层次**。最经典的配对是衬线标题 + 无衬线正文（或反过来）。例如：Playfair Display（Serif）做页面大标题 + Inter（Sans-serif）做正文和控件文字。在任意单一项目中，尽量不要使用超过 2 种字体家族，过多的字体会让界面看起来支离破碎、缺乏整体感。

## 系统字体 vs Web 字体

**系统字体**（如 -apple-system、SF Pro、Segoe UI、苹方）的显著优势是零加载时间、零网络开销，渲染锐利。适用于功能性 UI、后台管理系统等对性能有要求的场景。

**Web 字体**（如从 Google Fonts 引入的 Inter、Noto Sans SC）提供了更丰富的字形选择和精细的字体设计，但需要额外的加载时间。使用 Web 字体时注意：通过 `font-display: swap` 确保文字在字体加载期间保持可见；尽量只加载项目中实际使用的字重（而不是引入全部 9 种粗细）；中文 Web 字体文件体积较大，使用 `unicode-range` 子集化可以有效减小文件大小。

## 中文排版的特殊要点

中文排版有不同于西文的若干规则：

- **字距**：中文字间距通常无需调整（字符本身就是方块形的），但中英文混排时应在英文单词前后添加一个空格。
- **标点避头尾**：中文标点不应出现在行首（如逗号、句号、右引号），也不应孤立在行尾（如左引号、左书名号）。CSS 的 `line-break` 和 `word-break` 属性可以解决大部分问题。
- **最小字号**：中文笔画比拉丁字母更复杂，12px 以下的中文几乎不可读。移动端最小中文字号建议 14px，桌面端 14-16px 为正文标准。
- **段落首行缩进**：中文传统排版会首行缩进两字，但在数字界面中，西式段落分隔（段落间距）越来越成为主流做法，两者可根据内容风格二选一，不应同时使用。

## 常见排版错误

新手设计师在字体排版上最常见的三类错误：**字号层级过多**——在一个页面中使用了 8-10 种不同字号，导致层次感混乱而非清晰；**对比度不足**——浅灰色文字放在浅灰背景上，用户阅读吃力；**间距不一致**——标题与上文段落间距不等于标题与下文段落的间距，违反了格式塔接近性原则，让用户难以辨识内容分组。解决这些问题的最好方法就是建立一个字体排版规范文档，将所有字号、行高、字重、颜色组合记录为固定搭配，并在全项目中严格执行。

## 在 Figma 中建立字体排版规范

Figma 的 Text Styles 功能让你将字体排版决策固化为可复用的设计资产。创建 Text Style 的操作流程如下：

1. 选中一段已经设置好字体、字号、字重、行高、字间距和颜色的文字。
2. 点击右侧属性面板中 Text 区域顶部的四孔图标（Style 按钮），选择 **+** 创建新样式。
3. 为样式命名，遵循 `类别/用途/层级` 的命名约定。例如：`Heading/H1/Desktop`、`Body/Default`、`Caption/Small`。
4. 创建完成后，在团队或项目级别发布为共享样式，其他设计师可以直接应用。

Text Styles 的真正价值不只是省去重复设置的时间，而是建立了"唯一修改点"——当产品需要全局将正文字号从 14px 调整到 15px 时，你只需修改 Body/Default 这一个 Style，所有引用它的文字图层都会自动更新。这就如同代码中的 CSS 变量或设计系统的 Design Token，让字体排版的一致性成为默认状态而非需要刻意维护的负担。
