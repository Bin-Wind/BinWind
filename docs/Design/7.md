# 间距系统与网格 (Grid) 布局

间距和网格是 UI 设计中"看不见的骨架"。用户不会直接注意到它们，但它们决定了界面看起来是专业还是业余，是舒适还是杂乱。一套一致的间距系统和网格布局就像音乐中的节拍器——它提供了底层的节奏感，让所有视觉元素在统一的韵律中排列。

## 8px 网格系统

8px 网格系统是当今 UI 设计界使用最广泛的间距基准。它的核心规则很简单：**所有间距、尺寸、位置坐标都应该是 8 的倍数**（经细微调整时可使用 4px）。为什么是 8？首先，8 的因数最多（可被 1、2、4、8 整除），意味着用它做单位能满足各种粒度需求。其次，大多数屏幕的像素密度适合 8 作为基本单位，它在 1x、2x、3x 分辨率下都能落在整数像素上，不会出现模糊的亚像素渲染。

### 标准间距尺度

以下是实践中常用的间距值序列，依次适用于从"紧密关联"到"区块分隔"的不同语义需求：

| Token 名称 | 值 | 典型用途 |
|---|---|---|
| spacing-1 | 4px | 微小元素间距，如图标与相邻文字 |
| spacing-2 | 8px | 紧密相关元素间距，如标签内的 padding |
| spacing-3 | 12px | 组件内部小间距 |
| spacing-4 | 16px | 标准内边距、卡片 padding |
| spacing-5 | 20px | 中等间距（非 8 倍数，但在特定场景实用） |
| spacing-6 | 24px | 段落间距、模块间标准分隔 |
| spacing-8 | 32px | 大模块分隔 |
| spacing-10 | 40px | 页内区域分隔 |
| spacing-12 | 48px | 大板块间距离 |
| spacing-16 | 64px | 首页 hero 与内容区间隔 |
| spacing-20 | 80px | 大板块留白 |
| spacing-24 | 96px | 超大留白，首页级 |

对于精细的 micro-spacing（如图标内间距、装饰线粗细），可以微调到 4px 甚至 2px，只要不破坏整体的 8px 节奏即可。

## 列网格（Column Grid）

列网格是最常用的 UI 布局网格类型，它由三个参数定义：列数、列间距（Gutter，列与列之间的空隙）和页边距（Margin，网格左右边缘到画板边界的距离）。

### Web 端：12 列网格

12 是 2、3、4、6 的倍数，意味着你可以轻松地将内容分为 2 列（各占 6 列宽度）、3 列（各占 4 列宽度）、4 列（各占 3 列宽度），甚至更不对称的比例（左侧 8 列 + 右侧 4 列的侧边栏布局）。对于桌面端 1440px 的设计宽度，常见设置是：Margin 左右各 80px（或 5.5%），Gutter 24px，单列宽度约 80px。

### 移动端：4 列网格

手机屏幕宽度有限，一般使用 4 列网格。以 iPhone 14（390px 宽）为例：Margin 左右各 16px，Gutter 16px，每列宽度约 77px。

### Figma 中设置列网格

选中 Frame，在右侧属性面板中找到 Layout Grid 区域，点击 + 添加 Grid，将类型从默认的 Grid 切换为 Columns。在弹出设置中填入 Column count（列数）、Gutter（间距）、Margin（页边距）即可。你可以在同一 Frame 上叠加 Rows（行网格）用于控制垂直方向的对齐。

## 垂直韵律（Vertical Rhythm）

垂直韵律是指页面上的所有元素按照一个固定的基线（通常为正文字号乘以行高的值）来排布。例如，正文字号 16px、行高 1.5，则基线为 24px。在这个系统中，所有标题、段落、图片、卡片的上下间距都应该是 24px 的倍数，让文字的每一行都落在相同的虚拟网格线上。这确保了：即使内容跨列，相邻两列的文字也能在同一水平线上对齐；页面在垂直方向上的滚动节奏统一、视觉稳定。

## 实战：用 12 列网格布局一个落地页

以下步骤演示如何用 12 列网格为一款 SaaS 产品构建一个典型的营销落地页：

1. **创建画板**：Frame 工具选择 Desktop 1440，宽度 1440px。
2. **设置网格**：在 Layout Grid 中添加 Columns，Count=12，Gutter=24，Margin=80，Type=Stretch。
3. **Hero 区域**：创建一个高度 600px 的区域。左侧文字区域占 5 列宽度（标题 + 描述 + CTA 按钮），右侧占 7 列宽度（产品插图或 3D 图占位）。
4. **特性展示区**：三列等宽卡片。每个卡片占 4 列（12/3=4），Gutter 24px 在卡片间自然形成了间距。
5. **Logo 墙**：将客户 Logo 排列在 12 列中，4 个 Logo 各占 3 列，居中对齐每个 Logo。
6. **Footer**：占满 12 列宽度，内部使用 4 列子网格分隔（链接分类 × 4）。
7. **响应式验证**：切换 Frame 宽度为 Tablet（768px，8 列）；切换为 Mobile（375px，4 列）。检查各元素在变窄时的布局表现——Hero 的左右布局是否变为上下堆叠？三列卡片是否变为单列？

## 为什么一致的间距如此重要

间距的一致性不是审美强迫症，而是有认知心理学基础的。**格式塔原理**中的"接近性"告诉我们：间距越小、元素越被感知为一组。当你在整个产品中使用相同的间距规则时，用户在潜意识中就学会了"阅读"布局——16px 间距意味着组件内部的紧凑结构，48px 间距意味着逻辑上的大板块分隔。一旦这个节拍被打乱（比如某个页面突然用 35px 替代 32px，用 18px 替代 16px），用户的视线节奏就会被打断，产生"这个页面看起来不对劲"的直觉反应——尽管他们可能无法说出原因。

将间距定义为 Design Token，在 Figma 和代码中同步维护，是保证间距一致性最可靠的方式。同时，养成在 Figma 中使用 Smart Selection 检查间距数值的习惯——选中相邻元素，查看粉红色间距标注上的数值是否符合预期的 Token 值。
