# Redis 缓存服务器部署

Redis 是一个高性能的键值存储数据库，以其亚毫秒级响应速度和丰富的数据结构支持而闻名。它广泛应用于缓存、会话存储、消息队列和实时计数等场景。本文介绍 Redis 的安装部署和核心使用方法。

## 为什么选择 Redis

Redis 的核心优势：
- **极致的速度**：纯内存操作，单线程模型避免锁竞争，单机 QPS 轻松达到 10 万+
- **丰富的数据结构**：不仅支持字符串，还支持哈希、列表、集合、有序集合等
- **持久化支持**：RDB 快照和 AOF 日志两种持久化方式
- **高可用**：Sentinel 哨兵模式和 Cluster 集群模式
- **原子操作**：所有命令都是原子的

## 安装 Redis

### Linux (Debian/Ubuntu)

```bash
sudo apt update
sudo apt install redis-server

# 启动并设置开机自启
sudo systemctl start redis-server
sudo systemctl enable redis-server
sudo systemctl status redis-server
```

### macOS

```bash
brew install redis
brew services start redis
```

### 从源码编译（安装最新版本）

```bash
wget https://download.redis.io/releases/redis-stable.tar.gz
tar xzf redis-stable.tar.gz
cd redis-stable
make -j$(nproc)
sudo make install
```

### 验证安装

```bash
redis-cli ping
# 返回 PONG 表示正常

redis-cli --version
# redis-cli 7.x.x
```

## 基本配置

Redis 主配置文件位于 `/etc/redis/redis.conf`，核心配置项：

```conf
# 绑定地址：生产环境只监听内网 IP
bind 127.0.0.1 192.168.1.100

# 端口
port 6379

# 密码认证（生产环境必须设置）
requirepass your_strong_password_here

# 守护进程模式
daemonize yes

# 最大内存（建议设为物理内存的 50%-70%）
maxmemory 2gb

# 内存淘汰策略：LRU 淘汰最少使用的键
maxmemory-policy allkeys-lru

# 日志文件
logfile /var/log/redis/redis-server.log

# 数据库数量（默认 16 个，0-15）
databases 16
```

修改配置后重启：
```bash
sudo systemctl restart redis-server
```

### 密码认证方式

配置密码后，连接 Redis 需要认证：

```bash
# 在 redis-cli 中
redis-cli -a your_strong_password_here
redis-cli
> AUTH your_strong_password_here

# 使用环境变量避免密码暴露在命令行
redis-cli --no-auth-warning -a $REDIS_PASSWORD
```

## 持久化配置

Redis 提供两种持久化方式，可以单独或组合使用。

### RDB（快照）

定期将内存数据快照保存到磁盘：

```conf
# /etc/redis/redis.conf
save 900 1      # 900 秒内至少 1 次修改则保存
save 300 10     # 300 秒内至少 10 次修改则保存
save 60 10000   # 60 秒内至少 10000 次修改则保存

dbfilename dump.rdb
dir /var/lib/redis

# 启用压缩
rdbcompression yes

# 保存失败时拒绝写入
stop-writes-on-bgsave-error yes
```

**优点**：文件紧凑，恢复速度快，适用于灾难恢复
**缺点**：可能丢失最后一次快照之后的数据

### AOF（追加文件）

记录每一条写命令到日志文件：

```conf
appendonly yes
appendfilename "appendonly.aof"

# 同步策略
# always   : 每条命令都同步，最安全但最慢
# everysec : 每秒同步一次（推荐，最多丢失 1 秒数据）
# no       : 由操作系统决定同步时机
appendfsync everysec

# 自动重写（压缩 AOF 文件）
auto-aof-rewrite-percentage 100  # 文件增长 100% 后自动重写
auto-aof-rewrite-min-size 64mb   # 文件至少 64MB 才触发重写
```

**优点**：数据安全性更高，默认每秒 fsync 最多丢失 1 秒数据
**缺点**：文件通常比 RDB 大，恢复速度较慢

### 推荐组合策略

生产环境建议 RDB + AOF 同时开启：RDB 用于灾难备份恢复，AOF 用于保证数据安全。

## 数据类型速查

Redis 支持 5 种核心数据类型，每种有其适用场景：

### String（字符串）
```bash
SET key value              # 设置键值
GET key                    # 获取值
SET name "John" EX 3600    # 设置值并指定过期时间（秒）
INCR counter               # 计数器自增（原子操作）
DECR counter               # 计数器自减
MSET k1 v1 k2 v2           # 批量设置
MGET k1 k2                 # 批量获取
APPEND key "suffix"        # 追加内容
```

**典型场景**：缓存、计数器、分布式锁

### Hash（哈希）
```bash
HSET user:100 name "John" age 30 email "john@example.com"
HGET user:100 name         # 获取单个字段
HGETALL user:100           # 获取所有字段和值
HDEL user:100 age          # 删除字段
HEXISTS user:100 email     # 检查字段是否存在
HINCRBY user:100 age 1     # 字段值自增
```

**典型场景**：对象存储（用户信息、配置项）

### List（列表）
```bash
LPUSH queue "task1"        # 左侧插入
RPUSH queue "task2"        # 右侧插入
LPOP queue                 # 左侧弹出
RPOP queue                 # 右侧弹出
LRANGE queue 0 -1          # 获取所有元素
LLEN queue                 # 列表长度
LTRIM queue 0 99           # 只保留前 100 个元素
```

**典型场景**：消息队列、最新消息列表

### Set（集合）
```bash
SADD tags "redis" "db" "cache"
SMEMBERS tags              # 列出所有成员
SISMEMBER tags "redis"     # 检查成员是否存在
SREM tags "db"             # 删除成员
SINTER set1 set2           # 交集
SUNION set1 set2           # 并集
SDIFF set1 set2            # 差集
SCARD tags                 # 集合大小
```

**典型场景**：标签系统、共同好友、独立访客统计

### Sorted Set（有序集合）
```bash
ZADD leaderboard 100 "player1" 90 "player2" 95 "player3"
ZRANGE leaderboard 0 -1 WITHSCORES    # 按分数升序排列
ZREVRANGE leaderboard 0 9 WITHSCORES  # 排名前 10
ZRANK leaderboard "player1"           # 获取排名
ZSCORE leaderboard "player1"          # 获取分数
ZINCRBY leaderboard 5 "player1"       # 增加分数
ZREM leaderboard "player2"            # 删除成员
```

**典型场景**：排行榜、优先级队列、延迟队列

## 键的过期管理

```bash
EXPIRE key 3600             # 设置过期时间（秒）
PEXPIRE key 60000           # 设置过期时间（毫秒）
TTL key                     # 查看剩余 TTL（秒），-1 表示永不过期，-2 表示不存在
PERSIST key                 # 移除过期时间，永久保存
EXPIREAT key 1700000000     # 指定 Unix 时间戳过期
```

## 常用应用场景

### 1. 缓存（Cache Aside 模式）

```python
# Python 伪代码：先查 Redis，未命中再查 MySQL
def get_user(user_id):
    # 1. 尝试从 Redis 获取
    key = f"user:{user_id}"
    cached = redis.get(key)
    if cached:
        return json.loads(cached)

    # 2. Redis 未命中，查数据库
    user = db.query(f"SELECT * FROM users WHERE id = {user_id}")

    # 3. 写入 Redis，设置 1 小时过期
    redis.setex(key, 3600, json.dumps(user))
    return user
```

### 2. 分布式会话存储

```python
# Express.js
const session = require('express-session');
const RedisStore = require('connect-redis').default;

app.use(session({
    store: new RedisStore({ client: redisClient }),
    secret: 'your-secret',
    resave: false,
    saveUninitialized: false,
    cookie: { maxAge: 86400000 }  // 24 小时
}));
```

### 3. 速率限制

```bash
# 实现每个 IP 每分钟最多 60 次请求
KEY = "rate_limit:192.168.1.1"
MULTI
    INCR $KEY
    EXPIRE $KEY 60
EXEC
# 如果返回值 > 60，拒绝请求
```

### 4. 简单消息队列

```bash
# 生产者
LPUSH myqueue "task_data"

# 消费者（阻塞式，无消息时等待）
BLPOP myqueue 30   # 等待 30 秒
```

## Sentinel 高可用模式概述

Redis Sentinel 用于监控、自动故障转移和通知，最小配置如下：

```bash
# sentinel.conf（每个 Sentinel 节点各一份）
port 26379
sentinel monitor mymaster 192.168.1.100 6379 2
sentinel auth-pass mymaster your_redis_password
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 10000
sentinel parallel-syncs mymaster 1
```

关键参数：
- `2`：至少 2 个 Sentinel 同意才算故障
- `down-after-milliseconds 5000`：5 秒无响应判定为主观下线
- `parallel-syncs 1`：故障转移后，新主库每次只同步一个从库

## 安全加固

1. **设置强密码**：`requirepass` 必须配置
2. **绑定内网 IP**：`bind` 不绑定 0.0.0.0
3. **禁用危险命令**：
```conf
rename-command FLUSHALL ""
rename-command FLUSHDB ""
rename-command CONFIG ""
rename-command KEYS ""
```
4. **防火墙隔离**：Redis 端口（6379）仅对应用服务器开放
5. **定期更新**：Redis 版本保持最新，及时修补已知漏洞
