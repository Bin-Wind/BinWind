# 防火墙配置 (UFW / iptables)

防火墙是服务器安全的第一道防线。Linux 下主要有两种防火墙管理工具：UFW（Uncomplicated Firewall）是 iptables 的友好前端，适合日常操作；iptables 则是底层工具，功能更强大也更复杂。本文兼顾两者，帮助你在不同场景下正确配置防火墙。

## UFW — 简化防火墙管理

UFW 是 Ubuntu 默认的防火墙工具，语法直观，覆盖大多数常见需求。

### 基本操作

```bash
# 安装（通常已预装）
sudo apt install ufw

# 查看状态
sudo ufw status
sudo ufw status verbose           # 详细状态
sudo ufw status numbered          # 带编号的规则列表

# 启用/禁用
sudo ufw enable                   # 启用防火墙（开机自启）
sudo ufw disable                  # 禁用防火墙
sudo ufw reload                   # 重载规则

# 重置（清除所有规则，恢复默认）
sudo ufw reset
```

### 默认策略

最佳实践是"默认拒绝，仅放行必要端口"：

```bash
# 拒绝所有入站连接
sudo ufw default deny incoming

# 允许所有出站连接
sudo ufw default allow outgoing
```

### 放行/拒绝端口与服务

```bash
# 按端口和协议放行
sudo ufw allow 22/tcp             # 放行 SSH
sudo ufw allow 80/tcp             # HTTP
sudo ufw allow 443/tcp            # HTTPS
sudo ufw allow 3306/tcp           # MySQL
sudo ufw allow 6379/tcp           # Redis

# 按服务名放行（读取 /etc/services）
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https

# 放行端口范围
sudo ufw allow 8000:8100/tcp      # 放行 8000-8100 端口
sudo ufw allow 6000:6060/udp

# 拒绝端口
sudo ufw deny 8080/tcp
sudo ufw deny 25                  # 拒绝 SMTP（不对外开放邮件服务）
```

### 按 IP 地址控制

```bash
# 允许特定 IP 访问任何端口
sudo ufw allow from 192.168.1.100

# 允许特定 IP 访问特定端口
sudo ufw allow from 192.168.1.100 to any port 22

# 允许整个子网
sudo ufw allow from 192.168.1.0/24

# 允许整个子网访问特定端口
sudo ufw allow from 192.168.1.0/24 to any port 3306

# 拒绝某个 IP
sudo ufw deny from 203.0.113.100

# 拒绝整个子网
sudo ufw deny from 203.0.113.0/24
```

### 删除规则

```bash
# 方法一：按规则内容删除
sudo ufw delete deny 8080/tcp
sudo ufw delete allow from 192.168.1.100

# 方法二：按编号删除（更精确）
sudo ufw status numbered
# 输出示例：
# [ 1] 22/tcp       ALLOW IN    Anywhere
# [ 2] 80/tcp       ALLOW IN    Anywhere
# [ 3] 8080/tcp     DENY IN     Anywhere

sudo ufw delete 3                 # 删除编号 3 的规则
```

### 应用配置文件

UFW 内置了常见应用的服务配置，一些软件安装后也会添加自己的配置：

```bash
# 查看可用应用配置
sudo ufw app list

# 查看应用配置详情
sudo ufw app info "Nginx Full"

# 按应用名放行
sudo ufw allow "Nginx Full"       # 同时放行 80 和 443
sudo ufw allow "OpenSSH"
```

常见应用配置：
- `Nginx HTTP`：80/tcp
- `Nginx HTTPS`：443/tcp
- `Nginx Full`：80 + 443/tcp
- `Apache Full`：80 + 443/tcp
- `OpenSSH`：22/tcp

### 日志与审计

```bash
# 查看 UFW 拦截日志
sudo ufw logging on               # 日志级别：off/low/medium/high/full
sudo ufw logging medium           # 设置日志级别
sudo tail -f /var/log/ufw.log     # 实时查看日志
```

### 常见配置案例

**Web 服务器**：
```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

**仅允许内网访问的数据库**：
```bash
sudo ufw default deny incoming
sudo ufw allow 22/tcp
sudo ufw allow from 10.0.0.0/8 to any port 3306
sudo ufw allow from 172.16.0.0/12 to any port 3306
sudo ufw enable
```

**私有 API 服务（仅特定 IP 访问）**：
```bash
sudo ufw default deny incoming
sudo ufw allow 22/tcp
sudo ufw allow from 203.0.113.50 to any port 8443
sudo ufw enable
```

## iptables — 底层防火墙控制

iptables 是 Linux 内核 netfilter 的用户空间工具，提供了精细的包过滤能力。

### 核心概念

iptables 使用表和链的概念组织规则：

| 链 | 说明 |
|------|------|
| `INPUT` | 处理入站数据包 |
| `OUTPUT` | 处理出站数据包 |
| `FORWARD` | 处理转发数据包（路由器/NAT 场景） |

### 基本命令

```bash
# 查看所有规则
sudo iptables -L
sudo iptables -L -v -n             # 详细输出 + 数字格式（更快）
sudo iptables -S                   # 以命令语法显示

# 默认策略
sudo iptables -P INPUT DROP        # 默认拒绝入站
sudo iptables -P FORWARD DROP      # 默认拒绝转发
sudo iptables -P OUTPUT ACCEPT     # 默认允许出站
```

### 常用规则

```bash
# 允许本地回环
sudo iptables -A INPUT -i lo -j ACCEPT

# 允许已建立/相关的连接
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 放行 SSH (22)
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 放行 HTTP (80) 和 HTTPS (443)
sudo iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

# 放行特定 IP 的 SSH 连接
sudo iptables -A INPUT -p tcp -s 192.168.1.100 --dport 22 -j ACCEPT

# 拒绝来自特定 IP 的访问
sudo iptables -A INPUT -s 203.0.113.100 -j DROP

# 防止端口扫描（限制新建连接速率）
sudo iptables -A INPUT -p tcp -m state --state NEW -m recent --set
sudo iptables -A INPUT -p tcp -m state --state NEW -m recent \
    --update --seconds 60 --hitcount 10 -j DROP

# 防止 Ping 洪水攻击
sudo iptables -A INPUT -p icmp --icmp-type echo-request \
    -m limit --limit 1/second -j ACCEPT

# 记录被丢弃的数据包
sudo iptables -A INPUT -j LOG --log-prefix "IPTables-Dropped: "
sudo iptables -A INPUT -j DROP
```

### 插入和删除规则

```bash
# 查看规则及行号
sudo iptables -L --line-numbers

# 在指定位置插入规则
sudo iptables -I INPUT 1 -p tcp --dport 22 -j ACCEPT  # 插入为第 1 条

# 按行号删除规则
sudo iptables -D INPUT 3

# 按规则内容删除
sudo iptables -D INPUT -s 203.0.113.100 -j DROP

# 清空所有规则（不改变默认策略）
sudo iptables -F
```

### 保存与恢复规则

iptables 规则在重启后会丢失，需要持久化：

```bash
# Debian/Ubuntu 安装 iptables-persistent
sudo apt install iptables-persistent

# 手动保存当前规则
sudo iptables-save > /etc/iptables/rules.v4
sudo ip6tables-save > /etc/iptables/rules.v6

# 手动恢复规则
sudo iptables-restore < /etc/iptables/rules.v4
```

使用 `iptables-persistent` 包后，规则会在系统启动时自动加载，也可通过 netfilter-persistent 服务管理：

```bash
sudo systemctl enable netfilter-persistent
sudo netfilter-persistent save
```

### 完整 iptables 入门脚本

以下是一个安全的 iptables 基础配置模板：

```bash
#!/bin/bash
# 清空现有规则
iptables -F
iptables -X
iptables -Z

# 默认策略：丢弃入站和转发，允许出站
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 允许本地回环
iptables -A INPUT -i lo -j ACCEPT

# 允许已建立和相关的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许 Ping
iptables -A INPUT -p icmp -j ACCEPT

# 开放服务端口
iptables -A INPUT -p tcp --dport 22 -j ACCEPT    # SSH
iptables -A INPUT -p tcp --dport 80 -j ACCEPT    # HTTP
iptables -A INPUT -p tcp --dport 443 -j ACCEPT   # HTTPS

# 记录并丢弃其余流量
iptables -A INPUT -j LOG --log-prefix "IPTABLES-DROP: "
iptables -A INPUT -j DROP
```

## 安全最佳实践

1. **默认拒绝原则**：`default deny incoming`，仅显式放行需要的端口
2. **最小暴露面**：只开放业务必需的端口，如 Web 服务器仅开 22/80/443
3. **限制来源 IP**：数据库和内部服务仅允许内网 IP 段访问
4. **限制连接速率**：防止暴力破解和 DDoS
5. **启用日志记录**：定期审查防火墙日志，发现异常访问模式
6. **测试前保留后门**：修改远程服务器防火墙前，开一个备用 SSH 连接，防止将自己锁在外面
7. **维护规则文档**：记录每条规则的用途，避免随时间的规则累积变成无人理解的配置
