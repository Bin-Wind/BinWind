# Linux 用户与权限管理

Linux 是多用户操作系统，完善的用户和权限管理是服务器安全的基础。不当的权限设置可能导致数据泄露或系统被提权攻击。本文系统介绍 Linux 下的用户管理、文件权限和 sudo 配置。

## 用户管理

### useradd — 创建用户

```bash
# 创建用户（默认值在 /etc/default/useradd 中定义）
sudo useradd alice

# 创建用户 + 常用选项
sudo useradd -m \            # 创建主目录 (/home/alice)
    -s /bin/bash \            # 指定登录 Shell
    -c "Alice Wang" \         # 备注（全名）
    -g developers \           # 指定主组
    -G docker,sudo \          # 附加组
    alice

# 创建系统用户（无主目录，UID 通常在 1-999 之间）
sudo useradd -r -s /usr/sbin/nologin myapp
```

### usermod — 修改用户

```bash
# 修改用户 Shell
sudo usermod -s /bin/bash alice

# 将用户添加到附加组
sudo usermod -aG docker alice       # -a 表示追加而不是替换
sudo usermod -aG sudo,www-data bob  # 同时添加多个组

# 锁定/解锁用户
sudo usermod -L alice               # 锁定用户（禁止登录）
sudo usermod -U alice               # 解锁用户

# 修改用户名
sudo usermod -l newname oldname

# 修改主目录
sudo usermod -d /new/home/dir -m username

# 修改用户备注
sudo usermod -c "New Full Name" username

# 设置用户过期日期
sudo usermod -e 2025-12-31 username
```

### userdel — 删除用户

```bash
sudo userdel username               # 仅删除用户，保留主目录
sudo userdel -r username            # 同时删除用户和主目录及邮件池
```

### passwd — 密码管理

```bash
# 修改当前用户密码
passwd

# 修改其他用户密码（root）
sudo passwd alice

# 锁定/解锁密码
sudo passwd -l alice                # 锁定密码
sudo passwd -u alice                # 解锁密码

# 设置密码过期
sudo passwd -e alice                # 下次登录强制改密码
sudo chage -M 90 alice              # 密码有效期 90 天

# 查看密码状态
sudo passwd -S alice
sudo chage -l alice
```

### 用户和组信息查看

```bash
id alice                            # 查看用户 UID、GID 及所属组
whoami                              # 当前用户名
who                                 # 当前登录的用户列表
w                                   # 当前登录用户及正在执行的操作
last                                # 最近的登录记录
cat /etc/passwd                     # 所有用户列表
cat /etc/shadow                     # 加密密码（仅 root 可读）
```

## 组管理

### groupadd — 创建组

```bash
sudo groupadd developers
sudo groupadd -g 1500 designers    # 指定 GID
sudo groupadd -r appgroup          # 创建系统组
```

### groupmod — 修改组

```bash
sudo groupmod -n newname oldname   # 重命名组
sudo groupmod -g 1600 developers   # 修改 GID
```

### groupdel — 删除组

```bash
sudo groupdel developers
# 注意：如果该组是某用户的主组，需先将用户移出该组
```

### 组成员管理

```bash
# 将用户添加到附加组
sudo usermod -aG developers alice

# 查看用户所属组
groups alice

# 查看组成员
getent group developers
grep developers /etc/group

# 移除用户从附加组（gpasswd）
sudo gpasswd -d alice developers

# 也可以直接编辑 /etc/group，手动删除用户名
```

## 文件权限

### 权限表示法

Linux 文件权限通过三组三元组表示：

```
-rwxr-xr--  1 alice  developers  4096 Jun 28 10:00 script.sh
 ┬ ┬ ┬
 │ │ └─ 其他用户权限 (r--)
 │ └─── 组权限 (r-x)
 └───── 所有者权限 (rwx)
```

- `r`（read）= 4，`w`（write）= 2，`x`（execute）= 1
- 权限数字是三者之和：`rwx` = 4+2+1 = 7，`rw-` = 4+2+0 = 6，`r--` = 4

### chmod — 修改权限

```bash
# 数字模式
chmod 755 script.sh             # rwxr-xr-x（所有者全权限，组和其他可读执行）
chmod 644 file.txt              # rw-r--r--（所有者读写，组和其他只读）
chmod 600 id_rsa                # rw-------（仅所有者可读写，SSH 私钥推荐）
chmod 777 public_upload/        # rwxrwxrwx（所有人全权限，尽量避免使用）

# 符号模式
chmod u+x script.sh             # 所有者（user）添加执行权限
chmod g-w file.txt              # 组（group）移除写权限
chmod o-rwx file.txt            # 其他用户（others）移除所有权限
chmod a+r file.txt              # 所有用户（all）添加读权限
chmod u=rwx,g=rx,o= script.sh  # 精确设置权限

# 递归修改
chmod -R 755 /var/www           # 递归修改目录及所有子文件
chmod -R u=rwX,g=rX,o= /var/www  # X=仅目录和已有执行权限的文件加 x
```

### chown — 修改所有者

```bash
# 修改所有者
sudo chown alice file.txt

# 同时修改所有者和组
sudo chown alice:developers file.txt
sudo chown alice.developers file.txt  # 同上（. 与 : 等价）

# 仅修改组
sudo chown :developers file.txt
sudo chgrp developers file.txt        # 同上

# 递归修改
sudo chown -R alice:www-data /var/www
```

### 常用权限对照表

| 权限 | 数字 | 适用场景 |
|------|------|----------|
| 700 | rwx------ | 个人脚本、私钥 |
| 755 | rwxr-xr-x | 可执行程序、目录 |
| 644 | rw-r--r-- | 普通文件 |
| 600 | rw------- | 配置文件含敏感信息、SSH 私钥 |
| 750 | rwxr-x--- | 组内共享的可执行文件 |
| 640 | rw-r----- | 组内共享的普通文件 |
| 400 | r-------- | 只读配置文件 |
| 777 | rwxrwxrwx | 不应使用！安全隐患极大 |

## 特殊权限位

### setuid（SUID）

文件执行时以**文件所有者**的身份运行，而不是执行者身份：

```bash
# 设置 SUID
chmod u+s /usr/bin/passwd        # 或 chmod 4755
ls -l /usr/bin/passwd
# -rwsr-xr-x 1 root root ... /usr/bin/passwd
#   ┬ s 代替了 x，表示 SUID 已设置

# 查找系统中的 SUID 文件（安全审计）
sudo find / -perm -4000 -type f 2>/dev/null
```

**安全风险**：SUID 程序如果存在漏洞，攻击者可以提权到 root。应定期审计和最小化 SUID 文件。

### setgid（SGID）

文件执行时以**文件所属组**的身份运行；目录上设置 SGID 后，新创建的文件自动继承目录的组：

```bash
chmod g+s /shared/project       # 或 chmod 2755

ls -ld /shared/project
# drwxr-sr-x 2 root developers ... /shared/project
#       ┬ s 代替了 x
```

**典型场景**：共享目录，确保团队成员创建的文件都属于同一组。

### sticky bit

仅对目录有效，设置后只有文件所有者才能删除该目录中的文件：

```bash
chmod +t /tmp                    # 或 chmod 1777

ls -ld /tmp
# drwxrwxrwt 10 root root ... /tmp
#          ┬ t 代替了 x
```

`/tmp` 和 `/var/tmp` 默认设置了 sticky bit。

## sudo 配置

sudo 允许普通用户以 root 或其他用户身份执行命令，配置通过 `visudo` 编辑 `/etc/sudoers` 文件：

```bash
sudo visudo                       # 使用默认编辑器
sudo EDITOR=vim visudo            # 指定编辑器

# 推荐方式：在 /etc/sudoers.d/ 中添加独立配置文件
sudo visudo -f /etc/sudoers.d/alice
```

### 常用配置示例

```bash
# 授予用户完整 sudo 权限
alice ALL=(ALL:ALL) ALL

# 免密 sudo
bob ALL=(ALL:ALL) NOPASSWD: ALL

# 限制特定命令
charlie ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx, \
                             /usr/bin/systemctl status nginx

# 仅允许以特定用户身份执行
deploy ALL=(www-data) NOPASSWD: /usr/bin/systemctl reload nginx

# 组授权
%developers ALL=(ALL) ALL
%devops ALL=(ALL) NOPASSWD: ALL
```

### sudo 安全审计

```bash
# 查看当前用户的 sudo 权限
sudo -l

# 查看其他用户的 sudo 权限（需要 root）
sudo -l -U alice

# 查看 sudo 使用历史
sudo journalctl -t sudo
cat /var/log/auth.log | grep sudo    # Debian/Ubuntu
cat /var/log/secure | grep sudo      # CentOS/RHEL
```

## /etc/passwd 与 /etc/shadow

### /etc/passwd 格式

```bash
root:x:0:0:root:/root:/bin/bash
alice:x:1001:1001:Alice Wang:/home/alice:/bin/bash
 ┬   ┬  ┬   ┬     ┬           ┬            ┬
 │   │  │   │     │           │            └─ Shell
 │   │  │   │     │           └─ 主目录
 │   │  │   │     └─ 备注（GECOS）
 │   │  │   └─ 主组 GID
 │   │  └─ UID
 │   └─ 密码占位符（x 表示密码在 /etc/shadow）
 └─ 用户名
```

### /etc/shadow 格式

```bash
alice:$6$salt$hashedpassword:19500:0:99999:7:::
  ┬        ┬                  ┬    ┬   ┬   ┬
  │        │                  │    │   │   └─ 保留字段
  │        │                  │    │   └─ 过期前警告天数
  │        │                  │    └─ 密码过期后锁定天数
  │        │                  └─ 密码有效天数（99999=永不过期）
  │        └─ 加密后的密码
  └─ 用户名
```

## umask — 默认权限掩码

umask 决定了新建文件和目录的默认权限：

```bash
# 查看当前 umask
umask
# 0022（常见默认值）

# 计算规则：新建文件默认 666，新建目录默认 777，然后减去 umask
# umask=022: 文件=666-022=644, 目录=777-022=755

# 临时修改
umask 027      # 文件=640, 目录=750（更安全）

# 永久修改：添加到 ~/.bashrc 或 /etc/profile
echo "umask 027" >> ~/.bashrc
```

**推荐值**：
- 个人工作站：`022`（文件 644，目录 755）
- 生产服务器：`027`（文件 640，目录 750）
- 高度安全环境：`077`（文件 600，目录 700）

## 最佳实践

1. **最小权限原则**：每个用户和服务只拥有完成其工作所需的最小权限
2. **禁用 root 直接登录**：通过 sudo 代替直接使用 root
3. **使用 sudo 组**：将需要管理权限的用户加入 sudo 或 wheel 组，通过组授权而非单独配置
4. **服务使用专用用户**：每个服务（Nginx、MySQL、应用）使用独立的非特权用户运行
5. **SSH 私钥 600 权限**：私钥权限必须为 600，否则 SSH 拒绝使用
6. **Web 目录权限**：文件 644，目录 755，所有者通常是 deploy 用户，组通常是 www-data
7. **定期审计**：检查 `/etc/passwd` 中的异常账户、SUID 文件、sudo 配置
8. **不共享账户**：每人使用独立账户，便于行为追溯和审计
