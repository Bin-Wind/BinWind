# 磁盘空间管理

磁盘空间耗尽是最常见的服务器故障之一。当磁盘写满时，服务会停止工作、数据库无法写入，甚至 SSH 登录都可能受影响。本文介绍 `df`、`du`、`ncdu` 等磁盘管理工具，以及 LVM 的扩容方法和磁盘满时的应急处理步骤。

## df：查看磁盘使用概况

```bash
# 人类可读格式查看磁盘使用情况
df -h

# 查看 inode 使用情况（即使空间未满，inode 耗尽也无法创建文件）
df -i

# 只查看本地文件系统（排除 tmpfs 等）
df -h -t ext4 -t xfs

# 指定文件系统类型
df -h -T
```

输出解读：重点关注 `Use%` 列，超过 85% 就应该引起警觉，超过 95% 需要立刻处理。

## du：定位磁盘消耗大户

```bash
# 查看指定目录的大小
du -sh /var/*

# 按大小排序，找出当前目录下的前 10 个大目录
du -sh * | sort -rh | head -10

# 显示多层目录的大小
du -h --max-depth=2 /home | sort -rh | head -20
```

## ncdu：交互式磁盘分析器

`ncdu`（NCurses Disk Usage）是一个 TUI 界面的磁盘分析工具，比手动跑 `du` 直观得多。

```bash
# 安装
sudo apt install ncdu

# 分析当前目录
ncdu

# 分析根目录
sudo ncdu /

# 将分析结果导出为文件（不进入交互界面）
ncdu -o output.json /var

# 从文件加载分析结果
ncdu -f output.json
```

在 `ncdu` 界面内，可以使用方向键浏览目录，按 `d` 删除文件/目录，按 `m` 切换排序方式，按 `?` 查看所有快捷键。

## find：搜索大文件

```bash
# 查找大于 500MB 的文件
sudo find / -type f -size +500M -exec ls -lh {} \; 2>/dev/null

# 查找大于 1GB 的文件
sudo find / -type f -size +1G 2>/dev/null

# 查找最近 7 天内修改过的大文件（>100MB）
sudo find / -type f -size +100M -mtime -7 2>/dev/null

# 查找指定目录下大于 100MB 的文件，按大小排序
find /var/log -type f -size +100M -exec du -h {} \; | sort -rh

# 查找并删除指定类型的大文件（谨慎操作！）
find /var/log -name "*.log" -size +1G -delete
```

## 常用清理目标

```bash
# 1. 清理系统日志
sudo journalctl --vacuum-size=200M        # journal 日志
sudo rm -rf /var/log/*.gz                 # 已压缩的旧日志

# 2. 清理临时文件
sudo find /tmp -type f -mtime +7 -delete

# 3. 清理 APT 缓存（Debian/Ubuntu）
sudo apt clean                            # 清空 /var/cache/apt/archives
sudo apt autoremove                       # 移除不再需要的依赖包

# 4. 清理旧内核
dpkg -l | grep linux-image | awk '/^ii/{print $2}' | head -n -2
sudo apt autoremove --purge               # 也会清理旧内核

# 5. Docker 空间清理
docker system prune -a -f                 # 清理未使用的镜像、容器、网络、构建缓存
docker volume prune -f                    # 清理未使用的卷

# 6. 查找并清理大日志文件
find /var/log -type f -name "*.log" -size +50M | while read f; do
    echo "Truncating $f"
    sudo truncate -s 0 "$f"
done
```

## LVM 基础与扩容

LVM（Logical Volume Manager）允许动态调整分区大小，不需要重启服务器。

核心概念：
- **PV**（Physical Volume）：物理卷，对应实际的磁盘或分区
- **VG**（Volume Group）：卷组，由多个 PV 组成，是容量池
- **LV**（Logical Volume）：逻辑卷，从 VG 中划分，实际挂载使用的设备

```bash
# 查看现有 LVM 布局
sudo pvs          # 查看物理卷
sudo vgs          # 查看卷组
sudo lvs          # 查看逻辑卷
sudo pvdisplay    # 详细信息
sudo vgdisplay
```

扩容逻辑卷的完整步骤：

```bash
# 场景：新增了一块 /dev/sdb 硬盘，要扩展到 /data 所在的逻辑卷

# 1. 创建物理卷
sudo pvcreate /dev/sdb

# 2. 扩展到卷组
sudo vgextend vg0 /dev/sdb

# 3. 扩展逻辑卷（增加 50G）
sudo lvextend -L +50G /dev/vg0/data

# 或者扩展到整个卷组的大小
sudo lvextend -l +100%FREE /dev/vg0/data

# 4. 扩展文件系统（ext4）
sudo resize2fs /dev/vg0/data

# 如果是 xfs 文件系统
sudo xfs_growfs /data
```

## 磁盘满的应急处理流程

1. **确认问题**：`df -h` 确认哪个分区满了；`df -i` 确认不是 inode 耗尽
2. **快速释放空间**：删除明确的垃圾文件（如 `/tmp` 下的临时文件、旧日志）
3. **定位元凶**：`du -sh /* | sort -rh | head -10` 逐层深入定位
4. **检查被删文件**：如果 `df -h` 和 `du -sh /` 的总和不匹配，说明存在被删除但未被释放的文件。使用 `lsof +L1` 找出这些文件，重启对应的进程即可释放
5. **扩容或迁移**：短期方案是 `lvextend` 扩容；长期方案可能涉及迁移部分数据到新盘或启用日志轮转限制
6. **预防**：添加磁盘使用率监控告警（可在 Prometheus 中配置 `node_filesystem_avail_bytes` 告警规则），提前发现趋势

```bash
# 监控告警参考：当根分区使用率超过 85% 时发送告警
# Prometheus 告警规则示例
alert: DiskSpaceLow
expr: (1 - node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"}) * 100 > 85
for: 5m
labels:
  severity: warning
annotations:
  summary: "磁盘空间不足，使用率超过 85%"
```

## 小结

磁盘管理需要养成习惯：定期 `df -h` 查看趋势，配置 `logrotate` 自动清理旧日志，用 `ncdu` 快速定位空间消耗点。LVM 提供了灵活的在线扩容能力，适合生产环境。最重要的是设置监控告警，在磁盘用完之前就采取行动。
