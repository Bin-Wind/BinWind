# Shell 脚本自动化入门

Shell 脚本是运维自动化的基础技能。无论是定时备份、批量处理还是服务监控，几行 Shell 脚本往往比写一个完整的程序更高效。本文从零开始，介绍 Bash 脚本的变量、条件判断、循环、函数和错误处理，最后给出一个实用的备份脚本示例。

## 脚本基础结构

```bash
#!/bin/bash
# 这是一个注释
# 作用：将当前日期写入日志文件

set -euo pipefail   # 严格模式（后面会详细说明）

echo "脚本开始执行"
date >> /var/log/myapp.log
echo "脚本执行完毕"
```

第一行 `#!/bin/bash` 称为 Shebang，告诉系统用哪个解释器执行脚本。赋予执行权限：

```bash
chmod +x script.sh
./script.sh
```

## 变量与参数

```bash
#!/bin/bash
# 变量赋值（等号两边不能有空格）
name="production"
count=42

# 使用变量：${} 是推荐写法，明确变量边界
echo "环境：${name}"
echo "数量：${count}"

# 特殊变量
echo "脚本名：$0"
echo "第一个参数：$1"
echo "参数个数：$#"
echo "所有参数：$@"
echo "上一条命令的退出码：$?"

# 参数展开：设置默认值
backup_dir="${1:-/tmp/backup}"    # 如果 $1 为空，使用默认值
echo "备份目录：${backup_dir}"

# 命令替换
current_time=$(date +%Y%m%d_%H%M%S)
file_count=$(ls -1 /var/log/*.log 2>/dev/null | wc -l)
```

## 条件判断

```bash
#!/bin/bash
file="/etc/nginx/nginx.conf"

# 文件测试（注意 [ ] 内的空格是必需的）
if [ -f "$file" ]; then            # 是否为普通文件
    echo "$file 存在"
elif [ -d "$file" ]; then          # 是否为目录
    echo "$file 是一个目录"
else
    echo "$file 不存在"
fi

# 常用文件测试运算符
# [ -f file ]  文件存在且为普通文件
# [ -d dir ]   目录存在
# [ -x file ]  文件可执行
# [ -r file ]  文件可读
# [ -s file ]  文件存在且非空
# [ -z str ]   字符串为空
# [ -n str ]   字符串非空
# [ str1 = str2 ]  字符串相等
# [ n1 -eq n2 ]    数值相等
# [ n1 -gt n2 ]    数值大于
# [ n1 -lt n2 ]    数值小于

# 现代写法：使用 [[ ]] 更安全（支持正则，不需要转义）
if [[ "$1" =~ ^[0-9]+$ ]]; then
    echo "参数是纯数字"
fi

# && 和 || 的短路用法
[ -d /backup ] || mkdir -p /backup     # 目录不存在则创建
[ $? -eq 0 ] && echo "成功" || echo "失败"
```

## 循环

```bash
#!/bin/bash

# for 循环：遍历列表
for file in /var/log/*.log; do
    echo "处理文件：${file}"
    gzip -f "$file"
done

# for 循环：C 风格
for ((i=1; i<=5; i++)); do
    echo "第 ${i} 次尝试"
done

# while 循环：逐行读取文件
while IFS= read -r line; do
    echo "行内容：${line}"
done < /etc/hosts

# while 循环：条件等待
retry=0
max_retry=10
while ! curl -s http://localhost:8080/health > /dev/null; do
    retry=$((retry + 1))
    if [ $retry -ge $max_retry ]; then
        echo "服务启动失败"
        exit 1
    fi
    echo "等待服务启动... (${retry}/${max_retry})"
    sleep 5
done
echo "服务已就绪"
```

## 函数

```bash
#!/bin/bash

# 定义函数
log_info() {
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')   # local 限定变量作用域
    echo "[INFO] ${timestamp} - $1"
}

log_error() {
    local timestamp=$(date '+%Y-%m-%d %H:%M:%S')
    echo "[ERROR] ${timestamp} - $1" >&2            # 输出到 stderr
}

# 带返回值的函数
is_port_open() {
    local host=$1
    local port=$2
    timeout 3 bash -c "echo >/dev/tcp/${host}/${port}" 2>/dev/null
    return $?      # 0=成功（端口开放），非0=失败
}

# 使用函数
log_info "开始检查服务状态"
if is_port_open "localhost" "80"; then
    log_info "Nginx 端口已开放"
else
    log_error "Nginx 端口未开放"
fi
```

## 错误处理

```bash
#!/bin/bash

# 1. 严格模式 —— 推荐在所有脚本中使用
set -e          # 任何命令退出码非零就终止脚本
set -u          # 使用未定义的变量时报错
set -o pipefail # 管道中任何命令失败都视为失败

# 也可以一行写完
set -euo pipefail

# 2. trap：捕获信号和异常退出，做清理工作
cleanup() {
    echo "清理临时文件..."
    rm -rf /tmp/myscript-*
}
trap cleanup EXIT         # 脚本退出时执行 cleanup
trap 'echo "收到 Ctrl+C, 退出"' INT   # 捕获 SIGINT

# 3. 关键操作的错误检查
if ! mysql -u root -e "FLUSH TABLES;"; then
    log_error "MySQL FLUSH 失败，中断备份"
    exit 1
fi
```

## 实战：数据库备份脚本

```bash
#!/bin/bash
set -euo pipefail

# ===== 配置 =====
BACKUP_DIR="${BACKUP_DIR:-/backup/mysql}"
RETENTION_DAYS="${RETENTION_DAYS:-7}"
MYSQL_USER="${MYSQL_USER:-root}"
MYSQL_PASS="${MYSQL_PASS:-}"
DATE_STAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/mysqldump_${DATE_STAMP}.sql.gz"

# ===== 函数 =====
log() { echo "[$(date '+%F %T')] $*"; }
die() { log "ERROR: $*"; exit 1; }

# ===== 前置检查 =====
[ -d "$BACKUP_DIR" ] || mkdir -p "$BACKUP_DIR"
which mysqldump >/dev/null || die "mysqldump 未安装"

# ===== 执行备份 =====
log "开始备份数据库..."
mysqldump \
    -u "$MYSQL_USER" \
    -p"$MYSQL_PASS" \
    --single-transaction \
    --quick \
    --all-databases \
    | gzip > "$BACKUP_FILE"

# 验证备份文件
if [ ! -s "$BACKUP_FILE" ]; then
    die "备份文件为空，备份可能失败"
fi
log "备份完成：${BACKUP_FILE} (大小: $(du -h "$BACKUP_FILE" | cut -f1))"

# ===== 清理过期备份 =====
log "清理 ${RETENTION_DAYS} 天前的旧备份..."
find "$BACKUP_DIR" -name "mysqldump_*.sql.gz" -mtime "+${RETENTION_DAYS}" -delete

log "备份任务完成"
```

将脚本加入 crontab 实现自动化：

```bash
# 每天凌晨 2 点执行备份
0 2 * * * /usr/local/bin/mysql_backup.sh >> /var/log/backup.log 2>&1
```

## 脚本调试技巧

```bash
# 执行时开启详细输出（每行命令都会打印）
bash -x myscript.sh

# 或者在脚本内指定行范围开启调试
set -x             # 开始跟踪
# ... 需要调试的代码 ...
set +x             # 停止跟踪

# 使用 shellcheck 进行静态检查
shellcheck myscript.sh
```

## 小结

Shell 脚本的精髓在于"自动化重复操作"。掌握变量、条件、循环和函数这四个基本构件后，再加上 `set -euo pipefail` 的严格模式和 `trap` 清理机制，就能写出健壮实用的运维脚本。从备份、部署到健康检查，Shell 脚本始终是最直接、最轻量的运维工具。
