# Git 服务器搭建（Gitea）

对于追求数据隐私、代码自主可控的团队来说，自建 Git 服务器是一个理想选择。Gitea 是一个轻量级的自托管 Git 服务，使用 Go 语言编写，部署方便、资源占用低，功能足以替代 GitHub/GitLab 的核心场景。本文将介绍从安装到备份的完整流程。

## 为什么选择 Gitea

- **轻量**：单二进制文件，最低 512MB 内存即可运行
- **私有仓库无限**：不受 GitHub 私有仓库数量或协作者限制
- **数据自主**：所有代码和数据都存放在自己的服务器上
- **功能齐全**：支持 Issues、Pull Requests、Wiki、CI/CD（Gitea Actions）、Webhooks
- **兼容性好**：API 兼容 GitHub，可直接使用 Git 命令行和图形客户端

## Docker 方式安装（推荐）

```bash
# 创建数据目录
mkdir -p /opt/gitea/{data,config}

# Docker Compose 部署
cat > /opt/gitea/docker-compose.yml << 'EOF'
version: "3"

services:
  gitea:
    image: gitea/gitea:latest
    container_name: gitea
    environment:
      - USER_UID=1000
      - USER_GID=1000
      - GITEA__server__DOMAIN=gitea.example.com
      - GITEA__server__SSH_DOMAIN=gitea.example.com
      - GITEA__server__ROOT_URL=https://gitea.example.com/
    restart: always
    volumes:
      - ./data:/data
      - ./config:/etc/gitea
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro
    ports:
      - "3000:3000"
      - "2222:22"          # SSH 端口（避免与主机 SSH 冲突）
    depends_on:
      - db

  db:
    image: postgres:15-alpine
    container_name: gitea-db
    restart: always
    environment:
      - POSTGRES_USER=gitea
      - POSTGRES_PASSWORD=your_secure_password
      - POSTGRES_DB=gitea
    volumes:
      - ./postgres:/var/lib/postgresql/data
EOF

cd /opt/gitea
docker compose up -d
```

服务启动后，访问 `http://your-server-ip:3000` 即可看到 Gitea 的安装页面。

## 二进制方式安装

```bash
# 下载 Gitea 二进制文件
wget -O /usr/local/bin/gitea https://dl.gitea.com/gitea/1.22.0/gitea-1.22.0-linux-amd64
chmod +x /usr/local/bin/gitea

# 创建运行用户
sudo adduser --system --group --home /var/lib/gitea git

# 创建目录结构
mkdir -p /var/lib/gitea/{custom,data,log}
chown -R git:git /var/lib/gitea
chmod -R 750 /var/lib/gitea
mkdir -p /etc/gitea
chown root:git /etc/gitea
chmod 770 /etc/gitea
```

## Systemd 服务配置

```ini
# /etc/systemd/system/gitea.service
[Unit]
Description=Gitea - Git with a cup of tea
After=network.target postgresql.service
Wants=postgresql.service

[Service]
User=git
Group=git
WorkingDirectory=/var/lib/gitea
ExecStart=/usr/local/bin/gitea web --config /etc/gitea/app.ini
Restart=always
RestartSec=5
Environment=USER=git HOME=/var/lib/gitea GITEA_WORK_DIR=/var/lib/gitea
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now gitea
sudo systemctl status gitea
```

## 初始配置

初次访问 Gitea 时，页面会引导你完成数据库和站点配置：

- **数据库**：选择 PostgreSQL，填入数据库连接信息（主机 `localhost`、数据库 `gitea`、用户名和密码）
- **站点名称**：自定义你的 Gitea 站点名称
- **域名和 URL**：如 `gitea.example.com` 和 `https://gitea.example.com/`
- **SSH 端口**：如果 Docker 映射为 2222，此处填写 2222
- **管理员账户**：在此页面创建第一个管理员用户

提交后，Gitea 将自动初始化数据库并重启。

## SSH 设置

要让 Gitea 支持 Git over SSH，需要在服务器上创建 `git` 用户并配置 SSH 密钥验证。

Docker 版本的 SSH 配置（使用主机 2222 端口）：

```bash
# 用户生成 SSH 密钥后，将公钥添加到 Gitea Web UI 的 SSH Keys 页面
# 客户端连接时使用 2222 端口
git clone ssh://git@gitea.example.com:2222/username/repo.git
```

二进制版本的 SSH 配置更简单，Gitea 直接使用系统 `git` 用户处理 SSH 连接，用户只需将公钥添加到 Web UI 即可。

## 备份策略

Gitea 的备份包含两部分：数据库和仓库数据。

```bash
#!/bin/bash
# Gitea 备份脚本
set -euo pipefail

BACKUP_DIR="/backup/gitea"
RETENTION_DAYS=30
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"

# 方式一：使用 Gitea dump 命令（推荐）
sudo -u git gitea dump -c /etc/gitea/app.ini --file "${BACKUP_DIR}/gitea_dump_${DATE}.zip"

# 方式二：分别备份
# 1. 备份数据库
sudo -u postgres pg_dump gitea | gzip > "${BACKUP_DIR}/gitea_db_${DATE}.sql.gz"

# 2. 备份仓库目录
tar -czf "${BACKUP_DIR}/gitea_repos_${DATE}.tar.gz" -C /var/lib/gitea data/

# 3. 备份配置文件
cp /etc/gitea/app.ini "${BACKUP_DIR}/app.ini.${DATE}"

# 清理旧备份
find "$BACKUP_DIR" -type f -mtime "+${RETENTION_DAYS}" -delete

echo "Gitea 备份完成: ${BACKUP_DIR}"
```

恢复备份：

```bash
# 解压 dump 文件
unzip gitea_dump_20250101_120000.zip

# 恢复数据库
psql -U gitea -h localhost < gitea-db.sql

# 恢复仓库
cp -r repos/* /var/lib/gitea/data/repositories/

# 修改权限
chown -R git:git /var/lib/gitea/data/
```

## 服务端 Git Hooks

Gitea 支持在仓库设置中配置 Git Hooks，用于实现自动检查、通知等功能：

```bash
# Gitea 服务端 Hook 示例：拒绝包含 TODO 标记的提交
#!/bin/bash
# 存储路径：/var/lib/gitea/data/gitea-repositories/{owner}/{repo}.git/hooks/pre-receive

while read oldrev newrev refname; do
    if git diff --name-only "$oldrev" "$newrev" | grep -q "secret.key"; then
        echo "错误：禁止提交 secret.key 文件"
        exit 1
    fi
done
```

通过 Web UI 配置 Webhook 也很方便，支持向 Jenkins、钉钉、企业微信等平台推送事件通知。

## 小结

Gitea 非常适合小中型团队或个人自建 Git 服务。Docker 部署只需几分钟就能完成，二进制部署资源占用更低。自带 Actions CI/CD 功能可以替代 GitHub Actions 的常见场景。定期备份仓库和数据库是运维的关键操作，建议结合 cron 定时任务自动化执行。
