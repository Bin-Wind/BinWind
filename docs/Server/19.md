# CI/CD 流水线搭建（GitHub Actions）

持续集成和持续部署（CI/CD）是现代软件开发的基础实践。GitHub Actions 深度集成在 GitHub 仓库中，无需额外服务器即可运行自动化流水线。本文将介绍 GitHub Actions 的核心概念和常见场景配置，帮助你快速搭建从测试到部署的自动化流水线。

## 工作流基础

GitHub Actions 的工作流定义在 `.github/workflows/` 目录下，YAML 格式。一个典型的目录结构：

```
.github/
└── workflows/
    ├── test.yml          # 测试流水线
    ├── deploy.yml         # 部署流水线
    └── nightly-build.yml  # 定时构建
```

## 基础工作流结构

```yaml
# .github/workflows/test.yml
name: Run Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: 检出代码
        uses: actions/checkout@v4

      - name: 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"

      - name: 安装依赖
        run: npm ci

      - name: 运行测试
        run: npm test

      - name: 上传测试报告
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: test-results
          path: test-results/
```

## 触发器

GitHub Actions 支持多种触发方式：

```yaml
on:
  # 代码推送
  push:
    branches: [main]
    tags: ["v*"]

  # Pull Request
  pull_request:
    branches: [main]
    types: [opened, synchronize, reopened]

  # 定时触发（使用 UTC 时间）
  schedule:
    - cron: "0 2 * * *"     # 每天凌晨 2 点（UTC）

  # 手动触发
  workflow_dispatch:
    inputs:
      environment:
        description: "部署环境"
        required: true
        type: choice
        options:
          - staging
          - production
```

## 常用 Action

```yaml
steps:
  # 检出代码
  - uses: actions/checkout@v4

  # Node.js 环境
  - uses: actions/setup-node@v4
    with:
      node-version: "20"

  # Python 环境
  - uses: actions/setup-python@v5
    with:
      python-version: "3.12"

  # Docker 构建和推送
  - uses: docker/setup-buildx-action@v3
  - uses: docker/login-action@v3
    with:
      registry: ghcr.io
      username: ${{ github.actor }}
      password: ${{ secrets.GITHUB_TOKEN }}
  - uses: docker/build-push-action@v5
    with:
      push: true
      tags: ghcr.io/${{ github.repository }}:latest
      cache-from: type=gha
      cache-to: type=gha,mode=max

  # SSH 远程部署
  - uses: appleboy/ssh-action@v1.0.0
    with:
      host: ${{ secrets.SERVER_HOST }}
      username: ${{ secrets.SERVER_USER }}
      key: ${{ secrets.SSH_PRIVATE_KEY }}
      script: |
        cd /opt/myapp
        docker compose pull
        docker compose up -d
```

## Secrets 管理

敏感信息（密码、密钥、Token）绝不能硬编码在工作流文件中，应该使用 GitHub Secrets：

```yaml
# 在 workflow 中使用 secrets
env:
  DATABASE_URL: ${{ secrets.DATABASE_URL }}

steps:
  - name: Deploy
    env:
      SSH_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
      API_TOKEN: ${{ secrets.API_TOKEN }}
    run: |
      echo "$SSH_KEY" > ~/.ssh/deploy_key
      chmod 600 ~/.ssh/deploy_key
```

Secrets 可以在两个层级设置：
- **Repository Secrets**：仅当前仓库可用，在 Settings > Secrets and variables > Actions 中配置
- **Environment Secrets**：绑定到特定环境（如 production），可以设置审批流程和保护规则

## Matrix 构建

Matrix 策略可以并行运行多个配置组合：

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 20, 22]
        os: [ubuntu-latest, windows-latest]
      fail-fast: false        # 一个版本失败不影响其他版本

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm test
```

## 完整示例：Node.js 应用到 Docker 部署

```yaml
# .github/workflows/ci-cd.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
    tags: ["v*"]
  pull_request:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # ===== 第一阶段：测试 =====
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 20, 22]

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: "npm"

      - run: npm ci
      - run: npm run lint
      - run: npm test
      - run: npm run build

  # ===== 第二阶段：构建并推送镜像 =====
  build-and-push:
    needs: test                       # 依赖 test 通过
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v4

      - name: 登录到 Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: 构建并推送镜像
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ===== 第三阶段：部署到服务器 =====
  deploy:
    needs: build-and-push
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    environment: production

    steps:
      - name: 部署到生产服务器
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /opt/myapp
            echo "${{ secrets.GITHUB_TOKEN }}" | docker login ghcr.io -u ${{ github.actor }} --password-stdin
            docker pull ghcr.io/${{ github.repository }}:${{ github.sha }}
            docker tag ghcr.io/${{ github.repository }}:${{ github.sha }} myapp:latest
            docker compose down
            docker compose up -d
            docker image prune -f
```

## 实用技巧

**Job 间共享数据**：

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Hello" > output.txt
      - uses: actions/upload-artifact@v4
        with:
          name: build-output
          path: output.txt

  deploy:
    needs: build
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: build-output
      - run: cat output.txt
```

**缓存依赖加速构建**：

```yaml
- uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-
```

**仅在特定路径变更时触发**：

```yaml
on:
  push:
    paths:
      - "src/**"
      - "package.json"
      - ".github/workflows/**"
    paths-ignore:
      - "docs/**"
      - "*.md"
```

## 小结

GitHub Actions 的核心思维是"事件驱动"：代码推送触发测试、标签推送触发发版、定时任务触发构建。通过合理拆分 Job 和利用 Matrix 并行化，可以构建高效稳定的 CI/CD 流水线。Secrets 管理、环境保护和部署审批是生产级流水线的必要考量。
