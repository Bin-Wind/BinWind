# Linux 常用命令速查手册

Linux 命令行是服务器运维的必备技能。本文汇总了日常工作中最常用的 Linux 命令，按功能分类整理，方便快速查阅。

## 文件与目录操作

### ls — 列出目录内容
```bash
ls -la          # 列出所有文件（含隐藏文件），显示详细信息
ls -lh          # 以人类可读的格式显示文件大小
ls -lt          # 按修改时间排序
ls -lS          # 按文件大小排序
```

### cd — 切换目录
```bash
cd /path/to/dir # 进入指定目录
cd ~            # 回到用户主目录
cd -            # 回到上一次所在的目录
cd ..           # 回到上级目录
```

### cp — 复制文件或目录
```bash
cp file1 file2              # 复制文件
cp -r dir1 dir2             # 递归复制目录
cp -p file1 file2           # 保留文件属性（权限、时间戳）
cp -a dir1 dir2             # 归档模式复制（保留所有属性）
```

### mv — 移动或重命名
```bash
mv oldname newname          # 重命名
mv file /target/path/       # 移动文件
mv -i source dest           # 覆盖前询问确认
```

### rm — 删除文件或目录
```bash
rm file                     # 删除文件
rm -r dir                   # 递归删除目录
rm -rf dir                  # 强制删除，不询问（慎用！）
rm -i file                  # 删除前逐一确认
```

### find — 查找文件
```bash
find /path -name "*.log"                # 按文件名查找
find /path -type f -mtime -7            # 查找7天内修改的文件
find /path -size +100M                  # 查找大于100MB的文件
find /path -name "*.tmp" -delete        # 查找并删除
find /path -type f -exec chmod 644 {} \;  # 对查找到的文件执行操作
```

### grep — 文本搜索
```bash
grep "pattern" file                     # 在文件中搜索模式
grep -r "pattern" /path                 # 递归搜索目录
grep -i "pattern" file                  # 忽略大小写
grep -v "pattern" file                  # 反向匹配（排除）
grep -n "pattern" file                  # 显示行号
grep -c "pattern" file                  # 统计匹配行数
grep -A 3 -B 2 "pattern" file          # 显示匹配行前后文
```

## 进程管理

### ps — 查看进程状态
```bash
ps aux                          # 显示所有进程的详细信息
ps aux | grep nginx             # 查找特定进程
ps -ef                          # 显示完整格式的进程信息
ps -eo pid,ppid,cmd,%mem,%cpu   # 自定义输出列
```

### top — 实时进程监控
```bash
top                     # 实时显示系统进程和资源使用
top -u username         # 只显示指定用户的进程
top -p PID              # 只监控特定进程
# 在 top 界面中: 按 1 查看各CPU核心, 按 M 按内存排序, 按 P 按CPU排序, 按 q 退出
```

### kill — 终止进程
```bash
kill PID                # 发送 TERM 信号（优雅终止）
kill -9 PID             # 发送 KILL 信号（强制终止）
kill -15 PID            # 同 kill，发送 TERM 信号
killall process_name    # 按名称终止所有匹配进程
pkill -f pattern        # 按命令行模式匹配并终止
```

## 磁盘与存储

### df — 查看磁盘空间使用
```bash
df -h                   # 以人类可读格式显示磁盘使用
df -h /data             # 查看指定目录所在分区的使用情况
df -i                   # 查看 inode 使用情况
```

### du — 查看目录/文件大小
```bash
du -sh /path            # 显示目录总大小
du -sh *                # 显示当前目录下各子目录/文件的大小
du -h --max-depth=1     # 只显示一层子目录的大小
du -sh * | sort -rh     # 按大小降序排列
```

## 权限管理

### chmod — 修改文件权限
```bash
chmod 755 script.sh             # rwxr-xr-x
chmod 644 file.txt              # rw-r--r--
chmod +x script.sh              # 添加执行权限
chmod -R 755 /var/www           # 递归修改目录及所有子文件
chmod u+w file                  # 给所有者添加写权限
```

### chown — 修改文件所有者
```bash
chown user:group file           # 同时修改所有者和所属组
chown -R user:group /path       # 递归修改
chown user file                 # 只修改所有者
```

## 压缩与归档

### tar — 打包压缩
```bash
tar -czvf archive.tar.gz /path      # 创建 gzip 压缩包
tar -xzvf archive.tar.gz            # 解压 gzip 压缩包
tar -cjvf archive.tar.bz2 /path     # 创建 bzip2 压缩包
tar -xzvf archive.tar.gz -C /dest   # 解压到指定目录
tar -tzvf archive.tar.gz            # 查看压缩包内容不解压
```

## 网络操作

### wget / curl — 文件下载与请求
```bash
wget https://example.com/file.tar.gz            # 下载文件
wget -c https://example.com/file.tar.gz          # 断点续传
wget -O custom_name.tar.gz URL                   # 指定保存文件名

curl https://api.example.com                     # 发送 GET 请求
curl -X POST -d "key=value" URL                  # 发送 POST 请求
curl -H "Content-Type: application/json" URL     # 添加请求头
curl -o output.txt URL                           # 保存到文件
curl -O URL                                      # 以远程文件名保存
curl -L URL                                      # 跟随重定向
```

### ssh — 远程登录
```bash
ssh user@host                   # 使用默认端口登录
ssh -p 2222 user@host           # 指定端口登录
ssh -i ~/.ssh/id_rsa user@host  # 使用指定私钥登录
ssh user@host "ls -la /tmp"     # 远程执行命令后退出
```

### scp — 远程文件传输
```bash
scp file.txt user@host:/remote/path/        # 上传文件到远程
scp user@host:/remote/file.txt ./           # 从远程下载文件
scp -r dir/ user@host:/remote/path/         # 递归传输目录
scp -P 2222 file.txt user@host:/path/       # 指定端口传输
```

## 系统服务管理

### systemctl — 管理 systemd 服务
```bash
systemctl status nginx          # 查看服务状态
systemctl start nginx           # 启动服务
systemctl stop nginx            # 停止服务
systemctl restart nginx         # 重启服务
systemctl reload nginx          # 重载配置（不中断服务）
systemctl enable nginx          # 设置开机自启
systemctl disable nginx         # 禁用开机自启
systemctl list-units --type=service  # 列出所有服务
journalctl -u nginx -f          # 查看服务日志（实时跟踪）
```

## 实用技巧

管道和重定向是 Linux 命令行的核心能力，将多个简单命令组合可以实现复杂的数据处理：

```bash
# 查找占用空间最大的10个文件
find / -type f -exec du -h {} + 2>/dev/null | sort -rh | head -10

# 统计 Nginx 日志中各 IP 的访问次数
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -20

# 查找并杀死僵尸进程
ps aux | grep defunct | awk '{print $2}' | xargs kill -9

# 批量重命名文件（将所有 .txt 改为 .md）
for f in *.txt; do mv "$f" "${f%.txt}.md"; done
```

熟练掌握以上命令，可以覆盖 80% 以上的日常运维场景。遇到不熟悉的参数时，`man command` 或 `command --help` 永远是最可靠的参考。
