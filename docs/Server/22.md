# 服务器安全加固清单

一台刚装完系统的 Linux 服务器暴露在公网上，平均几分钟内就会开始收到扫描和攻击尝试。安全不是一次性配置，而是一个持续的实践。本文整理了一份服务器安全加固的核心清单，每一项都可以直接执行。

## SSH 安全加固

SSH 是远程管理服务器的入口，也是攻击者最常尝试的突破口。

```bash
# 编辑 SSH 配置文件
sudo vim /etc/ssh/sshd_config
```

关键配置项：

```conf
# /etc/ssh/sshd_config
# === 禁止 root 直接登录 ===
PermitRootLogin no

# === 仅允许密钥认证（禁用密码登录） ===
PasswordAuthentication no
PubkeyAuthentication yes

# === 更改默认端口（降低扫描量，并非安全措施本身） ===
Port 2222

# === 限制登录用户 ===
AllowUsers deploy ops

# === 限制登录尝试 ===
MaxAuthTries 3
MaxSessions 5

# === 断开空闲连接 ===
ClientAliveInterval 300
ClientAliveCountMax 0
```

修改后验证配置并重启：

```bash
sudo sshd -t                       # 验证语法
sudo systemctl restart sshd         # 重启服务
```

注意：重启 SSH 前，务必保持一个活跃的 SSH 会话作为后备，以防配置错误导致自己被锁在外面。

## fail2ban 防暴力破解

`fail2ban` 监控日志文件，检测到多次失败的登录尝试后自动封禁来源 IP。

```bash
sudo apt install fail2ban

# 创建本地配置（覆盖默认值）
sudo tee /etc/fail2ban/jail.local << 'EOF'
[DEFAULT]
bantime = 3600                      # 封禁时间（秒）
findtime = 600                      # 统计窗口（秒）
maxretry = 5                        # 最大尝试次数
ignoreip = 127.0.0.1/8 192.168.0.0/16  # 白名单 IP

[sshd]
enabled = true
port = ssh
logpath = %(sshd_log)s
backend = %(sshd_backend)s

# 自定义 Nginx 防护
[nginx-http-auth]
enabled = true
port = http,https
logpath = /var/log/nginx/error.log

[nginx-botsearch]
enabled = true
port = http,https
logpath = /var/log/nginx/access.log
maxretry = 2
EOF

sudo systemctl enable --now fail2ban
```

常用管理命令：

```bash
sudo fail2ban-client status           # 查看总体状态
sudo fail2ban-client status sshd      # 查看 sshd jail 状态
sudo fail2ban-client set sshd unbanip 1.2.3.4   # 手动解封
```

## 防火墙：UFW

Ubuntu 的 UFW（Uncomplicated Firewall）是 iptables 的友好封装。

```bash
sudo apt install ufw

# 默认策略：拒绝入站，允许出站
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 开放必要的端口
sudo ufw allow ssh                   # 或指定端口 sudo ufw allow 2222/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 限制端口（内置速率限制，防止洪水攻击）
sudo ufw limit ssh

# 启用防火墙
sudo ufw enable
sudo ufw status verbose
```

更精细的规则：

```bash
# 允许特定 IP 访问
sudo ufw allow from 203.0.113.4 to any port 22

# 允许子网访问数据库端口
sudo ufw allow from 10.0.0.0/8 to any port 5432

# 删除规则
sudo ufw status numbered
sudo ufw delete 3
```

## 自动安全更新

无人值守更新确保安全补丁及时安装。

```bash
sudo apt install unattended-upgrades
sudo dpkg-reconfigure --priority=low unattended-upgrades   # 选择 "Yes"
```

手动配置更精细的策略：

```conf
# /etc/apt/apt.conf.d/50unattended-upgrades
Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}-security";   # 仅安全更新
    // "${distro_id}:${distro_codename}-updates";  # 稳定更新（通常不启用）
};

Unattended-Upgrade::AutoFixInterruptedDpkg "true";
Unattended-Upgrade::MinimalSteps "true";

# 自动重启（需要时）
Unattended-Upgrade::Automatic-Reboot "false";
Unattended-Upgrade::Automatic-Reboot-Time "02:00";

# 通知邮件
Unattended-Upgrade::Mail "admin@example.com";
```

## 安全审计工具

### Lynis：系统安全审计

```bash
sudo apt install lynis
sudo lynis audit system
```

Lynis 会扫描系统并给出安全评分和改进建议，涵盖文件权限、内核参数、已安装软件等方面。

### rkhunter：Rootkit 扫描

```bash
sudo apt install rkhunter
sudo rkhunter --update                   # 更新特征库
sudo rkhunter --check --sk               # 快速扫描
sudo rkhunter --checkall                 # 完整扫描
```

### 文件完整性监控：AIDE

AIDE（Advanced Intrusion Detection Environment）建立文件校验基准，之后检测文件被篡改：

```bash
sudo apt install aide

# 初始化数据库（建立基准）
sudo aideinit
sudo mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db

# 定期检查
sudo aide --check
```

## 最小化原则

```bash
# 列出所有监听端口，关闭不需要的服务
sudo ss -tlnp

# 列出所有 systemd 服务
systemctl list-unit-files --state=enabled

# 禁用不需要的服务
sudo systemctl disable --now bluetooth
sudo systemctl disable --now cups
sudo systemctl disable --now avahi-daemon

# 删除不需要的软件包
sudo apt purge telnetd rsh-server talkd
sudo apt autoremove
```

## 内核安全参数

```bash
# /etc/sysctl.d/99-security.conf
# 防止 IP 欺骗
net.ipv4.conf.all.rp_filter = 1

# 忽略 ICMP 广播请求（防止 SMURF 攻击）
net.ipv4.icmp_echo_ignore_broadcasts = 1

# 忽略 ICMP 重定向
net.ipv4.conf.all.accept_redirects = 0
net.ipv6.conf.all.accept_redirects = 0

# 启用 SYN Cookie 防护（防止 SYN Flood）
net.ipv4.tcp_syncookies = 1

# 禁用 IPv6（如果不需要）
net.ipv6.conf.all.disable_ipv6 = 1

# 记录 Martian 包（来源地址不合理的包）
net.ipv4.conf.all.log_martians = 1
```

应用配置：

```bash
sudo sysctl -p /etc/sysctl.d/99-security.conf
```

## 定期维护策略

```bash
# 每周检查清单
# 1. 查看最近登录记录
last -n 20
sudo lastb -n 20            # 失败的登录尝试

# 2. 检查异常的定时任务
sudo crontab -l             # root
cat /etc/crontab
ls /etc/cron.*/

# 3. 检查异常用户
sudo cat /etc/passwd | grep -v nologin | grep -v false

# 4. 查看失败的 systemd 服务
systemctl --failed

# 5. 安全更新状态
apt list --upgradable 2>/dev/null | grep -i security
```

## 小结

安全加固是一项系统工程，从 SSH 访问控制、fail2ban 防护、UFW 防火墙，到自动更新、安全审计和定期检查——每一项都是防线的一环。核心原则是"最小权限"：只开放必要的端口、只安装必要的软件、只给必要的人访问权限。建议将安全审计（Lynis）加入月度的运维计划，持续跟踪安全评分。
