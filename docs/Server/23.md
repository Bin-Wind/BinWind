# 文件同步工具（rsync）详解

`rsync` 是 Linux 最强大的文件同步工具，在备份、镜像、部署等场景中无处不在。它的增量传输算法让它只传输变化的部分，比 `scp` 和 `cp` 高效得多。本文介绍 rsync 的常用选项、远程同步和增量备份方案。

## 基础语法

```bash
# 基本格式
rsync [选项] 源 目标

# 本地同步：将 source/ 目录内容复制到 dest/
rsync -av source/ dest/

# 注意：source/ 表示复制目录内容（不包含目录本身）
#      source  表示复制整个目录（包含目录本身）
```

## 常用选项

| 选项 | 含义 |
|------|------|
| `-a` | 归档模式，等价于 `-rlptgoD`（递归、保留符号链接、权限、时间戳、组、所有者、设备文件） |
| `-v` | 详细输出 |
| `-z` | 传输时压缩（适合慢速网络） |
| `-P` | 等价于 `--partial --progress`（断点续传 + 显示进度） |
| `-h` | 人类可读的大小显示 |
| `-n` / `--dry-run` | 模拟运行，不实际传输 |
| `--delete` | 删除目标中源不存在的文件（保持完全同步） |
| `--exclude` | 排除匹配的文件或目录 |
| `--bwlimit` | 限速（单位 KB/s） |
| `--max-size` | 只传输小于指定大小的文件 |

```bash
# 显示进度、压缩、保留属性
rsync -avzP /data/ /backup/

# 增量同步：同时删除目标多余的旧文件
rsync -avzP --delete /data/ /mirror/

# 排除指定目录和文件类型
rsync -avzP \
    --exclude 'node_modules' \
    --exclude '.git' \
    --exclude '*.log' \
    --exclude '*.tmp' \
    /project/ /deploy/

# 限速同步（避免占满带宽影响业务）
rsync -avzP --bwlimit=1000 /large_files/ user@remote:/backup/

# 只传输小于 10MB 的文件
rsync -avzP --max-size=10M /source/ /dest/
```

## 远程同步（SSH）

rsync 默认通过 SSH 进行远程传输，安全且无额外配置。

```bash
# 从本地推送到远程
rsync -avz /local/dir/ user@remote:/remote/dir/

# 从远程拉取到本地
rsync -avz user@remote:/remote/dir/ /local/dir/

# 指定 SSH 端口
rsync -avz -e "ssh -p 2222" /local/ user@remote:/remote/

# 使用 SSH 密钥
rsync -avz -e "ssh -i ~/.ssh/deploy_key" /local/dist/ user@remote:/var/www/html/
```

常见的部署场景：

```bash
# 部署静态网站到生产服务器
rsync -avzP --delete \
    --exclude '.env' \
    --exclude '.htaccess' \
    -e "ssh -i ~/.ssh/deploy_key" \
    ./dist/ deploy@web-server:/var/www/site/
```

## 预览模式（Dry Run）

在正式执行前先用 `--dry-run` 确认会同步哪些文件，避免误操作。

```bash
# 先预览
rsync -avz --delete --dry-run /src/ /dest/

# 更详细的预览（列出每个文件的变化类型）
rsync -avz --delete --dry-run --itemize-changes /src/ /dest/
```

`--itemize-changes` 输出的变化类型标识：
- `>` 文件被传输（新增或更新）
- `*deleting` 文件将被删除
- `d` 目录
- `f` 文件
- `.` 属性未变化

## 增量备份方案：--link-dest

`--link-dest` 是 rsync 最强大的功能之一，通过硬链接实现空间高效的版本快照。未变更的文件创建硬链接（不占用额外磁盘空间），变更的文件正常复制。

```bash
#!/bin/bash
# 快照式增量备份脚本
set -euo pipefail

SRC="/data"
BACKUP_ROOT="/backup/snapshots"
DATE=$(date +%Y%m%d_%H%M%S)
CURRENT="${BACKUP_ROOT}/current"
NEW="${BACKUP_ROOT}/${DATE}"

# 首次备份：创建完整副本
if [ ! -d "$CURRENT" ]; then
    echo "首次备份，执行完整复制..."
    rsync -av "$SRC/" "$CURRENT/"
    cp -al "$CURRENT" "$NEW"
else
    # 增量备份：未变更文件使用硬链接
    rsync -av --delete \
        --link-dest="$CURRENT" \
        "$SRC/" "$NEW/"
fi

# 更新 current 符号链接
rm -f "$CURRENT"
ln -sf "$NEW" "$CURRENT"

echo "备份完成: ${NEW}"

# 删除 30 天前的旧快照
find "$BACKUP_ROOT" -maxdepth 1 -type d -name "202*" -mtime +30 -exec rm -rf {} \;
```

这种方案的优势：每次备份只占用变更文件的空间，可以低代价地保留长时间的历史版本。`ls -l` 会显示硬链接的文件 inode 的引用计数（大于 1 表示多个快照共享同一数据块）。

## 常用场景脚本

### 场景一：代码部署

```bash
#!/bin/bash
# deploy.sh —— 零停机部署
REMOTE="deploy@web01.example.com"
REMOTE_PATH="/var/www/app"
LOCAL_DIST="./dist"

# 同步到远程临时目录
rsync -avzP --delete \
    --exclude '.env' \
    --exclude 'storage/' \
    -e "ssh -i ~/.ssh/deploy" \
    "$LOCAL_DIST/" "$REMOTE:${REMOTE_PATH}_new"

# 切换符号链接（原子操作）
ssh -i ~/.ssh/deploy "$REMOTE" "
    ln -sfn ${REMOTE_PATH}_new ${REMOTE_PATH}_current;
    sudo systemctl reload php-fpm;
    # 保留旧版本便于回滚
    mv ${REMOTE_PATH}_new ${REMOTE_PATH}_$(date +%Y%m%d_%H%M%S)
"
```

### 场景二：定时备份到远程

```bash
# crontab 每天凌晨 2 点同步
# 0 2 * * * /usr/local/bin/nightly-sync.sh

#!/bin/bash
set -euo pipefail

SRC="/data/"
DEST="backup@backup-server:/backup/data"
LOG="/var/log/rsync-backup.log"

echo "===== $(date) 开始同步 =====" >> "$LOG"
rsync -avzP --delete --bwlimit=2000 \
    -e "ssh -i /root/.ssh/backup_key" \
    "$SRC" "$DEST" >> "$LOG" 2>&1
echo "===== $(date) 同步完成 =====" >> "$LOG"
```

### 场景三：两台服务器之间迁移数据

```bash
# 第一次全量同步（可中断可恢复）
rsync -avzP /data/ root@new-server:/data/

# 第二次增量同步（服务切换前的最终同步）
rsync -avzP --delete /data/ root@new-server:/data/

# 第三次最终同步（切换 DNS 前）
# 源端先暂停写入，然后快速增量
rsync -avzP --delete /data/ root@new-server:/data/
```

## 常见问题

```bash
# 权限拒绝：使用 sudo 保留所有权
sudo rsync -avz /source/ /dest/

# 大文件传输中断后续传
# -P 中的 --partial 会保留部分传输的文件，下次自动续传
rsync -avP large_file.iso user@remote:/backup/

# 复制到 FAT/NTFS 等不支持权限的文件系统时
rsync -rltDv --chmod=ugo=rwX /source/ /mnt/usb/

# 比对两个目录差异（不复制）
rsync -avnc --delete /dir1/ /dir2/
```

## 小结

rsync 的核心优势在于增量算法和灵活度。`-avzP` 组合覆盖了大多数日常场景，`--delete` 保证完全同步，`--link-dest` 实现空间高效的增量快照。配合 `--dry-run` 预览和 `--bwlimit` 限速，rsync 可以安全高效地处理从几 KB 的配置文件到几 TB 的数据迁移。
