# Kubernetes 入门：Pod 与 Deployment

Kubernetes（K8s）已经成为容器编排的事实标准。它将运维的复杂性抽象为声明式配置——你描述"想要什么状态"，K8s 负责"如何达到那个状态"。本文从核心概念出发，介绍 Pod、Deployment、Service 以及 kubectl 的基本操作。

## 什么是 Kubernetes

Kubernetes 解决的核心问题：在一组服务器上运行和管理容器化应用。它负责：

- **调度**：决定容器运行在哪台机器上
- **自愈**：容器挂了自动重启，节点故障自动迁移
- **伸缩**：根据负载增减容器数量
- **服务发现**：自动为容器提供 DNS 名称和负载均衡
- **配置管理**：统一管理应用配置和密钥

## 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **Pod** | K8s 最小调度单元，包含一个或多个容器 | 一个"逻辑主机" |
| **Deployment** | 管理 Pod 的副本数和更新策略 | 应用的"期望状态声明" |
| **Service** | 为一组 Pod 提供稳定的访问入口和负载均衡 | 一个"固定的访问地址" |
| **ConfigMap** | 非敏感的配置数据 | 环境变量或配置文件 |
| **Secret** | 敏感的配置数据（密码、Token） | Base64 编码的密钥 |
| **Namespace** | 资源隔离的逻辑分区 | 项目/环境的分隔 |

## Pod YAML

Pod 是最基础的工作负载单元，但通常不直接创建 Pod，而是通过 Deployment 管理。

```yaml
# pod.yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-demo
  labels:
    app: nginx
spec:
  containers:
    - name: nginx
      image: nginx:1.25-alpine
      ports:
        - containerPort: 80
      env:
        - name: NGINX_HOST
          value: "example.com"
      resources:
        requests:
          memory: "64Mi"
          cpu: "100m"
        limits:
          memory: "128Mi"
          cpu: "200m"
```

```bash
kubectl apply -f pod.yaml
kubectl get pods
kubectl describe pod nginx-demo
kubectl logs nginx-demo
kubectl exec -it nginx-demo -- sh
kubectl delete pod nginx-demo
```

## Deployment YAML

Deployment 管理 ReplicaSet（副本集），ReplicaSet 管理 Pod，形成一个三层结构。

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
  labels:
    app: web-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web-app
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1          # 滚动更新时额外可创建的 Pod 数
      maxUnavailable: 0    # 滚动更新时不可用的最大 Pod 数
  template:
    metadata:
      labels:
        app: web-app
    spec:
      containers:
        - name: web
          image: nginx:1.25-alpine
          ports:
            - containerPort: 80
          resources:
            requests:
              memory: "64Mi"
              cpu: "100m"
            limits:
              memory: "128Mi"
              cpu: "200m"
          readinessProbe:
            httpGet:
              path: /health
              port: 80
            initialDelaySeconds: 5
            periodSeconds: 10
          livenessProbe:
            httpGet:
              path: /health
              port: 80
            initialDelaySeconds: 15
            periodSeconds: 20
```

常用 Deployment 操作：

```bash
kubectl apply -f deployment.yaml
kubectl get deployments
kubectl get pods -l app=web-app               # 按标签筛选 Pod
kubectl describe deployment web-app

# 滚动更新
kubectl set image deployment/web-app web=nginx:1.26-alpine
kubectl rollout status deployment/web-app     # 观察更新进度
kubectl rollout history deployment/web-app    # 查看发布历史
kubectl rollout undo deployment/web-app       # 回滚到上一版本
kubectl rollout undo deployment/web-app --to-revision=2  # 回滚到指定版本

# 扩缩容
kubectl scale deployment web-app --replicas=5
```

## Service YAML

Service 为一组 Pod 提供稳定的访问入口。K8s 提供三种 Service 类型：

| 类型 | 用途 | 访问方式 |
|------|------|----------|
| **ClusterIP**（默认） | 集群内部访问 | 仅在集群内可访问 |
| **NodePort** | 通过节点 IP + 端口暴露 | `<NodeIP>:<NodePort>` |
| **LoadBalancer** | 云厂商负载均衡器 | 外部 IP 直接访问 |

```yaml
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: web-app-svc
spec:
  type: ClusterIP       # 或 NodePort / LoadBalancer
  selector:
    app: web-app        # 选择带有此标签的 Pod
  ports:
    - name: http
      protocol: TCP
      port: 80          # Service 自己监听的端口
      targetPort: 80    # Pod 中容器的端口
```

```bash
kubectl apply -f service.yaml
kubectl get svc
kubectl describe svc web-app-svc

# 在集群内访问 Service
kubectl run curl-test --image=curlimages/curl -it --rm -- sh
# 然后在容器内：
curl http://web-app-svc          # 通过 Service 名访问
curl http://web-app-svc.namespace.svc.cluster.local  # 完整 FQDN
```

## ConfigMap 和 Secret

```yaml
# configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_ENV: "production"
  DB_HOST: "db.internal"
  app.conf: |           # 多行配置文件
    [server]
    port = 8080
    workers = 4
---
# secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
type: Opaque
data:
  DB_PASSWORD: c2VjdXJlX3Bhc3N3b3Jk     # base64 编码值
  API_KEY: YXBpX2tleV8xMjM0NQo=
```

在 Deployment 中使用：

```yaml
spec:
  containers:
    - name: web
      image: nginx
      envFrom:
        - configMapRef:
            name: app-config
        - secretRef:
            name: app-secret
      env:
        - name: SPECIFIC_KEY
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: APP_ENV
      volumeMounts:
        - name: config
          mountPath: /etc/app
  volumes:
    - name: config
      configMap:
        name: app-config
```

## kubectl 常用操作速查

```bash
# 查看资源
kubectl get pods                          # 列出所有 Pod
kubectl get pods -o wide                  # 包含 IP 和节点信息
kubectl get all                           # 查看所有资源
kubectl get nodes                         # 查看节点
kubectl get ns                            # 查看命名空间

# 详细信息
kubectl describe pod <name>               # Pod 详情（事件、状态）
kubectl describe node <name>              # 节点详情

# 日志
kubectl logs <pod>                        # 查看日志
kubectl logs <pod> -f                     # 实时跟踪日志
kubectl logs <pod> -c <container>         # 多容器时指定容器
kubectl logs --previous <pod>             # 查看上一个容器实例的日志（crash 后）

# 进入容器
kubectl exec -it <pod> -- bash            # 进入容器执行命令

# 端口转发（本地调试）
kubectl port-forward pod/<pod-name> 8080:80    # 本地 8080 → Pod 80
kubectl port-forward svc/<svc-name> 8080:80    # 本地 8080 → Service 80

# 管理资源
kubectl apply -f file.yaml               # 创建/更新资源
kubectl delete -f file.yaml              # 删除资源
kubectl edit deployment <name>           # 在线编辑

# 查看 YAML 输出（可用于导出或备份）
kubectl get deployment web-app -o yaml
```

## 快速上手 Demo

```bash
# 1. 确保 K8s 集群可用
kubectl cluster-info

# 2. 创建 Deployment
kubectl create deployment nginx-demo --image=nginx:alpine --replicas=2

# 3. 暴露为 Service
kubectl expose deployment nginx-demo --port=80 --type=NodePort

# 4. 查看状态
kubectl get pods,svc

# 5. 访问服务（minikube 环境）
minikube service nginx-demo

# 6. 扩缩容
kubectl scale deployment nginx-demo --replicas=5

# 7. 查看 Pod 分布
kubectl get pods -o wide

# 8. 清理
kubectl delete service nginx-demo
kubectl delete deployment nginx-demo
```

## 小结

Kubernetes 的核心心智模型是"声明式"：用 YAML 描述 Pod、Deployment、Service 的目标状态，K8s 控制循环（control loop）持续将实际状态调整为目标状态。Pod 是最小调度单元，Deployment 管理 Pod 的副本和滚动更新，Service 提供稳定的访问入口。掌握这三个核心对象和 kubectl 基础命令，就已经具备了将应用部署到 K8s 的入门能力。
