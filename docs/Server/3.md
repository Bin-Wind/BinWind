# SSH 免密登录配置指南

每次通过 SSH 连接服务器都要输入密码，既繁琐又不安全。SSH 密钥认证不仅能免去输入密码的麻烦，还能显著提升安全性。本文将详细介绍如何配置 SSH 免密登录。

## 工作原理

SSH 密钥认证基于非对称加密：你持有一对密钥——私钥（留在本地，绝不外传）和公钥（部署到服务器）。连接时，服务器用公钥加密一个挑战，只有持有对应私钥的客户端才能解密应答，从而完成身份验证。

## 生成 SSH 密钥对

推荐使用 Ed25519 算法，它比 RSA 更安全且性能更好：

```bash
# 生成 Ed25519 密钥对（推荐）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 如果系统不支持 Ed25519，使用 RSA 4096
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

执行过程中会提示：
- **保存路径**：直接回车使用默认路径 `~/.ssh/id_ed25519`
- **密码短语（passphrase）**：可选，设置了则每次使用密钥时需要输入。生产环境建议设置以增加安全性

生成后查看密钥：
```bash
ls -la ~/.ssh/
# id_ed25519      — 私钥（保密，不可分享）
# id_ed25519.pub  — 公钥（可部署到服务器）
```

## 将公钥复制到服务器

有三种方式将公钥部署到目标服务器：

### 方式一：ssh-copy-id（推荐）
```bash
ssh-copy-id user@your-server.com

# 如果 SSH 端口不是默认的 22
ssh-copy-id -p 2222 user@your-server.com
```

执行后会提示输入一次密码，之后即可免密登录。

### 方式二：手动复制
```bash
cat ~/.ssh/id_ed25519.pub | ssh user@your-server.com "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

### 方式三：直接追加
```bash
# 先在服务器上创建 .ssh 目录
ssh user@your-server.com "mkdir -p ~/.ssh"

# 将本地公钥追加到服务器的 authorized_keys
scp ~/.ssh/id_ed25519.pub user@your-server.com:~/.ssh/tmp_key
ssh user@your-server.com "cat ~/.ssh/tmp_key >> ~/.ssh/authorized_keys && rm ~/.ssh/tmp_key"
```

## 权限设置

SSH 对文件权限要求非常严格，权限不正确会导致密钥认证失败：

```bash
# 在服务器上执行
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chown -R $USER:$USER ~/.ssh
```

本地私钥权限同样需要正确：
```bash
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

## 配置 SSH Config 简化连接

如果你管理多台服务器，为每台记住 IP、端口、用户名会很痛苦。在 `~/.ssh/config` 文件中配置别名可以大大简化操作：

```bash
# ~/.ssh/config
Host prod-web
    HostName 192.168.1.100
    User deploy
    Port 22
    IdentityFile ~/.ssh/id_ed25519

Host staging-api
    HostName staging.example.com
    User admin
    Port 2222
    IdentityFile ~/.ssh/staging_rsa

Host jump-box
    HostName 10.0.0.1
    User ops

# 通过跳板机访问内网服务器
Host internal-db
    HostName 192.168.100.50
    User dbuser
    ProxyJump jump-box
```

配置完成后，只需 `ssh prod-web` 即可连接，无需记忆完整命令。

## 常见问题排查

### 1. 权限过松（Permissions too open）
```
Permissions 0644 for '/home/user/.ssh/id_ed25519' are too open.
```
**解决**：私钥权限必须是 600：
```bash
chmod 600 ~/.ssh/id_ed25519
```

### 2. authorized_keys 权限错误
```
Authentication refused: bad ownership or modes for directory
```
**解决**：确保 .ssh 目录和 authorized_keys 权限正确：
```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

### 3. 使用了错误的密钥
```
Permission denied (publickey)
```
**解决**：使用 `-v` 参数查看详细调试信息，确认客户端提供了正确的密钥：
```bash
ssh -v user@server
ssh -vvv user@server   # 更详细的调试输出
```

### 4. 服务器端未开启密钥认证
**解决**：检查服务器的 `/etc/ssh/sshd_config`：
```bash
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
PasswordAuthentication yes   # 测试成功后建议关闭
```

修改后重启 SSH 服务：
```bash
sudo systemctl restart sshd
```

### 5. Connection refused（端口不通）
**解决**：检查服务器 SSH 服务是否运行、端口是否正确、防火墙是否放行：
```bash
sudo systemctl status sshd
sudo ufw status
```

## 安全加固建议

配置好免密登录后，建议进一步加固 SSH 安全：

```bash
# /etc/ssh/sshd_config
# 禁用 root 直接登录
PermitRootLogin no

# 测试免密登录成功后，禁用密码认证
PasswordAuthentication no

# 如果可能，更改默认端口（22→非标端口）
Port 2233

# 限制登录用户
AllowUsers deploy ops

# 设置登录超时
ClientAliveInterval 300
ClientAliveCountMax 2
```

每次修改 sshd_config 后，务必在另一个终端窗口中保持现有连接，以防配置错误导致无法登入。确认新配置正确后再关闭原会话。
