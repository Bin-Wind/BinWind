# Nginx HTTPS 证书配置 (Let's Encrypt)

HTTPS 已成为网站的标配，不仅保护用户数据安全，还影响搜索引擎排名。Let's Encrypt 提供免费的 SSL/TLS 证书，配合 Certbot 工具可以实现全自动的证书申请和续期。本文将详细介绍如何在 Nginx 上配置 Let's Encrypt 证书。

## 为什么选择 Let's Encrypt

Let's Encrypt 是由非营利组织 ISRG 运营的免费证书颁发机构，具有以下优势：
- **完全免费**，无需任何费用
- **自动化**，通过 ACME 协议自动验证和签发
- **受广泛信任**，证书被所有主流浏览器信任
- **短期证书**（90 天有效期），减少泄露风险

## 安装 Certbot

Certbot 是 Let's Encrypt 官方推荐的客户端工具：

```bash
# Debian/Ubuntu
sudo apt update
sudo apt install certbot python3-certbot-nginx

# CentOS/RHEL 7
sudo yum install epel-release
sudo yum install certbot python2-certbot-nginx

# CentOS/RHEL 8+
sudo dnf install certbot python3-certbot-nginx

# macOS
brew install certbot
```

## 申请证书

### 使用 Nginx 插件自动配置（最简单）

执行前确保 Nginx 已正确配置了域名，且该域名已解析到当前服务器：

```bash
# 单个域名
sudo certbot --nginx -d example.com

# 多个域名（主域名和 www 子域名）
sudo certbot --nginx -d example.com -d www.example.com

# 通配符证书需要 DNS 验证，不适用 --nginx 插件
```

交互过程中：
1. 输入邮箱（用于证书到期通知）
2. 同意服务条款
3. 选择是否自动将 HTTP 重定向到 HTTPS

### 仅获取证书（手动配置 Nginx）

如果希望手动管理 Nginx 配置：

```bash
sudo certbot certonly --nginx -d example.com

# 或使用 standalone 模式（需要暂时停止 Nginx）
sudo certbot certonly --standalone -d example.com

# 使用 webroot 模式（Nginx 正常运行）
sudo certbot certonly --webroot -w /var/www/html -d example.com
```

证书文件位置：
```
/etc/letsencrypt/live/example.com/
├── fullchain.pem    # 完整证书链（cert + chain）
├── privkey.pem      # 私钥
├── cert.pem         # 服务器证书
└── chain.pem        # 中间证书
```

## Nginx HTTPS 配置模板

获取证书后，手动配置一个安全的 HTTPS 站点：

```nginx
server {
    listen 80;
    server_name example.com www.example.com;
    # HTTP 重定向到 HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com www.example.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers on;

    # 会话缓存
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    resolver 8.8.8.8 8.8.4.4 valid=300s;
    resolver_timeout 5s;

    # HSTS（强制浏览器使用 HTTPS）
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;

    # 其他安全头
    add_header X-Frame-Options DENY always;
    add_header X-Content-Type-Options nosniff always;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 安全配置说明

### ssl_protocols
```nginx
ssl_protocols TLSv1.2 TLSv1.3;  # 禁用 TLSv1 和 TLSv1.1（不安全）
```

### ssl_ciphers
使用现代密码套件，优先 GCM 模式。可以用 Mozilla SSL Configuration Generator 生成推荐配置。

### HSTS（HTTP Strict Transport Security）
```nginx
add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;
```
- `max-age=63072000`：2 年有效期
- `includeSubDomains`：应用于所有子域名
- `preload`：允许加入浏览器的 HSTS 预加载列表

**提醒**：启用 `preload` 后，浏览器将强制使用 HTTPS，HTTP 访问将被拒绝。请确保所有子域名都已配置 HTTPS。

### OCSP Stapling
```nginx
ssl_stapling on;
ssl_stapling_verify on;
```

由服务器主动提供证书吊销状态，减少客户端单独查询 OCSP 服务器的开销，提升首次连接的响应速度。

## 证书自动续期

Let's Encrypt 证书有效期为 90 天，需要定期续期。Certbot 安装时会自动配置 systemd timer 或 cron 任务：

```bash
# 查看定时任务
sudo systemctl list-timers | grep certbot

# 手动测试续期流程（不会真正续期）
sudo certbot renew --dry-run

# 手动续期所有证书
sudo certbot renew
```

如果自动续期未配置，可以手动添加：

```bash
# 方式一：systemd timer（推荐）
sudo systemctl enable certbot.timer
sudo systemctl start certbot.timer

# 方式二：cron 任务
sudo crontab -e
# 添加以下行：每天凌晨 3 点检查并续期
0 3 * * * certbot renew --quiet --post-hook "systemctl reload nginx"
```

`--post-hook` 参数确保续期后自动重载 Nginx 配置，使新证书生效。

## 常见问题排查

### 1. 域名验证失败
```
Challenge failed for domain example.com
```
**原因**：域名未解析到当前服务器 IP，或 Nginx 未正确配置该域名的 server 块。
**解决**：
```bash
# 确认域名解析
dig example.com +short

# 确保 Nginx 正在监听 80 端口
sudo nginx -t && sudo nginx -s reload
```

### 2. 证书已过期
**解决**：手动强制续期：
```bash
sudo certbot renew --force-renewal
```

### 3. 443 端口被防火墙拦截
```bash
sudo ufw allow 443/tcp
sudo ufw allow 80/tcp   # Let's Encrypt 验证也需要 80 端口
```

### 4. OCSP Stapling 报错
```
ssl_stapling ignored, no OCSP responder URL in the server certificate
```
**解决**：检查 DNS 解析设置是否正常，确保 `resolver` 指令配置了可用的 DNS 服务器。

### 5. 证书速率限制
Let's Encrypt 同一域名每周最多申请 5 个证书。测试时使用 `--staging` 参数：
```bash
sudo certbot --staging --nginx -d example.com
```
