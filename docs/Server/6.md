# Docker 入门：从安装到第一个容器

Docker 是一个开源的容器化平台，它将应用及其依赖打包到一个轻量级、可移植的容器中，确保在任何环境中一致运行。本文面向 Docker 初学者，从零开始带你完成 Docker 的安装和基本使用。

## 什么是 Docker

简单来说，Docker 解决的问题就是"在我机器上明明可以跑"。"容器"比传统虚拟机更轻量——它共享宿主机内核，秒级启动，占用资源极少。核心概念：

- **镜像（Image）**：只读模板，包含运行应用所需的一切
- **容器（Container）**：镜像的运行实例
- **Dockerfile**：用于构建镜像的指令文件
- **仓库（Registry）**：存储和分发镜像的地方（如 Docker Hub）

## 安装 Docker

### macOS
直接下载安装 Docker Desktop：
```bash
brew install --cask docker
```
安装后启动 Docker Desktop 应用，等待状态栏图标显示"Docker Desktop is running"。

### Linux (Debian/Ubuntu)
```bash
# 卸载旧版本
sudo apt remove docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt update
sudo apt install ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加仓库
echo "deb [arch=$(dpkg --print-architecture) \
    signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

### 将当前用户加入 docker 组（Linux）
```bash
sudo usermod -aG docker $USER
# 重新登录后生效，或执行：
newgrp docker
```

### 验证安装
```bash
docker version         # 查看客户端和服务端版本
docker info            # 查看 Docker 引擎详细信息
docker run hello-world # 运行测试容器
```

## 镜像管理

### 拉取镜像
```bash
# 从 Docker Hub 拉取镜像
docker pull nginx:alpine         # 指定标签（alpine 是精简版）
docker pull node:18              # Node.js 18 版本
docker pull mysql:8.0            # MySQL 8.0
docker pull python:3.11-slim     # Python 精简版

# 查看本地镜像
docker images
# 或更详细的输出
docker image ls -a
```

### 删除镜像
```bash
docker rmi image_name:tag        # 按名称:标签删除
docker rmi image_id              # 按镜像 ID 删除
docker image prune               # 删除所有未使用的镜像（慎用）
docker image prune -a            # 删除所有未被容器引用的镜像
```

## 容器基本操作

### 运行容器
```bash
# 运行一个 Nginx 容器
docker run -d \
    --name my-nginx \
    -p 8080:80 \
    -v /host/path:/usr/share/nginx/html \
    nginx:alpine

# 参数说明：
# -d              后台运行（detach）
# --name          指定容器名称
# -p 8080:80      端口映射（宿主机:容器）
# -v              卷挂载（宿主机目录:容器目录）
```

现在访问 `http://localhost:8080` 即可看到 Nginx 欢迎页。

### 常见运行示例

```bash
# 运行 Node.js 应用
docker run -d --name my-node-app -p 3000:3000 \
    -v $(pwd):/app -w /app \
    node:18 node app.js

# 运行 MySQL
docker run -d --name my-mysql \
    -p 3306:3306 \
    -e MYSQL_ROOT_PASSWORD=secret123 \
    -e MYSQL_DATABASE=mydb \
    -v mysql_data:/var/lib/mysql \
    mysql:8.0

# 运行 Redis
docker run -d --name my-redis -p 6379:6379 \
    redis:alpine redis-server --requirepass mypassword

# 运行临时容器（退出后自动删除）
docker run --rm -it ubuntu:22.04 bash
```

### 查看容器
```bash
docker ps                  # 查看运行中的容器
docker ps -a               # 查看所有容器（包括已停止的）
docker ps -a --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

### 查看日志
```bash
docker logs my-nginx           # 查看全部日志
docker logs -f my-nginx        # 实时跟踪日志（Ctrl+C 退出）
docker logs --tail 50 my-nginx # 查看最后 50 行日志
docker logs --since 10m my-nginx # 查看最近 10 分钟的日志
```

### 进入容器内部
```bash
docker exec -it my-nginx /bin/sh   # Alpine 用 sh
docker exec -it my-nginx bash      # Ubuntu/Debian 用 bash
# 进入后可以执行任何命令，exit 退出
```

### 停止、启动、删除容器
```bash
docker stop my-nginx         # 停止容器
docker start my-nginx        # 启动已停止的容器
docker restart my-nginx      # 重启容器
docker rm my-nginx           # 删除容器（需先停止）
docker rm -f my-nginx        # 强制删除运行中的容器
docker container prune       # 删除所有已停止的容器
```

## Dockerfile 入门

Dockerfile 是用来自定义镜像的构建文件。以下是一个 Node.js 应用的基础示例：

```dockerfile
# Dockerfile
# 基础镜像
FROM node:18-alpine

# 设置工作目录
WORKDIR /app

# 复制 package.json 并安装依赖
COPY package*.json ./
RUN npm ci --only=production

# 复制源代码
COPY . .

# 暴露端口（文档性质，实际映射在 docker run 时指定）
EXPOSE 3000

# 定义默认启动命令
CMD ["node", "server.js"]
```

Dockerfile 核心指令：

| 指令 | 说明 |
|------|------|
| `FROM` | 指定基础镜像 |
| `RUN` | 构建时执行的命令 |
| `COPY` | 从宿主机复制文件到镜像 |
| `ADD` | 同 COPY，但支持 URL 和自动解压 |
| `WORKDIR` | 设置工作目录 |
| `ENV` | 设置环境变量 |
| `EXPOSE` | 声明容器监听的端口 |
| `CMD` | 容器启动时的默认命令 |
| `ENTRYPOINT` | 容器入口点（通常与 CMD 配合） |

### 构建和运行自定义镜像
```bash
# 构建镜像
docker build -t my-app:v1 .

# 查看构建历史
docker history my-app:v1

# 运行自定义镜像
docker run -d -p 3000:3000 --name my-app my-app:v1
```

## 实用清理命令

```bash
# 清理所有停止的容器、未使用的网络、悬挂镜像
docker system prune

# 更彻底的清理（包括所有未使用的镜像）
docker system prune -a --volumes

# 查看磁盘使用情况
docker system df
```
