# Docker Compose 多服务编排

现代应用通常由多个服务组成——Web 服务器、数据库、缓存、消息队列等。Docker Compose 允许你用 YAML 文件定义和管理这些服务的集合，一条命令就能启动整个技术栈。本文将介绍 Docker Compose 的核心概念和最佳实践。

## 安装 Docker Compose

Docker Compose 现在作为 Docker 插件发布，安装 Docker Desktop 或 Docker Engine 时通常已自带：

```bash
# 使用 Docker 插件方式（新版，推荐）
docker compose version

# 旧版独立命令方式（如果安装了 docker-compose）
docker-compose --version

# Linux 手动安装（如果未安装）
sudo apt install docker-compose-plugin    # Debian/Ubuntu
```

## docker-compose.yml 基础结构

Compose 文件的核心由三个顶级元素组成：

```yaml
# docker-compose.yml
version: "3.8"

services:       # 定义各个服务
  web:
    ...
  db:
    ...

networks:       # 定义自定义网络
  app-network:

volumes:        # 定义命名卷
  mysql_data:
```

`version` 字段虽已标记为可选（Compose V2），但为兼容性考虑仍建议保留。

## 实战：Node.js + MySQL + Redis 技术栈

以下是一个典型 Web 应用的多服务编排示例：

```yaml
# docker-compose.yml
version: "3.8"

services:
  # Node.js Web 应用
  web:
    build: .
    container_name: myapp-web
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_USER=${MYSQL_USER}
      - DB_PASSWORD=${MYSQL_PASSWORD}
      - DB_NAME=${MYSQL_DATABASE}
      - REDIS_HOST=redis
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_started
    volumes:
      - ./app:/app
    networks:
      - app-network
    restart: unless-stopped

  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: myapp-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${MYSQL_ROOT_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app-network
    restart: unless-stopped

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: myapp-redis
    command: redis-server --requirepass ${REDIS_PASSWORD} --appendonly yes
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app-network
    restart: unless-stopped

networks:
  app-network:
    driver: bridge

volumes:
  mysql_data:
    driver: local
  redis_data:
    driver: local
```

## 环境变量管理

Compose 支持从 `.env` 文件自动读取环境变量：

```bash
# .env 文件（不要提交到 Git）
MYSQL_ROOT_PASSWORD=SuperSecretRoot123
MYSQL_DATABASE=myapp
MYSQL_USER=myapp_user
MYSQL_PASSWORD=MyAppPass456
REDIS_PASSWORD=RedisPass789
```

在 `docker-compose.yml` 中使用 `${VARIABLE}` 语法引用。确保将 `.env` 添加到 `.gitignore`。

## depends_on 与健康检查

`depends_on` 控制服务启动顺序，但默认只等待容器启动不等待服务就绪。配合 `healthcheck` 可以确保依赖服务真正就绪后再启动：

```yaml
depends_on:
  mysql:
    condition: service_healthy   # 等待健康检查通过
  redis:
    condition: service_started   # 仅等待容器启动
```

`service_healthy` 需要 Docker Compose v3.8+ 或 Compose V2。对于不支持 `condition` 的版本，可以使用 `wait-for-it.sh` 或 `dockerize` 等工具在应用内部等待。

## 网络隔离

Compose 默认创建一个网络，所有服务自动加入。可以显式定义多个网络实现服务隔离：

```yaml
networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
  internal:
    driver: bridge
    internal: true   # 无法访问外部网络

services:
  web:
    networks:
      - frontend
      - backend

  mysql:
    networks:
      - backend       # 只连后端网络，从前端网络隔离
```

常见模式：Web 服务连接前端和后端网络，数据库只连接后端网络，从而实现数据库对公网的隔离。

## 命名卷与数据持久化

```yaml
volumes:
  mysql_data:
    driver: local     # 本地驱动，数据存储在宿主机

  nfs_data:
    driver: local
    driver_opts:
      type: nfs
      o: addr=192.168.1.100,rw
      device: ":/exports/data"
```

数据默认存储在 `/var/lib/docker/volumes/` 下，删除容器不会丢失卷数据。手动清理：

```bash
docker volume ls
docker volume prune
```

## 常用管理命令

```bash
# 启动所有服务（后台运行）
docker compose up -d

# 启动并重新构建镜像
docker compose up -d --build

# 查看服务状态
docker compose ps

# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f web

# 重启单个服务
docker compose restart web

# 停止所有服务
docker compose down

# 停止并删除卷（会丢失数据！）
docker compose down -v

# 进入服务容器
docker compose exec web bash
docker compose exec mysql mysql -u root -p

# 拉取所有服务的镜像更新
docker compose pull
```

## Compose 扩展能力

### 扩展/合并多个 Compose 文件

可以组合多个 Compose 文件，适合区分开发和生产配置：

```bash
# 基础配置 + 开发覆盖
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d

# 基础配置 + 生产覆盖
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

```yaml
# docker-compose.prod.yml
services:
  web:
    restart: always         # 覆盖 restart 策略
    deploy:                 # 仅 swarm 模式生效
      replicas: 3
```

### 复用定义（anchors & extensions）

```yaml
x-logging: &default-logging
  driver: json-file
  options:
    max-size: "10m"
    max-file: "3"

services:
  web:
    logging: *default-logging
  mysql:
    logging: *default-logging
```

## 最佳实践总结

1. **使用 `.env` 文件**管理敏感信息，绝不硬编码密码
2. **命名卷**用于需要持久化的数据（数据库、用户上传文件）
3. **健康检查**配合 `depends_on` 确保服务启动顺序
4. **网络隔离**将数据库等服务限制在后端网络中
5. **固定镜像版本**（如 `mysql:8.0`），避免 `latest` 标签导致意外升级
6. **生产环境**使用 `restart: always` 或 `unless-stopped` 保证服务自恢复
