# MySQL 数据库安装与基本配置

MySQL 是全球最流行的开源关系型数据库之一，广泛应用于各类 Web 应用和企业系统。本文覆盖 MySQL 在不同平台上的安装、安全初始化、用户管理以及核心配置优化。

## 安装 MySQL

### Linux (Debian/Ubuntu)

```bash
# 安装 MySQL Server
sudo apt update
sudo apt install mysql-server

# 查看安装版本
mysql --version

# 启动服务并设置开机自启
sudo systemctl start mysql
sudo systemctl enable mysql
sudo systemctl status mysql
```

默认情况下，MySQL 使用 `auth_socket` 插件认证 root 用户，这意味着只有系统 root 用户能免密登录 MySQL。若需改用密码认证：

```bash
sudo mysql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
FLUSH PRIVILEGES;
```

### Linux (CentOS/RHEL)

```bash
sudo yum install mysql-server        # CentOS 7
sudo dnf install mysql-server        # CentOS 8+
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码（CentOS 7 在安装时会生成）
sudo grep 'temporary password' /var/log/mysqld.log
```

### macOS

```bash
brew install mysql
brew services start mysql

# 安全初始化
mysql_secure_installation
```

## 安全初始化：mysql_secure_installation

安装完成后，强烈建议运行安全初始化脚本：

```bash
sudo mysql_secure_installation
```

脚本会引导你完成以下配置：

```text
VALIDATE PASSWORD COMPONENT    # 是否启用密码强度验证插件
Remove anonymous users?        # 删除匿名用户（建议 Y）
Disallow root login remotely?  # 禁止 root 远程登录（建议 Y）
Remove test database?          # 删除测试数据库（建议 Y）
Reload privilege tables?       # 重新加载权限表（建议 Y）
```

如果启用了密码强度验证，MySQL 要求密码满足以下规则：
- 至少 8 个字符
- 包含大写字母、小写字母、数字和特殊字符

## 创建数据库和用户

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE myapp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建本地用户
CREATE USER 'myapp'@'localhost' IDENTIFIED BY 'StrongPassword123!';

-- 创建远程用户（允许从任何 IP 连接）
CREATE USER 'myapp'@'%' IDENTIFIED BY 'StrongPassword123!';

-- 创建限定 IP 段的用户
CREATE USER 'myapp'@'192.168.1.%' IDENTIFIED BY 'StrongPassword123!';

-- 授予权限
GRANT ALL PRIVILEGES ON myapp.* TO 'myapp'@'localhost';
GRANT ALL PRIVILEGES ON myapp.* TO 'myapp'@'%';

-- 授予最小必要权限（推荐）
GRANT SELECT, INSERT, UPDATE, DELETE ON myapp.* TO 'myapp'@'localhost';

-- 立即生效
FLUSH PRIVILEGES;

-- 查看用户权限
SHOW GRANTS FOR 'myapp'@'localhost';

-- 删除用户
DROP USER 'myapp'@'%';
```

## 远程访问配置

MySQL 默认只监听本地连接（`127.0.0.1`），如需允许远程访问，需修改配置：

```bash
# 编辑 MySQL 配置文件
sudo vim /etc/mysql/mysql.conf.d/mysqld.cnf   # Debian/Ubuntu
sudo vim /etc/my.cnf                           # CentOS/RHEL
```

```ini
[mysqld]
# 修改绑定地址
bind-address = 0.0.0.0          # 监听所有网络接口
# 或者指定某个 IP
# bind-address = 192.168.1.100

# 如果 skip-networking = 1，需注释掉
# skip-networking
```

重启 MySQL 使配置生效：
```bash
sudo systemctl restart mysql
```

同时需要开放防火墙端口：
```bash
sudo ufw allow 3306/tcp
# 更安全的做法：仅允许特定 IP
sudo ufw allow from 192.168.1.0/24 to any port 3306
```

## 核心配置优化（my.cnf）

MySQL 配置文件中对性能影响最大的几个参数：

```ini
[mysqld]
# 连接数
max_connections = 500             # 最大连接数，根据应用并发量调整
max_connect_errors = 100          # 连接错误阈值，防止暴力破解

# InnoDB 缓冲池（最重要的性能参数）
# 建议设置为可用内存的 50%-70%
innodb_buffer_pool_size = 2G      # 专用数据库服务器建议设置为内存的 70%
innodb_buffer_pool_instances = 4  # 多实例可减少锁竞争，每个实例至少 1GB

# 日志相关
innodb_log_file_size = 512M       # 重做日志文件大小，调大减少磁盘 I/O
innodb_flush_log_at_trx_commit = 2 # 1=最安全,2=每秒钟刷盘,0=每秒刷盘
innodb_flush_method = O_DIRECT    # 避免双重缓存

# 线程缓存
thread_cache_size = 16

# 表缓存
table_open_cache = 4000
table_definition_cache = 2000

# 排序缓冲
sort_buffer_size = 4M
join_buffer_size = 4M

# 慢查询日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow-query.log
long_query_time = 2               # 超过2秒的查询记为慢查询
log_queries_not_using_indexes = 1 # 记录未使用索引的查询

# 二进制日志（主从复制必需）
server-id = 1
log_bin = /var/log/mysql/mysql-bin.log
binlog_expire_logs_seconds = 604800   # 7 天后自动清理
max_binlog_size = 100M
```

修改配置后务必验证并重启：
```bash
sudo mysql -e "SELECT @@innodb_buffer_pool_size / 1024 / 1024 / 1024 AS buffer_pool_gb;"
sudo systemctl restart mysql
```

## 字符集配置

统一使用 `utf8mb4` 字符集（MySQL 的 `utf8` 是不完整的，只支持最多 3 字节的字符）：

```ini
[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

[client]
default-character-set = utf8mb4

[mysql]
default-character-set = utf8mb4
```

验证字符集配置：
```sql
SHOW VARIABLES LIKE 'character_set%';
SHOW VARIABLES LIKE 'collation%';
```

对于已有的数据库和表，需要手动转换：
```sql
ALTER DATABASE myapp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
ALTER TABLE mytable CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 状态检查与监控

```sql
-- 查看服务器状态
SHOW STATUS;
SHOW GLOBAL STATUS LIKE 'Threads_connected';   -- 当前连接数
SHOW GLOBAL STATUS LIKE 'Threads_running';     -- 正在执行的线程数
SHOW STATUS LIKE 'Innodb_buffer_pool%';         -- 缓冲池使用情况
SHOW STATUS LIKE 'Innodb_rows_%';              -- InnoDB 读写行统计

-- 查看当前连接
SHOW PROCESSLIST;
SHOW FULL PROCESSLIST;                         -- 显示完整 SQL 语句

-- 终止指定连接
KILL connection_id;

-- 查看数据库大小
SELECT
    table_schema AS '数据库',
    ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS '大小(MB)'
FROM information_schema.tables
GROUP BY table_schema;

-- 查看表大小
SELECT
    table_name AS '表名',
    ROUND((data_length + index_length) / 1024 / 1024, 2) AS '大小(MB)',
    table_rows AS '行数'
FROM information_schema.tables
WHERE table_schema = 'myapp'
ORDER BY (data_length + index_length) DESC;
```

## 日常维护建议

1. **定期检查慢查询日志**，对执行慢的 SQL 进行优化
2. **监控连接数**，防止连接耗尽：`SHOW GLOBAL STATUS LIKE 'Max_used_connections';`
3. **定期备份**，使用 `mysqldump` 或 Percona XtraBackup
4. **监控磁盘空间**，确保 binlog 不会撑满磁盘
5. **不要直接用 root 用户**操作业务数据库，遵循最小权限原则
