# 程序员必看的技术视频频道

程序员的学习路径有两个阶段。第一阶段是"学语法"——文档、教程、官方示例、Stack Overflow 上你反复点进去的那个回答。但到了某一个点，你发现你不缺"怎样做"——你缺的是"为什么这样做""别人怎么想""前面的坑在哪里"。那个阶段，好的技术视频频道比三天不睡觉看书学得多。

以下是按领域组织的高信息密度频道——不是"编程入门教程"，而是给有基础的程序员看的东西。

---

## 技术大会演讲

大会演讲是浓缩了演讲者几年经验的三十分钟。如果选得好，一场演讲比一个月的随机搜索更有收获。

### GOTO Conferences

GOTO 的频道是软件架构领域的 TED——Martin Fowler、Kevlin Henney、Sam Newman 的经典会议录像都在上面。GOTO 的内容偏向架构设计、领域驱动设计、微服务、团队工程，每一场演讲的幻灯片和演讲者的表达节奏都打磨过。搜索"Martin Fowler Microservices"和"Kevlin Henney Seven Ineffective Coding Habits"开始。

### Strange Loop

Strange Loop 是更"野"的技术会议——不只在演示产品，在挑战"普遍被接受的假设"。Dan Abramov 在 Strange Loop 的"The Wet Codebase"演讲把前端开发的复杂性讲成了哲学。Gary Bernhardt 的"Wat"（四分半钟，历史上最好笑的技术演讲没有之一）。Strange Loop 2022 年是最后一届，但整个回放库都在 YouTube 上存着。

### JSConf 和 NDC

JSConf 的演讲大多数开社区脑洞的——Brendan Eich 讲 JavaScript 的诞生、Jenn Schiffer 讲像素艺术的数学。NDC 偏向 .NET 和系统级工程，但对全栈程序员来说讲 API 设计、数据库优化、分布式系统的那部分内容跨语言。

## 直播编码流

编程直播的价值不在看语法——在于看"一个老手在想什么"。打字不是重点，重点是"为什么打这行字之前停了十秒"。

### ThePrimeagen

Netflix 前工程师，Neovim 布道者。他的直播和视频是 Vim 绝技秀场 + 代码审查 + 偶尔暴躁吐槽的组合。"看看这个开源项目有多烂""在 Rust 里这样做是错的因为……"——你从他的话里听到的不是"这样"，而是"我不这样做的原因"。

### Tsoding Daily

俄罗斯裔程序员，用 Rust、Python、C 做大量的"今天我要从零实现一个 X"的直播。四小时录播、不删废话。你看的是过程不是结果——看他在终端里翻文档、走错方向、回退、咒骂、然后思路变清晰。这是"真实编程过程"，不是演示。

### 程序员鱼皮

中文区全栈路线很清晰的 UP 主——从前端到后端、从项目实战到面试指导。他的"手写系列"（手写 Vue 响应式、手写 Promise）比大多数付费课程讲得更透彻。适合快速上手一个方向并理解其基本思想后自己展开。

## 系统设计

### ByteByteGo

Alex Xu 的频道，内容来自他的畅销书《System Design Interview》。每一期用简洁的动画和图表解释一个系统设计概念——从 URL 短链接到设计一个聊天应用、从数据库索引到缓存淘汰策略。动画质量很高，每一期五到十分钟，信息密度适合"用一顿饭时间理解一个概念"。

### Gaurav Sen

印度工程师讲系统设计的标杆——比 ByteByteGo 更深入，每一期会讨论各种方案之间的 trade-off。推荐从"设计 Twitter"和"一致性哈希"的系列开始——他的板书式讲解在难度和清晰度之间找到了好位置。

---

## 底层：从芯片到操作系统

### Ben Eater

前 Khan Academy 工程师，自制了一个 8 位计算机——从面包板、逻辑门、时钟脉冲、七段显示器开始，全部无芯片、全手工布线。看这个系列不是因为你要造计算机——而是当你知道 CPU 的每个时钟脉冲在物理层面是什么时，"时序问题""内存延迟""指令周期"这些词突然不再抽象。推荐从"Build an 8-bit computer from scratch"播放列表第一集开始。

### Andreas Kling

前 Apple WebKit 工程师，从零开发 SerenityOS——一个完整的类 Unix 操作系统，包括图形界面和浏览器。他的直播视频是"操作系统开发日志"——一个人 push commit 的实时记录。看两期你就会对"操作系统到底在干嘛"有一个肌肉记忆级别的理解。

---

## 计算机科学

### Harvard CS50

David Malan 的 CS50 是世界上最好的计算机科学入门课程——而且全部免费在 YouTube。不是因为他把 CS 讲简单了——是因为他把 CS 讲得让你觉得"我一定要自己动手试试"。从 C 到 Python、从内存管理到网络协议——这不是一门课，这是一个完整的世界观构建。如果你认识想转行做编程的人，发给他们 CS50 的链接而不是任何从语法开始的教程。

### MIT 6.006 Introduction to Algorithms

Erik Demaine 的算法课——讲在黑板上的那种，不是幻灯片一页页翻过去。如果你想理解为什么某个算法的渐近复杂度是 O(n log n) 而不是感觉是 O(n^2)，这是最清晰的道路。

## 职业和生活

### Joma Tech

前 Facebook、Google 软件工程师，现全职内容创作者——但和其他"前 FAANG"博主不同，Joma 的视频一半是讽刺小品。那些"在 Google 工作的一天""面试你能去 Facebook 吗"是高级喜剧——因为程序员圈子里的自嘲和荒诞只有圈内人懂得怎么编成剧本。

### mayuko

日裔美国软件工程师，她的频道侧重于女性在科技行业的职业轨迹，以及"工程师身份以外的自我"。分享从硅谷到圣地亚哥的职业路径、技术焦虑、职业倦怠——不是"成功学"，是关于"持续"的日常记录。

## 几场必看的历史性演讲

这些演讲不在任何"教学大纲"里，但被每个知道它们的人反复推荐：

- **"The Mother of All Demos"（Douglas Engelbart, 1968）**：在九十分钟里首次公开展示了鼠标、超文本、视频会议、实时协作编辑。这是世界第一次看到"个人计算机能做什么"。
- **"Simple Made Easy"（Rich Hickey, Strange Loop 2011）**：简单不等于容易——区分这两个词的演讲，是架构思维的分水岭。"简单是客观的，容易是主观的"——听完你再也不会用简单和容易当同义词。
- **"The Wet Codebase"（Dan Abramov, Deconstruct 2019）**：前端复杂性不是你的错——是软件的本质。React 的核心开发者在告诉你，他认为我们所有人都在与之搏斗的东西到底是什么。
